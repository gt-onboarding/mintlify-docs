<div id="headers">
  ## 見出し
</div>

見出しはコンテンツを整理し、ナビゲーション用のアンカーを生成します。目次に表示され、ユーザーがドキュメントを素早くスキャンできるようにします。

<div id="creating-headers">
  ### 見出しの作成
</div>

見出しのレベルに応じて `#` 記号を使用します：

```mdx
## メインセクションヘッダー
### サブセクションヘッダー
#### サブサブセクションヘッダー
```

<Tip>
  後続の内容を明確に示す、説明的でキーワードを適切に含んだ見出しを使いましょう。ユーザーの操作性と検索エンジン最適化（SEO）の両方の向上に役立ちます。
</Tip>

<div id="disabling-anchor-links">
  ### アンカーリンクを無効にする
</div>

デフォルトでは、見出しにはクリック可能なアンカーリンクが含まれており、ユーザーは特定のセクションへ直接リンクできます。HTML または React の見出しで `noAnchor` プロップを使用すると、これらのアンカーリンクを無効にできます。

<CodeGroup>
  ```mdx HTML header example
  <h2 noAnchor>
  アンカーリンクなしの見出し
  </h2>
  ```

  ```mdx React header example
  <Heading level={2} noAnchor>
  アンカーリンクなしの見出し
  </Heading>
  ```
</CodeGroup>

`noAnchor` を使用すると、見出しにはアンカー用のチップが表示されず、見出しテキストをクリックしてもアンカーリンクはクリップボードにコピーされません。

## テキストの書式設定

テキストの強調やスタイル指定のための多くのMarkdown書式に対応しています。

<div id="basic-formatting">
  ### 基本的な書式設定
</div>

テキストに次の書式を適用します：

| スタイル | 構文 | 例 | 結果 |
|-------|--------|---------|--------|
| **太字** | `**text**` | `**important note**` | **important note** |
| *斜体* | `_text_` | `_emphasis_` | *emphasis* |
| ~~取り消し線~~ | `~text~` | `~deprecated feature~` | ~~deprecated feature~~ |

<div id="combining-formats">
  ### 書式の組み合わせ
</div>

複数の書式スタイルを組み合わせられます:

```mdx
**_太字と斜体_**
**~~太字と取り消し線~~**
*~~斜体と取り消し線~~**
```

***太字と斜体***<br />
**~~太字と取り消し線~~**<br />
*~~斜体と取り消し線~~*

<div id="superscript-and-subscript">
  ### 上付き文字と下付き文字
</div>

数式や脚注には HTML タグを使用します：

| 種類 | 構文 | 例 | 結果 |
|------|--------|---------|--------|
| 上付き文字 | `<sup>text</sup>` | `example<sup>2</sup>` | example<sup>2</sup> |
| 下付き文字 | `<sub>text</sub>` | `example<sub>n</sub>` | example<sub>n</sub> |

## リンク

リンクは、ユーザーがページ間を移動したり、外部リソースにアクセスしたりする際に役立ちます。わかりやすいリンクテキストを用いて、アクセシビリティとユーザー体験を向上させましょう。

<div id="internal-links">
  ### 内部リンク
</div>

ドキュメント内の他のページへは、ルート相対パスを使用してリンクします：

```mdx
[クイックスタート](/quickstart)
[ステップ](/components/steps)
```

[クイックスタート](/ja/quickstart)<br />
[手順](/ja/components/steps)

<Note>
  `[page](../page)` のような相対リンクは読み込みが遅く、ルート相対リンクほど最適化できないため、使用は避けてください。
</Note>

<div id="external-links">
  ### 外部リンク
</div>

外部リソースには、完全な URL を記載してください。

```mdx
[Markdownガイド](https://www.markdownguide.org/)
```

[Markdownガイド](https://www.markdownguide.org/)

<div id="broken-links">
  ### リンク切れ
</div>

[CLI](/ja/installation) を使って、ドキュメント内のリンク切れをチェックできます：

```bash
mint broken-links
```

<div id="blockquotes">
  ## 引用
</div>

引用は、コンテンツ内の重要な情報や引用、例を際立たせるために使います。

<div id="single-line-blockquotes">
  ### 単一行の引用ブロック
</div>

引用ブロックを作成するには、テキストの先頭に `>` を付けます：

```mdx
> これはメインコンテンツから際立つ引用文です。
```

> これは本文から際立つ引用です。

<div id="multi-line-blockquotes">
  ### 複数行の引用ブロック
</div>

長めの引用や複数段落にわたる場合:

```mdx
> これは複数行のブロック引用の最初の段落です。
>
> これは2番目の段落で、`>`を含む空行で区切られています。
```

> これは複数行のブロック引用における最初の段落です。
>
> これは2つ目の段落で、`>` を付けた空行で区切られています。

<Tip>
  視覚的な効果と意味を保つため、ブロック引用の使用は控えめにしましょう。メモ、警告、その他の情報には [callouts](/ja/components/callouts) の利用を検討してください。
</Tip>

<div id="mathematical-expressions">
  ## 数式
</div>

数式や方程式のレンダリングに LaTeX をサポートしています。

<div id="inline-math">
  ### インライン数式
</div>

インラインの数式には、`$`（ドル記号1つ）を使用します。

```mdx
ピタゴラスの定理は、直角三角形において $(a^2 + b^2 = c^2)$ が成り立つことを示しています。
```

ピタゴラスの定理は、直角三角形では $(a^2 + b^2 = c^2)$ が成り立つとするものです。

<div id="block-equations">
  ### ブロック数式
</div>

独立した数式には、`$$`（ダブルドル記号）を使用します：

```mdx
$$
E = mc^2
$$
```

$$
E = mc^2
$$

<Info>
  LaTeX を利用するには、正しい数式の構文が必要です。詳細な構文ガイドラインは [LaTeX ドキュメント](https://www.latex-project.org/help/documentation/)を参照してください。
</Info>

<div id="line-breaks-and-spacing">
  ## 改行とスペーシング
</div>

コンテンツの可読性を高めるために、余白や改行を適切に調整します。

<div id="paragraph-breaks">
  ### 段落の区切り
</div>

段落は空行で区切ります。

```mdx
これは最初の段落です。

これは2番目の段落で、空白行で区切られています。
```

これは最初の段落です。

これは空行で区切られた2つ目の段落です。

<div id="manual-line-breaks">
  ### 手動改行
</div>

段落内で強制的に改行するには、HTML の `<br />` タグを使用します。

```mdx
この行はここで終わります。<br />
この行は新しい行から始まります。
```

この行はここで終わります。<br />
この行は次の行から始まります。

<Tip>
  多くの場合、手動の改行よりも空行で段落を分けたほうが可読性が高くなります。
</Tip>

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="content-organization">
  ### コンテンツの整理
</div>

* 見出しで明確な情報階層を構築する
* 見出しの階層構造を正しく守る（H2 から H4 へ飛ばさない）
* 内容を的確に表し、キーワードを盛り込んだ見出しテキストを書く

<div id="text-formatting">
  ### テキストの書式設定
</div>

* 強調には太字を使い、段落全体には使わない
* 斜体は用語、タイトル、または軽い強調に限定する
* コンテンツの理解を妨げる過度な装飾は避ける

<div id="links">
  ### リンク
</div>

* 「こちら」や「続きを読む」ではなく、内容を表すリンクテキストを書く
* 内部リンクにはルート相対パスを使う
* リンク切れを防ぐため、定期的にリンクを検証・テストする
