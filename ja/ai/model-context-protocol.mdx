<div id="about-mcp-servers">
  ## MCPサーバーについて
</div>

Model Context Protocol（MCP）は、AIアプリケーションとドキュメントなどの外部サービス間の接続を標準化するオープンなプロトコルです。Mintlifyは、ドキュメントおよびOpenAPI仕様からMCPサーバーを自動生成し、Claude、Cursor、GooseなどのあらゆるMCPクライアントがドキュメントやAPIに接続できる、より広範なAIエコシステム向けにコンテンツを準備します。

MCPサーバーは、AIアプリケーションがドキュメントを検索し、APIと連携できるツール群を提供します。

<div id="accessing-your-mcp-server">
  ## MCPサーバーへのアクセス
</div>

<Note>
  MCPサーバーは公開ドキュメントに対してのみ生成できます。エンドユーザー認証が必要なドキュメントは、サーバーの生成対象にはなりません。
</Note>

MCPサーバーは自動的に生成され、ドキュメントのURLにある`/mcp`パスでホストされます。たとえば、MintlifyのMCPサーバーは`https://mintlify.com/docs/mcp`で利用できます。

[MCPのダッシュボード](https://dashboard.mintlify.com/products/mcp)で、MCPサーバーのURLを確認・コピーできます。

<div id="configuring-your-mcp-server">
  ## MCPサーバーの設定
</div>

すべてのMCPサーバーにはデフォルトで`search`ツールが含まれており、他のツールからドキュメント情報を照会できます。

[ProまたはEnterpriseプラン](https://mintlify.com/pricing?ref=mcp)をご利用の場合、OpenAPI仕様のエンドポイントをMCPツールとして公開できます。

エンドポイントをMCPツールとして公開するには、ファイルレベルまたはエンドポイントレベルで`x-mint`拡張内の`mcp`オブジェクトを使用します。たとえば、MintlifyのMCPサーバーには、アシスタントのチャット作成、ステータス更新の取得、アップデートのトリガーといったツールが含まれています。

MCPサーバーはセキュリティファーストの方針に従っており、APIエンドポイントはデフォルトでは公開されません。MCPツールとして利用可能にするには、エンドポイントを明示的に有効化する必要があります。AIツール経由での公開アクセスに適した安全なエンドポイントのみを公開してください。

<ResponseField name="mcp" type="object">
  エンドポイントのMCP設定。

  <Expandable title="MCP">
    <ResponseField name="enabled" type="boolean">
      エンドポイントをMCPツールとして公開するかどうか。ファイルレベルの設定より優先されます。
    </ResponseField>

    <ResponseField name="name" type="string">
      MCPツール名。
    </ResponseField>

    <ResponseField name="description" type="string">
      MCPツールの説明。
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="file-level-configuration">
  ### ファイル単位の設定
</div>

OpenAPI仕様ファイルで、デフォルトですべてのエンドポイントにMCPを有効化し、必要に応じて特定のエンドポイントを除外します:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": false // このエンドポイントではMCPを無効化
          }
        },
        // ...
      }
    }
  }
}
```

<div id="endpoint-level-configuration">
  ### エンドポイント単位の設定
</div>

特定のエンドポイントで MCP を有効にします:

```json
{
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": true,
            "name": "get-users",
            "description": "ユーザー一覧を取得"
          },
          // ...
        }
      }
    },
    "/api/v1/delete": {
      "delete": {
        // `x-mint: mcp` がないため、このエンドポイントは MCP ツールとして公開されません
        // ...
      }
    }
  }
}
```

<div id="using-your-mcp-server">
  ## MCPサーバーの利用方法
</div>

ユーザーは、利用しているAIツールにあなたのMCPサーバーを接続する必要があります。

1. MCPサーバーのURLを公開します。
2. ユーザーがMCPサーバーのURLをコピーして、自分のツールに追加します。
3. ユーザーはツール経由であなたのドキュメントおよびAPIエンドポイントにアクセスします。

ユーザーがあなたのMCPサーバーに接続しやすくするための方法として、次のようなものがあります：

<Tabs>
  <Tab title="コンテキストメニュー">
    ドキュメントの任意のページからユーザーが MCP サーバーに接続できるよう、[コンテキストメニュー](/ja/ai/contextual-menu)にオプションを追加します。

    | オプション | 識別子 | 説明 |
    | :----- | :--------- | :---------- |
    | **MCP サーバー URL をコピー** | `mcp` | ユーザーのクリップボードに MCP サーバーの URL をコピーします。 |
    | **Cursor に接続** | `cursor` | MCP サーバーを Cursor にインストールします。 |
    | **VS Code に接続** | `vscode` | MCP サーバーを VS Code にインストールします。 |
  </Tab>

  <Tab title="Claude">
    <Steps>
      <Step title="MCP サーバーの URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバーの URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバーの URL を公開します。">
        ユーザー向けに、MCP サーバーの URL と Claude への接続手順を含むガイドを作成します。

        1. Claude の設定で [Connectors](https://claude.ai/settings/connectors) ページに移動します。
        2. **Add custom connector** を選択します。
        3. MCP サーバー名と URL を追加します。
        4. **Add** を選択します。
        5. Claude 利用時に、添付ボタン（プラスアイコン）を選択します。
        6. MCP サーバーを選択します。
      </Step>
    </Steps>

    詳細は [Model Context Protocol のドキュメント](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server)をご覧ください。
  </Tab>

  <Tab title="Claude Code">
    <Steps>
      <Step title="MCP サーバーの URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバーの URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバーの URL を公開します。">
        ユーザー向けに、MCP サーバーの URL と Claude Code に接続するコマンドを含むガイドを作成します。

        ```bash
        claude mcp add --transport http <name> <url>
        ```
      </Step>
    </Steps>

    詳細は [Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers)をご覧ください。
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="MCP サーバーの URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバーの URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバーの URL を公開します。">
        ユーザー向けに、MCP サーバーの URL と Cursor への接続手順を含むガイドを作成します。

        1. <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>（Windows では <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>）でコマンドパレットを開きます。
        2. "Open MCP settings" を検索します。
        3. **Add custom MCP** を選択します。`mcp.json` ファイルが開きます。
        4. `mcp.json` でサーバーを設定します:

        ```json
        {
          "mcpServers": {
            "<your-mcp-server-name>": {
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    詳細は [Cursor のドキュメント](https://docs.cursor.com/en/context/mcp#installing-mcp-servers)をご覧ください。
  </Tab>

  <Tab title="VS Code">
    <Steps>
      <Step title="MCP サーバーの URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバーの URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバーの URL を公開します。">
        ユーザー向けに、MCP サーバーの URL と VS Code への接続手順を含むガイドを作成します。

        1. `.vscode/mcp.json` ファイルを作成します。
        2. `mcp.json` でサーバーを設定します:

        ```json
        {
          "servers": {
            "<your-mcp-server-name>": {
              "type": "http",
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    詳細は [VS Code のドキュメント](https://code.visualstudio.com/docs/copilot/chat/mcp-servers)をご覧ください。
  </Tab>
</Tabs>

<div id="example-connecting-to-the-mintlify-mcp-server">
  ### 例: Mintlify MCP サーバーへの接続
</div>

Mintlify MCP サーバーに接続して、Mintlify API と対話したり、ドキュメントを検索したりできます。これにより、ローカル環境での Mintlify の使い方に関するより正確な回答が得られ、また、ユーザーが自分たちの MCP サーバーに接続する方法の手本にもなります。

<Tabs>
  <Tab title="Contextual menu">
    このページ上部のコンテキストメニューを開き、**Connect to Cursor** または **Connect to VS Code** を選択して、任意の IDE に Mintlify MCP サーバーを接続します。
  </Tab>

  <Tab title="Claude">
    Mintlify MCP サーバーを Claude で使用するには:

    <Steps>
      <Step title="Mintlify MCP サーバーを Claude に追加する">
        1. Claude の設定で [Connectors](https://claude.ai/settings/connectors) ページに移動します。
        2. **Add custom connector** を選択します。
        3. Mintlify MCP サーバーを追加します:

        - Name: `Mintlify`
        - URL: `https://mintlify.com/docs/mcp`

        4. **Add** を選択します。
      </Step>

      <Step title="チャットで MCP サーバーにアクセスする">
        1. Claude 使用中に、添付ボタン（プラスアイコン）を選択します。
        2. Mintlify MCP サーバーを選択します。
        3. Mintlify について Claude に質問します。
      </Step>
    </Steps>

    詳細は [Model Context Protocol のドキュメント](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server)を参照してください。
  </Tab>

  <Tab title="Claude Code">
    Mintlify MCP サーバーを Claude Code で使用するには、次のコマンドを実行します:

    ```bash
    claude mcp add --transport http Mintlify https://mintlify.com/docs/mcp
    ```

    次を実行して接続をテストします:

    ```bash
    claude mcp list
    ```

    詳細は [Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers)を参照してください。
  </Tab>

  <Tab title="Cursor">
    <PreviewButton href="cursor://anysphere.cursor-deeplink/mcp/install?name=mintlify&config=eyJ1cmwiOiJodHRwczovL21pbnRsaWZ5LmNvbS9kb2NzL21jcCJ9">Install in Cursor</PreviewButton>

    Mintlify MCP サーバーを Cursor に接続するには、**Install in Cursor** ボタンをクリックします。MCP サーバーを手動で接続する場合は、次の手順に従ってください:

    <Steps>
      <Step title="MCP 設定を開く">
        1. <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>（Windows の場合は <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>）でコマンドパレットを開きます。
        2. "Open MCP settings" を検索します。
        3. **Add custom MCP** を選択します。`mcp.json` ファイルが開きます。
      </Step>

      <Step title="Mintlify MCP サーバーを設定する">
        `mcp.json` に次を追加します:

        ```json
        {
          "mcpServers": {
            "Mintlify": {
              "url": "https://mintlify.com/docs/mcp"
            }
          }
        }
        ```
      </Step>

      <Step title="接続をテストする">
        Cursor のチャットで「What tools do you have available?」と尋ねます。Cursor に Mintlify MCP サーバーが利用可能なツールとして表示されます。
      </Step>
    </Steps>

    詳細は Cursor のドキュメントの [Installing MCP servers](https://docs.cursor.com/en/context/mcp#installing-mcp-servers) を参照してください。
  </Tab>

  <Tab title="VS Code">
    <PreviewButton href="https://vscode.dev/redirect/mcp/install?name=mintlify&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Fmintlify.com%2Fdocs%2Fmcp%22%7D">Install in VS Code</PreviewButton>

    Mintlify MCP サーバーを VS Code に接続するには、**Install in VS Code** ボタンをクリックします。MCP サーバーを手動で接続する場合は、`.vscode/mcp.json` ファイルを作成し、次を追加します:

    ```json
    {
      "servers": {
        "Mintlify": {
          "type": "http",
          "url": "https://mintlify.com/docs/mcp"
        }
      }
    }
    ```

    詳細は [VS Code のドキュメント](https://code.visualstudio.com/docs/copilot/chat/mcp-servers) を参照してください。

<div id="authentication">
  ## 認証
</div>

MCP で API エンドポイントを有効化すると、サーバーは OpenAPI の `securitySchemes` と `securityRequirement` に定義された認証要件を反映します。各種キーはツール側で直接扱われ、Mintlify によって保存・処理されることはありません。

ユーザーが AI ツールに保護されたエンドポイントの呼び出しを指示した場合、ツールはその時点でユーザーに必要な認証情報の提供を求めます。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

<AccordionGroup>
  <Accordion title="MCP サーバーで検索ツールしか表示されない">
    OpenAPI 仕様があるにもかかわらず、MCP サーバーに検索ツールしか表示されない場合:

    1. OpenAPI 仕様が正しく、有効かつアクセス可能であることを確認してください。
    2. 対象のエンドポイントで `x-mint.mcp.enabled: true` を明示的に有効化していることを確認してください。
    3. デプロイのログで OpenAPI の処理エラーがないか確認してください。

    OpenAPI の処理に失敗した場合でも、機能を維持するためにサーバーは検索ツールのみで稼働を継続します。
  </Accordion>

  <Accordion title="認証に関する問題">
    ユーザーから認証の問題が報告された場合:

    1. OpenAPI 仕様に適切な `securitySchemes` の定義が含まれていることを確認してください。
    2. 有効化したエンドポイントが、指定した認証方式で正しく動作することを確認してください。
  </Accordion>

  <Accordion title="ツールの説明が不足している、または不明瞭">
    AI ツールが API エンドポイントを効果的に利用できていない場合:

    1. エンドポイントに詳細な `summary` と `description` フィールドを追加してください。
    2. パラメータ名と説明が分かりやすいことを確認してください。
    3. MCP ダッシュボードで、エンドポイントがツールとしてどのように表示されるかを確認してください。
  </Accordion>
</AccordionGroup>
