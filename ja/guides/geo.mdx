ChatGPT、Perplexity、Google AI Overviews などのAI搭載型の回答エンジンと、従来型の検索エンジンの双方に向けて、ドキュメントを最適化しましょう。

Generative Engine Optimization（GEO）は、包括的なコンテンツと構造化された情報を通じてAIシステムに引用されることに重きを置くのに対し、従来のSEOは検索結果の順位向上を目標とします。

<div id="geo-quickstart">
  ## GEOクイックスタート
</div>

<div id="initial-setup">
  ### 初期セットアップ
</div>

1. `docs.json` の設定でドキュメントがインデックス対象になっていることを確認する
2. 既存ページを見直し、説明文やタイトルの不足がないかを確認する

<div id="content-improvements">
  ### コンテンツ改善
</div>

1. 適切なページに**比較表を追加**する
2. よくある質問に答えられているか**見出しを見直す**
3. 関連トピック間の**内部リンクを強化**する
4. **AIツールでテスト**して正確性を検証する

<div id="geo-best-practices">
  ## GEO のベストプラクティス
</div>

一般に、読みやすく構成の整ったドキュメントは、GEO の観点でも高い評価を得られます。まずは常にユーザーのために書くことを優先し、コンテンツがユーザーのニーズに応えていれば、AI ツール向けの最適化にも自然とつながります。最適化のための最適化ではなく、本当に役立つコンテンツを作ることが、従来型・AI の両方の検索エンジンから評価されます。

次の点に注力しましょう:

- キーワード一致ではなく、ユーザーのニーズに沿ったコンテンツ
- 構造化され、ざっと見て要点が把握できる情報
- 質問に対する直接的な回答

<div id="format-for-clarity">
  ### 明確さのための書式
</div>

次の書式上の工夫は、AIツールがコンテンツを正確に解析・理解するのに役立ちます。

- 見出しレベルを飛ばさない（H1 → H2 → H3）
- "it" や "this" の代わりに、具体的な対象名を用いる
- コードブロックに使用言語（プログラミング言語）を指定する
- 画像には内容を説明する代替テキスト（alt）を付ける
- 関連する概念へリンクし、AIが関係性を把握できるようにする

<div id="answer-questions-directly">
  ### 質問に直接答える
</div>

特定のユーザーの質問に的確に応えるコンテンツを書きましょう:

- セクションは最も重要な結論から始める
- よくある質問に合致するわかりやすい見出しを使う
- 複雑なトピックは番号付きの手順に分ける

<div id="mintlify-configuration">
  ## Mintlifyの設定
</div>

これらの機能を使ってGEOを改善しましょう。

<div id="add-descriptive-page-metadata">
  ### わかりやすいページメタデータを追加する
</div>

frontmatter に明確なタイトルと説明を含めてください:

```mdx
---
title: "API authentication guide"
description: "Complete guide to implementing API authentication with code examples"
---
```

<div id="configure-global-indexing-settings">
  ### グローバルなインデックス設定を行う
</div>

`docs.json` に以下を追加します:

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt ファイルは、サイトマップが検索エンジンを支援するのと同様に、AI システムがドキュメントの構造を理解するのに役立ちます。Mintlify はドキュメント向けに LLMs.txt ファイルを自動生成します。設定は不要です。

<div id="testing-your-documentation">
  ## ドキュメントをテストする
</div>

製品やドキュメントに関する質問で各種AIツールを試し、ドキュメントがどの程度正しく参照・引用されているかを確認します。

**アシスタントにドキュメントについて具体的な質問を投げてみましょう：**

- 「このAPIで認証を設定するにはどうすればよいですか？」
- 「インストール手順をステップごとに案内してください」

**ツールが次の点を満たしているか確認してください：**

- 正確なコードサンプル
- 誤りのない手順に沿った解説
