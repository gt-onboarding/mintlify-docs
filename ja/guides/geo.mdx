従来の検索エンジンに加え、ChatGPT、Perplexity、Google AI Overviews のような AI 生成回答エンジンの双方に向けて、ドキュメントを最適化しましょう。

Generative Engine Optimization（GEO）は、包括的なコンテンツと構造化された情報によって AI システムに引用されることに重点を置くのに対し、従来の SEO は検索結果での順位向上を目的とします。

<div id="geo-quickstart">
  ## GEO クイックスタート
</div>

<div id="initial-setup">
  ### 初期設定
</div>

1. `docs.json` の設定でドキュメントがインデックス対象になっていることを確認する
2. 既存のページを監査し、説明やタイトルの不足がないか確認する

<div id="content-improvements">
  ### コンテンツの改善
</div>

1. 適切なページに**比較表を追加**する
2. **見出しを見直し**、よくある質問に答えているか確認する
3. 関連トピック間の**内部リンクを強化**する
4. **AI ツールでテスト**して正確性を検証する

<div id="geo-best-practices">
  ## GEO のベストプラクティス
</div>

一般に、よく書かれ適切に構造化されたドキュメントは、GEO に強くなります。まずはユーザーのために書くことを優先し、コンテンツがユーザーのニーズを満たしていれば、AI ツール向けの最適化にも自然と近づけます。単なる「最適化のための最適化」ではなく、本当に役立つコンテンツを作成することが、従来型と AI の両方の検索エンジンに評価されます。

次に注力しましょう:

- キーワード一致ではなく、ユーザーのニーズに沿ったコンテンツ
- 構造化され、スキャンしやすい情報
- 質問への端的な回答

<div id="format-for-clarity">
  ### 明確さのための書式
</div>

次の書式の実践は、AI ツールがコンテンツを解析・理解するのに役立ちます:

- 見出しレベルを飛ばさない（H1 → H2 → H3）
- "it" や "this" の代わりに具体的なオブジェクト名を使う
- コードブロックに対応するプログラミング言語のラベルを付ける
- 画像には説明的な alt テキストを付ける
- 関連する概念にリンクし、AI が関係性を把握できるようにする

<div id="answer-questions-directly">
  ### 質問に直接答える
</div>

特定のユーザーの質問に的確に応えるコンテンツを書きましょう。

- セクションは最も重要な要点から始める
- よくある質問に合致する、わかりやすい見出しを使う
- 複雑なトピックは番号付きの手順に分解する

<div id="mintlify-configuration">
  ## Mintlify の設定
</div>

これらの機能を使って GEO を改善しましょう。

<div id="add-descriptive-page-metadata">
  ### 記述的なページメタデータを追加する
</div>

フロントマターにわかりやすいタイトルと説明を含めます:

```mdx
---
title: "API authentication guide"
description: "Complete guide to implementing API authentication with code examples"
---
```

<div id="configure-global-indexing-settings">
  ### グローバルなインデックス設定を設定する
</div>

`docs.json` に次を追加します:

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt は、サイトマップが検索エンジンを支援するのと同様に、AI システムがドキュメントの構造を理解するのに役立つファイルです。Mintlify はドキュメント向けの LLMs.txt を自動生成します。設定は不要です。

<div id="testing-your-documentation">
  ## ドキュメントのテスト
</div>

製品やドキュメントに関する質問で各種AIツールを試し、ドキュメントの参照・引用のされ方を確認しましょう。

**アシスタントにドキュメントについて具体的な質問を投げてみてください：**

- 「このAPIで認証を設定するにはどうすればよいですか？」
- 「インストール手順をステップごとに説明してください」

**ツールが次を提供しているか確認してください：**

- 正確なコードサンプル
- 正確なステップバイステップの手順
