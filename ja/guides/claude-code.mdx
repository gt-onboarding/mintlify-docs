---
title: "Claude Code"
description: "Claude Code を設定して、ドキュメントの作成・レビュー・更新を支援する"
---

Claude Code は、ドキュメントの保守を支援するエージェント型のコマンドラインツールです。新規コンテンツの作成、既存ページのレビュー、ドキュメントの最新状態の維持を行えます。

プロジェクトに `CLAUDE.md` ファイルを追加し、時間をかけて洗練していくことで、Claude Code にドキュメントの標準やワークフローを理解させられます。

<div id="getting-started">
  ## はじめに
</div>

**前提条件:**

- Claude の有効なサブスクリプション（Pro、Max、または API アクセス）

**セットアップ:**

1. Claude Code をインストールします:

```bash
  npm install -g @anthropic-ai/claude-code
  ```

2. ドキュメントのディレクトリに移動します。
3. （任意）以下の `CLAUDE.md` をプロジェクトに追加します。
4. `claude` を実行して開始します。

<div id="claudemd-template">
  ## CLAUDE.md テンプレート
</div>

ドキュメントディレクトリのルートに `CLAUDE.md` ファイルを保存して、Claude Code がプロジェクトを理解できるようにします。このファイルは、ドキュメントの標準・好み・ワークフローについて Claude Code に学習させます。詳細は Anthropic のドキュメント「[Claude のメモリを管理](https://docs.anthropic.com/en/docs/claude-code/memory)」を参照してください。

以下のサンプルテンプレートをコピーするか、ドキュメントの要件に合わせて調整してください。

```mdx
# Mintlifyドキュメント

## 協働関係
- アイデアに対して異議を唱えることができます。これにより、より良いドキュメントにつながる可能性があります。異議を唱える際は、情報源を引用し、理由を説明してください
- 推測するのではなく、必ず確認を求めてください
- 決して嘘をついたり、推測したり、でっち上げたりしないでください

## プロジェクトコンテキスト
- フォーマット: YAMLフロントマターを含むMDXファイル
- 設定: ナビゲーション、テーマ、設定用のdocs.json
- コンポーネント: Mintlifyコンポーネント

## コンテンツ戦略
- ユーザーの成功に必要な分だけドキュメント化する - 多すぎず、少なすぎず
- 正確性と使いやすさを優先する
- 可能な限りコンテンツを普遍的なものにする
- 新しいコンテンツを追加する前に、既存のコンテンツを検索する。戦略的な理由がない限り、重複を避ける
- 一貫性を保つために既存のパターンを確認する
- 最小限の妥当な変更から始める

## docs.json

- docs.jsonファイルとサイトナビゲーションを構築する際は、[docs.jsonスキーマ](https://mintlify.com/docs.json)を参照してください

## ページのフロントマター要件
- title: 明確で説明的なページタイトル
- description: SEO/ナビゲーション用の簡潔な要約

## 執筆基準
- 二人称(「あなた」)を使用
- 手順的なコンテンツの冒頭に前提条件を記載
- 公開前にすべてのコード例をテストする
- 既存ページのスタイルとフォーマットに合わせる
- 基本的なユースケースと高度なユースケースの両方を含める
- すべてのコードブロックに言語タグを付ける
- すべての画像に代替テキストを付ける
- 内部リンクには相対パスを使用する

## Gitワークフロー
- コミット時に--no-verifyを使用しないでください
- 作業を開始する前に、コミットされていない変更の処理方法を尋ねる
- 変更に適した明確なブランチが存在しない場合は、新しいブランチを作成する
- 開発中は頻繁にコミットする
- プリコミットフックをスキップまたは無効化しないでください

## 禁止事項
- MDXファイルのフロントマターを省略しない
- 内部リンクに絶対URLを使用しない
- テストされていないコード例を含めない
- 推測しない - 常に確認を求める
```


<div id="sample-prompts">
  ## サンプルプロンプト
</div>

Claude Code のセットアップが完了したら、以下のプロンプトを試して、一般的なドキュメント作業にどう役立つかを確認してください。これらの例はそのままコピーして貼り付けるか、用途に合わせて調整して使えます。

<div id="convert-notes-to-polished-docs">
  ### メモを完成度の高いドキュメントに変換
</div>

ラフな下書きを、フロントマターとコンポーネントを備えた適切なMarkdownページに仕上げます。

**プロンプト例:**

```text wrap
このテキストを適切にフォーマットされたMDXページに変換してください：[ここにテキストを貼り付けてください]
```

<div id="review-docs-for-consistency">
  ### ドキュメントの一貫性をレビュー
</div>

スタイル、フォーマット、コンポーネントの使い方に関する改善提案を受け取ります。

**プロンプト例:**

```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### 機能が変わったらドキュメントを更新する
</div>

プロダクトの変更に合わせてドキュメントを常に最新に保ちましょう。

**プロンプト例:**

```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

<div id="generate-comprehensive-code-examples">
  ### 網羅的なコード例を生成する
</div>

エラーハンドリング付きの多言語コード例を作成します。

**プロンプト例:**

```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Claude Code の拡張
</div>

手動で Claude Code にプロンプトするだけでなく、既存のワークフローにも統合できます。

<div id="automation-with-github-actions">
  ### GitHub Actions による自動化
</div>

コードの変更に合わせて Claude Code を自動実行し、ドキュメントを最新の状態に保ちます。プルリクエストでドキュメントのレビューを自動で開始したり、API の変更を検知した際にサンプルを更新したりできます。

<div id="multi-instance-workflows">
  ### 複数インスタンスのワークフロー
</div>

タスクごとに Claude Code のセッションを分けて使いましょう。たとえば、新規コンテンツ作成用とレビュー・品質保証用でそれぞれ別のセッションにします。こうすることで一貫性を保ち、単一セッションでは見落としかねない問題を検知できます。

<div id="team-collaboration">
  ### チームでのコラボレーション
</div>

洗練した `CLAUDE.md` ファイルをチームと共有し、全ての貢献者でドキュメントの基準を一貫させましょう。チームでは、プロジェクトに特化したプロンプトやワークフローを整備し、ドキュメント作成プロセスに組み込むことがよくあります。

<div id="custom-commands">
  ### カスタムコマンド
</div>

プロジェクトやチーム固有の、よく使うドキュメント作業向けに再利用可能なスラッシュコマンドを、`.claude/commands/` に作成します。
