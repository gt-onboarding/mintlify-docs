title: "非表示ページ"
description: "ナビゲーションからページを除外する"

---

非表示ページはサイトのナビゲーションからは表示されませんが、URLを知っていれば誰でも引き続きアクセスできます。

非表示ページは、サイト上でアクセス可能にしておきたい、またはAIツールのコンテキストとして参照させたいが、ナビゲーションからは見つからないようにしたいコンテンツに適しています。

厳密なアクセス制御が必要なコンテンツには、[認証](/ja/authentication-personalization/authentication-setup) を設定してください。

特定のユーザーグループにのみページを非表示にしたい場合は、パーソナライゼーションを使用して[ページの可視性](/ja/authentication-personalization/overview#page-visibility) を制御してください。

<div id="hiding-a-page">
  ## ページを非表示にする
</div>

`docs.json` のナビゲーションに含まれていないページは非表示になります。ページを非表示にするには、ナビゲーション構造からそのページを削除してください。

非表示のページも、ファイルパスに基づく通常のページと同じ URL 構造を使用します。たとえば、`guides/hidden-page.mdx` は `docs.yoursite.com/guides/hidden-page` でアクセスできます。

[非表示ページの例](/ja/guides/hidden-page-example)をご覧ください。

<Note>
  サイドバー、ドロップダウン、タブなどの一部のナビゲーション要素は、非表示ページでは空に表示されたり、レイアウトが崩れたりする場合があります。
</Note>

<div id="hiding-a-group-of-pages">
  ## ページグループを非表示にする
</div>

`docs.json` ファイルで `hidden` プロパティを `true` に設定すると、ページグループは非表示になります。

```json highlight={4}
"groups": [
  {
    "group": "Getting started",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "Guides",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

この例では、`Getting started` グループは非表示で、`Guides` グループは表示されます。

<div id="hiding-a-tab">
  ### タブを非表示にする
</div>

`docs.json` ファイルに `hidden` プロパティを追加すると、タブを非表示にできます：

```json highlight={4}
"tabs": [
  {
    "tab": "Home",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="search-seo-and-ai-indexing">
  ## 検索、SEO、AIによるインデックス
</div>

既定では、非表示ページは検索エンジン、ドキュメント内の内部検索、そしてアシスタントのコンテキスト用インデックスから除外されます。非表示ページを検索結果およびアシスタントのコンテキストに含めるには、`docs.json` に `seo` プロパティを追加します:

```json
"seo": {
    "indexing": "all"
}
```

特定のページを除外するには、そのフロントマターに `noindex: "true"` を追加します。
