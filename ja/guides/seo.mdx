<div id="content-basics">
  ## コンテンツの基本
</div>

検索エンジンが読み取りやすい文章と構成にしましょう。

* **見出しと小見出し:** 意味の通る階層見出しを順序どおりに用いてコンテンツを構造化します。各ページには、frontmatter の `title:` プロパティから生成される H1 が設定されています。
* **短い段落と箇条書き:** 大きなテキスト塊は読みやすいセクションに分割します。適切な場面では箇条書きや番号付きリストを使用します。
* **内部リンク:** 説明的なアンカーテキストで関連コンテンツへリンクします。たとえば、「Click here」ではなく「Learn more about rate limiting」のようにします。

<div id="technical-seo-basics">
  ## テクニカルSEOの基本
</div>

コンテンツを最適化したら、技術面でもドキュメントがしっかり機能するようにしましょう。

以下の基本的なテクニカルSEOの取り組みで、ドキュメントの発見性を高められます:

* **メタタグとディスクリプション:** 各ページにSEOを意識したタイトル（50〜60文字）とディスクリプション（150〜160文字）を作成します。ほとんどの[メタタグ](/ja/optimize/seo)は自動生成されます。
* **画像の代替テキスト（alt）:** 画像には関連キーワードを含む説明的な代替テキストを付けましょう。例えば、単なる「diagram」ではなく「OAuth 2.0 API authentication flow」のようにします。これはSEOとアクセシビリティの向上に役立ちます。
* **サイトマップ:** サイトマップが最新であることを確認します。Mintlifyはサイトマップを自動生成しますが、カスタム形式を使いたい場合は手動で作成することもできます。作成後、検索エンジンは時間をかけてサイトマップをインデックスしますが、Google Search Consoleに直接送信すると処理を早められます。

<div id="page-performance">
  ## ページパフォーマンス
</div>

[Google PageSpeed Insights](https://pagespeed.web.dev) などのツールを使って、テクニカルSEOの改善点を特定しましょう。

高度な最適化の例:

* **メディアの高速化:** 画像は WebP や AVIF などで圧縮し、ページの読み込みを高速化します（理想的には 3 秒未満）。
* **構造化データ（スキーママークアップ）:** ページにスキーママークアップ（HowTo、FAQ など）を追加し、検索エンジンによる内容の理解と評価を向上させます。

<div id="keyword-research">
  ## キーワードリサーチ
</div>

オーガニックトラフィックを増やすには、ユーザーがドキュメントにたどり着くためのキーワードを把握することに時間を割きましょう。

* **キーワードリサーチ:** [Google Keyword Planner](https://ads.google.com/intl/en_us/home/tools/keyword-planner/) や [Keywords Everywhere](https://keywordseverywhere.com) などの無料ツールで、一般的な語句やロングテールキーワードを特定します。
* **キーワードを自然に組み込む:** 見出し、小見出し、本文全体に無理なくキーワードを配置します。詰め込みは避けましょう。ドキュメントは検索エンジンではなくユーザーのために書くべきです。
