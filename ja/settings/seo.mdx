Mintlifyは、多くのSEOベストプラクティスを自動で対応します。

- メタタグの生成
- サイトマップと`robots.txt`ファイルの生成
- セマンティックなHTML構造
- モバイル最適化

`docs.json`またはページのフロントマターに`metatags`フィールドを追加すると、サイトのメタタグを自由にカスタマイズできます。

<div id="global-meta-tags">
  ## グローバルなメタタグ
</div>

`docs.json` に `metadata` フィールドを追加すると、デフォルトのメタタグを設定できます。

```json
"seo": {
    "metatags": {
        "og:image": "デフォルトのメタタグ画像へのリンク"
    }
}
```

<div id="page-specific-meta-tags">
  ## ページ固有のメタタグ
</div>

次のように、ページのメタデータにメタタグを追加します:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  コロンを含むメタタグは、ページのメタデータに記載する際は引用符で囲む必要があります。
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## サイトマップとrobots.txtファイル
</div>

Mintlifyは自動的に`sitemap.xml`ファイルと`robots.txt`ファイルを生成します。ドキュメントサイトのURLに`/sitemap.xml`を付けると、サイトマップを確認できます。

デフォルトでは、`docs.json`に含まれるページのみが対象です。非表示のリンクも含めるには、`docs.json`に`seo.indexing`を追加します:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### カスタムのsitemap.xmlおよびrobots.txtファイル
</div>

カスタムの`sitemap.xml`または`robots.txt`を追加するには、プロジェクトのルートに`sitemap.xml`または`robots.txt`を作成してください。カスタムファイルを追加すると、同名の自動生成ファイルが上書きされます。カスタムファイルを削除すると、デフォルトのファイルが再び使用されます。

<div id="disabling-indexing">
  ## インデックスの無効化
</div>

ページを検索エンジンにインデックスさせたくない場合は、ページの[frontmatter](ja/pages)に以下を追加してください:

```
---
noindex: true
---
```

また、`docs.json`内で`metadata.robots`フィールドを`"noindex"`に設定すると、ドキュメント内のすべてのページに対して`noindex`を指定できます:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEOのベストプラクティス
</div>

<AccordionGroup>
  <Accordion title="わかりやすいタイトルと説明文を書く">
    - 明確でわかりやすいページタイトルを使う（50〜60文字）
    - 魅力的な説明文を書く（150〜160文字）
    - 関連キーワードを含める
    - 各ページのタイトルと説明文をユニークにする
  </Accordion>

  <Accordion title="コンテンツ構造を最適化する">
    - 適切な見出し階層を使う（H1 → H2 → H3）
    - まずは読者のために、次に検索エンジンのために書く
    - 見出しと本文に関連キーワードを含める
    - URLは短く、わかりやすく、階層的に整理する
    - 長文は小見出しやリストで区切る
  </Accordion>

  <Accordion title="内部リンクの戦略">
    - ドキュメント内の関連ページへリンクする
    - 「click here」ではなく、内容を示すアンカーテキストを使う
    - 関連する概念を相互にリンクしてトピッククラスターを作る
    - 自動クロスリファレンス機能を活用する
  </Accordion>

  <Accordion title="画像のSEO">
    - 画像には説明的なファイル名を付ける
    - アクセシビリティとSEOのために必ずalt属性（代替テキスト）を設定する
    - 高速表示のために画像ファイルを最適化する
    - コンテンツを補強する適切な画像を使用する
  </Accordion>
</AccordionGroup>
