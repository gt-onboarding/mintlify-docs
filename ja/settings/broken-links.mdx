docs フォルダ内のファイルのパスを変更すると、そのページの URL パスも変更されます。これは、ドキュメント構成を見直したり、サイドバーのタイトルを変更したりする際に起こり得ます。

<div id="broken-links">
  ## リンク切れ
</div>

CLI でリンク切れを検出できます。[CLI をインストール](/ja/installation)して、次のコマンドを実行します:

```bash
mint broken-links
```

CLI は、ドキュメント内に存在しない相対リンクを検出します。

<div id="redirects">
  ## リダイレクト
</div>

`docs.json` ファイルに `redirects` フィールドを追加して、301 リダイレクトを設定します。

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

これにより、`/source/path` から `/destination/path` へ恒久的にリダイレクトされ、元のページの過去の SEO を失わずに済みます。

ワイルドカードのパスに一致させるには、パラメータの後に `*` を付けます。次の例では、`/beta/:slug*` は `/beta/introduction` に一致し、`/v2/introduction` にリダイレクトします。

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
