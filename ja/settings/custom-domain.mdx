ドキュメントをカスタムドメインで公開するには、Mintlify の設定で使用したいカスタムドメインを指定し、ドメインプロバイダー側で DNS を設定してください。

<Info>
  `mintlify.com/docs` のようなカスタムサブディレクトリを設定したい場合は、[カスタムサブディレクトリ](/ja/advanced/subpath/cloudflare)のドキュメントをご覧ください。
</Info>

<div id="dashboard-settings">
  ## ダッシュボードの設定
</div>

1. [ダッシュボード](https://dashboard.mintlify.com) に移動します
2. **Settings** を選択します
3. **Domain Setup** を選択します
4. 使用したいカスタムドメインを入力します（例: `docs.mintlify.com`）

<div id="configuring-your-dns">
  ## DNS の設定
</div>

1. ドメインプロバイダーのサイトで、対象ドメインの DNS 設定ページを開きます。
2. 次の値を用いて、新しい DNS レコードを作成します:

```bash
CNAME | docs | cname.vercel-dns.com.
```

<div id="provider-specific-settings">
  ## プロバイダー別の設定
</div>

<AccordionGroup>
  <Accordion title="Vercel の検証">
    ドメインプロバイダーが Vercel の場合、検証用の `TXT` レコードを追加する必要があります。カスタムドメインを送信すると、この情報はダッシュボードに表示されるほか、メールでも送信されます。
  </Accordion>

  <Accordion title="Cloudflare のセキュリティ設定">
    DNS プロバイダーが Cloudflare の場合、HTTPS 設定で「Full (strict)」のセキュリティオプションを有効にする必要があります。
  </Accordion>
</AccordionGroup>
