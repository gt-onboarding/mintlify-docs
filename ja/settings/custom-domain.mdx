カスタムドメインでドキュメントを公開するには、Mintlify の設定で使用したいカスタムドメインを指定し、ドメインプロバイダー側で DNS 設定を行ってください。

<Info>
  `mintlify.com/docs` のようなカスタムサブディレクトリを設定したい場合は、[カスタムサブディレクトリ](/ja/advanced/subpath/cloudflare)のドキュメントをご覧ください。
</Info>

<div id="dashboard-settings">
  ## ダッシュボード設定
</div>

1. [ダッシュボード](https://dashboard.mintlify.com) にアクセスします
2. **Settings** を選択します
3. **Domain Setup** を選択します
4. 使用したいカスタムドメインを入力します（例: `docs.mintlify.com`）

<div id="configuring-your-dns">
  ## DNS の設定
</div>

1. ドメインプロバイダーのサイトで、対象ドメインの DNS 設定ページを開きます。
2. 次の値で新しい DNS レコードを作成します：

```bash
CNAME | docs | cname.vercel-dns.com.
```

<div id="provider-specific-settings">
  ## プロバイダー別の設定
</div>

<AccordionGroup>
  <Accordion title="Vercel verification">
    ドメインプロバイダーが Vercel の場合、確認用の `TXT` レコードを追加する必要があります。カスタムドメインを提出すると、この情報がダッシュボードに表示され、メールでも送信されます。
  </Accordion>

  <Accordion title="Cloudflare security settings">
    DNS プロバイダーが Cloudflare の場合、https の設定で「Full (strict)」のセキュリティオプションを有効にする必要があります。
  </Accordion>
</AccordionGroup>
