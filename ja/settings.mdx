`docs.json` ファイルを使うと、Markdown ファイルの集合を、ナビゲーションしやすくカスタマイズ可能なドキュメントサイトに変換できます。この必須の構成ファイルは、スタイリング、ナビゲーション、各種連携などを制御します。ドキュメントの設計図と捉えてください。

`docs.json` の設定は、すべてのページに対してグローバルに適用されます。

<div id="setting-up-your-docsjson">
  ## `docs.json` のセットアップ
</div>

はじめるには、`theme`、`name`、`colors.primary`、`navigation` を指定するだけで十分です。その他のフィールドは任意で、ドキュメントのニーズに応じて追加できます。

最適な編集体験のために、`docs.json` ファイルの先頭にスキーマリファレンスを含めてください。これにより、多くのコードエディターでオートコンプリート、検証、役立つツールチップが有効になります。

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // ナビゲーション構造
  }
  // 残りの設定
}
```

<div id="customization">
  ### カスタマイズ
</div>

<ResponseField name="theme" required>
  サイトのレイアウトテーマ。

  次のいずれか：`mint`、`maple`、`palm`、`willow`、`linden`、`almond`、`aspen`。

  詳細は[テーマ](ja/themes)を参照してください。
</ResponseField>

<ResponseField name="name" type="string" required>
  プロジェクト、組織、または製品の名称。
</ResponseField>

<ResponseField name="colors" type="object" required>
  ドキュメントで使用する配色。適用方法はテーマによって異なります。`primary` のみを指定した場合は、すべてのカラー要素に適用されます。

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      ドキュメントのプライマリカラー。一般的にライトモードで強調に使用されます（テーマによって一部異なります）。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ダークモードでの強調に使用されるカラー。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ライト/ダーク両モードで、ボタンやホバー状態に使用されるカラー（テーマによって一部異なります）。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  SEO と AI インデックス作成のためのサイトの説明。
</ResponseField>

<ResponseField name="logo" type="string or object">
  ライトモードおよびダークモード用のロゴ。

  <Expandable title="Logo">
    <ResponseField name="light" type="string" required>
      ライトモード用ロゴファイルへのパス。拡張子を含めてください。例：`/logo.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用ロゴファイルへのパス。拡張子を含めてください。例：`/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      ロゴをクリックした際に遷移するURL。未指定の場合、ロゴはホームページにリンクします。例：`https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string or object">
  ファビコンファイルへのパス（拡張子を含む）。適切なサイズに自動リサイズされます。
  単一ファイル、またはライト/ダークモードで別々のファイルを指定できます。例：`/favicon.png`

  <Expandable title="Favicon">
    <ResponseField name="light" type="string" required>
      ライトモード用のファビコンファイルへのパス。拡張子を含めてください。例：`/favicon.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用のファビコンファイルへのパス。拡張子を含めてください。例：`/favicon-dark.png`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="thumbnails" type="object">
  ソーシャルメディアやページプレビュー用のサムネイルのカスタマイズ。

  <Expandable title="Thumbnails">
    <ResponseField name="appearance" type="&#x22;light&#x22; | &#x22;dark&#x22;">
      サムネイルのビジュアルテーマ。未指定の場合、`colors` フィールドで定義されたサイトの配色が使用されます。
    </ResponseField>

    <ResponseField name="background" type="string">
      サムネイルの背景画像。相対パスまたは絶対URLを使用できます。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="styling" type="object">
  ビジュアルスタイルの設定。

  <Expandable title="Styling">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      ページのアイブローのスタイル。セクション名を表示するには `section`、完全なナビゲーションパスを表示するには `breadcrumbs` を選択します。デフォルトは `section` です。
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22;">
      コードブロックのテーマ。サイトのテーマに合わせるには `system`、常にダークのコードブロックにするには `dark` を選択します。デフォルトは `system` です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="icons" type="object">
  アイコンライブラリの設定。

  <Expandable title="Icons">
    <ResponseField name="library" type="&#x22;fontawesome&#x22; | &#x22;lucide&#x22;" required>
      ドキュメント全体で使用するアイコンライブラリ。既定は `fontawesome` です。

      <Note>
        ライブラリ設定に関係なく、各アイコンごとに外部ホストのアイコンURL、プロジェクト内のアイコンファイルへのパス、または JSX 互換の SVG コードを指定できます。
      </Note>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fonts" type="object">
  ドキュメントのフォント設定。既定のフォントは [Inter](https://fonts.google.com/specimen/Inter) です。

  <Expandable title="Fonts">
    <ResponseField name="family" type="string" required>
      「Open Sans」「Playfair Display」などのフォントファミリー。
    </ResponseField>

    <ResponseField name="weight" type="number">
      400 や 700 などのフォントウェイト。可変フォントは 550 のような細かなウェイトもサポートします。
    </ResponseField>

    <ResponseField name="source" type="string (uri)">
      フォントソースのURL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動的に読み込まれるため、ソースURLは不要です。
    </ResponseField>

    <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
      フォントファイル形式。
    </ResponseField>

    <ResponseField name="heading" type="object">
      見出し用のフォント設定を個別に上書きします。

      <Expandable title="Heading">
        <ResponseField name="family" type="string" required>
          「Open Sans」「Playfair Display」などのフォントファミリー。
        </ResponseField>

        <ResponseField name="weight" type="number">
          400、700 などのフォントウェイト。可変フォントは 550 のような細かなウェイトもサポートします。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          フォントソースのURL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動的に読み込まれるため、ソースURLは不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイル形式。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="body" type="object">
      本文用のフォント設定を個別に上書きします。

      <Expandable title="Body">
        <ResponseField name="family" type="string" required>
          「Open Sans」「Playfair Display」などのフォントファミリー。
        </ResponseField>

        <ResponseField name="weight" type="number">
          400、700 などのフォントウェイト。可変フォントは 550 のような細かなウェイトもサポートします。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          フォントソースのURL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動的に読み込まれるため、ソースURLは不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイル形式。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="appearance" type="object">
  ライト/ダークモードの切り替え設定。

  <Expandable title="Appearance">
    <ResponseField name="default" type="&#x22;system&#x22; | &#x22;light&#x22; | &#x22;dark&#x22;">
      既定のテーマモード。`system` を選ぶとユーザーのOS設定に合わせ、`light` または `dark` を選ぶと特定のモードを強制します。既定は `system` です。
    </ResponseField>

    <ResponseField name="strict" type="boolean">
      ライト/ダークモードの切り替えトグルを非表示にするかどうか。既定は `true` です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="background" type="object">
  背景色と装飾の設定。

  <Expandable title="Background">
    <ResponseField name="image" type="string or object">
      サイトの背景画像。単一ファイル、またはライトモードとダークモードで別々のファイルを指定できます。

      <Expandable title="Image">
        <ResponseField name="light" type="string" required>
          ライトモード用の背景画像のパス。拡張子を含めてください。例: `/background.png`。
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモード用の背景画像のパス。拡張子を含めてください。例: `/background-dark.png`。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="decoration" type="&#x22;gradient&#x22; | &#x22;grid&#x22; | &#x22;windows&#x22;">
      テーマの背景装飾。
    </ResponseField>

    <ResponseField name="color" type="object">
      ライトモードとダークモード用のカスタム背景色。

      <Expandable title="Color">
        <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ライトモードの背景色。

          `#` で始まる 16 進数コードである必要があります。
        </ResponseField>

        <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ダークモードの背景色。

          `#` で始まる 16 進数コードである必要があります。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## `mint.json` からのアップグレード
</div>

ドキュメントプロジェクトで非推奨の `mint.json` を使用している場合は、以下の手順に従って `docs.json` へアップグレードしてください。

<Steps>
  <Step title="CLI をインストールまたは更新する">
    まだ [CLI](/ja/installation) をインストールしていない場合は、インストールしてください：

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    すでに CLI をインストールしている場合は、最新版になっているか確認してください：

    ```bash
    mint update
    ```
  </Step>

  <Step title="docs.json ファイルを作成する">
    ドキュメントのリポジトリで次を実行します：

    ```bash
    mint upgrade
    ```

    このコマンドは既存の `mint.json` から `docs.json` を作成します。生成されたファイルを見直し、すべての設定が正しいか確認してください。
  </Step>

  <Step title="mint.json ファイルを削除する">
    `docs.json` の設定が正しいことを確認したら、古い `mint.json` ファイルを安全に削除できます。
  </Step>
</Steps>
