`docs.json` ファイルを使うと、Markdown ファイルの集まりを、ナビゲーション可能でカスタマイズされたドキュメントサイトに変換できます。この必須の設定ファイルは、スタイル、ナビゲーション、インテグレーションなどを制御します。ドキュメントの設計図だと考えてください。

`docs.json` の設定は、すべてのページに一括で適用されます。

<div id="setting-up-your-docsjson">
  ## `docs.json` のセットアップ
</div>

はじめるには、`theme`、`name`、`colors.primary`、そして `navigation` を指定するだけで十分です。その他のフィールドは任意で、ドキュメントの要件が増えるにつれて追加できます。

最適な編集体験のために、`docs.json` ファイルの先頭にスキーマ参照を含めてください。これにより、ほとんどのコードエディターでオートコンプリート、検証、役立つツールチップが有効になります。

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // ナビゲーション構造
  }
  // その他の設定
}
```

<div id="customization">
  ### カスタマイズ
</div>

<ResponseField name="theme" required>
  サイトのレイアウトテーマ。

  次のいずれか: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  詳細は [Themes](/ja/customize/themes) を参照してください。
</ResponseField>

<ResponseField name="name" type="string" required>
  プロジェクト、組織、または製品の名称。
</ResponseField>

<ResponseField name="colors" type="object" required>
  ドキュメントで使用する色。テーマによって適用方法が異なります。`primary` のみを指定した場合、すべてのカラー要素に適用されます。

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      ドキュメントのプライマリカラー。一般的にライトモードでの強調に使用され、テーマによって多少の違いがあります。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ダークモードでの強調に使用される色。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ライト/ダーク両モードで、ボタンやホバー状態に使用される色。テーマによって多少の違いがあります。

      `#` で始まる16進数コードである必要があります。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  SEO と AI インデックス用のサイトの説明。
</ResponseField>

<ResponseField name="logo" type="string or object">
  ライトモードとダークモードの両方で使用するロゴを設定します。

  <Expandable title="Logo">
    <ResponseField name="light" type="string" required>
      ライトモード用のロゴファイルへのパス。拡張子を含めてください。例: `/logo.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用のロゴファイルへのパス。拡張子を含めてください。例: `/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      ロゴをクリックしたときに遷移する URL。未指定の場合、ロゴはホームページにリンクします。例: `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string or object">
  拡張子を含む favicon ファイルへのパス。適切な favicon サイズに自動リサイズされます。単一ファイル、またはライト/ダークモードで別々のファイルを指定できます。例: `/favicon.png`

  <Expandable title="Favicon">
    <ResponseField name="light" type="string" required>
      ライトモード用の favicon ファイルへのパス。拡張子を含めてください。例: `/favicon.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用の favicon ファイルへのパス。拡張子を含めてください。例: `/favicon-dark.png`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="thumbnails" type="object">
  ソーシャルメディアおよびページプレビュー向けのサムネイルのカスタマイズ。

  <Expandable title="Thumbnails">
    <ResponseField name="appearance" type="&#x22;light&#x22; | &#x22;dark&#x22;">
      サムネイルのビジュアルテーマ。未指定の場合、`colors` フィールドで定義されたサイトの配色が使用されます。
    </ResponseField>

    <ResponseField name="background" type="string">
      サムネイルの背景画像。相対パスまたは絶対 URL を指定できます。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="styling" type="object">
  ビジュアルスタイルの設定。

  <Expandable title="Styling">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      ページのアイブロー（上部の補助見出し）の表示スタイル。セクション名を表示するには `section`、完全なナビゲーションパスを表示するには `breadcrumbs` を選びます。デフォルトは `section` です。
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      コードブロックのテーマ設定。デフォルトは `"system"`。

      **簡易設定:**

      * `"system"`: 現在のサイトモード（ライト/ダーク）に合わせる
      * `"dark"`: 常にダークモードを使用

      **カスタムテーマ設定:**

      * 文字列を使って、すべてのコードブロックに単一の [Shiki テーマ](https://shiki.style/themes) を指定
      * オブジェクトを使って、ライト/ダークで別々の [Shiki テーマ](https://shiki.style/themes) を指定

      <ResponseField name="theme" type="string">
        ライト/ダーク両方で使う単一の Shiki テーマ名。

        ```json
        "styling": {
          "codeblocks": {
            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

      <ResponseField name="theme" type="object">
        ライトモードとダークモードで異なるテーマを指定します。

        <Expandable title="theme">
          <ResponseField name="light" type="string" required>
            ライトモード用の Shiki テーマ名。
          </ResponseField>

          <ResponseField name="dark" type="string" required>
            ダークモード用の Shiki テーマ名。
          </ResponseField>

          ```json
          "styling": {
            "codeblocks": {
              "theme": {
                "light": "github-light",
                "dark": "github-dark"
              }
            }
          }
          ```
        </Expandable>
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="icons" type="object">
  アイコンライブラリの設定。

  <Expandable title="Icons">
    <ResponseField name="library" type="&#x22;fontawesome&#x22; | &#x22;lucide&#x22;" required>
      ドキュメント全体で使用するアイコンライブラリ。デフォルトは `fontawesome`。

      <Note>
        ライブラリ設定に関わらず、個々のアイコンについては、外部ホスティングされたアイコンの URL、プロジェクト内のアイコンファイルへのパス、または JSX 互換の SVG コードを指定できます。
      </Note>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fonts" type="object">
  ドキュメント用にカスタムフォントを設定します。デフォルトのフォントはテーマによって異なります。

  <Expandable title="Fonts">
    <ResponseField name="family" type="string" required>
      フォントファミリー。例: &quot;Open Sans&quot;。[Google Fonts](https://fonts.google.com) のファミリー名に対応しています。
    </ResponseField>

    <ResponseField name="weight" type="number">
      ウェイト。例: 400 や 700。可変フォントでは 550 のような細かなウェイトも指定できます。
    </ResponseField>

    <ResponseField name="source" type="string (uri)">
      次のいずれか:

      * ホストされているフォントへの URL。例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2
      * ローカルフォントファイルへのパス。例: `/fonts/Hubot-Sans.woff2`

      [Google Fonts](https://fonts.google.com) は Google Font の `family` 名を指定すると自動的に読み込まれるため、source の URL は不要です。
    </ResponseField>

    <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
      フォントファイルの形式。`source` フィールドを使用する場合は必須です。
    </ResponseField>

    <ResponseField name="heading" type="object">
      見出し用のフォント設定を個別に上書きします。

      <Expandable title="Heading">
        <ResponseField name="family" type="string" required>
          フォントファミリー。例: &quot;Open Sans&quot;, &quot;Playfair Display&quot;。[Google Fonts](https://fonts.google.com) のファミリー名に対応しています。
        </ResponseField>

        <ResponseField name="weight" type="number">
          ウェイト。例: 400, 700。可変フォントでは 550 のような細かなウェイトも指定できます。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          次のいずれか:

          * ホストされているフォントへの URL。例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2
          * ローカルフォントファイルへのパス。例: `/fonts/Hubot-Sans.woff2`

          [Google Fonts](https://fonts.google.com) は Google Font の `family` 名を指定すると自動的に読み込まれるため、source の URL は不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイルの形式。`source` フィールドを使用する場合は必須です。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="body" type="object">
      本文用のフォント設定を個別に上書きします。

      <Expandable title="Body">
        <ResponseField name="family" type="string" required>
          フォントファミリー。例: &quot;Open Sans&quot;, &quot;Playfair Display&quot;。[Google Fonts](https://fonts.google.com) のファミリー名に対応しています。
        </ResponseField>

        <ResponseField name="weight" type="number">
          ウェイト。例: 400, 700。可変フォントでは 550 のような細かなウェイトも指定できます。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          次のいずれか:

          * ホストされているフォントへの URL。例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2
          * ローカルフォントファイルへのパス。例: `/fonts/Hubot-Sans.woff2`

          [Google Fonts](https://fonts.google.com) は Google Font の `family` 名を指定すると自動的に読み込まれるため、source の URL は不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイルの形式。`source` フィールドを使用する場合は必須です。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="appearance" type="object">
  ライト/ダークモードの切り替え設定。

  <Expandable title="Appearance">
    <ResponseField name="default" type="&#x22;system&#x22; | &#x22;light&#x22; | &#x22;dark&#x22;">
      デフォルトのテーマモード。`system` を選ぶとユーザーの OS 設定に合わせ、`light` または `dark` を選ぶと特定のモードを強制します。デフォルトは `system` です。
    </ResponseField>

    <ResponseField name="strict" type="boolean">
      ライト/ダークモードの切り替えトグルを非表示にするかどうか。デフォルトは `true`。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="background" type="object">
  背景色と装飾の設定。

  <Expandable title="Background">
    <ResponseField name="image" type="string or object">
      サイトの背景画像。ライトモードとダークモードで、単一ファイルまたは別々のファイルを指定できます。

      <Expandable title="Image">
        <ResponseField name="light" type="string" required>
          ライトモード用の背景画像のパス。拡張子を含めてください。例: `/background.png`
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモード用の背景画像のパス。拡張子を含めてください。例: `/background-dark.png`
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="decoration" type="&#x22;gradient&#x22; | &#x22;grid&#x22; | &#x22;windows&#x22;">
      テーマの背景装飾。
    </ResponseField>

    <ResponseField name="color" type="object">
      ライトモードとダークモード用のカスタム背景色。

      <Expandable title="Color">
        <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ライトモードの背景色。

          `#` で始まる16進数のコードである必要があります。
        </ResponseField>

        <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ダークモードの背景色。

          `#` で始まる16進数のコードである必要があります。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## `mint.json` からのアップグレード
</div>

ドキュメントプロジェクトで非推奨の `mint.json` を使用している場合は、以下の手順に従って `docs.json` にアップグレードしてください。

<Steps>
  <Step title="CLI をインストールまたは更新する">
    まだ [CLI](/ja/installation) をインストールしていない場合は、次を実行してインストールしてください:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    すでに CLI をインストールしている場合は、次を実行して最新状態であることを確認してください:

    ```bash
    mint update
    ```
  </Step>

  <Step title="docs.json ファイルを作成する">
    ドキュメントのリポジトリで次を実行します:

    ```bash
    mint upgrade
    ```

    このコマンドは既存の `mint.json` から `docs.json` を作成します。生成されたファイルを確認し、すべての設定が正しいことを確認してください。
  </Step>

  <Step title="mint.json ファイルを削除する">
    `docs.json` の設定が正しいことを確認したら、古い `mint.json` は安全に削除できます。
  </Step>
</Steps>
