<Update label="October 17" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "アシスタントの改善、ナビゲーションの強化、APIプレイグラウンドの更新、バグ修正" }}>
  ## アシスタントとAIの改善

  * 精度向上と文脈に応じた応答のためにアシスタントのプロンプトをアップグレード
  * アシスタントの応答における判断を改善するため、信頼度スコアリングを追加
  * 高速化のため、ドキュメントを並列クエリするアシスタント検索を強化
  * アシスタントが消える不具合を修正し、小さなページでの視認性を改善

  ## ナビゲーションとUIの強化

  * 一貫したレイアウトのため、ナビゲーションコンポーネントに安定したスクロールバーガターを追加
  * ページ上部へ素早くスクロールできるよう &quot;On this page&quot; セクションをクリック可能に
  * すべてのテーマでアシスタントバーの位置を修正
  * コンテンツの重なりを防ぐようアシスタントバーを改善
  * Almond テーマのワイドページの最大幅を修正
  * ナビゲーショングループの見た目を統一するため、ドロップダウンの矢印アイコンのサイズを調整

  ## APIプレイグラウンドとOpenAPIの改善

  * 新しいAPIリファレンスページデザイン用のフィーチャーフラグを追加
  * コンポジションを正しく扱えるよう OpenAPI スキーマグラフユーティリティを改善
  * より良いAPIドキュメントのため、オペレーションデータにエンドポイントタイプを追加
  * Markdown 内に enum とテーブルがある場合の APIプレイグラウンドの説明レンダリングを修正
  * より良いレイアウトのため、APIプレイグラウンドで説明のレンダリング位置を調整

  ## バグ修正と信頼性

  * ロケール切り替え時にホームページへリダイレクトせず同じページに留まるよう修正
  * 壊れたリンク検出 CLI が CI/CD 連携のためコード 1 で終了するよう修正
  * ローカルフォントパスの先頭にスラッシュが正しく付与されるよう修正
  * `docs.json` のリダイレクト処理で先頭のスラッシュを無視するよう修正
  * より良い SEO のため `/index` をカノニカル URL から除去する処理を修正
  * アコーディオン内の番号付きリストの位置を修正
  * `<span>` タグのスタイルを `<p>` タグのように振る舞うよう修正
  * リスト内の最初の `<p>` タグの display: block を修正
  * インデックスページで区分名が表示されるよう OG 画像の表示を修正
  * 相対パスで `BASE_PATH` を含めるようアイコンパスを修正
  * パフォーマンス向上のためアシスタントの背景ぼかし削除を改善
</Update>

<Update label="September 13" tags={["New releases", "Improvements"]} rss={{ title: "Weekly Updates", description: "AI 404 suggestions, assistant web search with external sources, security enhancement, and reliability fixes" }}>
  ## 主要リリース

  * **大幅な強化**: 404ページでのAIによるページ提案。[無効なリンクにアクセス → AIエージェントがパスを解析 → 意味的に近いページを提案](https://x.com/mintlify/status/1966625627773059495)
  * **メジャーリリース**: アシスタントのウェブ検索で外部ソースを含められるようになりました\

  * スライディングウィンドウが正しく機能せず、アシスタントが誤ってレート制限される不具合を修正
  * 空の`text`ブロックを正しく処理できるよう、アシスタントのツール呼び出しの不具合を修正
  * MCPサーバー名とツール呼び出しの連結により、MCPクライアントが課す60文字制限を超えることがある不具合を修正
  * アシスタントメニューの高さがビューポートの数倍になり、延々とスクロールしてしまう不具合を修正
  * ダッシュボードでアシスタントの支出額が小数点以下2桁を超えて表示されることがある不具合を修正

  ## ウェブエディタとデプロイ

  * 接続されたGitホスティングリポジトリで`write permissions`を持つユーザーのみが変更できるよう、エディタのセキュリティを強化
  * 名前に`=`を含むブランチでプレビューデプロイが動作しない不具合を修正
  * プレビューデプロイ作成時に長いブランチ名がモーダルからはみ出す不具合を修正
  * サインアップ招待で、メールのクエリパラメータが入力欄に自動入力される利便性向上
  * コンテキストメニューからページをコピーできないSafariでの不具合を修正

  ## APIプレイグラウンドとナビゲーション

  * [`docs.jsonのナビゲーショングループでexpandedフィールドを使用してデフォルトで展開状態にできるようになりました`](https://mintlify.com/docs/navigation#default-expanded-state)

  ## SEOとUI

  * 各サイトのドキュメントサイト本体と同じURLから配信することで、検索エンジンにfaviconが表示されない不具合を修正
  * YouTubeの埋め込みが読み込み時に点滅する不具合を修正
  * フィードバックメニューを展開して記述式回答を含めた際に、目次でレイアウトシフトが発生する不具合を修正
  * 通知バナーを閉じた際に、Mapleテーマでテキストがトップバー上部にはみ出す不具合を修正
  * MapleおよびWillowテーマのサイドバーにログイン/ログアウトボタンを追加してアクセス性を向上

  * アシスタントのアナリティクスエクスポートはバックグラウンドで実行され、より信頼性の高い体験のためメールで送信されるようになりました
</Update>

  ## バグ修正と信頼性

  * `x-robots-tag noindex` ヘッダーが欠けていたために Google が生の `*.md` ファイルをインデックスしていた不具合を修正
  * 保護されたドキュメントでの OAuth において、フロー完了後に開始ページへリダイレクトされないことがあった不具合を修正
  * 認証保護されたドキュメントのプレビューで、ナビゲーションバー全体が表示されない不具合を修正
  * 新しい画像 CDN における SVG の取り扱いに関する不具合を修正

  ## コンポーネントとスタイリングの強化

  * `SidebarNavGroupDivider` に対するカスタムスタイル用の新しい CSS セレクタを追加
  * セキュリティが定義された MDX の API ページ向けの新しい回帰テストにより、安定性をさらに向上

  ## パフォーマンス改善

  * レイテンシ削減のため、KaTeX の CSS を cdnjs から自社の CloudFront 上の CDN に移行し、パフォーマンスを改善
</Update>

  * MDX のトランスパイルを、未キャッシュの NextJS サーバーレス環境での各ページ読み込み時ではなくデプロイ時に実行するよう変更し、未キャッシュページの初回バイト到達時間を改善しました。
  * コンテンツベースのハッシュにより、MDX に変更がない場合の再トランスパイルを防止し、大量ページを持つお客様の更新ワークフロー時間を約 50% 短縮します（5 分超のデプロイはおおよそ半減）
  * ダッシュボードでのプレビュー用デプロイの表示が、バックエンドでのデータベースインデックス追加とクエリ並列化により高速化
  * 各ページの `rsc` ペイロード内に重複していた `navigation` データを排除してページサイズを削減—ページ数が多い場合や複雑なナビゲーション構造で特に効果が顕著
  * より積極的なプリフェッチにより、即時にページが読み込まれる頻度が向上

  ## APIプレイグラウンドと OpenAPI の強化

  * OpenAPI から MCP への変換をバックエンドに移し、ホストされた MCP サーバーにツールを含められるようにしました（より明確なドキュメントと設定オプションを近日公開予定）
  * APIプレイグラウンドに Ruby サポートを追加
  * 新機能として、[docs.json の指定だけで API ページを自動生成](/ja/api-playground/openapi-setup#auto-populate-api-pages)できるようになりました。新しい MDX ファイルの作成は不要です。
  * OpenAPI スペックからのドキュメントナビゲーションで [`webhook pages`](/ja/api-playground/openapi-setup#webhooks) をサポート
  * Anthropic、OpenAI、その他のプロバイダへ遷移する際、Markdown リンクからアンカー指定を除去して AI モデルのコンテキストを最適化

  ## ウェブエディタの改善

  * ファイルの作成/リネーム時、Enter キーを押さなくてもクリックでフォーカスが外れたタイミングで変更が保存されるように
  * URL を特定のブランチに変更すると、意図した場所ではなく最後にアクティブだったブランチにリダイレクトされていたブランチナビゲーションを修正
  * `/` を含むブランチタイトルを正しく URL エンコードしてナビゲーションの不具合を防止
  * モノレポ用ダッシュボードのエディタにおける `Ctrl+K` リンクショートカットが docs リポジトリパスを先頭に付けて壊れたリンクを作成していた問題を修正

  ## アナリティクスと LLM 統合

  * カスタム `llms.txt` および `llms-full.txt` をサポート—ドキュメントリポジトリのルートに追加すると、LLM カスタマイズ用に `/llms.txt` と `/llms-full.txt` エンドポイントで配信
  * [Hightouch のアナリティクス統合](/ja/integrations/analytics/hightouch#hightouch)を追加
  * コンテキストメニューのアナリティクス計測を強化（ダッシュボードでの可視化は近日対応）
  * 正しい配信を保証するため `llms.txt` と `llms-full.txt` の e2e テストを追加

  ## コンポーネントとスタイリングの強化

  * 見出しのカスタムスタイル適用のため、`h{1-4}` タグでカスタムクラス名をサポート
  * カスタムページモードで `h{1-4}` タグがチップ付きの `Heading` コンポーネントとしてレンダリングされる問題を修正
  * カスタム CSS のターゲティング用に [breadcrumbs](/ja/organize/navigation#breadcrumbs) に CSS セレクタを追加
  * 高さ 56px で比率を維持できるよう寸法を解析し、引き伸ばされていた Open Graph 画像を修正
  * コンテキストメニュー有効時の表記を `VSCode` から `VS Code` に修正
  * カスタムコンポーネント内の見出しがセマンティックな見出しとともに目次に表示されていた問題を修正

  * 空の OpenAPI JSON ファイルに遭遇した際の CLI エラー `Cannot convert undefined or null to object` を解消

  * ソースマップを削除して CLI のダウンロード速度を改善

  * 新機能およびエッジケースに対する包括的なテストカバレッジ

  <Frame>

  1. CLI を最新バージョンに更新する

  ```bash
  npm i mint@latest -g
  ```

  1. ドキュメントのリポジトリで次を実行

  ```bash
  mint upgrade
  ```

  1. 旧 `mint.json` ファイルを削除し、変更をプッシュする
