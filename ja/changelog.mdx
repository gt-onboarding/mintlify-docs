<Update label="8月24日 - 8月30日" tags={["Improvements"]} rss={{ title: "週間アップデート", description: "フィードバック収集の強化、ナビゲーションの改善、認証のバグ修正" }}>
  ## 主要リリース: フィードバック収集の強化

  - **大幅な改善**: 読者&#x306F;_&#x30B5;ムズアップ/ダウ&#x30F3;_&#x306E;選択後に、選択式や自由記述を含む、より詳細なフィードバックを送れるようになりました。コードブロックへのフィードバック収集にも対応し、すべての回答はダッシュボードのアナリティクスで確認できます。\
    _注: この機能の有効化をご希望の場合はお問い合わせください。_

  ## ナビゲーションと利便性の向上

  - 末尾スラッシュや重複スラッシュを無視するようにし、docs.jsonで厳密に揃える必要がなくなりました
  - `h1-6` のHTMLタグに `noAnchor` 属性を追加できるようになり、アンカーリンクの付与を避けられます
  - Palmテーマに、Stripeに近い左下のフローティング言語セレクターを追加
  - docs.json に新フィールド [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups) を追加。グループ展開時にナビゲーショングループの最初のページへ自動遷移するかどうかを制御できます
  - 入れ子の番号付きリストが decimal/Roman と alpha で交互にスタイル切り替えされるように改善

  ## バグ修正と信頼性

  - ページにJSコンポーネントがある場合にスクロール位置アンカーリンクが正しく動作しない不具合を修正
  - `x-robots-tag noindex` ヘッダーが欠落していたために Google が生の `*.md` ファイルをインデックスしてしまう不具合を修正
  - 保護されたドキュメントでのOAuthにおいて、フロー完了後に開始ページへリダイレクトされない不具合を修正
  - 認証保護されたドキュメントのプレビューで、ナビゲーションバー全体が表示されない不具合を修正
  - 新しい画像CDNにおけるSVGの取り扱いに関する不具合を修正

  ## コンポーネントとスタイリングの強化

  - `SidebarNavGroupDivider` 向けにカスタムスタイル用の新しいCSSセレクターを追加
  - セキュリティが定義されたMDXベースのAPIページに対する新しい回帰テストにより、安定性を向上

  ## パフォーマンス改善

  - KaTeXのCSSをcdnjsからCloudFront上の自社CDNへ移行し、レイテンシ削減によるパフォーマンスを改善
</Update>

<Update label="8月17日 - 8月23日" tags={["Improvements"]} rss={{ title: "週間アップデート", description: "画像レイアウトの修正、認証の強化、モバイルナビの改善、Webエディタの安定性向上" }}>
  ## 画像処理の改善

  - **大幅な改善**: 幅・高さ属性が未指定でも、デフォルトで画像によるレイアウトシフトが発生しなくなりました—自動サイズ調整によりページ読み込み中のコンテンツジャンプを防止します
  - リポジトリ内のすべての静的ファイル（PDF、TXT、XMLなど）が、デプロイ時に自動でアップロード・配信されるようになり、アセットを完全にカバーします

  ## Webエディタとデプロイの強化

  - Webエディタのブランチ作成フローを修正し、新規作成ブランチへ正しく遷移し、そのまま滞在するようにしました
  - マージコンフリクトのダイアログを強化し、適切なエスケープ機能を追加。コンフリクトの解消にページ再読み込みは不要になりました
  - 部分更新時に変更ページのみキャッシュ無効化することで更新ワークフローを最適化し、デプロイ時間を短縮

  ## 認証とナビゲーションの改善

  - カスタムサブディレクトリでの認証を新たにサポート。`https://yourdomain.com/docs` でドキュメントを配信している場合でも、認証がシームレスに機能します
  - サイドバーでリンクが1つだけ設定されている場合に誤表示される不具合を修正
  - モバイルナビゲーションを全面的に改善: ボタンの中央揃えと適切なマージン/パディング、ドロップダウンメニューの間隔を改善、空セクションの不要な区切り線とマージンを削除、Mapleテーマのギャップ/パディング問題を修正

  ## コンポーネントとスタイリングの修正

  - `<h1-6>` タグが誤ってHeadingコンポーネントに変換され、カスタムスタイリングが崩れる問題を解消
  - ダッシュボードにワンクリックのアシスタント設定トグルを追加し、管理を容易化

  ## 技術的な改善と信頼性

  - 更新ワークフローのログシステムを強化し、より迅速なデバッグと問題解決を可能に
  - 個別ファイル取得からリポジトリのクローンに切り替えることで、10件以上のOpenAPI/AsyncAPI仕様を持つお客様で発生していたGitHubのレート制限を解消
  - バックアップLLMのサポート、レート制限エラー処理の強化、より堅牢な検索ツール機能により、アシスタントの信頼性を向上
</Update>

<Update label="August 10 - August 16" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "パフォーマンス最適化、LLM 連携、Web エディタの磨き込み、API プレイグラウンドの強化" }}>
  ## パフォーマンスとビルドの最適化

  - MDX のトランスパイルは、キャッシュされていない NextJS サーバーレス環境での各ページ読み込み時ではなくデプロイ時に実行されるようになり、キャッシュされていないページの最初のバイトまでの時間が改善されました。
  - コンテンツベースのハッシュにより、MDX に変更がない場合の再トランスパイルを防止し、大量ページを持つお客様の更新ワークフロー時間を約 50% 短縮します（5 分超のデプロイはおおよそ半減）
  - ダッシュボードでのプレビューデプロイの閲覧が、バックエンドでのデータベースインデックス追加とクエリの並列化により高速化
  - 各ページの `rsc` ペイロード内の重複した `navigation` データを排除してページサイズを削減—ページ数が多い場合やナビゲーションが複雑な場合にパフォーマンス向上が顕著
  - より積極的なプリフェッチにより、即時のページ読み込みが発生しやすくなりました

  ## APIプレイグラウンドと OpenAPI の強化

  - OpenAPI から MCP への変換をバックエンドに移行し、ホストされた MCP サーバーがツールを含められるように（まもなく、より明確なドキュメントと設定オプションを提供予定）
  - APIプレイグラウンドに Ruby サポートを追加
  - 新機能として、[docs.json だけで API ページを指定](/ja/api-playground/openapi-setup#auto-populate-api-pages)できるようになり、新しい mdx ファイルを作成する必要がなくなりました。
  - OpenAPI 仕様からのドキュメントナビゲーションにおける[`webhook` ページ](/ja/api-playground/openapi-setup#webhooks)をサポート
  - Anthropic、OpenAI、その他のプロバイダへ遷移する際、Markdown リンクからアンカー指定を除去して AI モデルのコンテキストを最適化

  ## Web エディタの改善

  - ファイルの作成/名称変更で、Enter キーを押さずにクリックでフォーカスが外れたタイミングで変更を保存
  - 特定のブランチへの URL 変更が、意図した遷移先ではなく最後にアクティブだったブランチへリダイレクトされる問題を修正
  - ナビゲーション破損を防ぐため、`/` を含むブランチタイトルを適切に URL エンコード
  - monorepo ダッシュボードの Editor における `Ctrl+K` リンクショートカットがドキュメントリポジトリのパスを前置して壊れたリンクを作成していた問題を修正

  ## アナリティクスと LLM 連携

  - カスタム `llms.txt` および `llms-full.txt` をサポート—ドキュメントリポジトリのルートに追加すると、LLM カスタマイズ用に `/llms.txt` と `/llms-full.txt` エンドポイントで提供
  - [Hightouch アナリティクス連携](/ja/integrations/analytics/hightouch#hightouch)を追加
  - コンテキストメニューのアナリティクストラッキングを強化（ダッシュボードでの表示は近日公開）
  - 正しい配信を保証するため、`llms.txt` と `llms-full.txt` の e2e テストを追加

  ## コンポーネントとスタイリングの強化

  - カスタム見出しスタイル適用のため、`h{1-4}` タグでカスタム classnames をサポート
  - カスタムページモードで `h{1-4}` タグがチップ付きの `Heading` コンポーネントとしてレンダリングされていた問題を修正
  - カスタム CSS のターゲティング用に [breadcrumbs](/ja/navigation#breadcrumbs) へ CSS セレクタを追加
  - 高さ 56px でプロポーションを維持するため、寸法を解析して引き伸ばされる open-graph 画像を修正
  - 有効化時のコンテキストメニュー内で `VSCode` を `VS Code` に修正
  - カスタムコンポーネント内の見出しがセマンティックな見出しと一緒に目次に表示される問題を修正

  ## バグ修正と信頼性向上

  - 生成上の問題を引き起こす文字をサニタイズすることで、特定のページタイトルでの PDF レンダリング問題を修正
  - 空の OpenAPI JSON ファイルに遭遇した際の CLI エラー `Cannot convert undefined or null to object` を解消
  - 生成された値で上書きされていたカスタム `docs.json` の open-graph メタタグを修正
  - RSS リンクに origin + pathname を使用することで、アンカーリンク着地時の RSS フィードボタンのクリック動作を修正
  - ソースマップを削除して CLI のダウンロード速度を改善

  ## 技術的な改善

  - 早期のリグレッション検知のため、CI パイプラインにビジュアルテストを追加
  - エラーハンドリングとデバッグ機能を強化
  - 新機能およびエッジケースに対する包括的なテストカバレッジを実施
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API、OAuth の磨き込み、アセット配信、MCP ツール群" }}>
  ## 認証の改善

  - グループレベルの公開アクセス：`docs.json` でページグループ全体を公開にでき、各ページに `public: true` を付ける必要がなくなります（[詳細](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  - [OAuth 設定での `logoutURL` のサポート](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)：上流のクッキーを削除し、サインアウトを完了
  - OAuth エラー時は、指定した `logoutURL` にリダイレクトして認証フローを再開
  - コールバック前の OAuth/JWT フローで一瞬 500 エラーが表示される問題を修正
  - 誤設定防止のため、OAuth/JWT 認証設定の URL から自動で `https://` を除去

  ## APIプレイグラウンドの拡張

  - 新しい [Search API エンドポイント](https://mintlify.com/docs/api-reference/assistant/search)：ドキュメント上にエージェントや MCP サーバーを構築可能に
  - `openapi` と `asyncapi` ファイルが指定されたパスで配信されるようになりました（例：`https://mydocsurl.extension/{openapi-or-file-name}.json`）
  - [OpenAPI ファイル内の `x-mint` フィールド](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension)で、生成フィールドの上書き、前文コンテンツのカスタマイズ、コードサンプル内のエンドポイント URL の変更が可能に
  - OpenAPI 設定で、[`x-mcp` は `x-mint.mcp` に変更](https://mintlify.com/docs/api-playground/openapi-setup#mcp)：どのルートを MCP ツールとして公開するかを制御

  ## アシスタントの更新

  - 新しいメッセージがストリーミングされると、古いメッセージのアクションメニュー（コピーやサムズアップなどのオプションを含む）が消える問題を修正
  - 先週の[ホスト型 MCP サーバーのリリース](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)後に発生した、ネストされた `/mcp/...` ページのアクセシビリティ問題を修正

  ## パフォーマンスと信頼性

  - ドキュメントリポジトリ内のすべての画像・動画アセットが、ドメイン上の適切なパスで配信されるようになりました。たとえば、リポジトリに `/assets/marketing/my-logo.png` がある場合、`https://mydocsurl.extension/assets/marketing/my-logo.png` で提供されます。
  - Mintlify ダッシュボードのログイン画面でメールアドレス欄が自動フォーカスされ、すぐに入力を開始できます（ちょっとした使い勝手の改善）
  - ナビゲーション読み込み時間のパフォーマンス改善のため、Redis 上でカスタムドメインとサブドメインの両方を取り扱い（レイテンシ約 50ms 削減）
  - PDF エクスポートの信頼性向上のため再試行ロジックを追加
  - クッキー同意ポップアップが承認または却下後に再表示される問題を修正—最初の選択が尊重されるようになりました
  - Safari でページをクリップボードにコピーできない問題を、`navigator.write` に MIME の `type` を指定することで修正

  ## 技術的な改善

  - Windows と pnpm 向けの CLI バグ修正に加え、リグレッション防止のための CI テストを追加
  - エラーログ出力を改善—デバッグ時のエンジニアリングチームの利便性が向上
  - `contentDirectory` ファイルが欠落している場合の broken-link CI アクションの軽微な修正
  - 先週の認証保護プレビュー修正に起因するリグレッション（ナビゲーション UI でアクティブタブが正しく設定されない）を修正
  - テーマのライト背景色がアクティブタブのアイコンに適用されない問題を修正
  - ダッシュボードで認証タイプを変更すると更新後に以前の保存タイプへ戻ってしまう問題を修正—新しい選択が保存後も保持されます
  - カスタム UI ライブラリを持つエンタープライズ顧客向けの内部 DX 改善—当社がコンポーネントを取り込んだり、短納期のリクエストに対応したりしやすくなりました
</Update>
