---
title: "MDX のセットアップ"
description: "`MDX` を使って API エンドポイントのドキュメントページを生成する"
---

OpenAPI 仕様を使わずに、個々の `MDX` ファイルで API エンドポイントを手動定義できます。この方法はカスタムコンテンツに柔軟ですが、保守性が高く機能も豊富なため、ほとんどのプロジェクトでは OpenAPI 仕様ファイルから API ドキュメントを生成することを推奨します。ただし、小規模な API のドキュメント化やプロトタイピングには、`MDX` ページでの作成が有用です。

`MDX` で API エンドポイントのページを生成するには、`docs.json` で API 設定を行い、各エンドポイントごとに個別の `MDX` ファイルを作成し、`<ParamFields />` などのコンポーネントでパラメータを定義します。これらの定義から、Mintlify はインタラクティブな APIプレイグラウンド、リクエスト例、レスポンス例を生成します。

<Steps>
  <Step title="API を構成する">
    `docs.json` ファイルで、ベース URL と認証方式を定義します:

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // 複数のベース URL がある場合は文字列配列
        "auth": {
          "method": "key",
          "name": "x-api-key" // オプション: bearer, basic, key
        }
      }
    }
    ```

    APIプレイグラウンドを非表示にしたい場合は、`display` フィールドを使用します。プレイグラウンドを非表示にする場合は認証方式を設定する必要はありません。

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    API 設定の全一覧は [Settings](/ja/organize/settings#api-configurations) を参照してください。
  </Step>

  <Step title="エンドポイントページを作成する">
    各 API エンドポイントページには対応する `MDX` ファイルが必要です。各ファイルの先頭に `title` と `api` を定義します:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    パスパラメータは、パラメータ名を `{}` で囲んでパスに追加することで指定できます:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      `docs.json` に `server` フィールドが設定されている場合は、`/v1/endpoint` のような相対パスを使えます。
    </Note>

    フロントマターに `playground` を追加することで、ページ単位で APIプレイグラウンドのグローバルな表示モードを上書きできます:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    * `playground: 'interactive'` - インタラクティブなプレイグラウンドを表示。
    * `playground: 'simple'` - プレイグラウンドなしでコピー可能なエンドポイントを表示。
    * `playground: 'none'` - プレイグラウンドを非表示。
  </Step>

  <Step title="エンドポイントをドキュメントに追加する">
    `docs.json` の `navigation` フィールドにパスを追加して、サイドバーにエンドポイントページを表示します。ドキュメント構成の詳細は [Navigation](/ja/organize/navigation) を参照してください。
  </Step>
</Steps>

<div id="enabling-authentication">
  ## 認証の有効化
</div>

`docs.json` に認証方式を追加して、すべてのページでグローバルに有効化することも、ページごとに設定することもできます。

ページの認証方式は、両方が設定されている場合、グローバル設定を上書きします。

<div id="bearer-token">
  ### ベアラー・トークン
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "ページのタイトル"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

  ### APIキー

<CodeGroup>

<div id="none">
  ### None
</div>

`none` 認証メソッドは、docs.json でデフォルトを設定した後に、特定のエンドポイントだけ認証を無効にするのに便利です。

<CodeGroup>
  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
