Escolha entre três abordagens para gerenciar o acesso e personalizar sua documentação com base nas informações do usuário e nas suas necessidades de segurança:

- **Autenticação completa**: Proteção total de privacidade para todo o conteúdo, com personalização completa.
- **Autenticação parcial**: Controle de acesso página a página, com personalização completa.
- **Personalização**: Personalização de conteúdo com **nenhuma garantia de segurança**. Todo o conteúdo permanece publicamente acessível.

<div id="use-case-examples">
  ### Exemplos de casos de uso
</div>

**Escolha autenticação completa quando:**

- Estiver criando documentação interna da empresa que contenha informações confidenciais
- Estiver documentando APIs proprietárias que exigem verificação do usuário
- Estiver criando guias de implementação específicos para clientes

**Escolha autenticação parcial quando:**

- Oferecer guias públicos de primeiros passos com recursos avançados privados
- Operar um produto freemium em que usuários premium recebem documentação adicional
- Publicar documentação de código aberto com seções empresariais privadas

**Escolha personalização quando:**

- Criar documentação pública de API que exiba exemplos específicos do usuário
- Construir sites de marketing que personalizam o conteúdo com base em perfis de usuário
- Oferecer tutoriais públicos que se adaptem às preferências ou ao nível de habilidade do usuário

<div id="handshake-methods">
  ## Métodos de handshake
</div>

Escolha o método que melhor se adapta à sua infraestrutura e aos seus requisitos de segurança.

| Método | Disponível para | Complexidade de configuração | Ideal para |
|:-------|:--------------|:-----------------|:----------|
| **JWT** | Todas as abordagens | Média | Fluxos de login personalizados, controle máximo de segurança |
| **OAuth 2.0** | Todas as abordagens | Alta | Provedores de autenticação de terceiros, segurança corporativa |
| **Mintlify Dashboard** | Apenas autenticação | Baixa | Equipes que já utilizam o painel da Mintlify |
| **Senha** | Apenas autenticação | Baixa | Acesso compartilhado simples sem personalização |
| **Sessão compartilhada** | Apenas personalização | Baixa | Apps com autenticação baseada em sessão já existente |

<div id="when-to-use-each-method">
  ### Quando usar cada método
</div>

**JWT**: Use quando você já possui um sistema de autenticação e quer controle total sobre o fluxo de login. Ideal para gerenciamento personalizado de usuários ou quando é necessário desacoplar o acesso à documentação do seu aplicativo principal.

**OAuth 2.0**: Use quando quiser aproveitar provedores de autenticação de terceiros (Google, GitHub, etc.) ou precisar de padrões de segurança de nível empresarial. Melhor para organizações que já utilizam infraestrutura OAuth.

**Mintlify Dashboard**: Use quando quem edita a documentação também é quem a consulta. Perfeito para equipes internas que já gerenciam o conteúdo pelo dashboard da Mintlify.

**Password**: Use para controle de acesso simples quando não é necessário rastrear usuários individualmente ou personalizar conteúdo. Indicado para contratados, usuários beta ou cenários de acesso temporário.

**Shared Session**: Use quando quiser login contínuo entre seu aplicativo e a documentação. Ideal quando os usuários já estão autenticados no seu aplicativo principal e você quer personalizar a experiência deles na documentação.

<div id="content-customization">
  ## Personalização de conteúdo
</div>

As três abordagens oferecem recursos de personalização, incluindo conteúdo dinâmico em MDX, preenchimento antecipado de chave de API e controles de visibilidade de páginas. Para orientações detalhadas de implementação, consulte [Configuração de personalização](/pt-BR/authentication-personalization/personalization-setup).
