Acesse a aba de feedback pela página **Insights** no seu [painel](https://dashboard.mintlify.com/products/insights).

<div id="managing-feedback">
  ## Gerenciando feedback
</div>

Para feedback contextual e sobre trechos de código, você pode definir o status de um feedback e adicionar notas internas para acompanhar seu trabalho na resolução do feedback dos usuários.

<div id="changing-feedback-status">
  ### Alterando o status do feedback
</div>

Selecione o status ao lado de um feedback para marcá-lo como **Pendente**, **Em andamento**, **Resolvido** ou **Descartado**.

Boas práticas para definir status de feedback:

* **Pendente**: O feedback está aguardando revisão.
* **Em andamento**: O feedback foi validado e está em andamento.
* **Resolvido**: O feedback foi resolvido.
* **Descartado**: O feedback foi descartado por não ser acionável, irrelevante ou impreciso.

<div id="filtering-by-status">
  ### Filtrando por status
</div>

Use o filtro de status para controlar quais feedbacks são exibidos. Limpe um status para ocultar todos os feedbacks com esse status. Por padrão, todos os feedbacks são exibidos.

<div id="adding-internal-notes">
  ### Adicionando notas internas
</div>

Clique em um feedback para adicionar uma nota interna. Essas notas são visíveis apenas para pessoas com acesso ao seu painel.

Use as notas para adicionar informações para colaboração, vincular chamados relevantes de suporte ou engenharia, ou registrar qualquer outra informação útil.

<div id="using-feedback-data">
  ## Usando dados de feedback
</div>

Analise seus dados de feedback para:

* **Identificar conteúdo de sucesso**: Páginas com mais feedback positivo mostram o que funciona bem na sua documentação.
  * **Priorizar melhorias**: Páginas com mais feedback negativo indicam quais conteúdos podem precisar de atenção.
  * **Agir**: Faça atualizações na documentação com base no feedback direto dos usuários.
