A Mintlify trata automaticamente de muitas boas práticas de SEO, incluindo:

* Geração de meta tags
* Geração de sitemap e do arquivo `robots.txt`
* Estrutura HTML semântica
* Otimização para dispositivos móveis

Você pode personalizar completamente as meta tags do seu site adicionando o campo `metatags` ao `docs.json` ou ao frontmatter de uma página.

<div id="global-meta-tags">
  ## Metatags globais
</div>

Para definir metatags padrão para todas as páginas, adicione o campo `metatags` ao seu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "link para sua imagem padrão de meta tag"
    }
}
```

<div id="set-a-canonical-url">
  ### Defina uma URL canônica
</div>

Se você estiver usando um domínio personalizado, defina a meta tag `canonical` para garantir que os mecanismos de pesquisa indexem seu domínio preferido. Uma URL canônica informa aos mecanismos de pesquisa qual versão da sua documentação é a principal. Isso melhora o SEO quando sua documentação está acessível por várias URLs e evita problemas de conteúdo duplicado.

```json
"seo": {
    "metatags": {
        "canonical": "https://www.seu-dominio-customizado-aqui.com"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metatags específicas da página
</div>

Para definir metatags específicas de uma página, adicione-as ao frontmatter da página.

```mdx
---
title: "Título da sua página de exemplo"
"og:image": "link para a imagem da sua meta tag"
---
```

<Note>
  Metatags com dois-pontos devem estar entre aspas.
</Note>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps e arquivos robots.txt
</div>

O Mintlify gera automaticamente um arquivo `sitemap.xml` e um arquivo `robots.txt`. Você pode visualizar seu sitemap adicionando `/sitemap.xml` ao URL do seu site de documentação.

Por padrão, apenas as páginas presentes no seu `docs.json` são incluídas. Para incluir links ocultos, adicione `seo.indexing` ao seu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps e arquivos robots.txt personalizados
</div>

Para adicionar um `sitemap.xml` ou `robots.txt` personalizado, crie um arquivo `sitemap.xml` ou `robots.txt` na raiz do seu projeto. Ao adicionar um arquivo personalizado, ele substituirá o arquivo gerado automaticamente com o mesmo nome. Se você excluir um arquivo personalizado, o arquivo padrão voltará a ser usado.

<div id="disabling-indexing">
  ## Desativando a indexação
</div>

Se você quiser impedir que uma página seja indexada por mecanismos de busca, inclua o seguinte no [frontmatter](/pt-BR/organize/pages) da sua página:

```
---
noindex: true
---
```

Você também pode definir `noindex` para todas as páginas da sua documentação configurando o campo `metatags.robots` como `"noindex"` no seu `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Melhores práticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escreva títulos e descrições descritivos">
    * Use títulos de página claros e descritivos (50–60 caracteres)
    * Escreva descrições atraentes (150–160 caracteres)
    * Inclua palavras-chave relevantes
    * Torne cada título e descrição de página únicos
  </Accordion>

  <Accordion title="Otimize a estrutura do conteúdo">
    * Use a hierarquia adequada de títulos (H1 → H2 → H3)
    * Escreva primeiro para pessoas, depois para mecanismos de busca
    * Inclua palavras-chave relevantes nos títulos e no conteúdo
    * Mantenha URLs curtas, descritivas e organizadas hierarquicamente
    * Quebre conteúdos longos com subtítulos e listas
  </Accordion>

  <Accordion title="Estratégia de links internos">
    * Crie links para páginas relacionadas dentro da documentação
    * Use texto âncora descritivo em vez de &quot;clique aqui&quot;
    * Crie clusters de tópicos ao vincular conceitos relacionados
    * Use os recursos automáticos de referência cruzada
  </Accordion>

  <Accordion title="SEO para imagens">
    * Use nomes de arquivo descritivos para as imagens
    * Sempre inclua texto alternativo (alt) para acessibilidade e SEO
    * Otimize o tamanho dos arquivos de imagem para carregamento mais rápido
    * Use imagens relevantes que reforcem seu conteúdo
  </Accordion>
</AccordionGroup>
