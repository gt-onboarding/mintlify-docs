Quando você altera o caminho de um arquivo na pasta de docs, o caminho da URL dessa página também muda. Isso pode acontecer ao reestruturar a documentação ou ao alterar o título da barra lateral.

<div id="broken-links">
  ## Links quebrados
</div>

Detecte links quebrados com nossa CLI. [Instale a CLI](/pt-BR/installation) e execute o comando:

```bash
mint broken-links
```

A CLI identifica qualquer link relativo na sua documentação que não exista.

<div id="redirects">
  ## Redirecionamentos
</div>

Configure redirecionamentos 301 adicionando o campo `redirects` ao seu arquivo `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Isso redireciona permanentemente `/source/path` para `/destination/path`, para que você não perca o SEO acumulado da página original.

Para corresponder a um caminho curinga, use `*` após um parâmetro. Neste exemplo, `/beta/:slug*` corresponde a `/beta/introduction` e o redireciona para `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
