A Mintlify lida automaticamente com diversas boas práticas de SEO, incluindo:

- Geração de meta tags
- Geração de sitemap e arquivo `robots.txt`
- Estrutura HTML semântica
- Otimização para dispositivos móveis

Você pode personalizar completamente as meta tags do seu site adicionando o campo `metatags` ao seu `docs.json` ou ao frontmatter de uma página.

<div id="global-meta-tags">
  ## Metatags globais
</div>

Você pode definir metatags padrão adicionando o campo `metadata` ao seu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "link para a imagem padrão da sua meta tag"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metatags específicas da página
</div>

Adicione metatags aos metadados da página da seguinte forma:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  Metatags que contêm dois-pontos precisam estar entre aspas quando você as coloca nos metadados da página.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps e arquivos robots.txt
</div>

A Mintlify gera automaticamente um arquivo `sitemap.xml` e um arquivo `robots.txt`. Você pode visualizar seu sitemap adicionando `/sitemap.xml` à URL do seu site de documentação.

Por padrão, somente as páginas listadas no seu `docs.json` são incluídas. Para incluir links ocultos, adicione `seo.indexing` ao seu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps e arquivos robots.txt personalizados
</div>

Para adicionar um `sitemap.xml` ou `robots.txt` personalizado, crie o arquivo `sitemap.xml` ou `robots.txt` na raiz do seu projeto. Ao adicionar um arquivo personalizado, ele substituirá o arquivo gerado automaticamente com o mesmo nome. Se você excluir o arquivo personalizado, o arquivo padrão voltará a ser usado.

<div id="disabling-indexing">
  ## Desativando a indexação
</div>

Se você deseja impedir que uma página seja indexada por mecanismos de busca, inclua o seguinte no [frontmatter](pt-BR/pages) da página:

```
---
noindex: true
---
```

Você também pode definir `noindex` para todas as páginas da sua documentação configurando o campo `metadata.robots` como `"noindex"` no `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Melhores práticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escreva títulos e descrições claros e descritivos">
    - Use títulos de página claros e descritivos (50–60 caracteres)
    - Escreva descrições atraentes (150–160 caracteres)
    - Inclua palavras-chave relevantes
    - Torne o título e a descrição de cada página únicos
  </Accordion>

  <Accordion title="Otimize a estrutura do conteúdo">
    - Use a hierarquia adequada de cabeçalhos (H1 → H2 → H3)
    - Escreva para pessoas primeiro, mecanismos de busca depois
    - Inclua palavras-chave relevantes nos cabeçalhos e no conteúdo
    - Mantenha URLs curtas, descritivas e organizadas hierarquicamente
    - Divida conteúdos longos com subtítulos e listas
  </Accordion>

  <Accordion title="Estratégia de links internos">
    - Crie links para páginas relacionadas dentro da documentação
    - Use texto âncora descritivo em vez de “clique aqui”
    - Forme clusters de tópicos interligando conceitos relacionados
    - Utilize os recursos automáticos de referência cruzada
  </Accordion>

  <Accordion title="SEO para imagens">
    - Use nomes de arquivo descritivos para as imagens
    - Sempre inclua texto alternativo (alt text) para acessibilidade e SEO
    - Otimize o tamanho dos arquivos de imagem para carregamento mais rápido
    - Use imagens relevantes que reforcem seu conteúdo
  </Accordion>
</AccordionGroup>
