As implantações de pré-visualização permitem ver como as alterações na sua documentação ficarão antes de fazer o merge em produção. Cada pré-visualização gera uma URL compartilhável que é atualizada automaticamente à medida que você envia novas alterações.

<div id="creating-preview-deployments">
  ## Criando ambientes de pré-visualização
</div>

Ambientes de pré-visualização são criados automaticamente por meio de pull requests ou manualmente pelo seu painel.

<div id="automatic-previews">
  ### Prévias automáticas
</div>

Quando você abre um pull request, o bot da Mintlify adiciona automaticamente um link para visualizar a implantação de prévia no seu pull request. A prévia é atualizada sempre que você envia novos commits para a branch.

<div id="manual-previews">
  ### Pré-visualizações manuais
</div>

Você pode criar manualmente uma pré-visualização para qualquer branch.

1. Acesse o seu [dashboard](https://dashboard.mintlify.com/).
2. Selecione **Previews**.
3. Selecione **Create preview**.
4. Insira o nome da sua branch.
5. Selecione **Create deployment**.

<div id="redeploying-a-preview">
  ## Reimplantar uma prévia
</div>

Reimplante uma prévia para atualizar o conteúdo ou tentar novamente após uma implantação com falha.

1. Selecione a prévia no seu [dashboard](https://dashboard.mintlify.com/).
2. Selecione **Reimplantar**.

<div id="sharing-preview-deployments">
  ## Compartilhamento de implantações de pré-visualização
</div>

As implantações de pré-visualização podem ser acessíveis publicamente ou restritas a usuários autenticados.

**Acesso público**: As URLs de pré-visualização são públicas por padrão. Compartilhe um link de pré-visualização com qualquer pessoa que precise revisar suas alterações.

**Acesso restrito**: Ative a autenticação da implantação de pré-visualização para limitar o acesso apenas a usuários autenticados da Mintlify. Consulte [Configuração de autenticação](/pt-BR/authentication-personalization/authentication-setup) para mais informações.
