A API REST da Mintlify permite interagir programaticamente com sua documentação, acionar atualizações e incorporar experiências de chat com IA.

<div id="endpoints">
  ## Endpoints
</div>

- [Trigger update](/pt-BR/api-reference/update/trigger): Acione uma atualização do seu site quando quiser.
- [Get update status](/pt-BR/api-reference/update/status): Consulte o status de uma atualização e outros detalhes da sua documentação.
- [Generate assistant message](/pt-BR/api-reference/assistant/create-assistant-message): Incorpore o Assistente, treinado na sua documentação, em qualquer aplicativo de sua escolha.
- [Search documentation](/pt-BR/api-reference/assistant/search): Pesquise na sua documentação.

<div id="authentication">
  ## Autenticação
</div>

Você pode gerar uma chave de API pelo [dashboard](https://dashboard.mintlify.com/settings/organization/api-keys). As chaves de API são associadas à organização inteira e podem ser usadas em várias implantações.

<div id="admin-api-key">
  ### Chave de API de administrador
</div>

A chave de API de administrador é usada nos endpoints [Trigger update](/pt-BR/api-reference/update/trigger) e [Get update status](/pt-BR/api-reference/update/status).

As chaves de API de administrador começam com o prefixo `mint_`. Mantenha suas chaves de API de administrador em sigilo.

<div id="assistant-api-key">
  ### Chave da API do Assistente
</div>

A chave da API do Assistente é usada nos endpoints [Generate assistant message](/pt-BR/api-reference/assistant/create-assistant-message) e [Search documentation](/pt-BR/api-reference/assistant/search).

As chaves da API do Assistente começam com o prefixo `mint_dsc_`.

A **chave** da API do Assistente é um token de servidor que deve ser mantido em sigilo.

O **token** da API do Assistente é público e pode ser referenciado no seu código de frontend.

<Note>
  Chamadas usando o token da API do Assistente podem gerar custos: seja consumindo seus créditos do Assistente de IA ou incorrendo em excedentes.
</Note>
