Use CSS para estilizar elementos HTML ou adicione CSS e JavaScript personalizados para ajustar completamente a aparência e a experiência da sua documentação.

<div id="styling-with-tailwind-css">
  ## Estilizando com Tailwind CSS
</div>

Use o Tailwind CSS v3 para estilizar elementos HTML. Você pode controlar o layout, o espaçamento, as cores e outras propriedades visuais. Algumas classes comuns são:

* `w-full` - Largura total
* `aspect-video` - Proporção 16:9
* `rounded-xl` - Cantos bem arredondados
* `block`, `hidden` - Controle de exibição
* `dark:hidden`, `dark:block` - Visibilidade no modo escuro

Valores arbitrários do Tailwind CSS não são compatíveis. Para valores personalizados, use a prop `style`.

```html
<img style={{ width: '350px', margin: '12px auto' }} src="/path/image.jpg" />
```


<div id="custom-css">
  ## CSS personalizado
</div>

Adicione arquivos CSS ao seu repositório e os nomes de classe definidos serão aplicados e ficarão disponíveis em todos os seus arquivos `MDX`.

<div id="adding-stylecss">
  ### Adicionando `style.css`
</div>

Por exemplo, você pode adicionar o arquivo `style.css` a seguir para personalizar o estilo da barra de navegação e do rodapé.

```css
#navbar {
  background: "#fffff2";
  padding: 1rem;
}

footer {
  margin-top: 2rem;
}
```

<div id="custom-javascript">
  ## JavaScript personalizado
</div>

O JavaScript personalizado permite adicionar código executável de forma global. É equivalente a inserir uma tag `<script>` com código JS em todas as páginas.

<div id="adding-custom-javascript">
  ### Adicionando JavaScript personalizado
</div>

Qualquer arquivo `.js` dentro do diretório de conteúdo da sua documentação será incluído em todas as páginas. Por exemplo, você pode adicionar o arquivo `ga.js` abaixo para ativar o [Google Analytics](https://marketingplatform.google.com/about/analytics) em toda a documentação.

```js
window.dataLayer = window.dataLayer || [];
function gtag() {
  dataLayer.push(arguments);
}
gtag('js', new Date());

gtag('config', 'TAG_ID');
```

<Warning>
  Use com cautela para não introduzir vulnerabilidades de segurança.
</Warning>
