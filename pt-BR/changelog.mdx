<Update label="31 de agosto - 13 de setembro" tags={["Novos lançamentos", "Melhorias"]} rss={{ title: "Atualizações semanais", description: "Sugestões de IA para 404, busca na web do assistente com fontes externas, melhorias de segurança e correções de confiabilidade" }}>
  ## Principais lançamentos

  * **Grande melhoria**: páginas sugeridas por IA em páginas 404, [quando alguém acessa um link quebrado → o agente de IA lê o caminho → sugere páginas semanticamente semelhantes](https://x.com/mintlify/status/1966625627773059495)
  * **Grande lançamento**: a busca na web do Assistente agora pode incluir fontes externas\
    *Observação: Entre em contato conosco para ativar este recurso no seu site.*

  ## Assistente e MCP

  * Corrigido um bug em que o Assistente era limitado por taxa incorretamente devido à janela de tempo não deslizar corretamente
  * Corrigido um bug nas chamadas de ferramentas do Assistente para lidar corretamente com blocos `text` vazios
  * Corrigido um bug em que o nome do servidor MCP concatenado com chamadas de ferramenta às vezes ultrapassava o limite de 60 caracteres imposto pelos clientes MCP
  * Corrigido um bug em que o menu do Assistente tinha uma altura várias vezes maior que a janela de visualização e rolava indefinidamente
  * Corrigido um bug em que os valores de gasto do Assistente podiam ser exibidos com mais de duas casas decimais no painel

  ## Editor web e implantações

  * Melhoria de segurança no editor: apenas usuários com `write permissions` no repositório de hospedagem git conectado podem fazer alterações
  * Corrigido um bug em que as implantações de pré-visualização não funcionavam para branches com `=` no nome
  * Corrigido um bug em que nomes de branch longos extrapolavam os modais na criação de implantações de pré-visualização
  * Melhoria de usabilidade: o parâmetro de consulta de e-mail preenche automaticamente o campo em convites de cadastro
  * Corrigido um bug em que copiar uma página pelo menu contextual não funcionava no Safari

  ## Playground de API e navegação

  * Vários códigos de resposta no Playground de API agora são exibidos em um select estilizado controlado em vez do select padrão do sistema quando em foco
  * Agora você pode usar o [campo `expanded` em grupos de navegação no seu `docs.json` para deixá-los abertos por padrão](https://mintlify.com/docs/navigation#default-expanded-state)

  ## SEO e UI

  * Corrigido um bug em que favicons não apareciam nos mecanismos de busca ao servi-los pela mesma URL do próprio site de documentação para cada site
  * Corrigido um bug em que incorporações do YouTube piscavam ao carregar
  * Corrigido um bug em que expandir o menu de feedback para incluir respostas escritas causava deslocamento de layout no sumário
  * Corrigido um bug em que texto vazava acima da barra superior no tema Maple ao dispensar o banner de notificação
  * Temas Maple e Willow aprimorados com a adição de botões de login/logout na barra lateral para facilitar o acesso

  ## Analytics e exportações

  * Corrigidos problemas de confiabilidade na visualização e exportações de analytics do Assistente
  * As exportações de analytics do Assistente agora são executadas em segundo plano e enviadas por e-mail para uma experiência mais confiável

<Update label="August 24 - August 30" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Coleta de feedback aprimorada, melhorias na navegação e correções de bugs de autenticação" }}>
  ## Lançamento principal: coleta de feedback aprimorada

  * **Grande melhoria**: leitores agora podem dar feedback mais detalhado após selecionar *polegar para cima/para baixo*, incluindo opções e comentários escritos. Você também pode coletar feedback em blocos de código e ver todas as respostas nas análises do seu painel.\
    *Observação: entre em contato conosco para ativar este recurso no seu site.*

  ## Melhorias na navegação e na usabilidade

  * Melhoria de usabilidade para ignorar barras finais e barras duplas, assim você não precisa se preocupar em acertá-las exatamente no seu docs.json
  * Agora você pode adicionar o atributo `noAnchor` às suas tags HTML `h1-6` para evitar que tenham um link âncora
  * O tema Palm agora tem um seletor de idioma flutuante no canto inferior esquerdo, similar à abordagem da Stripe
  * Adicionado um novo campo ao docs.json chamado [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups), que permite controlar se os usuários serão ou não direcionados para a primeira página em um grupo de navegação quando o grupo for expandido

  ## Correções de bugs e confiabilidade

  * Corrigido um bug em que links de âncora para posição de rolagem não funcionavam corretamente quando havia componentes JS em uma página
  * Corrigido um bug em que o Google estava indexando arquivos brutos `*.md` por estarem sem o cabeçalho `x-robots-tag noindex`
  * Corrigido um bug com OAuth em docs protegidas em que você não era redirecionado de volta à página inicial após concluir o fluxo com sucesso
  * Corrigido um bug nas pré-visualizações de docs protegidas por autenticação em que você não conseguia ver toda a barra de navegação
  * Correções de bugs na forma como SVGs são processados com nossa nova CDN de imagens

  ## Aprimoramentos de componentes e de estilo

  * Adicionado um novo seletor CSS para estilos personalizados em `SidebarNavGroupDivider`
  * Novos testes de regressão para páginas de API definidas em MDX que têm segurança configurada garantirão um grau maior de estabilidade

  ## Melhorias de performance

  * Melhoria de performance ao mover o CSS do KaTeX de cdnjs para nossa própria CDN na CloudFront, reduzindo a latência
</Update>

<Update label="August 17 - August 23" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Correções no layout de imagens, aprimoramentos de autenticação, melhorias na navegação móvel e estabilidade do Editor web" }}>
  ## Melhorias no tratamento de imagens

  * **Grande melhoria**: imagens não causam mais deslocamento de layout por padrão, mesmo quando os atributos de largura e altura não são especificados — o dimensionamento automático evita saltos de conteúdo durante o carregamento da página
  * Todos os arquivos estáticos no seu repositório (PDF, TXT, XML, etc.) agora são automaticamente enviados e servidos quando você faz o deploy, garantindo cobertura completa de assets

  ## Aprimoramentos no Editor web e na implantação

  * Corrigido o fluxo de criação de branch no Editor web para navegar corretamente para e permanecer nos branches recém-criados
  * Diálogo de conflito de merge aprimorado com funcionalidade adequada de escape; não são mais necessários recarregamentos de página para dispensar conflitos
  * Otimizado o desempenho do fluxo de atualização, invalidando o cache apenas das páginas alteradas durante atualizações parciais, reduzindo o tempo de implantação

  ## Melhorias de autenticação e navegação

  * Novo suporte para autenticação em subdiretórios personalizados; se você serve docs em `https://yourdomain.com/docs`, a autenticação agora funciona perfeitamente
  * Corrigido um bug na exibição da barra lateral que aparecia incorretamente quando apenas um link estava configurado
  * Reformulação abrangente da navegação móvel: botões centralizados com margens/espacamentos adequados, espaçamento aprimorado em menus suspensos, remoção de divisores e margens desnecessários para seções vazias e correção de problemas de espaçamento no tema Maple

  ## Correções de componentes e de estilo

  * Resolvido o problema de conversão de tags `<h1-6>` que as transformava incorretamente em componentes Heading e prejudicava a estilização personalizada
  * Alternância de configuração do Assistente com um clique adicionada ao painel para facilitar o gerenciamento

  ## Melhorias técnicas e confiabilidade

  * Sistema de logs aprimorado para fluxos de atualização, possibilitando depuração mais rápida e resolução de problemas
  * Corrigido o rate limiting do GitHub para clientes com 10+ especificações OpenAPI/AsyncAPI, trocando a busca de arquivos individuais pela clonagem do repositório
  * Confiabilidade do Assistente aprimorada com suporte a LLM de backup, tratamento aprimorado de erros de limite de taxa e ferramenta de busca mais robusta
</Update>

<Update label="10 de agosto - 16 de agosto" tags={["Improvements"]} rss={{ title: "Atualizações semanais", description: "Otimizações de performance, integrações com LLMs, melhorias no editor web e aprimoramentos no playground de API" }}>
  ## Otimizações de performance e build

  * A transpilação de MDX agora ocorre no momento da implantação, em vez de a cada carregamento de página em ambientes serverless do NextJS sem cache, melhorando o time to first byte para páginas não armazenadas em cache.
  * O hashing baseado em conteúdo evita a retranspilação quando o MDX não mudou, reduzindo em ~50% os tempos de atualização para clientes com grande número de páginas (implantações acima de 5 minutos devem ser aproximadamente reduzidas pela metade)
  * A visualização de Implantações de pré-visualização no dashboard agora está mais rápida com índices de banco de dados adicionais e paralelização de consultas no nosso backend
  * Reduzimos o tamanho da página eliminando dados `navigation` duplicados no payload `rsc` de cada página — ganhos de performance mais perceptíveis com alto número de páginas ou estruturas de navegação complexas
  * Prefetch mais agressivo permite carregamentos instantâneos de página com mais frequência

  ## Aprimoramentos no Playground de API e em OpenAPI

  * A conversão de OpenAPI para MCP foi movida para o backend, permitindo que servidores MCP hospedados contenham ferramentas (em breve, documentação e opções de configuração mais claras)
  * Adicionamos suporte a Ruby no Playground de API
  * Adicionamos um recurso pelo qual agora você pode [especificar páginas de API usando apenas seu docs.json](/pt-BR/api-playground/openapi-setup#auto-populate-api-pages), sem criar nenhum novo arquivo MDX.
  * Suporte para páginas de [`webhook`](/pt-BR/api-playground/openapi-setup#webhooks) na navegação dos docs a partir de especificações OpenAPI
  * Contexto de modelo de IA otimizado, removendo âncoras de links em markdown ao navegar para Anthropic, OpenAI ou outros provedores

  ## Melhorias no Editor web

  * A criação/renomeação de arquivos agora salva as alterações ao clicar fora, em vez de exigir a tecla Enter
  * Corrigida a navegação de branches em que alterar a URL para uma branch específica redirecionava para a última branch ativa em vez do destino pretendido
  * Titles de branch com `/` agora são corretamente codificados na URL para evitar falhas de navegação
  * Corrigido o atalho de link `Ctrl+K` no Editor do dashboard de monorepo, que estava adicionando o caminho do repositório de docs e criando links quebrados

  ## Analytics e integrações com LLMs

  * Suporte a `llms.txt` e `llms-full.txt` personalizados — adicione à raiz do repositório de docs para servir nos endpoints `/llms.txt` e `/llms-full.txt` para personalização de LLM
  * Adicionada [integração de analytics com Hightouch](/pt-BR/integrations/analytics/hightouch#hightouch)
  * Aprimorado o rastreamento de analytics do Menu contextual (visualização no dashboard em breve)
  * Adicionados testes e2e para `llms.txt` e `llms-full.txt` para garantir a entrega correta

  ## Aprimoramentos de Componentes e estilização

  * Suporte a classnames personalizados em tags `h{1-4}` para aplicar estilos de heading personalizados
  * Corrigida a renderização de tags `h{1-4}` como componentes `Heading` com chips no modo de página personalizada
  * Adicionados seletores CSS para [breadcrumbs](/pt-BR/navigation#breadcrumbs) para direcionamento de CSS personalizado
  * Corrigidas imagens open graph esticadas, analisando dimensões para manter as proporções a 56px de altura
  * Corrigido de `VSCode` para `VS Code` no Menu contextual quando habilitado
  * Corrigido o aparecimento de headings dentro de componentes personalizados no sumário juntamente com headings semânticos

  ## Correções de bugs e confiabilidade

  * Corrigidos problemas de renderização de PDF com certos títulos de páginas, sanitizando caracteres que causavam problemas na geração
  * Resolvido o erro do CLI `Cannot convert undefined or null to object` ao encontrar arquivos OpenAPI JSON vazios
  * Corrigidas metatags open graph personalizadas em `docs.json` que estavam sendo sobrescritas pelas geradas
  * Corrigidos cliques no botão de feed RSS ao acessar links âncora, usando origin + pathname para links de RSS
  * Melhorada a velocidade de download do CLI removendo sourcemaps

  ## Melhorias técnicas

  * Adicionados testes visuais ao pipeline de CI para detecção mais precoce de regressões
  * Aprimorados o tratamento de erros e as capacidades de depuração
  * Cobertura de testes abrangente para novos recursos e edge cases
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## Melhorias de autenticação

  * Acesso público em nível de grupo: torne grupos inteiros de páginas públicos via `docs.json`, assim você não precisa de `public: true` em cada página ([saiba mais](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level))
  * Suporte a [`logoutURL` na configuração de OAuth](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3) para excluir cookies upstream e concluir o logout
  * Em erros de OAuth, os usuários são redirecionados para o `logoutURL` especificado para reiniciar o fluxo de autenticação
  * Corrigido um flash de erro 500 durante fluxos OAuth/JWT antes do callback
  * Remoção automática de `https://` de URLs na configuração de autenticação OAuth/JWT para evitar má configuração

  ## Aprimoramentos no Playground de API

  * Novo [endpoint da Search API](https://mintlify.com/docs/api-reference/assistant/search) para que você possa criar agentes e servidores MCP sobre a sua documentação
  * Arquivos `openapi` e `asyncapi` agora são servidos nos caminhos especificados (por exemplo, `https://mydocsurl.extension/{openapi-or-file-name}.json`)
  * Agora você pode usar o [`x-mint` em seus arquivos OpenAPI](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) para substituir campos gerados, personalizar o conteúdo de prefácio ou alterar URLs de endpoints em exemplos de código
  * [`x-mcp` agora é `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp) nas configurações OpenAPI para controlar quais rotas são expostas como ferramentas MCP

  ## Atualizações do Assistente

  * Corrigido um problema em que o menu de ações (com opções como copiar e curtir) em mensagens mais antigas desaparecia quando novas começavam a chegar em streaming
  * Corrigida a acessibilidade de páginas aninhadas `/mcp/...` após o [lançamento de servidores MCP hospedados](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server) da semana passada

  ## Desempenho e confiabilidade

  * Todos os assets de imagem e vídeo presentes no seu repositório de docs agora são servidos no caminho apropriado no seu domínio. Por exemplo, se você tiver `/assets/marketing/my-logo.png` no seu repositório, ele estará disponível em `https://mydocsurl.extension/assets/marketing/my-logo.png`.
  * O campo de e-mail na tela de login do dashboard da Mintlify agora recebe foco automático para que você possa começar a digitar imediatamente *(melhoria de usabilidade)*
  * Armazenamento de domínios personalizados e subdomínios no Redis para melhorar o desempenho nos tempos de carregamento da navegação (~50 ms de redução de latência)
  * Adicionada lógica de tentativas para exportações em PDF para melhorar a confiabilidade
  * Corrigido o pop-up de consentimento de cookies reaparecendo após aceitação ou rejeição — a primeira seleção agora é respeitada
  * Corrigida a cópia de uma página para a área de transferência no Safari especificando um `type` MIME em `navigator.write`

  ## Melhorias técnicas

  * Correções de bugs do CLI para Windows e pnpm, além de testes de CI para evitar regressões
  * Saída de logs de erro aprimorada — uma melhoria de usabilidade para nossa equipe de engenharia ao depurar
  * Correções menores na ação de CI de links quebrados quando arquivos de `contentDirectory` estão ausentes
  * Corrigida uma regressão causada pelas correções de pré-visualização protegida por autenticação da semana passada, em que a guia ativa não estava sendo definida corretamente na UI de navegação
  * Corrigida a cor de fundo clara do tema não sendo aplicada aos ícones da guia ativa
  * Corrigido um problema em que alterar o tipo de autenticação no dashboard atualizava e depois voltava ao tipo salvo anteriormente — agora a nova seleção persiste após salvar
  * Melhorias internas de DX para clientes enterprise com bibliotecas de UI personalizadas — agora é mais fácil para nós incluir seus componentes e atender solicitações em prazos menores
</Update>
