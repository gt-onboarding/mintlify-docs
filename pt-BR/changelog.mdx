<Update label="16 de agosto" tags={["Improvements"]} rss={{ title: "Atualizações semanais", description: "Otimizações de performance, integrações com LLMs, melhorias no editor web e aprimoramentos no playground de API" }}>
  ## Otimizações de performance e build

  * A transpilação de MDX agora ocorre no momento do deploy em vez de a cada carregamento de página em ambientes serverless do NextJS sem cache, melhorando o time to first byte para páginas não armazenadas em cache.
  * Hashing baseado em conteúdo evita retranspilação quando o MDX não mudou, reduzindo o tempo do fluxo de atualização em ~50% para clientes com grande volume de páginas (deploys acima de 5 minutos devem ser aproximadamente reduzidos pela metade)
  * A visualização de deploys de preview no dashboard agora está mais rápida com índices de banco de dados e paralelização de queries adicionados ao nosso backend
  * Reduzimos o tamanho das páginas eliminando dados duplicados de `navigation` no payload `rsc` de cada página — ganhos de performance mais perceptíveis com grande volume de páginas ou estruturas de navegação complexas
  * Prefetching mais agressivo permite carregamentos instantâneos de página com mais frequência

  ## Aprimoramentos no Playground de API e em OpenAPI

  * A conversão de OpenAPI para MCP foi movida para o backend, permitindo que servidores MCP hospedados contenham ferramentas (aguarde documentação e opções de configuração mais claras em breve)
  * Adicionamos suporte a Ruby no Playground de API
  * Adicionamos um recurso que permite [especificar páginas de API usando apenas seu docs.json](/pt-BR/api-playground/openapi-setup#auto-populate-api-pages) sem criar novos arquivos MDX.
  * Suporte a [`webhook pages`](/pt-BR/api-playground/openapi-setup#webhooks) na navegação da documentação a partir de especificações OpenAPI
  * Contexto de modelo de IA otimizado ao remover especificações de links âncora de links em Markdown ao navegar para Anthropic, OpenAI ou outros provedores

  ## Melhorias no Editor web

  * A criação/renomeação de arquivos agora salva as alterações ao clicar fora, em vez de exigir pressionar Enter
  * Corrigida a navegação de branches em que mudar a URL para um branch específico redirecionava para o último branch ativo em vez do destino pretendido
  * Codificação adequada em URL para títulos de branch contendo `/` para evitar quebras na navegação
  * Corrigido o atalho de link `Ctrl+K` no Editor do dashboard em monorepo, que estava prefixando o caminho do repositório de docs e criando links quebrados

  ## Analytics e integrações com LLMs

  * Suporte a `llms.txt` e `llms-full.txt` personalizados — adicione-os na raiz do repositório de docs para servir nos endpoints `/llms.txt` e `/llms-full.txt` para customização de LLM

  * Adicionados testes e2e para `llms.txt` e `llms-full.txt` para garantir a entrega correta

  ## Aprimoramentos de componentes e estilos

  * Suporte a classnames personalizados nas tags `h{1-4}` para aplicar estilos de títulos personalizados
  * Corrigido o render das tags `h{1-4}` como componentes `Heading` com chips no modo de página personalizada
  * Adicionados seletores CSS para [breadcrumbs](/pt-BR/organize/navigation#breadcrumbs) visando CSS personalizado
  * Corrigidas imagens open-graph esticadas, analisando dimensões para manter as proporções a 56px de altura
  * Corrigido `VSCode` para `VS Code` no menu contextual quando habilitado
  * Corrigido o aparecimento de títulos dentro de componentes personalizados no sumário junto a títulos semânticos

  * Temas: personalize o estilo com temas pré-configurados. Basta adicionar o tema Quill, Prism ou Venus ao seu arquivo `docs.json` e o estilo da sua documentação será atualizado.
  * Search V2: consulte diretamente descrições e títulos de endpoints OpenAPI para ir às páginas de Referência de API, remova Páginas ocultas da busca e aproveite a nossa barra de busca atualizada.
  * Ramificações no Editor web: crie branches no nosso Editor web sem precisar de um IDE.
  * Personalização do usuário: autentique usuários com Shared Session ou JWT para mostrar conteúdo personalizado, como pré-preencher chaves de API ou exibir conteúdo específico para clientes.
  * Upgrades de automação OpenAPI: para preencher automaticamente páginas de Playground de API, você pode adicionar um campo `openapi` a um objeto nas arrays de tabs ou anchors no `docs.json`.
</Update>

<Update label="April 2024">
  ## Modo personalizado

  Adicione uma configuração aos metadados para remover todos os elementos, exceto a barra superior. Exemplos de uso:

  * Criar uma landing page global personalizada com componentes customizados
  * Adicionar vídeos em tela cheia ou galerias de imagens
  * Incorporar elementos de demonstração em iFrame personalizados para adicionar interatividade à sua documentação

  Confira nossa [documentação do Modo personalizado](organize/pages#custom-mode).
</Update>
