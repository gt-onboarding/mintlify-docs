<Update label="24 de agosto - 30 de agosto" tags={["Improvements"]} rss={{ title: "Atualizações semanais", description: "Coleta de feedback aprimorada, melhorias na navegação e correções de bugs de autenticação" }}>
  ## Lançamento principal: Coleta de feedback aprimorada

  - **Grande melhoria**: Os leitores agora podem fornecer feedback mais detalhado após selecionar _polegar para cima/baixo_, incluindo opções e comentários escritos. Você também pode coletar feedback em blocos de código e visualizar todas as respostas nas análises do seu painel.\
    _Observação: Entre em contato conosco para habilitar este recurso no seu site._

  ## Melhorias de navegação e usabilidade

  - Melhoria de usabilidade para ignorar barras finais e barras duplas, para que você não precise se preocupar em acertá-las exatamente no seu docs.json
  - Agora você pode adicionar o atributo `noAnchor` às tags HTML `h1-6` para evitar que tenham link de âncora
  - O tema Palm agora tem um seletor de idioma flutuante no canto inferior esquerdo, semelhante à abordagem da Stripe
  - Adicionado um novo campo ao docs.json chamado [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups), que permite controlar se os usuários serão direcionados para a primeira página de um grupo de navegação quando o grupo for expandido
  - Melhoria de usabilidade para fazer listas ordenadas aninhadas alternarem estilos de decimal/romano para alfabético

  ## Correções de bugs e confiabilidade

  - Corrigido um bug em que links âncora de posição de rolagem não funcionavam corretamente quando havia componentes JS na página
  - Corrigido um bug em que o Google estava indexando arquivos `*.md` brutos por estarem sem o cabeçalho `x-robots-tag noindex`
  - Corrigido um bug com OAuth em docs protegidas que não redirecionava você de volta para a página de origem após concluir o fluxo com sucesso
  - Corrigido um bug nas prévias de docs protegidas por autenticação em que não era possível ver toda a barra de navegação
  - Correções de bugs no tratamento de SVGs com nossa nova CDN de imagens

  ## Aprimoramentos de componentes e estilo

  - Adicionado um novo seletor CSS para estilos personalizados em `SidebarNavGroupDivider`
  - Novos testes de regressão para páginas de API em MDX que têm segurança definida garantirão um maior grau de estabilidade

  ## Melhorias de desempenho

  - Melhoria de desempenho ao mover o CSS do KaTeX do cdnjs para nossa própria CDN na CloudFront, reduzindo a latência

<Update label="17 de agosto - 23 de agosto" tags={["Improvements"]} rss={{ title: "Atualizações semanais", description: "Correções de layout de imagem, aprimoramentos de autenticação, melhorias na navegação móvel e estabilidade do editor web" }}>
  ## Melhorias no tratamento de imagens

  - **Grande melhoria**: As imagens não causam mais deslocamento de layout por padrão, mesmo quando os atributos de largura e altura não estão especificados — o dimensionamento automático evita saltos de conteúdo durante o carregamento das páginas
  - Todos os arquivos estáticos do seu repositório (PDF, TXT, XML, etc.) agora são enviados e servidos automaticamente quando você faz o deploy, garantindo cobertura completa de assets

  ## Aprimoramentos no editor web e no deploy

  - Correção no fluxo de criação de branches no editor web para navegar corretamente para e permanecer nas branches recém-criadas
  - Janela de conflito de merge aprimorada com funcionalidade adequada de escape, sem necessidade de recarregar a página para dispensar conflitos
  - Otimização do fluxo de atualização ao invalidar o cache apenas das páginas alteradas durante atualizações parciais, reduzindo o tempo de deploy

  ## Melhorias de autenticação e navegação

  - Novo suporte para autenticação em subdiretórios personalizados; se você publica as docs em `https://yourdomain.com/docs`, a autenticação agora funciona perfeitamente
  - Corrigido um bug na exibição da barra lateral que aparecia incorretamente quando apenas um link estava configurado
  - Reformulação abrangente da navegação móvel: botões centralizados com margens/preenchimento adequados, espaçamento aprimorado em menus suspensos, remoção de divisores e margens desnecessárias para seções vazias e correção de gaps/preenchimento no tema Maple

  ## Correções de componentes e estilo

  - Resolvido o problema de conversão das tags `<h1-6>` que as transformava incorretamente em componentes Heading e comprometia estilos personalizados
  - Alternância de configuração do Assistente com um clique adicionada ao painel para facilitar o gerenciamento

  ## Melhorias técnicas e confiabilidade

  - Sistema de logging aprimorado para fluxos de atualização, permitindo depuração mais rápida e resolução de problemas
  - Correção de limitação de taxa do GitHub para clientes com 10+ especificações OpenAPI/AsyncAPI, substituindo a busca de arquivos individuais por clonagem do repositório
  - Confiabilidade do Assistente aprimorada com suporte a LLM de backup, tratamento aprimorado de erros de limite de taxa e funcionalidade de ferramenta de pesquisa mais robusta
</Update>

<Update label="10 de agosto - 16 de agosto" tags={["Improvements"]} rss={{ title: "Atualizações semanais", description: "Otimizações de desempenho, integrações com LLM, melhorias no editor web e aprimoramentos no playground de API" }}>
  ## Otimizações de desempenho e build

  - A transpilação de MDX agora ocorre no momento do deployment em vez de a cada carregamento de página em ambientes serverless do NextJS sem cache, melhorando o time to first byte para páginas não armazenadas em cache.
  - Hashing baseado em conteúdo evita retranspilação quando o MDX não mudou, reduzindo os tempos do fluxo de atualização em ~50% para clientes com grande número de páginas (deployments acima de 5 minutos devem ser aproximadamente reduzidos pela metade)
  - A visualização de deployments de preview no dashboard agora está mais rápida com índices de banco de dados e paralelização de consultas no nosso backend
  - Redução do tamanho da página ao eliminar dados `navigation` duplicados no payload `rsc` de cada página — ganhos de desempenho mais perceptíveis com grande número de páginas ou estruturas de navegação complexas
  - Pré-busca mais agressiva permite carregamentos instantâneos de página com mais frequência

  ## Aprimoramentos do Playground de API e OpenAPI

  - Conversão de OpenAPI para MCP movida para o backend, permitindo que servidores MCP hospedados contenham ferramentas (aguarde documentação e opções de configuração mais claras em breve)
  - Adicionado suporte a Ruby no Playground de API
  - Adicionamos um recurso para que você possa [especificar páginas de API usando apenas seu docs.json](/pt-BR/api-playground/openapi-setup#auto-populate-api-pages) sem criar novos arquivos MDX.
  - Suporte para páginas de [`webhook`](/pt-BR/api-playground/openapi-setup#webhooks) na navegação da documentação a partir de especificações OpenAPI
  - Contexto de modelo de IA otimizado ao remover especificações de links âncora de links em markdown ao navegar para Anthropic, OpenAI ou outros provedores

  ## Melhorias no editor web

  - A criação/renomeação de arquivos agora salva as alterações ao clicar fora, em vez de exigir pressionar a tecla Enter
  - Corrigida a navegação de branches em que alterar a URL para um branch específico redirecionava para o último branch ativo em vez do destino pretendido
  - Títulos de branches contendo `/` agora são devidamente codificados na URL para evitar quebras de navegação
  - Corrigido o atalho de link `Ctrl+K` no Editor do dashboard de monorepo, que estava prefixando o caminho do repositório de docs e criando links quebrados

  ## Integrações de Analytics e LLM

  - Suporte a `llms.txt` e `llms-full.txt` personalizados — adicione na raiz do repositório de docs para servir nos endpoints `/llms.txt` e `/llms-full.txt` para customização de LLM
  - Adicionada [integração de analytics Hightouch](/pt-BR/integrations/analytics/hightouch#hightouch)
  - Rastreamento de analytics do menu contextual aprimorado (visualização no dashboard em breve)
  - Adicionados testes e2e para `llms.txt` e `llms-full.txt` para garantir a entrega correta

  ## Aprimoramentos de componentes e estilização

  - Suporte a nomes de classe personalizados em tags `h{1-4}` para aplicar estilos de título personalizados
  - Corrigidas tags `h{1-4}` renderizando como componentes `Heading` com chips no modo de página personalizada
  - Seletores CSS adicionados a [breadcrumbs](/pt-BR/navigation#breadcrumbs) para direcionamento CSS personalizado
  - Corrigidas imagens open-graph esticadas ao analisar dimensões para manter as proporções com 56px de altura
  - Correção de `VSCode` para `VS Code` no menu contextual quando habilitado
  - Corrigido: títulos dentro de componentes personalizados aparecendo no sumário junto com títulos semânticos

  ## Correções de bugs e confiabilidade

  - Corrigidos problemas de renderização de PDF com certos títulos de página ao sanitizar caracteres que causavam falhas na geração
  - Corrigido erro do CLI `Cannot convert undefined or null to object` ao encontrar arquivos OpenAPI JSON vazios
  - Corrigidas metatags open-graph personalizadas de `docs.json` que estavam sendo sobrescritas pelas geradas
  - Corrigidos cliques no botão de feed RSS ao chegar em links âncora, usando origin + pathname para os links de RSS
  - Velocidade de download do CLI aprimorada ao remover sourcemaps

  ## Melhorias técnicas

  - Testes visuais adicionados ao pipeline de CI para detecção antecipada de regressões
  - Tratamento de erros e recursos de depuração aprimorados
  - Cobertura de testes abrangente para novos recursos e casos extremos

  ## Melhorias de autenticação

  - Acesso público em nível de grupo: torne grupos inteiros de páginas públicos via `docs.json`, assim você não precisa definir `public: true` em cada página ([saiba mais](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level))

  - Em erros de OAuth, os usuários são redirecionados para o `logoutURL` especificado para reiniciar o fluxo de autenticação
  - Corrigido um flash de erro 500 durante fluxos OAuth/JWT antes do callback
  - Remoção automática de `https://` de URLs na configuração de autenticação OAuth/JWT para evitar configurações incorretas

  - [`x-mcp` agora é `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp) nas configurações OpenAPI para controlar quais rotas são expostas como ferramentas MCP

  - Corrigida a cópia de uma página para a área de transferência no Safari especificando um `type` MIME em `navigator.write`

  - Corrigida uma regressão causada pelas correções de preview protegido por autenticação da semana passada, em que a aba ativa não estava sendo definida corretamente na UI de navegação

  ## Novo esquema de configuração `docs.json`

  <Frame>
    ![docs.json screenshot](/images/changelog/docsjson.png)
  </Frame>

  Apresentamos um novo esquema `docs.json` como substituto de `mint.json`, para oferecer melhor versionamento em vários níveis, compreensão visual mais fácil e terminologia mais consistente. Para mais informações sobre o que mudou, [confira nosso blog](https://mintlify.com/blog/refactoring-mint-json-into-docs-json).

  Atualize de `mint.json` para `docs.json` com as etapas a seguir:

  1. Certifique-se de que o seu CLI está na versão mais recente

  ```bash
  npm i mint@latest -g
  ```

  1. No repositório da sua documentação, execute

  ```bash
  mint upgrade
  ```

  1. Exclua o seu arquivo `mint.json` antigo e faça push das suas alterações
