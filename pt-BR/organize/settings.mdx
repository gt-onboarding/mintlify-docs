O arquivo `docs.json` permite transformar uma coleção de arquivos Markdown em um site de documentação navegável e personalizado. Esse arquivo de configuração obrigatório controla o estilo, a navegação, as integrações e muito mais. Pense nele como o plano diretor da sua documentação.

As configurações em `docs.json` se aplicam globalmente a todas as páginas.

<div id="setting-up-your-docsjson">
  ## Configurando seu `docs.json`
</div>

Para começar, você precisa apenas especificar `theme`, `name`, `colors.primary` e `navigation`. Os demais campos são opcionais e podem ser adicionados conforme as necessidades da sua documentação aumentarem.

Para a melhor experiência de edição, inclua a referência do schema no início do seu arquivo `docs.json`. Isso habilita autocompletar, validação e dicas úteis na maioria dos editores de código:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Sua Documentação",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Sua estrutura de navegação
  }
  // O restante de sua configuração
}
```

<ResponseField name="theme" required>
  O tema de layout do seu site.

  Um dos seguintes: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Veja [Temas](/pt-BR/customize/themes) para mais informações.
</ResponseField>

<ResponseField name="name" type="string" required>
  O nome do seu projeto, organização ou produto.
</ResponseField>

<ResponseField name="colors" type="object" required>
  As cores usadas na sua documentação. As cores são aplicadas de forma diferente em cada tema. Se você fornecer apenas uma cor primária, ela será usada para todos os elementos de cor.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      A cor primária da sua documentação. Geralmente usada para ênfase no modo claro, com alguma variação por tema.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A cor usada para ênfase no modo escuro.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A cor usada para botões e estados de hover nos modos claro e escuro, com alguma variação por tema.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      Configuração de tema para blocos de código. O padrão é `"system"`.

      **Configuração simples:**

      * `"system"`: Acompanha o modo atual do site (claro ou escuro)
      * `"dark"`: Sempre usar o modo escuro

      **Configuração de tema personalizada:**

      * Use uma string para especificar um único [tema Shiki](https://shiki.style/themes) para todos os blocos de código
      * Use um objeto para especificar [temas Shiki](https://shiki.style/themes) distintos para os modos claro e escuro

      <ResponseField name="theme" type="string">
        Um único nome de tema Shiki para usar tanto no modo claro quanto no escuro.

        ```json

          "codeblocks": {
            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

        Temas distintos para os modos claro e escuro.

        <Expandable title="theme">
          <ResponseField name="light" type="string" required>
            Um nome de tema Shiki para o modo claro.
          </ResponseField>

          <ResponseField name="dark" type="string" required>
            Um nome de tema Shiki para o modo escuro.
          </ResponseField>

          ```json
          "styling": {
            "codeblocks": {
              "theme": {
                "light": "github-light",
                "dark": "github-dark"
              }
            }
          }
          ```
        </Expandable>
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Atualizando a partir de `mint.json`
</div>

Se o seu projeto de documentação usa o arquivo `mint.json` descontinuado, siga estas etapas para migrar para o `docs.json`.

<Steps>
  <Step title="Instalar ou atualizar a CLI">
    Se você ainda não instalou a [CLI](/pt-BR/installation), instale agora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Se você já tem a CLI instalada, certifique-se de que ela esteja atualizada:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Criar o arquivo docs.json">
    No seu repositório de documentação, execute:

    ```bash
    mint upgrade
    ```

    Esse comando criará um arquivo `docs.json` a partir do seu `mint.json` atual. Revise o arquivo gerado para garantir que todas as configurações estejam corretas.
  </Step>

  <Step title="Excluir o arquivo mint.json">
    Depois de verificar que o `docs.json` está configurado corretamente, você pode excluir com segurança o antigo arquivo `mint.json`.
  </Step>
</Steps>
