Otimize sua documentação tanto para mecanismos de busca tradicionais quanto para mecanismos de resposta com IA, como ChatGPT, Perplexity e Google AI Overviews.

A Otimização para Motores Generativos (GEO) prioriza ser citada por sistemas de IA por meio de conteúdo abrangente e informações estruturadas, enquanto o SEO tradicional visa melhorar o ranqueamento nos resultados de busca.

<div id="geo-quickstart">
  ## Introdução rápida ao GEO
</div>

<div id="initial-setup">
  ### Configuração inicial
</div>

1. **Garanta que sua documentação esteja sendo indexada** nas configurações do seu `docs.json`
2. **Revise as páginas atuais** para identificar títulos e descrições ausentes

<div id="content-improvements">
  ### Melhorias de conteúdo
</div>

1. **Adicionar tabelas de comparação** às páginas relevantes
2. **Revisar títulos** para garantir que respondam às dúvidas mais comuns
3. **Melhorar os links internos** entre tópicos relacionados
4. **Testar com ferramentas de IA** para verificar a precisão

<div id="geo-best-practices">
  ## Melhores práticas de GEO
</div>

Em geral, documentação bem escrita e bem estruturada terá um GEO sólido. Ainda assim, priorize escrever para seus usuários e, se o seu conteúdo estiver atendendo às necessidades deles, você estará no caminho certo para otimizar para ferramentas de IA. Criar conteúdo genuinamente útil, em vez de otimizar por otimizar, é recompensado tanto por mecanismos de busca tradicionais quanto por mecanismos de IA.

Concentre-se em:

- Conteúdo alinhado às necessidades do usuário, e não em correspondência de palavras-chave
- Informações estruturadas e fáceis de percorrer
- Respostas diretas às perguntas

<div id="format-for-clarity">
  ### Formato para clareza
</div>

Estas práticas de formatação ajudam as ferramentas de IA a analisar e entender seu conteúdo:

- Não salte níveis de cabeçalho (H1 → H2 → H3)
- Use nomes específicos de objetos em vez de “it” ou “this”
- Rotule os blocos de código com sua linguagem de programação
- Forneça texto alternativo descritivo para as imagens
- Inclua links para conceitos relacionados para ajudar a IA a entender os relacionamentos

<div id="answer-questions-directly">
  ### Responda a perguntas diretamente
</div>

Escreva conteúdo que atenda a perguntas específicas dos usuários:

- Comece as seções com o principal ponto a ser levado
- Use títulos descritivos que correspondam às dúvidas mais comuns
- Divida tópicos complexos em etapas numeradas

<div id="mintlify-configuration">
  ## Configuração da Mintlify
</div>

Use estes recursos para melhorar o GEO.

<div id="add-descriptive-page-metadata">
  ### Adicione metadados descritivos à página
</div>

Inclua títulos e descrições claros no frontmatter:

```mdx
---
title: "Guia de autenticação de API"
description: "Guia completo para implementar autenticação de API com exemplos de código"
---
```

<div id="configure-global-indexing-settings">
  ### Configurar configurações globais de indexação
</div>

Adicione ao seu `docs.json`:

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

Arquivos LLMs.txt ajudam sistemas de IA a entender a estrutura da sua documentação, de modo semelhante a como sitemaps ajudam mecanismos de busca. A Mintlify gera automaticamente arquivos LLMs.txt para a sua documentação. Nenhuma configuração é necessária.

<div id="testing-your-documentation">
  ## Testando sua documentação
</div>

Teste várias ferramentas de IA com perguntas sobre seu produto e sua documentação para ver quão bem sua documentação está sendo citada.

**Faça perguntas específicas aos assistentes de IA sobre sua documentação:**

- "Como configuro a autenticação usando esta API?"
- "Explique o processo de instalação passo a passo"

**Verifique se as ferramentas fornecem:**

- Exemplos de código corretos
- Instruções passo a passo precisas
