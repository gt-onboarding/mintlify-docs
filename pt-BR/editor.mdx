Acesse o editor visual pelo seu [dashboard](https://dashboard.mintlify.com/editor) para gerenciar sua documentação diretamente no navegador.

* **Edição WYSIWYG**: Faça alterações na sua documentação usando um editor WYSIWYG (what you see is what you get) que mostra como seu conteúdo ficará quando publicado.
* **Sincronização com Git**: Todas as alterações são sincronizadas automaticamente com seu repositório Git para manter o controle de versões.
* **Colaboração em equipe**: Várias pessoas podem trabalhar na documentação simultaneamente.
* **Integração de componentes**: Adicione destaques, blocos de código e outros componentes com comandos de barra (/).
* **Sem necessidade de configuração**: Comece a escrever imediatamente a partir do seu dashboard.

<div id="overview">
  ## Visão geral
</div>

Veja como você normalmente vai trabalhar no editor visual:

<Steps>
  <Step title="Escolha sua branch">
    Crie uma branch ou faça alterações diretamente na branch de deploy. Recomendamos criar uma branch para que você possa visualizar suas alterações antes de publicá-las.
  </Step>

  <Step title="Abra seu arquivo">
    Navegue até um arquivo existente na barra lateral ou crie um novo usando o explorador de arquivos.
  </Step>

  <Step title="Edite seu conteúdo">
    Faça alterações no editor visual. Pressione <kbd>/</kbd> para abrir o menu de componentes.
  </Step>

  <Step title="Pré-visualize suas alterações">
    O modo visual mostra como suas alterações aparecerão no seu site em produção. Use-o para verificar se tudo está correto.
  </Step>

  <Step title="Publique suas alterações">
    Se você estiver trabalhando na branch de deploy, publique suas alterações diretamente do editor visual. Em outras branches, crie um pull request para revisão antes da publicação.
  </Step>
</Steps>

</div>

O modo visual oferece uma experiência WYSIWYG, em que as mudanças feitas no editor refletem exatamente como sua documentação publicada será exibida. Esse modo é ideal para visualizar em tempo real como suas alterações ficarão.

<div id="organize-your-navigation">
  ### Organize sua navegação
</div>

Edite o arquivo `docs.json` para adicionar novas páginas e remover páginas excluídas da navegação do seu site. Consulte [Navigation](/pt-BR/organize/navigation) para mais informações sobre como organizar páginas.

**Exemplo: adicionar uma página Themes ao grupo Profile**

Neste exemplo, você criou uma nova página intitulada Themes e deseja adicioná-la ao grupo Profile na sua documentação. Adicione o caminho da nova página ao array `pages` do grupo `Profile` no arquivo `docs.json` para que ela apareça na navegação do seu site.

```json title="Adding a Themes page to the Profile group" highlight="18"
{
    "navigation": {
        "groups": [
            {
                "group": "Primeiros passos",
                "pages": [
                    "index",
                    "quickstart",
                    "installation"
                ]
            },
            {
                "group": "Perfil",
                "pages": [
                    "settings",
                    "account-types",
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```

<div id="edit-content">
  ### Editar conteúdo
</div>

Faça alterações nas suas páginas usando o modo visual ou o modo Markdown no editor.

No modo visual, pressione <kbd>/</kbd> para abrir o menu de componentes. Adicione blocos de conteúdo, avisos, blocos de código e outros componentes para personalizar sua documentação.

<Frame>
  <img src="/images/editor/component-menu-light.png" alt="O menu de componentes expandido em destaque no editor visual do Mintlify" className="block dark:hidden" />

  <img src="/images/editor/component-menu-dark.png" alt="O menu de componentes expandido em destaque no editor visual do Mintlify" className="hidden dark:block" />

No modo Markdown, você edita diretamente o `MDX` das suas páginas. Isso pode ser útil quando você precisa:

* Definir propriedades específicas de componentes
* Trabalhar com componentes aninhados complexos
* Copiar e colar conteúdo `MDX` de outras fontes

Consulte [Formatar texto](/pt-BR/create/text) e [Formatar código](/pt-BR/create/code) para mais informações sobre como escrever usando a sintaxe Markdown.

<div id="publish-your-changes">
  ## Publique suas alterações
</div>

O branch em que você está trabalhando determina como o editor publica suas alterações:

* **Branch de deploy**: Ao publicar, seu site em produção é atualizado imediatamente.
* **Outros branches**: Ao publicar, é criado um pull request para você revisar as alterações antes de enviá-las para produção.

<div id="pull-requests-and-reviewing-changes">
  ### Pull requests e revisão de mudanças
</div>

Pull requests permitem propor mudanças a partir da sua branch para que outras pessoas possam revisá-las antes de fazer o merge na sua documentação em produção. Isso ajuda a garantir que suas mudanças estejam corretas e dá à sua equipe a oportunidade de colaborar no conteúdo.

<Tip>
  Mesmo trabalhando sozinho, os pull requests são valiosos para pré-visualizar mudanças e manter um histórico claro de atualizações.
</Tip>

<div id="create-a-pull-request">
  #### Crie um pull request
</div>

<Steps>
  <Step title="Salve seu trabalho">
    Selecione **Save Changes** para salvar todas as alterações no seu branch.
  </Step>

  <Step title="Crie o pull request">
    Selecione **Publish Pull Request** na barra de ferramentas do editor.
  </Step>

  <Step title="Adicione um título e uma descrição">
    Escreva um título e uma descrição claros explicando:

    * Quais alterações você fez
    * Por que as fez
    * Quais áreas específicas precisam de revisão
  </Step>

  <Step title="Crie e compartilhe">
    Selecione **Publish Pull Request**. O editor fornecerá um link para visualizar seu pull request.

    <Frame>
      <img src="/images/editor/pull-request-light.png" alt="Botão Publish Pull Request destacado no editor visual do Mintlify" className="block dark:hidden" />

      <img src="/images/editor/pull-request-dark.png" alt="Botão Publish Pull Request destacado no editor visual do Mintlify" className="hidden dark:block" />
    </Frame>
  </Step>
</Steps>

<div id="review-pull-requests">
  #### Revisar pull requests
</div>

Depois que seu pull request for criado:

1. **Revisar alterações**: Você e os membros da sua equipe podem revisar seu pull request no provedor Git, como GitHub ou GitLab.
2. **Deixar feedback**: Adicione comentários ou solicite alterações.
3. **Fazer alterações adicionais**: Faça alterações adicionais no editor visual. Ao salvar, o editor envia as mudanças para o seu pull request.
4. **Aprovar**: Aprove o pull request quando estiver satisfeito com as alterações.
5. **Fazer merge**: Faça o merge do pull request quando estiver pronto para implementar suas alterações em produção.

<div id="troubleshooting">
  ## Solução de problemas
</div>

Encontre soluções para problemas comuns que você pode enfrentar ao usar o editor visual.

<AccordionGroup>
  <Accordion title="Alterações não aparecem após a publicação">
    **Causas possíveis:**

    * A implantação ainda está em andamento
    * Problemas de cache do navegador
    * Erros de build ou de implantação

    **Soluções:**

    1. Verifique o status da implantação no seu painel.
    2. Faça uma atualização forçada no navegador (<kbd>Ctrl</kbd> + <kbd>F5</kbd> ou <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>R</kbd>)
    3. Limpe o cache do seu navegador.
  </Accordion>

  <Accordion title="Erros de permissão ao publicar">
    **Causas possíveis:**

    * Permissões insuficientes no repositório Git
    * Problemas de autenticação com o seu provedor Git

    **Soluções:**

    1. Verifique se você tem o acesso correto ao repositório.
    2. Confira se a sua integração com o Git está configurada corretamente.
    3. Consulte a documentação de [Permissões do Editor](/pt-BR/dashboard/permissions).
  </Accordion>

  <Accordion title="Problemas ao carregar o editor">
    **Causas possíveis:**

    * Problemas de conectividade de rede
    * Repositórios de documentação muito grandes

    **Soluções:**

    1. Verifique sua conexão com a internet.
    2. Recarregue a página.
    3. Entre em contato com o suporte se o problema persistir.
  </Accordion>

  <Accordion title="Arquivos não carregam ou exibem erros">
    **Causas possíveis:**

    * Sintaxe MDX inválida nos arquivos
    * Arquivos ausentes ou corrompidos
    * Arquivos muito grandes causando estouro de tempo (timeout)

    **Soluções:**

    1. Verifique a sintaxe do arquivo em busca de erros de formatação em MDX.
    2. Verifique se o arquivo existe no seu repositório.
  </Accordion>
</AccordionGroup>
