<div id="verceljson-file">
  ## arquivo vercel.json
</div>

O arquivo `vercel.json` é o arquivo de configuração da Vercel que permite personalizar como seu projeto é compilado e implantado. Ele fica no diretório raiz do seu projeto e controla vários aspectos da implantação, incluindo roteamento, redirecionamentos, cabeçalhos e configurações de build.

Usamos a configuração `rewrites` para fazer proxy de solicitações do seu domínio principal para a sua documentação.

As rewrites permitem mapear solicitações recebidas para destinos diferentes sem alterar a URL no navegador. Quando alguém visita `yoursite.com/docs`, a Vercel buscará internamente conteúdo de `your-subdomain.mintlify.dev/docs`, mas o usuário ainda verá `yoursite.com/docs` no navegador. Isso é diferente de redirecionamentos, que enviariam os usuários para uma URL completamente diferente.

<div id="configuration">
  ## Configuração
</div>

Para hospedar sua documentação em um subcaminho personalizado `/docs` usando a Vercel, adicione a seguinte configuração ao arquivo `vercel.json`:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

- **`source`**: O padrão de rota no seu domínio que aciona o rewrite.
- **`destination`**: O destino para o qual a solicitação deve ser encaminhada (proxy).
- **`:match*`**: Um curinga que captura quaisquer segmentos de caminho após `/docs/`.

Para mais informações, consulte [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) na documentação da Vercel.

<div id="using-external-proxies-with-vercel">
  ## Uso de proxies externos com a Vercel
</div>

Se você usa um proxy externo (como Cloudflare ou AWS CloudFront) à frente da sua implantação na Vercel, é necessário configurá-lo corretamente para evitar conflitos com a verificação de domínio da Vercel e o provisionamento de certificados SSL.

Uma configuração incorreta do proxy pode impedir a Vercel de provisionar certificados SSL do Let's Encrypt e causar falhas na verificação de domínio.

Consulte os [provedores compatíveis](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) na documentação da Vercel.

<div id="required-path-allowlist">
  ### Lista obrigatória de caminhos permitidos
</div>

Seu proxy externo deve permitir o tráfego para estes caminhos específicos sem bloquear, redirecionar ou aplicar cache agressivo:

- `/.well-known/acme-challenge/*` - Necessário para a verificação de certificado do Let's Encrypt
- `/.well-known/vercel/*` - Necessário para a verificação de domínio da Vercel

Esses caminhos devem passar diretamente para sua implantação na Vercel, sem modificações.

<div id="header-forwarding-requirements">
  ### Requisitos para encaminhamento de cabeçalhos
</div>

Certifique-se de que seu proxy encaminhe corretamente o cabeçalho `Host`. Sem o encaminhamento apropriado do cabeçalho, as solicitações de verificação falharão.

<div id="testing-your-proxy-setup">
  ### Testando sua configuração de proxy
</div>

Para verificar se o proxy está configurado corretamente:

1. Teste se `https://[yourdomain].com/.well-known/vercel/` retorna uma resposta.
2. Certifique-se de que os certificados SSL estão sendo provisionados corretamente no seu painel da Vercel.
3. Verifique se a verificação de domínio é concluída com sucesso.
