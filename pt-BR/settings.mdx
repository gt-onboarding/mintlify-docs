O arquivo `docs.json` permite transformar uma coleção de arquivos Markdown em um site de documentação navegável e personalizado. Esse arquivo de configuração obrigatório controla o estilo, a navegação, as integrações e muito mais. Pense nele como o plano da sua documentação.

As configurações em `docs.json` se aplicam globalmente a todas as páginas.

<div id="setting-up-your-docsjson">
  ## Configurando o seu `docs.json`
</div>

Para começar, basta especificar `theme`, `name`, `colors.primary` e `navigation`. Os demais campos são opcionais e você pode adicioná-los conforme as necessidades da sua documentação evoluem.

Para uma melhor experiência de edição, inclua a referência de schema no início do arquivo `docs.json`. Isso ativa o preenchimento automático, a validação e dicas úteis na maioria dos editores de código:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Sua estrutura de navegação
  }
  // O restante da sua configuração
}
```

<ResponseField name="theme" required>
  O tema de layout do seu site.

  Um dos seguintes: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Consulte [Temas](pt-BR/themes) para mais informações.
</ResponseField>

<ResponseField name="name" type="string" required>
  O nome do seu projeto, organização ou produto.
</ResponseField>

<ResponseField name="colors" type="object" required>
  As cores usadas na sua documentação. As cores são aplicadas de maneira diferente em cada tema. Se você fornecer apenas uma cor primária, ela será usada para todos os elementos de cor.

  <Expandable title="Cores">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      A cor primária da sua documentação. Geralmente usada para ênfase no modo claro, com alguma variação por tema.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A cor usada para ênfase no modo escuro.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A cor usada para botões e estados de foco/hover nos modos claro e escuro, com alguma variação por tema.

      Deve ser um código hexadecimal começando com `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Atualizando a partir de `mint.json`
</div>

Se o seu projeto de documentação usa o arquivo `mint.json` descontinuado, siga estas etapas para atualizar para `docs.json`.

<Steps>
  <Step title="Instalar ou atualizar a CLI">
    Se você ainda não instalou a [CLI](/pt-BR/installation), instale agora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Se você já tem a CLI instalada, certifique-se de que ela esteja atualizada:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Criar o arquivo docs.json">
    No repositório da sua documentação, execute:

    ```bash
    mint upgrade
    ```

    Esse comando criará um arquivo `docs.json` a partir do `mint.json` existente. Revise o arquivo gerado para garantir que todas as configurações estejam corretas.
  </Step>

  <Step title="Excluir o arquivo mint.json">
    Depois de verificar que o `docs.json` está configurado corretamente, você pode excluir com segurança o antigo arquivo `mint.json`.
  </Step>
</Steps>
