Lorsque vous modifiez le chemin d’un fichier dans votre dossier de docs, l’URL de cette page change également. Cela peut se produire lors d’une restructuration de vos docs ou d’une modification du titre dans la barre latérale.

<div id="broken-links">
  ## Liens cassés
</div>

Détectez les liens cassés avec notre CLI. [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

La CLI repère tous les liens relatifs dans votre documentation qui pointent vers des ressources inexistantes.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Cela redirige définitivement `/source/path` vers `/destination/path`, afin de ne pas perdre le référencement existant de la page d’origine.

Pour faire correspondre un chemin avec un joker, utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
