Lorsque vous modifiez le chemin d’un fichier dans votre dossier docs, l’URL de cette page change également. Cela peut se produire lors d’une restructuration de la documentation ou d’une modification du titre de la barre latérale.

<div id="broken-links">
  ## Liens brisés
</div>

Repérez les liens brisés avec notre CLI. [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

La CLI détecte tous les liens relatifs dans votre documentation qui pointent vers des ressources inexistantes.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Cela redirige de manière permanente `/source/path` vers `/destination/path`, afin de ne pas perdre le référencement (SEO) acquis par la page d’origine.

Pour faire correspondre un chemin générique (wildcard), utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et la redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
