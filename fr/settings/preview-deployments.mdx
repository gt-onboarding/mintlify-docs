</Info>

Les aperçus de déploiement vous permettent de voir l’apparence des modifications apportées à votre documentation avant leur fusion en production. Chaque aperçu génère une URL partageable qui se met automatiquement à jour à mesure que vous poussez de nouveaux changements.

<div id="creating-preview-deployments">
  ## Création de déploiements de prévisualisation
</div>

Les déploiements de prévisualisation sont créés automatiquement via des pull requests ou manuellement depuis votre tableau de bord.

<div id="automatic-previews">
  ### Aperçus automatiques
</div>

Lorsque vous créez une pull request, le bot Mintlify ajoute automatiquement un lien pour consulter le déploiement de prévisualisation dans votre pull request. Cette prévisualisation se met à jour à chaque nouveau commit poussé sur la branche.

<div id="manual-previews">
  ### Aperçus manuels
</div>

Vous pouvez créer manuellement un aperçu pour n’importe quelle branche.

1. Accédez à votre [tableau de bord](https://dashboard.mintlify.com/).
2. Sélectionnez **Aperçus**.
3. Sélectionnez **Créer un aperçu**.
4. Saisissez le nom de votre branche.
5. Sélectionnez **Créer un déploiement**.

<div id="redeploying-a-preview">
  ## Redéployer un aperçu
</div>

Redéployez un aperçu pour actualiser le contenu ou réessayer après un échec de déploiement.

1. Sélectionnez l’aperçu depuis votre [tableau de bord](https://dashboard.mintlify.com/).
2. Sélectionnez **Redéployer**.

<div id="sharing-preview-deployments">
  ## Partage des déploiements de prévisualisation
</div>

Les déploiements de prévisualisation peuvent être accessibles publiquement ou réservés aux utilisateurs authentifiés.

**Accès public** : Les URL de prévisualisation sont visibles publiquement par défaut. Partagez un lien de prévisualisation avec toute personne devant examiner vos modifications.

**Accès restreint** : Activez l’authentification des déploiements de prévisualisation pour limiter l’accès aux seuls utilisateurs Mintlify authentifiés. Voir [Configuration de l’authentification](/fr/authentication-personalization/authentication-setup) pour plus d’informations.
