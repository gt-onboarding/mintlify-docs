Chaque page est un fichier MDX qui associe du contenu Markdown à des composants React pour vous permettre de créer une documentation riche et interactive.

<div id="page-metadata">
  ## Métadonnées de page
</div>

Chaque page commence par un frontmatter, les métadonnées YAML délimitées par `---` au début du fichier. Ces métadonnées définissent l’apparence et le comportement de votre page.

Utilisez le frontmatter pour contrôler :

* Titres et descriptions de page
* Titres, icônes et tags de la barre latérale
* Mises en page
* Balises meta SEO
* Métadonnées personnalisées

<ResponseField name="title" type="string" required>
  Le titre de votre page qui apparaît dans la navigation et les onglets du navigateur.
</ResponseField>

<ResponseField name="description" type="string">
  Une brève description du contenu de la page. S’affiche sous le titre et améliore le SEO.
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  Un titre court affiché dans la navigation de la barre latérale.
</ResponseField>

<ResponseField name="icon" type="string">
  L’icône à afficher.

  Options :

  * Nom d’[icône Font Awesome](https://fontawesome.com/icons)
  * Nom d’[icône Lucide](https://lucide.dev/icons)
  * URL d’une icône hébergée à l’externe
  * Chemin vers un fichier d’icône dans votre projet
</ResponseField>

<ResponseField name="iconType" type="string">
  Le style d’icône [Font Awesome](https://fontawesome.com/icons). Utilisé uniquement avec les icônes Font Awesome.

  Options : `regular`, `solid`, `light`, `thin`, `sharp-solid`, `duotone`, `brands`.
</ResponseField>

<ResponseField name="tag" type="string">
  Un tag qui apparaît à côté du titre de votre page dans la barre latérale.
</ResponseField>

<ResponseField name="<custom>" type="string">
  Tout frontmatter YAML valide. Par exemple : `product: "API"` ou `version: "1.0.0"`.
</ResponseField>

```yaml Example YAML frontmatter wrap
---
title: "À propos du frontmatter"
description: "Le frontmatter est l'ensemble des métadonnées qui contrôlent l'apparence et le comportement de votre page"
sidebarTitle: "Frontmatter"
icon: "book"
tag: "NEW"
---
```

<div id="page-mode">
  ## Mode d’affichage
</div>

Contrôlez la façon dont votre page s’affiche grâce au paramètre `mode`.

<div id="default">
  ### Par défaut
</div>

Si aucun mode n’est défini, une mise en page standard avec une barre latérale de navigation et une table des matières est utilisée.

```yaml
---
title: "Titre de page par défaut"
---
```

<div id="wide">
  ### Large
</div>

Le mode large masque la table des matières. C’est utile pour les pages sans titres ou si vous préférez profiter de l’espace horizontal supplémentaire. Le mode large est disponible pour tous les thèmes.

```yaml
---
title: "Titre de page large"
mode: "wide"
---
```

<div id="custom">
  ### Personnalisé
</div>

Le mode personnalisé offre une mise en page minimaliste qui supprime tous les éléments à l’exception de la barre de navigation supérieure. C’est une page blanche pour créer des pages de destination ou tout autre type de mise en page unique nécessitant une navigation réduite au minimum. Le mode personnalisé est disponible pour tous les thèmes.

```yaml
---
title: "Titre de page personnalisé"
mode: "custom"
---
```

<div id="frame">
  ### Frame
</div>

Le mode Frame propose une mise en page proche du mode personnalisé, tout en conservant la navigation de la barre latérale. Ce mode de page permet d’utiliser du HTML et des composants personnalisés tout en préservant l’expérience de navigation par défaut. Le mode Frame est uniquement disponible avec les thèmes Aspen et Almond.

```yaml
---
title: "Titre de la page Frame"
mode: "frame"
---
```

<div id="center">
  ### Centré
</div>

Le mode Centré supprime la barre latérale et la table des matières, et centre le contenu. Il est utile pour les journaux de modifications (changelogs) ou d’autres pages où vous souhaitez mettre l’accent sur le contenu. Le mode Centré est disponible avec les thèmes Mint et Linden.

```yaml
---
title: "Titre de page centré"
mode: "center"
---
```

<div id="api-pages">
  ## Pages d’API
</div>

Créez des bacs à sable API interactifs en ajoutant une spécification d’API à votre frontmatter, `api` ou `openapi`.

```yaml
---
openapi: "GET /endpoint"
---
```

En savoir plus sur la création de [documentation d’API](/fr/api-playground/overview).

<div id="external-links">
  ## Liens externes
</div>

Ajoutez des liens vers des sites externes directement dans votre navigation à l’aide de la métadonnée `url`.

```yaml
---
title: "Package npm"
url: "https://www.npmjs.com/package/mint"
---
```

<div id="search-engine-optimization">
  ## Optimisation pour les moteurs de recherche
</div>

La plupart des balises méta SEO sont générées automatiquement. Vous pouvez définir manuellement des balises méta pour améliorer le référencement de votre site, le partage sur les réseaux sociaux et la compatibilité des navigateurs.

<Note>
  Les balises méta contenant des deux-points doivent être entourées de guillemets.
</Note>

```yaml
---
"twitter:image": "/images/social-preview.jpg"
---
```

Consultez [SEO](/fr/optimize/seo) pour la liste complète des options de métadonnées SEO.

<div id="internal-search-keywords">
  ## Mots-clés de recherche internes
</div>

Améliorez la découvrabilité d’une page spécifique dans la recherche intégrée en ajoutant des `keywords` à vos métadonnées. Ces mots-clés n’apparaissent pas dans le contenu de la page ni dans les résultats de recherche, mais lorsque des utilisateurs recherchent ces termes, la page leur est proposée en résultat.

```yaml
---
keywords: ['configuration', 'installation', 'premiers pas']
---
```
