<Info>
  L’agent est disponible dans les [offres Pro et Custom](https://mintlify.com/pricing?ref=agent) pour toute personne ayant accès à votre tableau de bord.
</Info>

L’agent crée des pull requests proposant des modifications à votre documentation en fonction de vos requêtes. Lorsque vous lui soumettez une demande, il s’appuie sur votre documentation, vos dépôts connectés et vos messages Slack pour produire un contenu conforme aux bonnes pratiques de la rédaction technique et au schéma Mintlify. Accédez à l’agent depuis votre espace de travail Slack ou intégrez-le à des applications personnalisées via l’API.

Utilisez l’agent pour :

* Rédiger du nouveau contenu à partir de vos requêtes, de liens vers des pull requests ou de fils de discussion Slack
* Mettre à jour des exemples de code et des références d’API obsolètes
* Rechercher et actualiser le contenu existant
* Répondre aux questions sur votre documentation et sur des sujets de rédaction technique

Pour commencer, ajoutez l’agent à votre espace de travail Slack et mentionnez-le avec `@mintlify` dans un canal.

<div id="add-the-agent-to-your-slack-workspace">
  ## Ajouter l’agent à votre espace de travail Slack
</div>

<Note>
  Si le propriétaire de votre espace de travail Slack exige une approbation d’un administrateur pour installer des applications, demandez-lui d’approuver l’application Mintlify avant de la connecter.
</Note>

1. Accédez à la page [agent](https://dashboard.mintlify.com/products/agent) de votre tableau de bord.
2. Cliquez sur le bouton **Connect**.
3. Suivez les instructions Slack pour ajouter l’application `mintlify` à votre espace de travail.
4. Suivez les instructions Slack pour lier votre compte Mintlify à votre espace de travail Slack.
5. Vérifiez que l’agent fonctionne et répond lorsque vous :
   * Lui envoyez un message direct.
   * Le mentionnez avec `@mintlify` dans un canal.

<div id="connect-repositories-as-context">
  ## Connecter des dépôts comme contexte
</div>

L’agent ne peut accéder qu’aux dépôts que vous connectez via l’application GitHub Mintlify. Modifiez les dépôts auxquels l’agent a accès dans les [paramètres de l’application GitHub](https://github.com/apps/mintlify/installations/new).

<div id="write-effective-prompts">
  ## Rédiger des prompts efficaces
</div>

Considérez l’agent comme un Assistant utile qui a besoin de votre orientation pour accomplir des tâches. Donnez-lui des instructions claires et du contexte. Des tâches plus ciblées sont plus faciles à réaliser, donc découpez les projets complexes en étapes plus petites.

Rendez vos prompts spécifiques et orientés vers les résultats. Des prompts génériques comme `@mintlify Improve the onboarding page` appliquent de bonnes pratiques générales, mais peuvent ne pas améliorer le contenu de la manière précise que vous aviez en tête.

Essayez des prompts fondés sur les résultats que vous souhaitez que vos utilisateurs atteignent ou sur les problèmes qu’ils rencontrent. Par exemple :

* `@mintlify A lot of users have trouble installing the CLI. Review the onboarding page and update the docs so that users can easily install the CLI`
* `@mintlify Developers keep getting 401 errors when following our authentication guide. Review the auth docs and add clearer examples showing how to properly format the API key`

Utilisez des prompts plus larges pour la maintenance générale du contenu, comme corriger les fautes de frappe, mettre à jour les redirections ou renommer une fonctionnalité dans l’ensemble de votre documentation. Par exemple :

* `@mintlify Find and fix all typos in the docs`
* `@mintlify change all unordered lists to use * instead of -`

<div id="specify-a-subdomain">
  ## Spécifier un sous-domaine
</div>

Si vous avez plusieurs sites de documentation avec leurs propres sous-domaines, incluez le paramètre `subdomain` dans votre message pour préciser sur quel sous-domaine l’agent doit intervenir.

Utilisez le format `@mintlify subdomain=<your-subdomain> <your-prompt>` pour demander à l’agent d’agir sur un sous-domaine spécifique.

Exemples :

* `@mintlify subdomain=public-docs Add a new section to the quickstart about inviting collaborators based on this PR` : Demande à l’agent de mettre à jour le quickstart uniquement sur le sous-domaine `public-docs`.
* `@mintlify subdomain=customer-docs Update the auth docs for the new authentication method` : Demande à l’agent de mettre à jour la documentation d’authentification uniquement sur le sous-domaine `customer-docs`.

<div id="agent-workflows">
  ## Workflows de l’agent
</div>

L’agent prend en charge de nombreuses tâches de documentation. Ces workflows montrent différentes manières d’intégrer l’agent à votre processus de documentation. Choisissez une approche adaptée au fonctionnement actuel de votre équipe et ajustez-la à vos besoins spécifiques.

<div id="iterate-on-a-prompt-in-a-slack-thread">
  ### Itérez sur une invite dans un fil Slack
</div>

Faites une demande à l’agent, puis continuez à le mentionner avec `@mintlify` dans le même fil pour affiner et itérer sur la pull request qu’il crée.

Par exemple : `@mintlify Notre page de démarrage rapide a besoin d’une nouvelle section sur l’invitation de collaborateurs`. Puis : `@mintlify La nouvelle section devrait s’appeler "Inviter des collaborateurs"`. Suivi de toute autre itération.

<div id="start-with-the-agent-finish-manually">
  ### Commencez avec l’agent, terminez manuellement
</div>

Demandez à l’agent de lancer un projet, puis récupérez la branche qu’il crée et finalisez la tâche dans votre environnement local ou l’éditeur web. L’agent peut vous aider à démarrer, puis vous pouvez prendre le relais pour terminer la tâche.

Par exemple : `@mintlify Update the quickstart page to include information about inviting collaborators`, puis passez sur la branche pour apporter des modifications supplémentaires avec la méthode de votre choix.

<div id="update-docs-when-merging-feature-changes">
  ### Mettre à jour la documentation lors de la fusion de changements de fonctionnalité
</div>

Lorsque vous fusionnez une pull request de fonctionnalité, partagez le lien du PR avec l’agent pour mettre à jour la documentation concernée.

Par exemple : `@mintlify Cette PR ajoute une nouvelle méthode d’authentification. Mets à jour la doc pour inclure le nouveau flux d’authentification : [lien de la PR]`.

### Générer des notes de version à partir d&#39;une pull request

Demandez à l’agent, à partir d’une pull request précise, de générer des notes de version ou des mises à jour du journal des modifications en fonction de l’historique des commits.

Par exemple : `@mintlify Generate release notes for this PR: [PR link]`.

<div id="generate-code-examples">
  ### Générer des exemples de code
</div>

Demandez à l’agent de générer des exemples de code pour des fonctionnalités dans l’ensemble de votre documentation ou sur des pages spécifiques.

Par exemple : `@mintlify Génère un exemple de code pour rendre la méthode d’authentification plus facile à comprendre`.

<div id="review-existing-content">
  ### Passer en revue le contenu existant
</div>

Demandez à l’agent d’auditer le contenu existant pour vérifier l’exactitude technique, le style, la grammaire ou d’autres points.

Par exemple : `@mintlify Review the API rate limiting section. We changed limits last month`.

<div id="respond-to-user-feedback">
  ### Répondre aux retours des utilisateurs
</div>

Fournissez à l’agent les retours de vos utilisateurs pour apporter des mises à jour ciblées à votre documentation.

Par exemple : `@mintlify Les utilisateurs se perdent à l’étape 3 du guide d’installation. Qu’est-ce qui pourrait la rendre peu claire ?`.

<div id="automate-with-the-api">
  ### Automatisez avec l’API
</div>

Intégrez l’agent à vos outils d’automatisation existants pour mettre automatiquement à jour la documentation lors des modifications de code, déclencher des revues de contenu ou synchroniser les mises à jour de la documentation sur plusieurs dépôts.

Découvrez comment dans le tutoriel [Mettre à jour automatiquement la documentation lors de la fusion du code](/fr/guides/automate-agent).
