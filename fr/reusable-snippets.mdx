L’un des principes fondamentaux du développement logiciel est le principe DRY (« Don’t Repeat Yourself »), qui s’applique également à la documentation. Si vous constatez que vous répétez le même contenu à plusieurs endroits, créez un extrait personnalisé pour maintenir votre contenu synchronisé.

<div id="creating-a-custom-snippet">
  ## Créer un extrait personnalisé
</div>

**Pré-requis** : vous devez créer votre fichier d’extrait dans le répertoire `snippets` pour que l’import fonctionne.

Toute page située dans le répertoire `snippets` sera traitée comme un extrait et ne sera pas
rendue en tant que page autonome. Si vous souhaitez créer une page autonome
à partir de l’extrait, importez l’extrait dans un autre fichier et utilisez-le comme
composant.

<div id="default-export">
  ### Export par défaut
</div>

1. Ajoutez le contenu que vous souhaitez réutiliser dans votre fichier de snippet. Vous pouvez également définir des variables à renseigner via des props
   lors de l’import du snippet. Dans cet exemple, notre variable est word.

```typescript snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages. 
```

2. Importez le snippet dans votre fichier de destination.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## En-tête

Lorem impsum dolor sit amet.

<MySnippet/>

```

<div id="exporting-with-variables">
  ### Exportation avec variables
</div>

1. Vous pouvez, si vous le souhaitez, ajouter des variables qui seront transmises via des props lors de l’import de l’extrait. Dans cet exemple, notre variable est word.

```typescript snippets/my-snippet.mdx
My keyword of the day is {word}.
```

2. Importez l’extrait dans votre fichier de destination en passant la variable. La propriété sera renseignée selon votre définition.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Header

Lorem impsum dolor sit amet.

<MySnippet word="bananas" />

```

<div id="reusable-variables">
  ### Variables réutilisables
</div>

1. Exportez une variable depuis votre fichier de fragment :

```typescript snippets/path/to/custom-variables.mdx
export const myName = 'my name';

export const myObject = { fruit: 'strawberries' };
```

2. Importez le fragment dans votre fichier de destination et utilisez la variable :

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

Hello, my name is {myName} and I like {myObject.fruit}.
```

<div id="jsx-snippets">
  ### Extraits JSX
</div>

1. Exportez un composant JSX depuis votre fichier d’extraits. (Consultez [React components](/fr/react-components) pour en savoir plus) :

```js icon=square-js snippets/my-jsx-snippet.jsx
export const MyJSXSnippet = () => {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  )
}
```

<Note>
  Important : lors de la création d’extraits JSX, utilisez la syntaxe de fonction fléchée (`=>`) plutôt que des déclarations de fonction. Le mot-clé `function` n’est pas pris en charge dans ce contexte.
</Note>

2. Importez l’extrait dans votre fichier de destination et utilisez le composant :

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { MyJSXSnippet } from '/snippets/my-jsx-snippet.jsx';

<MyJSXSnippet />
```
