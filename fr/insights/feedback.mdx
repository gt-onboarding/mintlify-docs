<Info>
  Pour collecter et afficher des retours, vous devez d’abord activer le feedback depuis la page [Add-ons](https://dashboard.mintlify.com/products/addons) de votre tableau de bord.
</Info>

L’onglet Feedback affiche les votes quantitatifs de type pouce levé ou pouce baissé que votre documentation a reçus, ainsi que tout retour qualitatif fourni par les utilisateurs. Utilisez ces informations pour évaluer la qualité de votre documentation et l’améliorer.

Accédez à l’onglet Feedback en vous rendant sur la page **Insights** de votre [tableau de bord](https://dashboard.mintlify.com/products/insights).

* **Feedback sur les extraits de code uniquement** : Feedback spécifique sur les extraits de code.

<div id="managing-feedback">
  ## Gestion des retours
</div>

Pour les retours contextuels et sur des extraits de code, vous pouvez définir le statut d’un retour et ajouter des notes internes afin de suivre l’avancement de la résolution des retours des utilisateurs.

<div id="changing-feedback-status">
  ### Modifier le statut des commentaires
</div>

Sélectionnez le statut à côté d’un commentaire pour le marquer comme **En attente**, **En cours**, **Résolu** ou **Ignoré**.

Bonnes pratiques pour définir les statuts des commentaires :

* **En attente** : le commentaire est en attente de revue.
* **En cours** : le commentaire a été validé et est en cours de traitement.
* **Résolu** : le commentaire a été résolu.
* **Ignoré** : le commentaire a été ignoré car non exploitable, non pertinent ou inexact.

<div id="filtering-by-status">
  ### Filtrer par statut
</div>

Utilisez le filtre de statut pour contrôler les retours affichés. Supprimez un statut pour masquer tous les retours ayant ce statut. Par défaut, tous les retours sont affichés.

<div id="adding-internal-notes">
  ### Ajouter des notes internes
</div>

Cliquez sur un commentaire pour ajouter une note interne. Ces notes ne sont visibles que par les personnes ayant accès à votre tableau de bord.

Utilisez-les pour ajouter des informations utiles à la collaboration, associer des tickets de support ou d’ingénierie pertinents, ou consigner toute autre information utile.

<div id="using-feedback-data">
  ## Exploiter les retours
</div>

Analysez vos données de retour pour :

* **Identifier le contenu le plus efficace** : les pages recevant le plus d’avis positifs montrent ce qui fonctionne bien dans votre documentation.
  * **Prioriser les améliorations** : les pages recevant le plus d’avis négatifs indiquent les contenus qui nécessitent votre attention.
  * **Passer à l’action** : mettez à jour la documentation à partir des retours directs des utilisateurs.
