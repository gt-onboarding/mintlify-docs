Les composants `<RequestExample>` et `<ResponseExample>` affichent des blocs de code dans la barre latérale droite afin de créer une mise en page en deux colonnes qui garde les exemples visibles pendant que les utilisateurs font défiler votre contenu. Conçus pour la documentation d’API, ces composants fonctionnent toutefois sur toutes les pages.

Cas d’utilisation courants :

- Documentation d’un endpoint d’API avec des exemples de requête et de réponse
- Exemples de configuration accompagnés d’un texte explicatif
- Extraits de code auxquels les utilisateurs se réfèrent en suivant les instructions
- Exemples « avant/après » dans les tutoriels

Sur mobile, les composants `<RequestExample>` et `<ResponseExample>` s’affichent comme des blocs de code classiques et peuvent être ignorés lors du défilement.

<div id="requestexample">
  ## RequestExample
</div>

Utilisez `<RequestExample>` pour épingler des exemples de code dans la barre latérale droite. Ce composant fonctionne de façon similaire au composant [CodeGroup](/fr/components/code-groups), mais affiche le code dans la barre latérale plutôt qu’en ligne.

Vous pouvez inclure plusieurs blocs de code dans un même `<RequestExample>`. Chaque bloc de code doit avoir un attribut title.

````mdx RequestExample
<RequestExample>

```bash Request
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

<div id="responseexample">
  ## ResponseExample
</div>

Le composant `<ResponseExample>` fixe des exemples de code dans la barre latérale droite, sous tout contenu `<RequestExample>` présent sur la même page.

````mdx ResponseExample
<ResponseExample>

```json Response
{ "status": "success" }
```

</ResponseExample>
````
