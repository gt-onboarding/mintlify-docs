---
title: "Champs de réponse"
description: "Afficher les valeurs de réponse de l'API"
---

Le composant `<ResponseField>` est conçu pour définir les valeurs renvoyées par une API. Beaucoup de documentations utilisent également `<ResponseField>` sur les pages lorsque vous devez lister les types d’éléments.

<ResponseField name="response" type="string" required>
  Exemple de champ de réponse
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Exemple de champ de réponse
</ResponseField>
```

<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Le nom de la valeur de réponse.
</ResponseField>

<ResponseField name="type" type="string" required>
  Type attendu de la valeur de réponse — cela peut être n’importe quelle chaîne.
</ResponseField>

<ResponseField name="default" type="string">
  La valeur par défaut.
</ResponseField>

<ResponseField name="required" type="boolean">
  Affiche « required » à côté du nom du champ.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indique si un champ est obsolète ou non.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Étiquettes affichées avant le nom du champ
</ResponseField>

<ResponseField name="post" type="string[]">
  Étiquettes affichées après le nom du champ
</ResponseField>
