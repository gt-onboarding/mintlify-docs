Le fichier `docs.json` vous permet de transformer une collection de fichiers Markdown en un site de documentation navigable et personnalisé. Ce fichier de configuration obligatoire contrôle le style, la navigation, les intégrations, et plus encore. Pensez-y comme au plan de votre documentation.

Les paramètres de `docs.json` s’appliquent globalement à toutes les pages.

<div id="setting-up-your-docsjson">
  ## Configuration de votre `docs.json`
</div>

Pour commencer, vous n’avez besoin d’indiquer que `theme`, `name`, `colors.primary` et `navigation`. Les autres champs sont facultatifs et vous pourrez les ajouter au fur et à mesure que vos besoins en documentation évoluent.

Pour une meilleure expérience d’édition, incluez la référence de schéma en haut de votre fichier `docs.json`. Cela active l’autocomplétion, la validation et des info-bulles utiles dans la plupart des éditeurs de code :

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // La structure de votre navigation
  }
  // Le reste de votre configuration
}
```

  ### Personnalisation
</div>

<ResponseField name="theme" required>
  Le thème de mise en page de votre site.

  L’un des suivants : `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Voir [Thèmes](fr/themes) pour plus d’informations.
</ResponseField>

<ResponseField name="name" type="string" required>
  Le nom de votre projet, de votre organisation ou de votre produit.
</ResponseField>

<ResponseField name="colors" type="object" required>
  Les couleurs utilisées dans votre documentation. Leur application varie selon les thèmes. Si vous ne fournissez qu’une couleur principale, elle sera utilisée pour tous les éléments de couleur.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      La couleur principale de votre documentation. Généralement utilisée pour la mise en évidence en mode clair, avec quelques variations selon le thème.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      La couleur utilisée pour la mise en évidence en mode sombre.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      La couleur utilisée pour les boutons et les états de survol en modes clair et sombre, avec quelques variations selon le thème.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  Description de votre site pour le SEO et l’indexation par l’IA.
</ResponseField>

<ResponseField name="logo" type="string or object">
  Votre logo pour les modes clair et sombre.

    <ResponseField name="dark" type="string" required>
      Chemin vers le fichier de votre logo pour le mode sombre. Incluez l’extension du fichier. Exemple : `/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      L’URL vers laquelle rediriger lorsque l’on clique sur le logo. Si non fourni, le logo renverra vers votre page d’accueil. Exemple : `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Mise à niveau depuis `mint.json`
</div>

Si votre projet de documentation utilise le fichier `mint.json` désormais déprécié, suivez ces étapes pour passer à `docs.json`.

<Steps>
  <Step title="Installer ou mettre à jour la CLI">
    Si vous n&#39;avez pas installé la [CLI](/fr/installation), installez-la maintenant :

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Si la CLI est déjà installée, assurez-vous qu’elle est à jour :

    ```bash
    mint update
    ```
  </Step>

  <Step title="Créer votre fichier docs.json">
    Dans votre dépôt de documentation, exécutez :

    ```bash
    mint upgrade
    ```

    Cette commande créera un fichier `docs.json` à partir de votre `mint.json` existant. Passez en revue le fichier généré pour vérifier que tous les paramètres sont corrects.
  </Step>

  <Step title="Supprimer votre fichier mint.json">
    Après avoir confirmé que votre `docs.json` est correctement configuré, vous pouvez supprimer en toute sécurité votre ancien fichier `mint.json`.
  </Step>
</Steps>
