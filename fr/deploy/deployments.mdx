Votre site de documentation est déployé automatiquement lorsque vous poussez des modifications vers votre dépôt connecté. Cela nécessite que l’application GitHub Mintlify soit correctement installée et connectée.

Si vos dernières modifications n’apparaissent pas sur votre site en production, vérifiez d’abord que l’application GitHub est installée sur le compte ou l’organisation propriétaire de votre dépôt de documentation. Consultez [Dépannage GitHub](/fr/deploy/github#troubleshooting) pour plus d’informations.

Si l’application GitHub est connectée mais que les modifications ne se déploient toujours pas, vous pouvez déclencher manuellement une reconstruction depuis votre tableau de bord.

<div id="manually-triggering-a-deployment">
  ## Déclencher manuellement un déploiement
</div>

<Steps>
  <Step title="Vérifiez que votre dernier commit a abouti.">
    Assurez-vous que votre dernier commit apparaît dans votre dépôt de documentation et qu’aucune erreur ne s’est produite.
  </Step>

  <Step title="Déclenchez manuellement un déploiement.">
    Accédez à votre [tableau de bord](https://dashboard.mintlify.com) et cliquez sur le bouton de déploiement.

  </Step>
</Steps>
