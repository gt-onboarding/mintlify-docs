Mintlify utilise une application GitHub pour synchroniser automatiquement votre documentation avec votre dépôt GitHub.

<div id="installing-the-github-app">
  ## Installation de l’app GitHub
</div>

Installez l’app GitHub Mintlify sur le compte ou l’organisation GitHub qui détient votre dépôt de documentation. Vous devez être propriétaire de l’organisation ou disposer des autorisations d’administrateur sur un dépôt pour installer l’app. Si vous n’avez pas les autorisations nécessaires, le propriétaire du dépôt doit approuver la demande d’installation.

Installez l’app GitHub Mintlify depuis votre [tableau de bord](https://dashboard.mintlify.com/settings/organization/github-app).

<Tip>
Nous vous recommandons d’accorder l’accès uniquement au dépôt où votre documentation est hébergée.
</Tip>

<div id="permissions">
  ## Autorisations
</div>

Lorsque vous installez l’application GitHub, il vous sera demandé d’accorder les autorisations suivantes :

Autorisations en lecture :

* `metadata` : Informations de base sur le dépôt

Autorisations en lecture et en écriture :

* `checks` : Créer des vérifications d’état sur les pull requests
* `code` : Lire les modifications de fichiers lorsque vous validez des commits sur votre branche de documentation
* `deployments` : Générer des déploiements de prévisualisation pour les pull requests
* `pull requests` : Créer des branches et des pull requests depuis l’Éditeur web
* `workflows` : Gérer GitHub Actions pour les déploiements de la documentation

<Info>
L’application n’accède qu’aux dépôts auxquels vous lui accordez explicitement l’accès. Si des règles de protection des branches sont activées, l’application ne peut pas pousser directement vers les branches protégées.
</Info>

<div id="managing-repository-access">
  ## Gestion de l’accès au dépôt
</div>

Lors de l’installation de notre application GitHub, vous pouvez accorder l’accès à l’ensemble de vos dépôts ou uniquement à certains. Nous recommandons de n’accorder l’accès qu’aux dépôts où se trouve votre documentation. Vous pouvez modifier cette sélection à tout moment dans vos [paramètres de l’application GitHub](https://github.com/apps/mintlify/installations/new).

<div id="configuring-docs-source">
  ## Configuration de la source de la documentation
</div>

Modifiez l’organisation, le dépôt ou la branche à partir desquels votre documentation est générée dans la section [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) de votre tableau de bord.

<div id="troubleshooting">
  ## Résolution des problèmes
</div>

<div id="github-app-connection-issues">
  ### Problèmes de connexion de l’application GitHub
</div>

Si vous rencontrez des problèmes avec l’application GitHub, réinitialiser la connexion peut résoudre la plupart des cas.

<Steps>
  <Step title="Désinstallez l’application Mintlify via GitHub.">
    1. Dans GitHub, allez à [installations](https://github.com/settings/installations) et sélectionnez **Configure** à côté de l’application Mintlify. Faites défiler vers le bas et sélectionnez **Uninstall**.
    2. Allez à [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) et sélectionnez **Revoke** à côté de l’application Mintlify.
  </Step>
  <Step title="Réinstallez l’application Mintlify.">
    1. Dans votre tableau de bord Mintlify, allez à [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) et installez l’application GitHub.
    2. Autorisez votre compte dans la section [My Profile](https://dashboard.mintlify.com/settings/account) de votre tableau de bord.
  </Step>
</Steps>

<div id="feedback-add-ons-are-unavailable">
  ### Les modules complémentaires de feedback sont indisponibles
</div>

Si vous ne pouvez pas activer les options de suggestions d’édition ou de signalement de problèmes dans votre tableau de bord et que votre dépôt est public, revalidez vos paramètres Git.

<Steps>
  <Step title="Accéder aux paramètres Git">
    Accédez à [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) dans votre tableau de bord.
  </Step>
  <Step title="Revalider vos paramètres">
    Cliquez sur la coche verte dans le coin du cadre des paramètres Git pour revalider les paramètres de votre dépôt. Cela forcera la mise à jour des paramètres de votre dépôt afin d’indiquer si votre dépôt est public ou privé.
