description: "Outils pour éviter les liens invalides"
---

Lorsque vous modifiez le chemin d’un fichier dans votre dossier docs, l’URL de la page change également. Cela peut se produire lorsque vous réorganisez votre documentation ou modifiez le titre dans la barre latérale.

<div id="broken-links">
  ## Liens brisés
</div>

Détectez les liens brisés avec notre CLI. [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

L’interface en ligne de commande (CLI) identifie tous les liens relatifs inexistants dans votre documentation.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/chemin/source",
    "destination": "/chemin/destination"
  }
]
```

Cela redirige de façon permanente `/source/path` vers `/destination/path`, afin de ne pas perdre le référencement (SEO) acquis par la page d’origine.

Pour faire correspondre un chemin générique, utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
