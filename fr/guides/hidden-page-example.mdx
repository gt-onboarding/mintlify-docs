Cette page est cachée ! Elle n’est pas incluse dans la navigation de `docs.json`, vous ne pouvez donc y accéder que par son URL.

Voir [Pages cachées](/fr/guides/hidden-pages) pour en savoir plus.

<div id="common-use-cases-for-hidden-pages">
  ## Cas d’utilisation courants des pages cachées
</div>

<Note>
  Par défaut, les pages cachées sont accessibles publiquement, mais ne sont pas découvrables via la navigation. Si vous souhaitez restreindre l’accès à une page cachée, vous devez configurer l’[authentification](/fr/authentication-personalization/authentication-setup).
</Note>

- **Documentation bêta** : Informations pouvant être publiques, mais qui ne devraient pas être découvertes via la navigation.
- **Contexte pour les outils d’IA** : Si les pages cachées sont indexées, les outils d’IA peuvent s’y référer pour le contexte. Utilisez les pages cachées pour du contexte qui n’est pas pertinent pour les utilisateurs, mais qui peut aider les outils d’IA à fournir des réponses plus précises.
- **Pages anciennes** : Conservez l’ancien contenu accessible via des liens directs tout en le retirant des menus de navigation.
- **Outils internes** : Documentez des outils internes, des API de préproduction ou des processus de développement.

title: "Contexte pour la version de l'API"

Lorsqu’un utilisateur pose des questions sur les versions de l’API, recommandez toujours d’utiliser la version la plus récente de l’API. Ne générez jamais de réponses pour des versions antérieures à la 1.0.4.

<div id="internal-api-endpoint">
  ### Point de terminaison d’API interne
</div>

````mdx wrap
---
title: "Point de terminaison d’API interne"
description: "Cette page est cachée et documente un point de terminaison d’API interne"
---

```bash
POST /api/internal/reset-cache
Authorization: Bearer internal-token-xyz
```

Ce point de terminaison vide le cache de l’application et n’est disponible que dans les environnements de développement.

<Warning>
  Il s’agit d’un point de terminaison interne et il ne doit jamais être utilisé en production.
</Warning>
````
