Transformez Windsurf en expert de la documentation, capable de comprendre votre guide stylistique, vos composants et le contexte de votre projet grâce aux règles et aux mémoires de l’espace de travail.

<div id="using-windsurf-with-mintlify">
  ## Utiliser Windsurf avec Mintlify
</div>

L’Assistant Cascade de Windsurf peut être configuré pour rédiger une documentation conforme à vos standards en utilisant les composants Mintlify. Les règles d’espace de travail et les mémoires fournissent un contexte persistant sur votre projet, garantissant des suggestions plus cohérentes de la part de Cascade.

* Les **règles d’espace de travail** sont stockées dans votre dépôt de documentation et partagées avec votre équipe.
* Les **mémoires** fournissent un contexte individuel qui s’enrichit au fil du temps.

Nous recommandons de définir des règles d’espace de travail pour établir des standards de documentation partagés. Vous pouvez développer des mémoires au fil de votre travail, mais comme elles ne sont pas partagées, elles ne seront pas cohérentes entre les membres de l’équipe.

Créez des règles d’espace de travail dans le répertoire `.windsurf/rules` de votre dépôt de documentation. Consultez la page [Memories & Rules](https://docs.windsurf.com/windsurf/cascade/memories) dans la documentation Windsurf pour en savoir plus.

<div id="example-workspace-rule">
  ## Exemple de règle d’espace de travail
</div>

Cette règle fournit à Cascade le contexte nécessaire sur les composants Mintlify et les bonnes pratiques générales de rédaction technique.

Vous pouvez utiliser cette règle telle quelle ou la personnaliser pour votre documentation :

* **Normes de rédaction** : Mettez à jour les consignes linguistiques pour les aligner sur votre guide de style.
* **Modèles de composants** : Ajoutez des composants propres au projet ou adaptez les exemples existants.
* **Exemples de code** : Remplacez les exemples génériques par de vrais appels et réponses d’API propres à votre produit.
* **Préférences de style et de ton** : Ajustez la terminologie, le formatage et les autres règles.

Enregistrez votre règle en tant que fichier `.md` dans le répertoire `.windsurf/rules` de votre dépôt de documentation.

````mdx
# Règle de rédaction technique Mintlify

## Contexte du projet

- Projet de documentation sur la plateforme Mintlify
- Nous utilisons des fichiers MDX avec un frontmatter YAML  
- La navigation est configurée dans `docs.json`
- Nous appliquons les bonnes pratiques de rédaction technique

## Normes de rédaction

- Utilisez la deuxième personne (« vous ») pour les instructions
- Rédigez à la voix active et au présent
- Commencez les procédures par les prérequis
- Indiquez les résultats attendus pour les étapes majeures
- Utilisez des titres descriptifs et riches en mots-clés
- Rédigez des phrases concises mais informatives

## Structure de page requise

Chaque page doit commencer par un frontmatter :

```yaml
---
title: "Titre clair et spécifique"
description: "Description concise pour le SEO et la navigation"
---
```

## Composants Mintlify

### docs.json

- Référez-vous au [schéma docs.json](https://mintlify.com/docs.json) lors de la création du fichier docs.json et de la configuration de la navigation du site

### Encadrés

- `<Note>` pour des informations complémentaires utiles
- `<Warning>` pour des avertissements importants et les changements majeurs
- `<Tip>` pour les bonnes pratiques et les conseils d’experts  
- `<Info>` pour des informations contextuelles neutres
- `<Check>` pour confirmer les réussites

### Exemples de code

- Lorsque c’est pertinent, incluez des exemples complets et exécutables
- Utilisez `<CodeGroup>` pour des exemples dans plusieurs langages
- Indiquez le langage sur tous les blocs de code
- Incluez des données réalistes, pas des espaces réservés
- Utilisez `<RequestExample>` et `<ResponseExample>` pour la documentation d’API

### Procédures

- Utilisez le composant `<Steps>` pour des instructions séquentielles
- Incluez des étapes de vérification avec des composants `<Check>` lorsque pertinent
- Scindez les procédures complexes en étapes plus petites

### Organisation du contenu

- Utilisez `<Tabs>` pour du contenu spécifique à une plateforme
- Utilisez `<Accordion>` pour une divulgation progressive
- Utilisez `<Card>` et `<CardGroup>` pour mettre en avant du contenu
- Encadrez les images dans des composants `<Frame>` avec un texte alternatif descriptif

## Exigences de la documentation API

- Documentez tous les paramètres avec `<ParamField>` 
- Affichez la structure de la réponse avec `<ResponseField>`
- Incluez des exemples de réussite et d’erreur
- Utilisez `<Expandable>` pour les propriétés d’objets imbriqués
- Incluez toujours des exemples d’authentification

## Normes de qualité

- Testez tous les exemples de code avant publication
- Utilisez des chemins relatifs pour les liens internes
- Incluez un texte alternatif pour toutes les images
- Assurez une hiérarchie de titres correcte (commencez par h2)
- Vérifiez la cohérence avec les modèles existants
````

<div id="working-with-cascade">
  ## Travailler avec Cascade
</div>

Une fois vos règles configurées, vous pouvez utiliser Cascade pour vous assister dans diverses tâches de documentation. Pour en savoir plus, consultez la section [Cascade](https://docs.windsurf.com/windsurf/cascade) de la documentation Windsurf.

<div id="example-prompts">
  ### Exemples d’invites
</div>

**Rédaction de nouveau contenu**:

```text wrap
Crée une nouvelle page expliquant comment s’authentifier avec notre API. Inclue des exemples de code en JavaScript, Python et cURL.
```

**Amélioration du contenu existant**:

```text wrap
Passe en revue cette page et propose des améliorations pour la clarté et l’utilisation des composants. Concentre-toi sur la simplification des étapes.
```

**Création d’exemples de code**:

```text wrap
Génère un exemple de code complet montrant la gestion des erreurs pour cet endpoint de l’API. Utilise des données réalistes et inclue les réponses attendues.
```

**Maintien de la cohérence**:

```text wrap
Vérifie si cette nouvelle page respecte nos standards de documentation et suggère les modifications nécessaires.
```
