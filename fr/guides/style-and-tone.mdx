<div id="writing-principles">
  ## Principes de rédaction
</div>

* **Soyez concis.** Les lecteurs consultent la documentation pour atteindre un objectif. Allez droit au but.
* **La clarté avant l’ingéniosité.** Soyez simple, direct et évitez le jargon ou des structures de phrase complexes.
* **Utilisez la voix active.** Plutôt que « Un fichier de configuration devrait être créé », écrivez « Créez un fichier de configuration ».
* **Facilitez le survol.** Utilisez des titres pour orienter le lecteur. Aérez les paragraphes trop denses. Servez-vous de listes à puces pour faciliter le balayage.
* **Écrivez à la deuxième personne.** S’adresser directement au lecteur facilite le suivi des instructions et rend la documentation plus personnelle.

<div id="common-writing-mistakes">
  ## Erreurs d’écriture courantes
</div>

* **Fautes d’orthographe et de grammaire.** Même quelques fautes d’orthographe et de grammaire dans votre documentation la rendent moins crédible et plus difficile à lire.
* **Terminologie incohérente.** Appeler un élément « API key » dans un paragraphe puis « API token » dans le suivant complique la lecture pour les utilisateurs.
* **Terminologie centrée produit.** Vos utilisateurs n’ont pas l’intégralité du contexte de votre produit. Utilisez un langage qui leur est familier.
* **Colloquialismes.** En particulier pour la localisation, les expressions familières nuisent à la clarté.

<div id="tips-for-enforcing-style">
  ## Conseils pour appliquer un style
</div>

Appuyez-vous sur des guides de style existants pour standardiser votre documentation :

* [Microsoft Style Guide](https://learn.microsoft.com/en-us/style-guide/welcome/)
* [Splunk Style Guide](https://docs.splunk.com/Documentation/StyleGuide/current/StyleGuide/Howtouse)
* [Google Developer Documentation Style Guide](https://developers.google.com/style)

Une fois que vous avez identifié les principes rédactionnels à appliquer, automatisez au maximum. Vous pouvez utiliser des [contrôles CI](/fr/deploy/ci) ou des linters comme [Vale](https://vale.sh).
