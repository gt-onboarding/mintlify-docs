---

Utiliza campos para documentar parámetros y respuestas de la API. Hay dos tipos de campos: campos de parámetros y campos de respuesta.

<div id="parameter-field">
  ## Campo de parámetro
</div>

El componente `<ParamField>` se usa para definir parámetros de tus API o SDK. Al agregar un `ParamField`, se añade automáticamente un [Área de pruebas de API](/es/api-playground/overview).

<ParamField path="param" type="string" required>
  Un ejemplo de campo de parámetro
</ParamField>

```mdx
<ParamField path="param" type="string" required>
  Un ejemplo de campo de parámetro
</ParamField>
```

<div id="properties">
  ### Propiedades
</div>

<ParamField body="query, path, body, or header" type="string">
  Indica si el parámetro es de consulta (query), ruta (path), cuerpo (body) o encabezado (header). A continuación, el
  nombre del parámetro.
</ParamField>

<ParamField body="type" type="string">
  Tipo esperado del valor del parámetro.

  Admite `number`, `string`, `boolean`, `object`.

  Los arrays se pueden definir con el sufijo `[]`. Por ejemplo, `string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Indica si el parámetro es obligatorio.
</ParamField>

<ParamField body="deprecated" type="boolean">
  Indica si el parámetro está obsoleto.
</ParamField>

<ParamField body="default" type="any">
  Valor predeterminado que se usa cuando el valor de la solicitud está vacío.
</ParamField>

<ParamField body="placeholder" type="string">
  Texto del placeholder para el campo en el área de pruebas.
</ParamField>

<ParamField body="children" type="string">
  Descripción del parámetro (compatible con Markdown).
</ParamField>

<div id="response-field">
  ## Campo de respuesta
</div>

El componente `<ResponseField>` define los valores de retorno de una API.

<ResponseField name="response" type="string" required>
  Un ejemplo de un campo de respuesta
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un ejemplo de un campo de respuesta
</ResponseField>
```

<div id="properties">
  ### Propiedades
</div>

<ResponseField name="name" type="string" required>
  El nombre del valor de la respuesta.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo esperado del valor de la respuesta. Puede ser cualquier cadena de texto.
</ResponseField>

<ResponseField name="default" type="string">
  El valor predeterminado.
</ResponseField>

<ResponseField name="required" type="boolean">
  Indica si la respuesta es obligatoria.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indica si un campo está en desuso.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Etiquetas que se muestran antes del nombre del campo.
</ResponseField>

<ResponseField name="post" type="string[]">
  Etiquetas que se muestran después del nombre del campo.
</ResponseField>
