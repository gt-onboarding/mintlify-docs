Mintlify gestiona automáticamente muchas prácticas recomendadas de SEO, incluyendo:

* Generación de metaetiquetas
* Generación del sitemap y del archivo `robots.txt`
* Estructura HTML semántica
* Optimización para dispositivos móviles

Puedes personalizar por completo las metaetiquetas de tu sitio agregando el campo `metatags` a tu `docs.json` o al frontmatter de una página.

<div id="global-meta-tags">
  ## Metaetiquetas globales
</div>

Puedes configurar metaetiquetas predeterminadas agregando el campo `metadata` a tu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "enlace a la imagen predeterminada para la metaetiqueta"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metadatos específicos de la página
</div>

Agrega tus meta tags a los metadatos de la página de esta manera:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  Las meta tags con dos puntos deben escribirse entre comillas cuando las incluyas en los metadatos de la página.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps y archivos robots.txt
</div>

Mintlify genera automáticamente un archivo `sitemap.xml` y un archivo `robots.txt`. Puedes ver tu sitemap añadiendo `/sitemap.xml` a la URL de tu sitio de documentación.

De forma predeterminada, solo se incluyen las páginas presentes en tu `docs.json`. Para incluir enlaces ocultos, añade `seo.indexing` a tu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps personalizados y archivos robots.txt
</div>

Para añadir un `sitemap.xml` o `robots.txt` personalizado, crea un archivo `sitemap.xml` o `robots.txt` en la raíz de tu proyecto. Añadir un archivo personalizado reemplazará el archivo generado automáticamente con el mismo nombre. Si eliminas el archivo personalizado, se volverá a usar el archivo predeterminado.

<div id="disabling-indexing">
  ## Desactivar la indexación
</div>

Si quieres impedir que una página sea indexada por los motores de búsqueda, puedes incluir lo siguiente en el [frontmatter](es/pages) de tu página:

```
---
noindex: true
---
```

También puedes establecer `noindex` para todas las páginas de tu documentación configurando el campo `metadata.robots` como `"noindex"` en tu `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Mejores prácticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escribe títulos y descripciones claras">
    * Usa títulos de página claros y descriptivos (50–60 caracteres)
    * Escribe descripciones convincentes (150–160 caracteres)
    * Incluye palabras clave relevantes
    * Asegúrate de que cada título y descripción de página sea único
  </Accordion>

  <Accordion title="Optimiza la estructura de tu contenido">
    * Usa una jerarquía de encabezados adecuada (H1 → H2 → H3)
    * Escribe primero para personas y después para motores de búsqueda
    * Incluye palabras clave relevantes en encabezados y contenido
    * Mantén las URL cortas, descriptivas y organizadas jerárquicamente
    * Divide el contenido extenso con subtítulos y listas
  </Accordion>

  <Accordion title="Estrategia de enlaces internos">
    * Enlaza a páginas relacionadas dentro de tu documentación
    * Usa texto de anclaje descriptivo en lugar de “haz clic aquí”
    * Crea clústeres de temas enlazando conceptos relacionados
    * Usa las funciones de interreferencia automática
  </Accordion>

  <Accordion title="SEO para imágenes">
    * Usa nombres de archivo descriptivos para las imágenes
    * Incluye siempre texto alternativo por accesibilidad y SEO
    * Optimiza el peso de las imágenes para una carga más rápida
    * Usa imágenes relevantes que refuercen tu contenido
  </Accordion>
</AccordionGroup>
