Los fragmentos de código generados automáticamente a partir de los SDK de Speakeasy pueden integrarse directamente en la documentación de referencia de API de Mintlify. Estos fragmentos de uso del SDK se muestran en la [zona de pruebas interactiva](/es/api-playground/overview) de los sitios de documentación creados con Mintlify.

<Frame>
  ![Una Zona de pruebas de API de Mintlify con fragmentos de código de Speakeasy.](/images/speakeasy/mintlify-with-speakeasy-openapi.png)
</Frame>

<div id="prerequisites">
  ## Requisitos previos
</div>

Para integrar Mintlify con Speakeasy, necesitarás lo siguiente:

- Un [repositorio de documentación de Mintlify](/es/quickstart#creating-the-repository).
  - Un SDK generado por Speakeasy con una [URL de muestras de código automatizadas](https://www.speakeasy.com/docs/code-samples/automated-code-sample-urls) configurada.

<div id="setting-up-the-integration">
  ## Configuración de la integración
</div>

Para integrar Speakeasy con Mintlify, debe obtener la URL pública de la especificación combinada de la API en el registro y actualizar su archivo de configuración `docs.json`.

<div id="get-the-apis-combined-spec-public-url-from-the-registry">
  ### Obtén la URL pública del spec combinado de la API desde el registro
</div>

Ve a tu [Speakeasy Dashboard](https://app.speakeasy.com) y abre la pestaña **API Registry**. Abre la entrada `*-with-code-samples` de la API.

<Frame>
  ![Captura de pantalla de la página Speakeasy API Registry. La pestaña API Registry está resaltada con un recuadro rojo y el número 1, y la entrada de la API está resaltada con un recuadro rojo y el número 2.](/images/speakeasy/openapi-registry-and-combined-spec.png)
</Frame>

<Note>
  Si la entrada no está etiquetada como **Combined Spec**, asegúrate de que la API tenga configurada una [URL automática de muestras de código](https://www.speakeasy.com/docs/code-samples/automated-code-sample-urls).
</Note>

En la página de la entrada del registro, copia la URL pública proporcionada.

<Frame>
  ![Captura de pantalla que muestra la entrada del registro del spec combinado con la función de copiar URL resaltada con un recuadro rojo.](/images/speakeasy/copy-combined-spec-url.png)
</Frame>

<div id="update-your-docsjson-configuration-file">
  ### Actualiza tu archivo de configuración `docs.json`
</div>

Añade la URL de la especificación combinada a una sección de **Anchors** o **Tabs** en tu archivo `docs.json`.

Añade la URL de la especificación combinada a un ancla actualizando el campo `anchor` en tu archivo `docs.json` de la siguiente manera:

```json docs.json
{
  "anchors": [
    {
      "name": "API Reference",
      // !mark
      "openapi": "SPEAKEASY_COMBINED_SPEC_URL",
      "url": "api-reference",
      "icon": "square-terminal"
    }
  ]
}
```

Añade la URL de la especificación combinada a una pestaña actualizando el campo `tab` en el archivo `docs.json` de la siguiente manera:

```json docs.json
{
  "tabs": [
    {
      "name": "API Reference",
      "url": "api-reference",
      // !mark
      "openapi": "SPEAKEASY_COMBINED_SPEC_URL"
    }
  ]
}
```

Ahora los fragmentos de código generados por Speakeasy pueden verse en tu documentación de la API e interactuar con ellos en la zona de pruebas.
