Mintlify gestiona automáticamente muchas buenas prácticas de SEO, entre ellas:

* Generación de metaetiquetas
* Generación del sitemap y del archivo `robots.txt`
* Estructura HTML semántica
* Optimización para dispositivos móviles

Puedes personalizar por completo las metaetiquetas de tu sitio añadiendo el campo `metatags` a tu `docs.json` o al frontmatter de una página.

<div id="global-meta-tags">
  ## Metatags globales
</div>

Para establecer metatags predeterminados para todas las páginas, agrega el campo `metatags` a tu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "enlace a tu imagen de metaetiqueta predeterminada"
    }
}
```

<div id="set-a-canonical-url">
  ### Configura una URL canónica
</div>

Si usas un dominio personalizado, establece la metaetiqueta `canonical` para asegurarte de que los motores de búsqueda indexen tu dominio preferido. Una URL canónica indica a los motores de búsqueda qué versión de tu documentación es la principal. Esto mejora el SEO cuando tu documentación es accesible desde varias URL y evita problemas de contenido duplicado.

```json
"seo": {
    "metatags": {
        "canonical": "https://www.tu-dominio-personalizado-aqui.com"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metadatos específicos de la página
</div>

Para establecer metadatos específicos de la página, añádelos al frontmatter de la página.

```mdx
---
title: "El título de tu página de ejemplo"
"og:image": "enlace a tu imagen de meta etiqueta"
---
```

<Note>
  Las metaetiquetas con dos puntos deben estar entre comillas.
</Note>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps y archivos robots.txt
</div>

Mintlify genera automáticamente un archivo `sitemap.xml` y un archivo `robots.txt`. Puedes ver tu sitemap agregando `/sitemap.xml` a la URL de tu sitio de documentación.

De forma predeterminada, solo se incluyen las páginas definidas en tu `docs.json`. Para incluir enlaces ocultos, agrega `seo.indexing` a tu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps personalizados y archivos robots.txt
</div>

Para añadir un `sitemap.xml` o `robots.txt` personalizado, crea un archivo `sitemap.xml` o `robots.txt` en la raíz de tu proyecto. Al añadir un archivo personalizado, se sobrescribirá el archivo generado automáticamente con el mismo nombre. Si eliminas el archivo personalizado, se volverá a usar el archivo predeterminado.

<div id="disabling-indexing">
  ## Desactivar la indexación
</div>

Si quieres impedir que una página sea indexada por los motores de búsqueda, puedes incluir lo siguiente en el [frontmatter](/es/organize/pages) de tu página:

```
---
noindex: true
---
```

También puedes aplicar `noindex` a todas las páginas de tu documentación estableciendo el campo `metatags.robots` en `"noindex"` en tu `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Mejores prácticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escribe títulos y descripciones informativos">
    * Usa títulos de página claros y descriptivos (50–60 caracteres)
    * Escribe descripciones atractivas (150–160 caracteres)
    * Incluye palabras clave relevantes
    * Haz que cada título y descripción de página sean únicos
  </Accordion>

  <Accordion title="Optimiza la estructura de tu contenido">
    * Usa una jerarquía de encabezados adecuada (H1 → H2 → H3)
    * Escribe pensando primero en las personas y luego en los motores de búsqueda
    * Incluye palabras clave relevantes en encabezados y contenido
    * Mantén las URL cortas, descriptivas y organizadas jerárquicamente
    * Divide el contenido extenso con subtítulos y listas
  </Accordion>

  <Accordion title="Estrategia de enlaces internos">
    * Enlaza a páginas relacionadas dentro de tu documentación
    * Usa texto de anclaje descriptivo en lugar de “haz clic aquí”
    * Crea clústeres de temas enlazando conceptos relacionados
    * Usa las funciones de referencia cruzada automáticas
  </Accordion>

  <Accordion title="SEO de imágenes">
    * Usa nombres de archivo descriptivos para las imágenes
    * Incluye siempre texto alternativo por accesibilidad y SEO
    * Optimiza el tamaño de los archivos de imagen para una carga más rápida
    * Usa imágenes relevantes que respalden tu contenido
  </Accordion>
</AccordionGroup>
