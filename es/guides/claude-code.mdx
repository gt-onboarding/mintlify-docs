Claude Code es una herramienta de línea de comandos con capacidades de agente que puede ayudarte a mantener tu documentación. Puede redactar contenido nuevo, revisar páginas existentes y mantener la documentación actualizada.

Puedes entrenar a Claude Code para que comprenda tus estándares y flujos de trabajo de documentación añadiendo un archivo `CLAUDE.md` a tu proyecto y perfeccionándolo con el tiempo.

<div id="getting-started">
  ## Primeros pasos
</div>

**Requisitos:**

- Suscripción activa a Claude (Pro, Max o acceso mediante API)

**Configuración:**

1. Instala Claude Code:

```bash
  npm install -g @anthropic-ai/claude-code
  ```

2. Ve al directorio de tu documentación.
3. (Opcional) Añade el archivo `CLAUDE.md` a tu proyecto.
4. Ejecuta `claude` para comenzar.

<div id="claudemd-template">
  ## Plantilla de CLAUDE.md
</div>

Guarda un archivo `CLAUDE.md` en la raíz de tu directorio de documentación para ayudar a Claude Code a comprender tu proyecto. Este archivo entrena a Claude Code en tus estándares de documentación, preferencias y flujos de trabajo. Consulta [Administrar la memoria de Claude](https://docs.anthropic.com/en/docs/claude-code/memory) en la documentación de Anthropic para obtener más información.

Copia esta plantilla de ejemplo o ajústala según las especificaciones de tu documentación:

```mdx
# Documentación de Mintlify

## Relación de trabajo
- Puedes cuestionar ideas: esto puede resultar en mejor documentación. Cita fuentes y explica tu razonamiento cuando lo hagas
- SIEMPRE pide aclaraciones en lugar de hacer suposiciones
- NUNCA mientas, adivines o inventes nada

## Contexto del proyecto
- Formato: archivos MDX con frontmatter YAML
- Configuración: docs.json para navegación, tema y ajustes
- Componentes: componentes de Mintlify

## Estrategia de contenido
- Documenta lo justo para el éxito del usuario: ni demasiado ni muy poco
- Prioriza la precisión y la usabilidad
- Haz que el contenido sea perenne siempre que sea posible
- Busca contenido existente antes de agregar algo nuevo. Evita la duplicación a menos que sea por una razón estratégica
- Revisa los patrones existentes para mantener la consistencia
- Comienza haciendo los cambios más pequeños que sean razonables

## docs.json

- Consulta el [esquema de docs.json](https://mintlify.com/docs.json) al construir el archivo docs.json y la navegación del sitio

## Requisitos de frontmatter para páginas
- title: Título de página claro y descriptivo
- description: Resumen conciso para SEO/navegación

## Estándares de escritura
- Voz en segunda persona ("tú")
- Prerrequisitos al inicio del contenido procedimental
- Prueba todos los ejemplos de código antes de publicar
- Mantén el estilo y formato de las páginas existentes
- Incluye casos de uso básicos y avanzados
- Etiquetas de lenguaje en todos los bloques de código
- Texto alternativo en todas las imágenes
- Rutas relativas para enlaces internos

## Flujo de trabajo de Git
- NUNCA uses --no-verify al hacer commit
- Pregunta cómo manejar los cambios sin commit antes de comenzar
- Crea una rama nueva cuando no exista una rama clara para los cambios
- Haz commit con frecuencia durante el desarrollo
- NUNCA omitas ni desactives los hooks de pre-commit

## No hacer
- Omitir frontmatter en ningún archivo MDX
- Usar URLs absolutas para enlaces internos
- Incluir ejemplos de código sin probar
- Hacer suposiciones: siempre pide aclaraciones
```


<div id="sample-prompts">
  ## Ejemplos de prompts
</div>

Una vez que tengas configurado Claude Code, prueba estos prompts para ver cómo puede ayudar con tareas comunes de documentación. Puedes copiar y pegar estos ejemplos tal cual o adaptarlos a tus necesidades específicas.

<div id="convert-notes-to-polished-docs">
  ### Convierte notas en documentación pulida
</div>

Convierte borradores en páginas MDX bien formateadas con componentes y frontmatter.

**Ejemplo de prompt:**

```text wrap
Convierte este texto en una página MDX correctamente formateada: [pega tu texto aquí]
```

<div id="review-docs-for-consistency">
  ### Revisar la documentación para mantener la coherencia
</div>

Obtén sugerencias para mejorar el estilo, el formato y el uso de Componentes (Mintlify).

**Ejemplo de prompt:**

```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### Actualiza la documentación cuando cambien las funcionalidades
</div>

Mantén la documentación al día a medida que tu producto evoluciona.

**Ejemplo de prompt:**

```text wrap
Nuestra API ahora requiere un parámetro de versión. Actualiza la documentación para incluir version=2024-01 en todos los ejemplos
```

<div id="generate-comprehensive-code-examples">
  ### Genera ejemplos de código completos
</div>

Crea ejemplos en varios lenguajes con manejo de errores.

**Ejemplo de indicación:**

```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Ampliación de Claude Code
</div>

Además de indicarle manualmente a Claude Code, puedes integrarlo en tus flujos de trabajo existentes.

<div id="automation-with-github-actions">
  ### Automatización con GitHub Actions
</div>

Ejecuta Claude Code automáticamente cuando haya cambios en el código para mantener la documentación al día. Puedes activar revisiones de la documentación en pull requests o actualizar los ejemplos cuando se detecten cambios en la API.

<div id="multi-instance-workflows">
  ### Flujos de trabajo multiinstancia
</div>

Usa sesiones separadas de Claude Code para distintas tareas: una para redactar contenido nuevo y otra para revisión y control de calidad. Esto ayuda a mantener la coherencia y a detectar problemas que una sola sesión podría pasar por alto.

<div id="team-collaboration">
  ### Colaboración en equipo
</div>

Comparte tu archivo `CLAUDE.md` perfeccionado con tu equipo para garantizar estándares de documentación uniformes entre todas las personas que contribuyen. Los equipos suelen desarrollar prompts e incluso flujos de trabajo específicos del proyecto que se integran en su proceso de documentación.

<div id="custom-commands">
  ### Comandos personalizados
</div>

Crea comandos de barra inclinada reutilizables en `.claude/commands/` para tareas de documentación frecuentes específicas de tu proyecto o equipo.
