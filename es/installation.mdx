<div id="installing-the-cli">
  ## Instalación de la CLI
</div>

<Info>
  **Requisito previo**: Instale [Node.js](https://nodejs.org/en) antes de continuar.
</Info>

<Steps>
  <Step title="Instalar la CLI.">
    Ejecute el siguiente comando para instalar la [CLI](https://www.npmjs.com/package/mint):

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Previsualizar localmente.">
    Vaya a su directorio de documentación (donde se encuentra el archivo `docs.json`) y ejecute el siguiente comando:

    ```bash
    mint dev
    ```

    Se abrirá una vista previa local de su documentación en `http://localhost:3000`.
  </Step>
</Steps>

Como alternativa, si no desea instalar la CLI globalmente, puede ejecutar un script de una sola vez:

```bash
npx mint dev
```

<div id="updates">
  ## Actualizaciones
</div>

Si la vista previa local no coincide con lo que ve en la versión de producción en la web, actualice su CLI local:

```bash
mint update
```

Si el comando `mint update` no está disponible en su versión local, reinstale la CLI con la versión más reciente:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Puertos personalizados
</div>

De forma predeterminada, la CLI utiliza el puerto 3000. Puedes personalizar el puerto con la opción `--port`. Para ejecutar la CLI en el puerto 3333, por ejemplo, usa este comando:

```bash
mint dev --port 3333
```

Si intentas ejecutar la CLI en un puerto que ya está en uso, se utilizará el siguiente puerto disponible:

```mdx
El puerto 3000 ya está en uso. Probando con 3001 en su lugar.
```

<div id="previewing-as-a-specific-group">
  ## Vista previa como un grupo específico
</div>

Si utiliza la autenticación parcial para restringir el acceso a su documentación, puede obtener una vista previa como un grupo de autenticación específico usando la opción `--group [groupname]`.

Por ejemplo, si tiene un grupo llamado `admin`, puede obtener una vista previa como miembro de ese grupo con el comando:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Comandos adicionales
</div>

Si bien `mint dev` es el comando más utilizado, hay otros comandos que puedes usar para administrar tu documentación.

<div id="finding-broken-links">
  ### Encontrar enlaces rotos
</div>

La CLI puede ayudar a validar los enlaces de referencia en su documentación. Para identificar cualquier enlace roto, use el siguiente comando:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### Comprobación de la especificación OpenAPI
</div>

Puede usar la CLI para comprobar su archivo de OpenAPI en busca de errores con el siguiente comando:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Puede proporcionar un nombre de archivo (p. ej., `./openapi.yaml`) o una URL (p. ej., `https://petstore3.swagger.io/api/v3/openapi.json`).

<div id="renaming-files">
  ### Cambiar el nombre de archivos
</div>

Puede renombrar y actualizar todas las referencias a archivos con el siguiente comando:

```bash
mint rename <oldFilename> <newFilename>
```

<div id="migrating-mdx-endpoint-pages">
  ### Migración de páginas de endpoints en MDX
</div>

Puede migrar páginas de endpoints en MDX a páginas generadas automáticamente a partir de su especificación de OpenAPI con el siguiente comando:

```bash
mint migrate-mdx
```

Este comando convierte páginas individuales de endpoints en MDX en páginas generadas automáticamente definidas en su `docs.json`, traslada el contenido de MDX a la extensión `x-mint` en su especificación de OpenAPI y actualiza su navegación. Consulte [Migrating from MDX](/es/api-playground/migrating-from-mdx) para obtener información detallada.

<div id="formatting">
  ## Formato
</div>

Durante el desarrollo local, recomendamos usar extensiones en su IDE para reconocer y formatear archivos `MDX`.

Si usa Cursor, Windsurf o VS Code, recomendamos la [extensión MDX para VS Code](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para el resaltado de sintaxis y [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para el formateo de código.

Si usa JetBrains, recomendamos el [complemento MDX para IntelliJ IDEA](https://plugins.jetbrains.com/plugin/14944-mdx) para el resaltado de sintaxis y configurar [Prettier](https://prettier.io/docs/webstorm) para el formateo de código.

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    Esto puede deberse a una versión desactualizada de Node. Prueba lo siguiente:

    1. Desinstala la versión actualmente instalada de la CLI de Mint: `npm uninstall -g mint`
    2. Actualiza a Node.js.
    3. Reinstala la CLI de Mint: `npm install -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    **Solución**: Ve al directorio raíz de tu dispositivo y elimina la carpeta `~/.mintlify`. Después, ejecuta `mint dev` nuevamente.
  </Accordion>

  <Accordion title="Error: permission denied">
    Esto se debe a no tener los permisos necesarios para instalar paquetes de Node globalmente.

    **Solución**: Intenta ejecutar `sudo npm i -g mint`. Se te pedirá tu contraseña, que es la que usas para desbloquear tu computadora.
  </Accordion>

  <Accordion title="The local preview doesn't look the same as my docs do on the web">
    Esto probablemente se deba a una versión desactualizada de la CLI.

    **Solución:** Ejecuta `mint update` para obtener los cambios más recientes.
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    Si tienes algún problema con el paquete de la CLI, primero ejecuta `npm ls -g`. Este comando muestra qué paquetes están instalados globalmente en tu máquina.

    Si no usas npm o no lo ves en la lista con -g, prueba con `which mint` para localizar la instalación.

    Si tienes instalado un paquete llamado `mint` y otro llamado `mintlify`, deberías desinstalar `mintlify`.

    1. Desinstala el paquete antiguo:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Limpia la caché de npm:

    ```bash
      npm cache clean --force
    ```

    3. Reinstala el paquete nuevo:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
