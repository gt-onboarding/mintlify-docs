El archivo `docs.json` te permite convertir una colección de archivos Markdown en un sitio de documentación navegable y personalizado. Este archivo de configuración requerido controla el estilo, la navegación, las integraciones y más. Piénsalo como el plano de tu documentación.

Los ajustes en `docs.json` se aplican globalmente a todas las páginas.

<div id="setting-up-your-docsjson">
  ## Configurar tu `docs.json`
</div>

Para empezar, solo necesitas especificar `theme`, `name`, `colors.primary` y `navigation`. Los demás campos son opcionales y puedes agregarlos a medida que aumenten las necesidades de tu documentación.

Para obtener la mejor experiencia de edición, incluye la referencia del esquema en la parte superior de tu archivo `docs.json`. Esto habilita el autocompletado, la validación y tooltips útiles en la mayoría de los editores de código:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Tus Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Tu estructura de navegación
  }
  // El resto de tu configuración
}
```

  ### Personalización

<ResponseField name="theme" required>
  El tema de diseño de tu sitio.

  Uno de los siguientes: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Consulta [Temas](/es/customize/themes) para obtener más información.
</ResponseField>

<ResponseField name="name" type="string" required>
  El nombre de tu proyecto, organización o producto.
</ResponseField>

<ResponseField name="colors" type="object" required>
  Los colores usados en tu documentación. Los colores se aplican de forma diferente según el tema. Si solo proporcionas un color principal, se usará para todos los elementos de color.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      El color principal de tu documentación. Generalmente se usa para énfasis en modo claro, con algunas variaciones según el tema.

      Debe ser un código hexadecimal que comience con `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      El color usado para énfasis en modo oscuro.

      Debe ser un código hexadecimal que comience con `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      El color usado para botones y estados de hover en modo claro y oscuro, con algunas variaciones según el tema.

      Debe ser un código hexadecimal que comience con `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  Descripción de tu sitio para SEO e indexación por IA.
</ResponseField>

<ResponseField name="logo" type="string or object">
  Configura tu logotipo tanto para modo claro como para modo oscuro.

  <Expandable title="Logo">
    <ResponseField name="light" type="string" required>
      Ruta al archivo de tu logotipo para modo claro. Incluye la extensión del archivo. Ejemplo: `/logo.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      Ruta al archivo de tu logotipo para modo oscuro. Incluye la extensión del archivo. Ejemplo: `/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      La URL a la que redirigir al hacer clic en el logotipo. Si no se proporciona, el logotipo enlazará a tu página de inicio. Ejemplo: `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string or object">
  Ruta a tu archivo de favicon, incluida la extensión del archivo. Se redimensiona automáticamente a los tamaños de favicon apropiados. Puede ser un único archivo o archivos separados para modo claro y modo oscuro. Ejemplo: `/favicon.png`

  <Expandable title="Favicon">
    <ResponseField name="light" type="string" required>

  </Expandable>
</ResponseField>

  <Expandable title="Thumbnails">
    <ResponseField name="appearance" type="&#x22;light&#x22; | &#x22;dark&#x22;">
      El tema visual de tus miniaturas. Si no se especifica, las miniaturas usan el esquema de color de tu sitio definido por el campo `colors`.

<ResponseField name="styling" type="object">
  Configuraciones de estilo visual.

  <Expandable title="Styling">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      Estilo del “eyebrow” de la página. Elige `section` para mostrar el nombre de la sección o `breadcrumbs` para mostrar la ruta de navegación completa. El valor predeterminado es `section`.
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      Configuración del tema de los bloques de código. El valor predeterminado es `"system"`.

      **Configuración simple:**

      * `"system"`: Coincide con el modo actual del sitio (claro u oscuro)
      * `"dark"`: Usa siempre el modo oscuro

      **Configuración de tema personalizada:**

      * Usa una cadena para especificar un único [tema de Shiki](https://shiki.style/themes) para todos los bloques de código
      * Usa un objeto para especificar [temas de Shiki](https://shiki.style/themes) distintos para los modos claro y oscuro

      <ResponseField name="theme" type="string">
        Un único nombre de tema de Shiki para usar tanto en modo claro como en modo oscuro.

        ```json
        "styling": {
          "codeblocks": {
            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

      <ResponseField name="theme" type="object">
        Temas separados para los modos claro y oscuro.

        <Expandable title="theme">
          <ResponseField name="light" type="string" required>
            Un nombre de tema de Shiki para el modo claro.
          </ResponseField>

          <ResponseField name="dark" type="string" required>
            Un nombre de tema de Shiki para el modo oscuro.
          </ResponseField>

          ```json
          "styling": {
            "codeblocks": {
              "theme": {
                "light": "github-light",
                "dark": "github-dark"
              }
            }
          }
          ```
        </Expandable>
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Actualizar desde `mint.json`
</div>

Si tu proyecto de documentación usa el archivo en desuso `mint.json`, sigue estos pasos para actualizar a `docs.json`.

<Steps>
  <Step title="Instala o actualiza la CLI">
    Si aún no has instalado la [CLI](/es/installation), instálala ahora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Si ya tienes instalada la CLI, asegúrate de que esté actualizada:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Crea tu archivo docs.json">
    En tu repositorio de documentación, ejecuta:

    ```bash
    mint upgrade
    ```

    Este comando creará un archivo `docs.json` a partir de tu `mint.json` existente. Revisa el archivo generado para asegurarte de que toda la configuración sea correcta.
  </Step>

  <Step title="Elimina tu archivo mint.json">
    Después de verificar que tu `docs.json` esté configurado correctamente, puedes eliminar con seguridad tu archivo `mint.json` anterior.
  </Step>
</Steps>
