---
title: "Despliegues"
description: "Soluciona problemas de tus despliegues"
---

Tu sitio de documentación se despliega automáticamente cuando haces push de cambios al repositorio conectado. Para ello, la aplicación de GitHub de Mintlify debe estar instalada y conectada correctamente.

Si tus últimos cambios no aparecen en el sitio en producción, primero verifica que la aplicación de GitHub esté instalada en la cuenta u organización propietaria del repositorio de la documentación. Consulta [Solución de problemas de GitHub](/es/deploy/github#troubleshooting) para más información.

Si la aplicación de GitHub está conectada pero los cambios siguen sin desplegarse, puedes activar manualmente una reconstrucción desde tu panel.

<div id="manually-triggering-a-deployment">
  ## Activar un despliegue manualmente
</div>

<Steps>
  <Step title="Verifica que tu último commit se haya realizado correctamente.">
    Comprueba que tu último commit aparezca en tu repositorio de documentación y que no haya arrojado errores.
  </Step>

  <Step title="Activa un despliegue manualmente.">
    Ve a tu [panel](https://dashboard.mintlify.com) y selecciona el botón de despliegue.

</Steps>
