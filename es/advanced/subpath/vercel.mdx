<div id="verceljson-file">
  ## archivo vercel.json
</div>

El archivo `vercel.json` es el archivo de configuración de Vercel que te permite personalizar cómo se compila y se despliega tu proyecto. Se encuentra en el directorio raíz de tu proyecto y controla diversos aspectos del despliegue, incluido el enrutamiento, las redirecciones, los encabezados y la configuración de compilación.

Usamos la configuración `rewrites` para hacer de proxy de las solicitudes desde tu dominio principal hacia tu documentación.

Las reescrituras te permiten dirigir las solicitudes entrantes a distintos destinos sin cambiar la URL en el navegador. Cuando alguien visita `yoursite.com/docs`, Vercel obtendrá internamente contenido de `your-subdomain.mintlify.dev/docs`, pero el usuario seguirá viendo `yoursite.com/docs` en su navegador. Esto es distinto de las redirecciones, que enviarían a los usuarios a una URL completamente diferente.

<div id="configuration">
  ## Configuración
</div>

Para alojar la documentación en una subruta personalizada `/docs` usando Vercel, agrega la siguiente configuración a tu archivo `vercel.json`:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

- **`source`**: El patrón de ruta en tu dominio que activa la reescritura.
- **`destination`**: El destino al que se debe enrutar (proxy) la solicitud.
- **`:match*`**: Un comodín que captura cualquier segmento de ruta después de `/docs/`.

Para obtener más información, consulta [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) en la documentación de Vercel.

<div id="using-external-proxies-with-vercel">
  ## Uso de proxies externos con Vercel
</div>

Si utiliza un proxy externo (como Cloudflare o AWS CloudFront) delante de su despliegue en Vercel, debe configurarlo correctamente para evitar conflictos con la verificación de dominio de Vercel y el aprovisionamiento de certificados SSL.

Una configuración incorrecta del proxy puede impedir que Vercel aprovisione certificados SSL de Let's Encrypt y provocar errores en la verificación de dominio.

Consulte los [proveedores admitidos](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) en la documentación de Vercel.

<div id="required-path-allowlist">
  ### Lista obligatoria de rutas permitidas
</div>

Tu proxy externo debe permitir el tráfico a estas rutas específicas sin bloquear, redirigir ni aplicar caché de forma agresiva:

- `/.well-known/acme-challenge/*` - Obligatorio para la verificación de certificados de Let's Encrypt
- `/.well-known/vercel/*` - Obligatorio para la verificación de dominios de Vercel

Estas rutas deben pasar directamente a tu implementación de Vercel sin modificaciones.

<div id="header-forwarding-requirements">
  ### Requisitos para el reenvío de encabezados
</div>

Asegúrate de que tu proxy reenvíe correctamente el encabezado `HOST`. Sin un reenvío adecuado de este encabezado, las solicitudes de verificación fallarán.

<div id="testing-your-proxy-setup">
  ### Probar la configuración de tu proxy
</div>

Para verificar que tu proxy esté configurado correctamente:

1. Comprueba que `https://[yourdomain].com/.well-known/vercel/` devuelva una respuesta.
2. Asegúrate de que los certificados SSL se estén aprovisionando correctamente en tu panel de Vercel.
3. Verifica que la verificación del dominio se complete correctamente.
