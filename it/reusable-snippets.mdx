Uno dei principi fondamentali dello sviluppo software è il DRY (Don't Repeat Yourself), che vale anche per la documentazione. Se ti capita di ripetere lo stesso contenuto in più punti, crea uno snippet personalizzato per mantenere i contenuti allineati.

<div id="creating-a-custom-snippet">
  ## Creazione di uno snippet personalizzato
</div>

**Precondizione**: devi creare il file dello snippet nella directory `snippets` affinché l'import funzioni.

Qualsiasi pagina nella directory `snippets` verrà trattata come snippet e non
verrà renderizzata come pagina autonoma. Se vuoi creare una pagina autonoma
a partire dallo snippet, importa lo snippet in un altro file e usalo come
componente.

<div id="default-export">
  ### Esportazione predefinita
</div>

1. Aggiungi al file dello snippet il contenuto che desideri riutilizzare. Facoltativamente, puoi aggiungere variabili da valorizzare tramite le props
   quando importi lo snippet. In questo esempio, la nostra variabile è word.

```typescript snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages. 
```

2. Importa lo snippet nel file di destinazione.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Intestazione

Lorem impsum dolor sit amet.

<MySnippet/>

```

<div id="exporting-with-variables">
  ### Esportazione con variabili
</div>

1. Facoltativamente, puoi aggiungere variabili che possono essere valorizzate tramite props quando importi lo snippet. In questo esempio, la nostra variabile è word.

```typescript snippets/my-snippet.mdx
My keyword of the day is {word}.
```

2. Importa lo snippet nel file di destinazione passando la variabile. La proprietà verrà valorizzata in base alla tua specifica.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Header

Lorem impsum dolor sit amet.

<MySnippet word="bananas" />

```

<div id="reusable-variables">
  ### Variabili riutilizzabili
</div>

1. Esporta una variabile dal file di snippet:

```typescript snippets/path/to/custom-variables.mdx
export const myName = 'my name';

export const myObject = { fruit: 'strawberries' };
```

2. Importa lo snippet nel file di destinazione e usa la variabile:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

Ciao, mi chiamo {myName} e mi piacciono le {myObject.fruit}.
```

<div id="jsx-snippets">
  ### Snippet JSX
</div>

1. Esporta un componente JSX dal file dello snippet. (Consulta [Componenti React](/it/react-components) per maggiori informazioni):

```js icon=square-js snippets/my-jsx-snippet.jsx
export const MyJSXSnippet = () => {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  )
}
```

<Note>
  Importante: quando crei snippet JSX, usa la sintassi con arrow function (`=>`) invece delle dichiarazioni di funzione. La parola chiave `function` non è supportata in questo contesto.
</Note>

2. Importa lo snippet dal file di destinazione e utilizza il componente:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { MyJSXSnippet } from '/snippets/my-jsx-snippet.jsx';

<MyJSXSnippet />
```
