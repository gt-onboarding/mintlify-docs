Quando modifichi il percorso di un file nella cartella dei tuoi docs, cambia anche il percorso URL di quella pagina. Questo può succedere quando ristrutturi la documentazione o cambi il titolo della barra laterale.

<div id="broken-links">
  ## Link non funzionanti
</div>

Individua i link non funzionanti con la nostra CLI. [Installa la CLI](/it/installation) ed esegui il comando:

```bash
mint broken-links
```

La CLI rileva eventuali link relativi nella tua documentazione che non esistono.

<div id="redirects">
  ## Reindirizzamenti
</div>

Configura i reindirizzamenti 301 aggiungendo il campo `redirects` al file `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Questo reindirizza in modo permanente `/source/path` verso `/destination/path`, così da non perdere il posizionamento SEO precedente della pagina originale.

Per gestire un percorso con wildcard, usa `*` dopo un parametro. In questo esempio, `/beta/:slug*` corrisponde a `/beta/introduction` e lo reindirizza a `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
