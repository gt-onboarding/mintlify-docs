<Info>
  I [piani Pro ed Enterprise](https://mintlify.com/pricing?ref=docs-ci) includono controlli CI per i repository GitHub.
</Info>

Usa i controlli CI per eseguire il linting della documentazione, individuare errori e segnalare avvisi prima della distribuzione. I controlli CI sono configurati per essere eseguiti sui commit del branch di distribuzione configurato o sulle pull request dirette a quel branch.

<div id="installation">
  ## Installazione
</div>

Per iniziare, segui i passaggi nella pagina [GitHub](/it/settings/github).

Per le GitHub App, puoi scegliere di concedere le autorizzazioni a un solo repository. Ti consigliamo vivamente di farlo, poiché ci serve l’accesso esclusivamente al repository in cui è ospitata la tua documentazione.

<div id="configuration">
  ## Configurazione
</div>

Configura i controlli CI abilitati per una distribuzione andando alla pagina [Add-ons](https://dashboard.mintlify.com/products/addons) della tua dashboard. Abilita i controlli che vuoi eseguire.

Quando abiliti i controlli, puoi scegliere di eseguirli con livello `Warning` o `Blocking`.

- Un controllo a livello `Warning` non restituirà mai uno stato di errore, anche in presenza di errori o suggerimenti.
- Un controllo a livello `Blocking` restituirà uno stato di errore se non viene superato o se vengono suggerite modifiche.

<div id="available-ci-checks">
  ## Controlli CI disponibili
</div>

<div id="broken-links">
  ### Link non funzionanti
</div>

Analogamente a come il [controllo dei link della CLI](/it/settings/broken-links#broken-links) opera sulla tua macchina locale, verifichiamo automaticamente la tua documentazione alla ricerca di link non funzionanti.

Per visualizzare i risultati di questo controllo, visita la pagina dei risultati dei check di GitHub per un commit specifico.

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) è un linter di prosa open source basato su regole che supporta vari formati di documento, tra cui Markdown e MDX.

Mintlify supporta l’esecuzione automatica di Vale in un check CI e la visualizzazione dei risultati come stato del check.

<div id="configuration">
  #### Configurazione
</div>

Se nella directory principale dei contenuti del tuo deployment è presente un file `.vale.ini`, utilizzeremo automaticamente quel file di configurazione. Utilizziamo inoltre automaticamente qualsiasi file di configurazione nel `stylesPath` specificato.

Se non disponi di un file di configurazione di Vale, verrà caricata automaticamente la configurazione predefinita.

```mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

  Tieni presente che, per motivi di sicurezza, non possiamo supportare alcuno `stylesPath` assoluto né `stylesPath` che includano valori `..`. Usa percorsi relativi e includi il `stylesPath` nel tuo repository.

<div id="packages">
  #### Pacchetti
</div>

Vale supporta una serie di [pacchetti](https://vale.sh/docs/keys/packages) che possono essere utilizzati per controllare errori di ortografia e di stile.
Qualsiasi pacchetto che includi nel tuo repository nel `stylesPath` corretto viene installato e utilizzato automaticamente nella configurazione di Vale.

Per i pacchetti non inclusi nel repository, puoi specificare qualsiasi pacchetto dal [registro dei pacchetti di Vale](https://vale.sh/explorer): verrà scaricato e utilizzato automaticamente nella configurazione di Vale.

<Warning>
  Tieni presente che, per motivi di sicurezza, non supportiamo il download automatico di pacchetti che non provengono dal [registro dei pacchetti di Vale](https://vale.sh/explorer).
</Warning>

<div id="vale-with-mdx">
  #### Vale con `MDX`
</div>

Vale non supporta nativamente `MDX`, ma l’autore di Vale ha fornito un’[estensione personalizzata](https://github.com/errata-ai/MDX) per abilitarne il supporto.

Se preferisci non usare questa estensione, ti consigliamo di aggiungere le seguenti righe al file `.vale.ini`:

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Per usare i commenti in-documento di Vale, utilizza i commenti in stile MDX `{/* ... */}`. Se nella configurazione imposti `CommentDelimiters = {/*, */}` [qui](https://vale.sh/docs/keys/commentdelimiters), Vale interpreterà automaticamente questi commenti durante il linting. In questo modo puoi usare facilmente le funzionalità integrate di Vale, come escludere righe o sezioni.

```mdx
{/* vale off */}

Questo testo sarà ignorato da Vale

{/* vale on */}
```

Se scegli di non usare `CommentDelimiters` ma vuoi comunque usare i commenti di Vale, devi racchiudere i commenti di Vale nei commenti MDX `{/* ... */}`. Ad esempio:

```mdx
{/* <!-- vale off --> */}

Questo testo sarà ignorato da Vale

{/* <!-- vale on --> */}
```

Questi tag di commento non sono supportati all’interno dei componenti Mintlify, ma possono essere usati ovunque al livello base di un documento.
