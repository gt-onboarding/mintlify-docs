<Info>
  I deployment di anteprima sono disponibili nei [piani Pro ed Enterprise](https://mintlify.com/pricing?ref=preview-deployments).
</Info>

I deployment di anteprima ti permettono di vedere come appariranno le modifiche alla documentazione prima del merge in produzione. Ogni anteprima genera un URL condivisibile che si aggiorna automaticamente man mano che invii nuove modifiche.

<div id="creating-preview-deployments">
  ## Creazione dei deployment di anteprima
</div>

I deployment di anteprima vengono creati automaticamente tramite pull request o manualmente dalla tua dashboard.

<div id="automatic-previews">
  ### Anteprime automatiche
</div>

Quando apri una pull request, il bot di Mintlify aggiunge automaticamente un link per visualizzare il deployment di anteprima nella pull request. L’anteprima si aggiorna ogni volta che fai push di nuovi commit sul branch.

<div id="manual-previews">
  ### Anteprime manuali
</div>

Puoi creare manualmente un’anteprima per qualsiasi branch.

1. Vai al tuo [dashboard](https://dashboard.mintlify.com/).
2. Seleziona **Anteprime**.
3. Seleziona **Crea anteprima**.
4. Inserisci il nome del branch.
5. Seleziona **Crea deployment**.

## Ridistribuire un&#39;anteprima

Ridistribuisci un&#39;anteprima per aggiornare i contenuti o riprovare dopo un deployment non riuscito.

1. Seleziona l&#39;anteprima dalla tua [dashboard](https://dashboard.mintlify.com/).
2. Seleziona **Redeploy**.

<div id="sharing-preview-deployments">
  ## Condivisione dei deployment di anteprima
</div>

I deployment di anteprima possono essere pubblici o limitati agli utenti autenticati.

**Accesso pubblico**: Gli URL di anteprima sono pubblicamente visibili per impostazione predefinita. Condividi un link di anteprima con chiunque debba esaminare le tue modifiche.

**Accesso limitato**: Abilita l&#39;autenticazione dei deployment di anteprima per consentire l&#39;accesso solo agli utenti Mintlify autenticati. Consulta [Configurazione dell&#39;autenticazione](/it/authentication-personalization/authentication-setup) per ulteriori informazioni.
