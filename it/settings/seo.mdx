Mintlify gestisce automaticamente molte best practice SEO, tra cui:

* Generazione dei meta tag
* Generazione della sitemap e del file `robots.txt`
* Struttura HTML semantica
* Ottimizzazione per dispositivi mobili

Puoi personalizzare completamente i meta tag del tuo sito aggiungendo il campo `metatags` al file `docs.json` o al frontmatter di una pagina.

<div id="global-meta-tags">
  ## Meta tag globali
</div>

Puoi impostare meta tag predefiniti aggiungendo il campo `metadata` al tuo `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "link all'immagine predefinita del meta tag"
    }
}
```

<div id="page-specific-meta-tags">
  ## Meta tag specifici della pagina
</div>

Aggiungi i meta tag ai metadati della pagina in questo modo:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  I meta tag che contengono i due punti devono essere racchiusi tra virgolette quando li inserisci nei metadati della pagina.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemap e file robots.txt
</div>

Mintlify genera automaticamente un file `sitemap.xml` e un file `robots.txt`. Puoi visualizzare la sitemap aggiungendo `/sitemap.xml` all&#39;URL del tuo sito di documentazione.

Per impostazione predefinita vengono incluse solo le pagine presenti nel `docs.json`. Per includere i link nascosti, aggiungi `seo.indexing` al `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemap personalizzati e file robots.txt
</div>

Per aggiungere un `sitemap.xml` o un `robots.txt` personalizzato, crea un file `sitemap.xml` o `robots.txt` nella directory principale del progetto. L’aggiunta di un file personalizzato sovrascriverà il file generato automaticamente con lo stesso nome. Se elimini il file personalizzato, verrà nuovamente utilizzato quello predefinito.

<div id="disabling-indexing">
  ## Disabilitare l’indicizzazione
</div>

Se vuoi impedire che una pagina venga indicizzata dai motori di ricerca, puoi includere quanto segue nel [frontmatter](it/pages) della pagina:

```
---
noindex: true
---
```

Puoi anche applicare `noindex` a tutte le pagine della documentazione impostando il campo `metadata.robots` su `"noindex"` nel file `docs.json`:

```json
"seo": {
  "metatags": {
    "robots": "noindex"
  }
}
```

<div id="seo-best-practices">
  ## Best practice SEO
</div>

<AccordionGroup>
  <Accordion title="Scrivi titoli e descrizioni efficaci">
    * Usa titoli di pagina chiari e descrittivi (50–60 caratteri)
    * Scrivi descrizioni accattivanti (150–160 caratteri)
    * Includi parole chiave pertinenti
    * Rendi unici titolo e descrizione di ogni pagina
  </Accordion>

  <Accordion title="Ottimizza la struttura dei contenuti">
    * Usa una gerarchia di titoli corretta (H1 → H2 → H3)
    * Scrivi prima per le persone, poi per i motori di ricerca
    * Inserisci parole chiave pertinenti nei titoli e nel contenuto
    * Mantieni gli URL brevi, descrittivi e organizzati gerarchicamente
    * Spezza i contenuti lunghi con sottotitoli ed elenchi
  </Accordion>

  <Accordion title="Strategia di linking interno">
    * Collega pagine correlate all’interno della documentazione
    * Usa un testo del link descrittivo invece di &quot;clicca qui&quot;
    * Crea cluster tematici collegando concetti correlati
    * Sfrutta le funzionalità automatiche di collegamenti incrociati
  </Accordion>

  <Accordion title="SEO per le immagini">
    * Usa nomi file descrittivi per le immagini
    * Includi sempre il testo alternativo per accessibilità e SEO
    * Ottimizza le dimensioni dei file immagine per un caricamento più rapido
    * Usa immagini pertinenti che supportino il contenuto
  </Accordion>
</AccordionGroup>
