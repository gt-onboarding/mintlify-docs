Mintlify gestisce automaticamente molte best practice SEO, tra cui:

* Generazione dei meta tag
* Generazione della sitemap e del file `robots.txt`
* Struttura HTML semantica
* Ottimizzazione per dispositivi mobili

Puoi personalizzare completamente i meta tag del tuo sito aggiungendo il campo `metatags` al file `docs.json` o al frontmatter di una pagina.

<div id="global-meta-tags">
  ## Meta tag globali
</div>

Per impostare meta tag predefiniti per tutte le pagine, aggiungi il campo `metatags` al file `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "link alla tua immagine meta tag predefinita"
    }
}
```

<div id="set-a-canonical-url">
  ### Imposta un URL canonico
</div>

Se utilizzi un dominio personalizzato, imposta il meta tag `canonical` per fare in modo che i motori di ricerca indicizzino il dominio che preferisci. Un URL canonico indica ai motori di ricerca quale versione della documentazione è quella principale. Questo migliora la SEO quando la documentazione è accessibile da più URL e previene problemi di contenuti duplicati.

```json
"seo": {
    "metatags": {
        "canonical": "https://www.il-tuo-dominio-personalizzato.com"
    }
}
```

<div id="page-specific-meta-tags">
  ## Meta tag specifici della pagina
</div>

Per impostare meta tag specifici della pagina, aggiungili al frontmatter della pagina.

```mdx
---
title: "Titolo della tua pagina di esempio"
"og:image": "link all'immagine del meta tag"
---
```

<Note>
  I meta tag con i due punti devono essere racchiusi tra virgolette.
</Note>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemap e file robots.txt
</div>

Mintlify genera automaticamente un file `sitemap.xml` e un file `robots.txt`. Puoi visualizzare la sitemap aggiungendo `/sitemap.xml` all’URL del tuo sito di documentazione.

Per impostazione predefinita vengono incluse solo le pagine presenti nel tuo `docs.json`. Per includere i collegamenti nascosti, aggiungi `seo.indexing` al tuo `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemap e file robots.txt personalizzati
</div>

Per aggiungere un `sitemap.xml` o un `robots.txt` personalizzato, crea un file `sitemap.xml` o `robots.txt` nella directory radice del progetto. L’aggiunta di un file personalizzato sostituirà il file generato automaticamente con lo stesso nome. Se elimini un file personalizzato, verrà nuovamente utilizzato il file predefinito.

<div id="disabling-indexing">
  ## Disabilitare l’indicizzazione
</div>

Se vuoi impedire che una pagina venga indicizzata dai motori di ricerca, puoi includere quanto segue nel [frontmatter](/it/organize/pages) della pagina:

```
---
noindex: true
---
```

Puoi anche specificare `noindex` per tutte le pagine della documentazione impostando il campo `metatags.robots` su `"noindex"` nel file `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Best practice SEO
</div>

<AccordionGroup>
  <Accordion title="Scrivi titoli e descrizioni efficaci">
    * Usa titoli di pagina chiari e descrittivi (50–60 caratteri)
    * Scrivi descrizioni accattivanti (150–160 caratteri)
    * Includi parole chiave pertinenti
    * Rendi unici il titolo e la descrizione di ogni pagina
  </Accordion>

  <Accordion title="Ottimizza la struttura dei contenuti">
    * Usa una gerarchia di titoli corretta (H1 → H2 → H3)
    * Scrivi prima per le persone, poi per i motori di ricerca
    * Includi parole chiave pertinenti nei titoli e nei contenuti
    * Mantieni gli URL brevi, descrittivi e organizzati gerarchicamente
    * Suddividi i contenuti lunghi con sottotitoli ed elenchi
  </Accordion>

  <Accordion title="Strategia di linking interno">
    * Collega pagine correlate all&#39;interno della documentazione
    * Usa testo del link descrittivo invece di &quot;clicca qui&quot;
    * Crea cluster tematici collegando concetti correlati
    * Sfrutta le funzionalità di riferimento incrociato automatico
  </Accordion>

  <Accordion title="SEO per le immagini">
    * Usa nomi file descrittivi per le immagini
    * Includi sempre testo alternativo per accessibilità e SEO
    * Ottimizza le dimensioni dei file immagine per velocizzare il caricamento
    * Usa immagini pertinenti che supportino i contenuti
  </Accordion>
</AccordionGroup>
