<Update label="August 16" tags={["Improvements"]} rss={{ title: "Aggiornamenti settimanali", description: "Ottimizzazioni delle prestazioni, integrazioni LLM, affinamenti dell'Editor web e miglioramenti dell'API playground" }}>
  ## Ottimizzazioni di prestazioni e build

  * La transpilation di MDX ora avviene al momento del deployment invece che a ogni caricamento di pagina negli ambienti serverless NextJS non in cache, migliorando il time to first byte per le pagine non in cache.
  * L&#39;hashing basato sul contenuto evita la ri-transpilazione quando MDX non è cambiato, riducendo i tempi del flusso di aggiornamento di circa il 50% per i clienti con molte pagine (i deployment oltre 5 minuti dovrebbero essere grossomodo dimezzati)
  * La visualizzazione dei deployment di anteprima nella dashboard è ora più rapida grazie a nuovi indici di database e alla parallelizzazione delle query nel nostro backend
  * Ridotte le dimensioni della pagina eliminando dati `navigation` duplicati nel payload `rsc` di ciascuna pagina—i guadagni di performance sono più evidenti con un alto numero di pagine o strutture di navigazione complesse
  * Un prefetching più aggressivo consente caricamenti istantanei della pagina più spesso

  ## Miglioramenti ad API playground e OpenAPI

  * Supporto per le [`webhook pages`](/it/api-playground/openapi-setup#webhooks) nella navigazione della documentazione a partire da specifiche OpenAPI
  * Contestualizzazione del modello AI ottimizzata rimuovendo le specifiche dei link di ancoraggio dai link markdown quando si passa ad Anthropic, OpenAI o altri provider
