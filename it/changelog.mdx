  ## Major release: Raccolta di feedback migliorata

  - **Miglioramento importante**: I lettori ora possono fornire feedback più dettagliati dopo aver selezionato _pollice su/giù_, inclusi opzioni e commenti scritti. Puoi anche raccogliere feedback sui blocchi di codice e visualizzare tutte le risposte nelle analytics della tua dashboard.\

  ## Miglioramenti alla navigazione e alla qualità della vita

  - Miglioramento della qualità della vita per ignorare le barre finali e le doppie barre in modo da non doverti preoccupare di inserirle esattamente nel tuo docs.json
  - Ora puoi aggiungere l’attributo `noAnchor` ai tag HTML `h1-6` per evitare che generino un link di ancoraggio
  - Il tema Palm ora ha un selettore della lingua flottante in basso a sinistra, simile all’approccio di Stripe
  - Aggiunto un nuovo campo in docs.json chiamato [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups) che consente di controllare se gli utenti vengono indirizzati alla prima pagina di un gruppo di navigazione quando il gruppo viene espanso

  ## Correzioni di bug e affidabilità

  - Corretto un bug per cui i link di ancoraggio della posizione di scorrimento non funzionavano correttamente quando erano presenti componenti JS in una pagina
  - Corretto un bug per cui Google indicizzava file `*.md` grezzi perché mancava l’intestazione `x-robots-tag noindex`

  - Corretto un bug nelle anteprime dei documenti protetti da autenticazione per cui non era possibile vedere l’intera barra di navigazione
  - Correzioni di bug relative alla gestione degli SVG con la nostra nuova CDN di immagini

  ## Miglioramenti a componenti e stile

  - Aggiunto un nuovo selettore CSS per stili personalizzati su `SidebarNavGroupDivider`
  - Nuovi test di regressione per le pagine API definite in MDX con sicurezza configurata garantiranno un grado maggiore di stabilità

  ## Miglioramenti delle prestazioni

  - Miglioramento delle prestazioni spostando il CSS di KaTeX da cdnjs alla nostra CDN su CloudFront per ridurre la latenza

<Update label="17 agosto - 23 agosto" tags={["Improvements"]} rss={{ title: "Aggiornamenti settimanali", description: "Correzioni del layout delle immagini, miglioramenti all'autenticazione, miglioramenti alla navigazione mobile e stabilità del web editor" }}>
  ## Miglioramenti alla gestione delle immagini

  - **Miglioramento importante**: Le immagini non causano più spostamenti del layout per impostazione predefinita, anche quando gli attributi di larghezza e altezza non sono specificati—il dimensionamento automatico evita salti di contenuto durante il caricamento delle pagine
  - Tutti i file statici nel tuo repository (PDF, TXT, XML, ecc.) sono ora caricati e serviti automaticamente al momento del deployment, garantendo una copertura completa delle risorse

  ## Miglioramenti al web editor e al deployment

  - Corretto il flusso di creazione dei branch nel web editor per passare correttamente ai branch appena creati e rimanervi
  - Migliorata la finestra di dialogo dei conflitti di merge con una corretta funzionalità di escape; non è più necessario ricaricare la pagina per chiudere i conflitti
  - Ottimizzato il flusso di aggiornamento migliorando le prestazioni: invalidazione della cache solo delle pagine modificate durante aggiornamenti parziali, riducendo i tempi di deployment

  ## Miglioramenti all'autenticazione e alla navigazione

  - Nuovo supporto per l’autenticazione su sottodirectory personalizzate; se servi la documentazione su `https://yourdomain.com/docs`, l’autenticazione ora funziona senza problemi
  - Corretto un bug di visualizzazione della sidebar che compariva in modo errato quando era configurato un solo link
  - Revisione completa della navigazione mobile: pulsanti centrati con margini/padding adeguati, spaziatura migliorata nei menu a discesa, rimossi divisori e margini non necessari per le sezioni vuote e corretti i problemi di gap/padding nel tema Maple

  ## Correzioni a componenti e stile

  - Risolto il problema di conversione dei tag `<h1-6>` che li trasformava erroneamente in componenti Heading, compromettendo lo stile personalizzato
  - Aggiunto un interruttore di configurazione dell’Assistente one-click nella dashboard per una gestione più semplice

  ## Miglioramenti tecnici e affidabilità

  - Sistema di logging migliorato per i flussi di aggiornamento, consentendo un debugging e una risoluzione dei problemi più rapidi
  - Risolto il rate limiting di GitHub per i clienti con oltre 10 specifiche OpenAPI/AsyncAPI passando dal recupero dei singoli file al cloning del repository
  - Migliorata l’affidabilità dell’Assistente con supporto LLM di backup, gestione migliorata degli errori di rate limit e funzionalità dello strumento di ricerca più robuste
</Update>

<Update label="10–16 agosto" tags={["Improvements"]} rss={{ title: "Aggiornamenti settimanali", description: "Ottimizzazioni delle prestazioni, integrazioni LLM, miglioramenti al web editor e all'Area di prova API" }}>
  ## Ottimizzazioni delle prestazioni e della build

  - La transpilation MDX ora avviene al momento del deployment invece che a ogni caricamento di pagina negli ambienti serverless NextJS non in cache, migliorando il time to first byte per le pagine non in cache.
  - L’hashing basato sul contenuto evita la nuova transpilation quando l’MDX non è cambiato, riducendo i tempi del flusso di aggiornamento di circa il 50% per i clienti con molte pagine (i deployment superiori a 5 minuti dovrebbero ridursi di circa la metà)
  - La visualizzazione dei deployment di anteprima nella dashboard è ora più rapida grazie all’aggiunta di indici al database e alla parallelizzazione delle query nel nostro backend
  - Dimensioni della pagina ridotte eliminando i dati `navigation` duplicati nel payload `rsc` di ciascuna pagina—i miglioramenti di performance sono più evidenti con un alto numero di pagine o strutture di navigazione complesse
  - Prefetching più aggressivo per abilitare caricamenti istantanei più spesso

  ## Area di prova API e miglioramenti a OpenAPI

  - La conversione da OpenAPI a MCP è stata spostata nel backend, consentendo ai server MCP ospitati di includere tool (a breve documentazione e opzioni di configurazione più chiare)
  - Aggiunto il supporto a Ruby nell’Area di prova API
  - Abbiamo aggiunto una funzionalità che consente ora di [specificare le pagine API usando solo il tuo docs.json](/it/api-playground/openapi-setup#auto-populate-api-pages) senza creare nuovi file MDX.
  - Supporto per le pagine [`webhook`](/it/api-playground/openapi-setup#webhooks) nella navigazione della documentazione a partire da specifiche OpenAPI
  - Contesto del modello AI ottimizzato rimuovendo le specifiche dei link ancora dai link markdown quando si utilizza Anthropic, OpenAI o altri provider

  ## Miglioramenti al web editor

  - La creazione/rinominazione dei file ora salva le modifiche al clic fuori invece di richiedere la pressione del tasto Invio
  - Corretta la navigazione dei branch in cui cambiare l’URL a un branch specifico reindirizzava all’ultimo branch attivo invece della destinazione prevista
  - Codifica corretta nell’URL dei titoli dei branch contenenti `/` per prevenire interruzioni di navigazione
  - Corretto lo shortcut `Ctrl+K` nell’Editor della dashboard monorepo che anteponeva il percorso del repo delle docs creando link non funzionanti

  ## Analytics e integrazioni LLM

  - Supporto a `llms.txt` e `llms-full.txt` personalizzati—aggiungili alla root del repo della documentazione per servirli agli endpoint `/llms.txt` e `/llms-full.txt` per la personalizzazione degli LLM
  - Aggiunta l’[integrazione analytics Hightouch](/it/integrations/analytics/hightouch#hightouch)
  - Tracciamento analytics del Menu contestuale migliorato (visualizzazione in dashboard in arrivo)
  - Aggiunti test end-to-end per `llms.txt` e `llms-full.txt` per garantirne la corretta erogazione

  ## Miglioramenti a componenti e styling

  - Supporto per class name personalizzate nei tag `h{1-4}` per applicare stili di intestazione personalizzati
  - Corretto il rendering dei tag `h{1-4}` come componenti `Heading` con chip in modalità pagina personalizzata
  - Aggiunti selettori CSS ai [breadcrumbs](/it/navigation#breadcrumbs) per targeting CSS personalizzato
  - Corrette immagini Open Graph deformate analizzando le dimensioni per mantenere le proporzioni a 56px di altezza
  - Corretto `VSCode` in `VS Code` nel Menu contestuale quando abilitato
  - Corrette intestazioni all’interno di componenti personalizzati che apparivano nell’indice insieme alle intestazioni semantiche

  ## Correzioni di bug e affidabilità

  - Risolti problemi di rendering PDF con alcuni titoli di pagina sanificando i caratteri che causavano errori di generazione
  - Risolto l’errore CLI `Cannot convert undefined or null to object` quando si incontravano file JSON OpenAPI vuoti
  - Corretti metatag Open Graph `docs.json` personalizzati che venivano sovrascritti da quelli generati
  - Corretti i clic sul pulsante del feed RSS quando si atterrava su link ancora usando origin + pathname per i link RSS
  - Migliorata la velocità di download della CLI rimuovendo le sourcemap

  ## Miglioramenti tecnici

  - Aggiunti test visivi alla pipeline CI per una rilevazione precoce delle regressioni
  - Migliorata la gestione degli errori e le funzionalità di debugging
  - Copertura di test completa per nuove funzionalità e casi limite

  ## Nuovo schema di configurazione `docs.json`

  <Frame>
    ![screenshot di docs.json](/images/changelog/docsjson.png)
  </Frame>

  Abbiamo introdotto un nuovo schema `docs.json` in sostituzione di `mint.json`, per supportare un versioning multilivello più efficace, una comprensione visiva più immediata e una terminologia più coerente. Per ulteriori dettagli su cosa è cambiato, [consulta il nostro blog](https://mintlify.com/blog/refactoring-mint-json-into-docs-json).

  Esegui l’aggiornamento da `mint.json` a `docs.json` seguendo questi passaggi:

  1. Assicurati che la tua CLI sia aggiornata all’ultima versione

  ```bash
  npm i mint@latest -g
  ```

  1. Nel repository della documentazione, esegui

  ```bash
  mint upgrade
  ```

  1. Elimina il vecchio file `mint.json` e invia le modifiche

  ## Verifiche CI

  Esegui automaticamente il linting della documentazione per trovare link interrotti, rilevare errori di ortografia e grammatica o applicare stili di scrittura con la tua configurazione di Vale. Scopri di più nella nostra [documentazione](it/settings/ci).

  ## Supporto .md per LLM

  Tutte le pagine della documentazione sono ora automaticamente disponibili come file Markdown in chiaro: è sufficiente aggiungere `.md` all’URL. Questo semplifica l’acquisizione di singole pagine della documentazione da parte degli LLM.
