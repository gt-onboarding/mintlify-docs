Usa il CSS per definire lo stile degli elementi HTML oppure aggiungi CSS e JavaScript personalizzati per adattare completamente l'aspetto e l'esperienza della tua documentazione.

<div id="styling-with-tailwind-css">
  ## Styling con Tailwind CSS
</div>

Usa Tailwind CSS v3 per stilizzare gli elementi HTML. Puoi controllare layout, spaziatura, colori e altre proprietà visive. Alcune classi comuni sono:

* `w-full` - Larghezza piena
* `aspect-video` - Rapporto 16:9
* `rounded-xl` - Angoli arrotondati ampi
* `block`, `hidden` - Controllo della visualizzazione
* `dark:hidden`, `dark:block` - Visibilità in modalità scura

I valori arbitrari di Tailwind CSS non sono supportati. Per valori personalizzati, usa la prop `style`.

```html
<img style={{ width: '350px', margin: '12px auto' }} src="/path/image.jpg" />
```


<div id="custom-css">
  ## CSS personalizzato
</div>

Aggiungi file CSS al tuo repository e i nomi delle classi definiti verranno applicati e resi disponibili in tutti i tuoi file `MDX`.

<div id="adding-stylecss">
  ### Aggiunta di `style.css`
</div>

Ad esempio, puoi aggiungere il seguente file `style.css` per personalizzare lo stile della barra di navigazione e del footer.

```css
#navbar {
  background: "#fffff2";
  padding: 1rem;
}

footer {
  margin-top: 2rem;
}
```

<div id="custom-javascript">
  ## JavaScript personalizzato
</div>

Il JavaScript personalizzato consente di aggiungere codice eseguibile personalizzato a livello globale. È l’equivalente di aggiungere un tag `<script>` con codice JS in ogni pagina.

<div id="adding-custom-javascript">
  ### Aggiunta di JavaScript personalizzato
</div>

Qualsiasi file `.js` nella directory dei contenuti della documentazione verrà incluso in ogni pagina. Ad esempio, puoi aggiungere il file `ga.js` seguente per abilitare [Google Analytics](https://marketingplatform.google.com/about/analytics) in tutta la documentazione.

```js
window.dataLayer = window.dataLayer || [];
function gtag() {
  dataLayer.push(arguments);
}
gtag('js', new Date());

gtag('config', 'TAG_ID');
```

<Warning>
  Utilizzalo con cautela per evitare di introdurre vulnerabilità di sicurezza.
</Warning>
