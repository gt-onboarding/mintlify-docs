Per ospitare la documentazione su un dominio personalizzato, imposta il dominio desiderato nelle impostazioni di Mintlify e configura i record DNS presso il tuo provider di domini.

<Info>
  Vuoi configurare una sottodirectory personalizzata come `mintlify.com/docs`? Consulta la documentazione sulla [sottodirectory personalizzata](/it/deploy/cloudflare).
</Info>

<div id="dashboard-settings">
  ## Impostazioni della dashboard
</div>

1. Vai alla tua [dashboard](https://dashboard.mintlify.com).
2. Seleziona **Settings**.
3. Seleziona **Domain Setup**.
4. Inserisci il dominio personalizzato che desideri. Ad esempio, `docs.mintlify.com`.

<div id="configure-your-dns">
  ## Configura il DNS
</div>

1. Vai alle impostazioni DNS del tuo dominio sul sito del provider del dominio.
2. Crea un nuovo record DNS con i seguenti valori:

```bash
CNAME | docs | cname.vercel-dns.com.
```

  </Accordion>

<div id="set-a-canonical-url">
  ## Imposta un URL canonico
</div>

Dopo aver configurato il DNS, imposta un URL canonico per assicurarti che i motori di ricerca indicizzino il tuo dominio preferito. Un URL canonico comunica ai motori di ricerca quale versione della documentazione è quella principale. Questo migliora la SEO quando la documentazione è accessibile da più URL e previene problemi di contenuti duplicati.

Aggiungi il meta tag `canonical` al file `docs.json`:

```json
"seo": {
    "metatags": {
        "canonical": "https://www.tuo-dominio-personalizzato.com"
    }
}
```

Sostituisci `https://www.your-custom-domain-here.com` con il tuo dominio personalizzato reale. Ad esempio, se il tuo dominio personalizzato è `docs.mintlify.com`, useresti:

```json
"seo": {
    "metatags": {
        "canonical": "https://docs.mintlify.com"
    }
}
```
