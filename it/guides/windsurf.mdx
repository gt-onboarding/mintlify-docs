Trasforma Windsurf in un esperto di documentazione che comprende le tue linee guida di stile, i componenti e il contesto del progetto grazie a regole dello spazio di lavoro e memorie.

<div id="using-windsurf-with-mintlify">
  ## Uso di Windsurf con Mintlify
</div>

L’Assistente AI Cascade di Windsurf può essere configurato per redigere la documentazione secondo i tuoi standard utilizzando i componenti di Mintlify. Le regole dell’area di lavoro e le memorie forniscono un contesto persistente sul tuo progetto, garantendo suggerimenti più coerenti da parte di Cascade.

* Le **regole dell’area di lavoro** sono archiviate nel repository della documentazione e condivise con il team.
* Le **memorie** forniscono un contesto individuale che si arricchisce nel tempo.

Consigliamo di definire regole dell’area di lavoro per standard di documentazione condivisi. Puoi sviluppare memorie man mano che lavori, ma poiché non sono condivise, potrebbero non essere coerenti tra i membri del team.

Crea le regole dell’area di lavoro nella directory `.windsurf/rules` del repository della documentazione. Consulta [Memories & Rules](https://docs.windsurf.com/windsurf/cascade/memories) nella documentazione di Windsurf per ulteriori informazioni.

<div id="example-workspace-rule">
  ## Esempio di regola dell'area di lavoro
</div>

Questa regola fornisce a Cascade il contesto sui componenti di Mintlify e sulle buone pratiche generali di technical writing.

Puoi utilizzare questa regola di esempio così com’è oppure personalizzarla per la tua documentazione:

* **Standard di scrittura**: Aggiorna le linee guida linguistiche per allinearle al tuo style guide.
* **Pattern dei componenti**: Aggiungi componenti specifici del progetto o modifica gli esempi esistenti.
* **Esempi di codice**: Sostituisci esempi generici con chiamate e risposte API reali per il tuo prodotto.
* **Preferenze di stile e tono**: Regola terminologia, formattazione e altre regole.

Salva la tua regola come file `.md` nella directory `.windsurf/rules` del repository della tua documentazione.

````mdx
# Mintlify technical writing rule

## Project context

- This is a documentation project on the Mintlify platform
- We use MDX files with YAML frontmatter  
- Navigation is configured in `docs.json`
- We follow technical writing best practices

## Writing standards

- Use second person ("you") for instructions
- Write in active voice and present tense
- Start procedures with prerequisites
- Include expected outcomes for major steps
- Use descriptive, keyword-rich headings
- Keep sentences concise but informative

## Required page structure

Every page must start with frontmatter:

```yaml
---
title: "Clear, specific title"
description: "Concise description for SEO and navigation"
---
```

## Mintlify components

### docs.json

- Refer to the [docs.json schema](https://mintlify.com/docs.json) when building the docs.json file and site navigation

### Callouts

- `<Note>` for helpful supplementary information
- `<Warning>` for important cautions and breaking changes
- `<Tip>` for best practices and expert advice  
- `<Info>` for neutral contextual information
- `<Check>` for success confirmations

### Code examples

- When appropriate, include complete, runnable examples
- Use `<CodeGroup>` for multiple language examples
- Specify language tags on all code blocks
- Include realistic data, not placeholders
- Use `<RequestExample>` and `<ResponseExample>` for API docs

### Procedures

- Use `<Steps>` component for sequential instructions
- Include verification steps with `<Check>` components when relevant
- Break complex procedures into smaller steps

### Content organization

- Use `<Tabs>` for platform-specific content
- Use `<Accordion>` for progressive disclosure
- Use `<Card>` and `<CardGroup>` for highlighting content
- Wrap images in `<Frame>` components with descriptive alt text

## API documentation requirements

- Document all parameters with `<ParamField>` 
- Show response structure with `<ResponseField>`
- Include both success and error examples
- Use `<Expandable>` for nested object properties
- Always include authentication examples

## Quality standards

- Test all code examples before publishing
- Use relative paths for internal links
- Include alt text for all images
- Ensure proper heading hierarchy (start with h2)
- Check existing patterns for consistency
````

<div id="working-with-cascade">
  ## Lavorare con Cascade
</div>

Una volta definite le regole, puoi usare Cascade per supportarti in varie attività di documentazione. Per ulteriori informazioni, consulta [Cascade](https://docs.windsurf.com/windsurf/cascade) nella documentazione di Windsurf.

<div id="example-prompts">
  ### Esempi di prompt
</div>

**Scrittura di nuovi contenuti**:

```text wrap
Crea una nuova pagina che spieghi come eseguire l’autenticazione con la nostra API. Includi esempi di codice in JavaScript, Python e cURL.
```

**Miglioramento dei contenuti esistenti**:

```text wrap
Rivedi questa pagina e suggerisci miglioramenti per la chiarezza e per l’uso dei componenti. Concentrati nel rendere i passaggi più facili da seguire.
```

**Creazione di esempi di codice**:

```text wrap
Genera un esempio di codice completo che mostri la gestione degli errori per questo endpoint API. Usa dati realistici e includi le risposte previste.
```

**Mantenimento della coerenza**:

```text wrap
Verifica che questa nuova pagina rispetti i nostri standard di documentazione e suggerisci eventuali modifiche necessarie.
```
