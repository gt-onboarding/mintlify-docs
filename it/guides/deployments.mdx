Il tuo sito di documentazione viene distribuito automaticamente quando esegui il push delle modifiche al repository collegato. Questo richiede che l’app GitHub di Mintlify sia installata e connessa correttamente.

Se le ultime modifiche non compaiono sul tuo sito live, verifica innanzitutto che l’app GitHub sia installata sull’account o sull’organizzazione proprietaria del repository della documentazione. Consulta la sezione [Risoluzione dei problemi di GitHub](/it/settings/github#troubleshooting) per maggiori informazioni.

Se l’app GitHub è connessa ma le modifiche continuano a non essere distribuite, puoi avviare manualmente una ricostruzione dalla tua dashboard.

<div id="manually-triggering-a-deployment">
  ## Avviare manualmente un deployment
</div>

<Steps>
  <Step title="Verifica che l'ultimo commit sia riuscito.">
    Controlla che l’ultimo commit compaia nel repository della documentazione e che non abbia generato errori.
  </Step>

  <Step title="Avvia manualmente un deployment.">
    Vai alla [dashboard](https://dashboard.mintlify.com) e seleziona il pulsante di deploy.
