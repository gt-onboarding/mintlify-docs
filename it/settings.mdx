Il file `docs.json` consente di trasformare una raccolta di file Markdown in un sito di documentazione navigabile e personalizzato. Questo file di configurazione obbligatorio controlla stile, navigazione, integrazioni e altro ancora. Consideralo come il blueprint della tua documentazione.

Le impostazioni in `docs.json` si applicano globalmente a tutte le pagine.

<div id="setting-up-your-docsjson">
  ## Configurazione del file `docs.json`
</div>

Per iniziare, è sufficiente specificare `theme`, `name`, `colors.primary` e `navigation`. Gli altri campi sono facoltativi e puoi aggiungerli man mano che aumentano le esigenze della tua documentazione.

Per la migliore esperienza di editing, includi il riferimento allo schema all’inizio del file `docs.json`. Questo abilita il completamento automatico, la validazione e utili suggerimenti contestuali nella maggior parte degli editor di codice:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Struttura di navigazione
  }
  // Resto della configurazione
}
```

<ResponseField name="theme" required>
  Il tema del layout del tuo sito.

  Una delle seguenti opzioni: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Consulta [Temi](it/themes) per ulteriori informazioni.
</ResponseField>

<ResponseField name="name" type="string" required>
  Il nome del tuo progetto, organizzazione o prodotto.
</ResponseField>

<ResponseField name="colors" type="object" required>
  I colori utilizzati nella documentazione. L’applicazione dei colori varia a seconda del tema. Se fornisci solo un colore primario, verrà usato per tutti gli elementi cromatici.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      Il colore primario della documentazione. In genere usato per l’enfasi in modalità chiara, con alcune variazioni in base al tema.

      Deve essere un codice esadecimale che inizia con `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Il colore usato per l’enfasi in modalità scura.

      Deve essere un codice esadecimale che inizia con `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Il colore usato per pulsanti e stati hover sia in modalità chiara che scura, con alcune variazioni in base al tema.

      Deve essere un codice esadecimale che inizia con `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

      * `cursor`: Installa il tuo server MCP ospitato in Cursor.
      * `vscode`: Installa il tuo server MCP ospitato in VSCode.

        Il menu contestuale è disponibile solo sui deployment di anteprima e di produzione.

<div id="upgrading-from-mintjson">
  ## Aggiornamento da `mint.json`
</div>

Se il tuo progetto di documentazione utilizza il file deprecato `mint.json`, segui questi passaggi per passare a `docs.json`.

<Steps>
  <Step title="Installa o aggiorna la CLI">
    Se non hai ancora installato la [CLI](/it/installation), installala ora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Se la CLI è già installata, assicurati che sia aggiornata:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Crea il file docs.json">
    Nel repository della tua documentazione, esegui:

    ```bash
    mint upgrade
    ```

    Questo comando creerà un file `docs.json` a partire dal `mint.json` esistente. Controlla il file generato per assicurarti che tutte le impostazioni siano corrette.
  </Step>

  <Step title="Elimina il file mint.json">
    Dopo aver verificato che `docs.json` è configurato correttamente, puoi eliminare in sicurezza il vecchio file `mint.json`.
  </Step>
</Steps>
