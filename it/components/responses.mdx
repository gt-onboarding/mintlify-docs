---
title: "Campi di risposta"
description: "Mostra i valori di risposta dell'API"
---

Il componente `<ResponseField>` è pensato per definire i valori restituiti da un'API. In molte documentazioni si usa anche `<ResponseField>` nelle pagine quando è necessario elencare i tipi di un elemento.

<ResponseField name="response" type="string" required>
  Un esempio di campo di risposta
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un esempio di campo di risposta
</ResponseField>
```

<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Il nome del valore di risposta.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo previsto del valore di risposta: può essere una qualsiasi stringa.
</ResponseField>

<ResponseField name="default" type="string">
  Il valore predefinito.
</ResponseField>

<ResponseField name="required" type="boolean">
  Mostra "required" accanto al nome del campo.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indica se un campo è obsoleto o meno.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Etichette visualizzate prima del nome del campo
</ResponseField>

<ResponseField name="post" type="string[]">
  Etichette visualizzate dopo il nome del campo
</ResponseField>
