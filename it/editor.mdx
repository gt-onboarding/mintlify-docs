Accedi all’editor visuale dalla tua [dashboard](https://dashboard.mintlify.com/editor) per gestire la documentazione direttamente nel browser.

* **Modifica WYSIWYG**: Apporta modifiche alla documentazione con un editor what-you-see-is-what-you-get (WYSIWYG) che mostra l’anteprima di come apparirà il contenuto una volta pubblicato.
* **Sincronizzazione con Git**: Tutte le modifiche vengono sincronizzate automaticamente con il tuo repository Git per mantenere il controllo delle versioni.
* **Collaborazione del team**: Più persone possono lavorare sulla documentazione contemporaneamente.
* **Integrazione dei componenti**: Aggiungi callout, blocchi di codice e altri componenti con i comandi con slash.
* **Nessuna configurazione necessaria**: Inizia a scrivere subito dalla dashboard.

<div id="overview">
  ## Panoramica
</div>

Ecco come lavori di solito nell’editor visuale:

<Steps>
  <Step title="Scegli il branch">
    Crea un branch o apporta modifiche direttamente al branch di deployment. Ti consigliamo di creare un branch così puoi visualizzare l’anteprima delle modifiche prima che vadano online.
  </Step>

  <Step title="Apri il file">
    Passa a un file esistente nella barra laterale oppure creane uno nuovo usando l’esplora file.
  </Step>

  <Step title="Modifica i contenuti">
    Apporta modifiche nell’editor visuale. Premi <kbd>/</kbd> per aprire il menu dei componenti.
  </Step>

  <Step title="Visualizza l’anteprima delle modifiche">
    La modalità visuale mostra come appariranno le modifiche sul tuo sito live. Usala per verificare che sia tutto corretto.
  </Step>

  <Step title="Pubblica le modifiche">
    Se stai lavorando sul branch di deployment, pubblica le modifiche direttamente dall’editor visuale. Sugli altri branch, crea una pull request per la revisione prima della pubblicazione.
  </Step>
</Steps>

<div id="organize-your-navigation">
  ### Organizza la navigazione
</div>

Modifica il file `docs.json` per aggiungere nuove pagine e rimuovere quelle eliminate dalla navigazione del sito. Consulta [Navigation](/it/organize/navigation) per maggiori informazioni su come organizzare le pagine.

**Esempio: aggiungere una pagina Themes al gruppo Profile**

In questo esempio, hai creato una nuova pagina intitolata Themes e vuoi aggiungerla al gruppo Profile nella documentazione. Aggiungi il percorso della nuova pagina all’array `pages` del gruppo `Profile` nel file `docs.json` affinché compaia nella navigazione del sito.

```json title="Adding a Themes page to the Profile group" highlight="18"
{
    "navigation": {
        "groups": [
            {
                "group": "Inizia da qui",
                "pages": [
                    "index",
                    "quickstart",
                    "installation"
                ]
            },
            {
                "group": "Profilo",
                "pages": [
                    "settings",
                    "account-types",
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```

  ## Pubblica le tue modifiche
</div>

Il branch su cui lavori determina come l’editor pubblica le modifiche:

* **Branch di deployment**: La pubblicazione aggiorna immediatamente il sito in produzione.
* **Altri branch**: La pubblicazione crea una pull request così puoi rivedere le modifiche prima di distribuirle in produzione.

<div id="pull-requests-and-reviewing-changes">
  ### Pull request e revisione delle modifiche
</div>

Le pull request ti permettono di proporre modifiche dal tuo branch, così che altre persone possano esaminarle prima di unirle alla documentazione in produzione. Questo aiuta a garantire che le modifiche siano corrette e offre al tuo team l&#39;opportunità di collaborare sui contenuti.

<Tip>
  Anche se lavori da solo, le pull request sono utili per visualizzare in anteprima le modifiche e mantenere una cronologia chiara degli aggiornamenti.
</Tip>

<div id="create-a-pull-request">
  #### Crea una pull request
</div>

<Steps>
  <Step title="Salva il tuo lavoro">
    Seleziona **Save Changes** per salvare tutte le modifiche sul tuo branch.
  </Step>

  <Step title="Crea la pull request">
    Seleziona **Publish Pull Request** dalla barra degli strumenti dell’editor.
  </Step>

  <Step title="Aggiungi un titolo e una descrizione">
    Scrivi un titolo e una descrizione chiari che spieghino:

    * Quali modifiche hai apportato
    * Perché le hai apportate
    * Eventuali aree specifiche che richiedono revisione
  </Step>

  <Step title="Crea e condividi">
    Seleziona **Publish Pull Request**. L’editor fornirà un link per visualizzare la tua pull request.

    <Frame>
      <img src="/images/editor/pull-request-light.png" alt="Pulsante Publish Pull Request evidenziato nell’editor visuale di Mintlify" className="block dark:hidden" />

      <img src="/images/editor/pull-request-dark.png" alt="Pulsante Publish Pull Request evidenziato nell’editor visuale di Mintlify" className="hidden dark:block" />
    </Frame>
  </Step>
</Steps>

<div id="review-pull-requests">
  #### Revisionare le pull request
</div>

Una volta creata la pull request:

1. **Rivedi le modifiche**: Tu e i membri del team potete rivedere la pull request nel vostro provider Git, ad esempio GitHub o GitLab.
2. **Lascia un feedback**: Aggiungi commenti o richiedi modifiche.
3. **Apporta ulteriori modifiche**: Effettua ulteriori modifiche nell’editor visuale. Quando salvi, l’editor le invia alla pull request.
4. **Approva**: Approva la pull request quando sei soddisfatto delle modifiche.
5. **Esegui il merge**: Esegui il merge della pull request quando sei pronto a distribuire le modifiche in produzione.

<div id="troubleshooting">
  ## Risoluzione dei problemi
</div>

Trova soluzioni ai problemi comuni che potresti incontrare usando l’editor visuale.

<AccordionGroup>
  <Accordion title="Le modifiche non compaiono dopo la pubblicazione">
    **Possibili cause:**

    * La distribuzione è ancora in corso
    * Problemi di cache del browser
    * Errori di build o di distribuzione

    **Soluzioni:**

    1. Controlla lo stato della distribuzione nella dashboard.
    2. Esegui un aggiornamento forzato del browser (<kbd>Ctrl</kbd> + <kbd>F5</kbd> o <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>R</kbd>)
    3. Svuota la cache del browser.
  </Accordion>

  <Accordion title="Errori di autorizzazione durante la pubblicazione">
    **Possibili cause:**

    * Permessi insufficienti sul repository Git
    * Problemi di autenticazione con il provider Git

    **Soluzioni:**

    1. Verifica di avere i permessi corretti sul repository.
    2. Controlla che l’integrazione con Git sia configurata correttamente.
    3. Consulta la documentazione su [Editor Permissions](/it/dashboard/permissions).
  </Accordion>

  <Accordion title="Problemi di caricamento dell'editor">
    **Possibili cause:**

    * Problemi di connettività di rete
    * Repository di documentazione di grandi dimensioni

    **Soluzioni:**

    1. Controlla la connessione a Internet.
    2. Aggiorna la pagina.
    3. Contatta il supporto se il problema persiste.
  </Accordion>

  <Accordion title="File che non si caricano o mostrano errori">
    **Possibili cause:**

    * Sintassi MDX non valida nei file
    * File mancanti o corrotti
    * File di grandi dimensioni che causano timeout

    **Soluzioni:**

    1. Controlla la sintassi del file per eventuali errori di formattazione MDX.
    2. Verifica che il file esista nel repository.
  </Accordion>
</AccordionGroup>
