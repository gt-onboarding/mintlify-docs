---
title: "Configurazione MDX"
description: "Genera pagine di documentazione per i tuoi endpoint API usando `MDX`"
---

Puoi definire manualmente gli endpoint API in singoli file `MDX` invece di usare una specifica OpenAPI. Questo metodo offre flessibilità per contenuti personalizzati, ma consigliamo di generare la documentazione API da un file di specifica OpenAPI per la maggior parte dei progetti perché è più facile da mantenere e più ricco di funzionalità. Tuttavia, creare pagine `MDX` per un'API può essere utile per documentare API di piccole dimensioni o per prototipare.

Per generare pagine per endpoint API usando `MDX`, configura le impostazioni della tua API in `docs.json`, crea singoli file `MDX` per ciascun endpoint e usa componenti come `<ParamFields />` per definire i parametri. A partire da queste definizioni, Mintlify genera aree di prova API interattive, esempi di richiesta ed esempi di risposta.

<Steps>
  <Step title="Configura la tua API">
    Nel file `docs.json`, definisci l'URL di base e il metodo di autenticazione:

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // array di stringhe per più URL di base
        "auth": {
          "method": "key",
          "name": "x-api-key" // opzioni: bearer, basic, key.
        }
      }
    }
    ```

    Se vuoi nascondere l'area di prova API, usa il campo `display`. Non è necessario includere un metodo di autenticazione se nascondi l'area di prova.

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    Trovi l'elenco completo delle configurazioni API in [Settings](/it/settings#api-configurations).
  </Step>

  <Step title="Crea le pagine dei tuoi endpoint">
    Ogni pagina di endpoint API dovrebbe avere un corrispondente file `MDX`. All'inizio di ogni file, definisci `title` e `api`:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    Puoi specificare i parametri di percorso aggiungendo il nome del parametro al percorso, racchiuso tra `{}`:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      Se hai configurato un campo `server` in `docs.json`, puoi usare percorsi relativi come `/v1/endpoint`.
    </Note>

    Puoi eseguire l'override della modalità di visualizzazione definita globalmente per l'area di prova API in una pagina aggiungendo `playground` al frontmatter:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    - `playground: 'interactive'` - Mostra l'area di prova interattiva.
    - `playground: 'simple'` - Mostra un endpoint copiabile senza area di prova.
    - `playground: 'none'` - Nasconde l'area di prova.
  </Step>

  <Step title="Aggiungi i tuoi endpoint alla documentazione">
    Aggiungi le pagine dei tuoi endpoint alla sidebar includendo i percorsi nel campo `navigation` del file `docs.json`. Scopri di più su come strutturare la documentazione in [Navigation](/it/navigation).
  </Step>
</Steps>

<div id="enabling-authentication">
  ## Abilitazione dell'autenticazione
</div>

Puoi aggiungere un metodo di autenticazione al file `docs.json` per abilitarlo globalmente su ogni pagina, oppure puoi impostarlo a livello di singola pagina.

Il metodo di autenticazione definito in una pagina ha priorità su quello globale se entrambi sono configurati.

<div id="bearer-token">
  ### Token bearer
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Titolo della pagina"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

<div id="none">
  ### Nessuna
</div>

Il metodo di autenticazione `none` è utile per disattivare l’autenticazione su un endpoint specifico dopo aver impostato un valore predefinito in docs.json.

<CodeGroup>
  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
