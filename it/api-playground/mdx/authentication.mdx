<div id="enabling-authentication">
  ## Abilitazione dell'autenticazione
</div>

Puoi aggiungere un metodo di autenticazione al file `docs.json` per abilitarlo globalmente su ogni pagina, oppure puoi impostarlo a livello di singola pagina.

Il metodo di autenticazione definito su una pagina ha la precedenza su quello globale se entrambi sono impostati.

<div id="bearer-token">
  ### Token bearer
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "Titolo della pagina"
authMethod: "bearer"
---
```

</CodeGroup>

<div id="basic-authentication">
  ### Autenticazione di base
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "basic"
      }
    }
}
```

```mdx Page Metadata
---
title: "Titolo della pagina"
authMethod: "basic"
---
```

</CodeGroup>

<div id="api-key">
  ### Chiave API
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "Titolo della pagina"
authMethod: "key"
---
```

</CodeGroup>

<div id="none">
  ### Nessuna
</div>

Il metodo di autenticazione "none" è utile per disattivare l'autenticazione su un endpoint specifico dopo aver definito un'impostazione predefinita in docs.json.

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
