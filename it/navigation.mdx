La proprietà [navigation](it/settings#param-navigation) in `docs.json` controlla la struttura e la gerarchia informativa della documentazione.

Con una configurazione della navigazione appropriata, puoi organizzare i contenuti in modo che gli utenti trovino esattamente ciò che cercano.

<div id="pages">
  ## Pagine
</div>

Le pagine sono il componente di navigazione più fondamentale. Le pagine corrispondono ai file MDX che compongono la documentazione.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

Nell'oggetto `navigation`, `pages` è un array in cui ogni elemento deve fare riferimento al percorso di un [file di pagina](it/pages).

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## Gruppi
</div>

Usa i gruppi per organizzare la navigazione della barra laterale in sezioni. I gruppi possono essere nidificati, etichettati con tag e rappresentati con icone.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

Nell'oggetto `navigation`, `groups` è un array in cui ogni elemento è un oggetto che richiede i campi `group` e `pages`. I campi `icon` e `tag` sono opzionali.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "icon": "play",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Writing content",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="tabs">
  ## Schede
</div>

Le schede creano sezioni distinte della documentazione con percorsi URL separati. Generano una barra di navigazione orizzontale nella parte superiore della documentazione che consente agli utenti di passare da una sezione all’altra.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

Nell’oggetto `navigation`, `tabs` è un array in cui ogni elemento è un oggetto che richiede il campo `tab` e può includere altri campi di navigazione come gruppi, pagine, icone o collegamenti a pagine esterne.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### Menu
</div>

I menu aggiungono elementi di navigazione a discesa a una scheda. Usa i menu per consentire agli utenti di accedere direttamente a pagine specifiche all'interno di una scheda.

Nell'oggetto `navigation`, `menu` è un array in cui ogni voce è un oggetto che richiede il campo `item` e può contenere altri campi di navigazione come gruppi, pagine, icone o collegamenti a pagine esterne.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Developer tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Core endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "Gli SDK vengono utilizzati per interagire con l'API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## Ancore
</div>

Le ancore aggiungono elementi di navigazione persistenti nella parte superiore della barra laterale. Usa le ancore per suddividere i contenuti, offrire accesso rapido a risorse esterne o creare call to action in evidenza.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

Nell'oggetto `navigation`, `anchors` è un array in cui ogni voce è un oggetto che richiede un campo `anchor` e può contenere altri campi di navigazione come gruppi, pagine, icone o collegamenti a pagine esterne.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Per le ancore che rimandano solo a collegamenti esterni, usa la keyword `global`. Le ancore in un oggetto `global` devono avere un campo `href` e non possono puntare a un percorso relativo.

Le ancore globali sono particolarmente utili per collegare risorse che non fanno parte della documentazione, ma che dovrebbero essere facilmente accessibili agli utenti, come un blog o un portale di supporto.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## Menu a discesa
</div>

I menu a discesa sono ospitati in un menu espandibile nella parte superiore della barra laterale di navigazione. Ogni elemento di un menu a discesa rimanda a una sezione della documentazione.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

Nell’oggetto `navigation`, `dropdowns` è un array in cui ogni voce è un oggetto che richiede il campo `dropdown` e può includere altri campi di navigazione come gruppi, pagine, icone o collegamenti a pagine esterne.

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="openapi">
  ## OpenAPI
</div>

Integra le specifiche OpenAPI direttamente nella struttura di navigazione per generare automaticamente la documentazione delle API. Crea sezioni API dedicate oppure inserisci le pagine degli endpoint all'interno di altri componenti di navigazione.

Imposta una specifica OpenAPI predefinita a qualsiasi livello della gerarchia di navigazione. Gli elementi figli erediteranno questa specifica, a meno che non ne definiscano una propria.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API reference",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Per maggiori informazioni su come fare riferimento agli endpoint OpenAPI nella documentazione, consulta la pagina [Configurazione OpenAPI](/it/api-playground/openapi-setup).

<div id="versions">
  ## Versioni
</div>

Suddividi la navigazione in versioni diverse. Le versioni sono selezionabili da un menu a discesa.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

Nell'oggetto `navigation`, `versions` è un array in cui ogni elemento è un oggetto che richiede il campo `version` e può contenere qualsiasi altro campo di navigazione.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

<div id="breadcrumbs">
  ## Breadcrumbs
</div>

I breadcrumb mostrano l’intero percorso di navigazione nella parte superiore delle pagine. Alcuni temi hanno i breadcrumb abilitati per impostazione predefinita, altri no. Puoi controllare se i breadcrumb sono abilitati per il tuo sito utilizzando la proprietà `styling` nel file `docs.json`.

<CodeGroup>
  ```json Display full breadcrumbs
  "styling": {
    "eyebrows": "breadcrumbs"
  }
  ```

  ```json Display parent section only
  "styling": {
    "eyebrows": "section"
  }
  ```
</CodeGroup>

<div id="interaction-configuration">
  ## Configurazione dell'interazione
</div>

Controlla come gli utenti interagiscono con gli elementi di navigazione utilizzando la proprietà `interaction` nel file `docs.json`.

<div id="enable-auto-navigation-for-groups">
  ### Abilitare la navigazione automatica per i gruppi
</div>

Quando un utente espande un gruppo di navigazione, alcuni temi passeranno automaticamente alla prima pagina del gruppo. Puoi modificare il comportamento predefinito del tema utilizzando l'opzione `drilldown`:

```json
{
  "interaction": {
    "drilldown": true  // Forza la navigazione alla prima pagina quando un utente espande un menu a discesa
  }
}
```

```json
{
  "interaction": {
    "drilldown": false // Non navigare mai, limita a espandere/comprimere
  }
}
```

- Imposta su `true` per forzare la navigazione automatica alla prima pagina quando viene selezionato un gruppo di navigazione.
- Imposta su `false` per impedire la navigazione e limitarsi a espandere o comprimere il gruppo quando viene selezionato.
- Lascia non impostato per utilizzare il comportamento predefinito del tema.

<div id="faq">
  ## Domande frequenti
</div>

<Accordion title="Quali cartelle di navigazione devo evitare?">
  Non utilizzare `/api` o `/mcp` come nomi del percorso radice (titoli o cartelle). I percorsi `/api` e `/mcp` sono riservati in produzione e faranno restituire errori 404 alle pagine se i relativi URL contengono `/api` o `/mcp`.
</Accordion>
