Per accedere alla documentazione, gli utenti devono prima autenticarsi.

<div id="authentication-modes">
  ## Modalità di autenticazione
</div>

Scegli tra autenticazione completa e parziale in base alle tue esigenze di controllo degli accessi.

**Autenticazione completa**: Tutte le pagine sono protette. Gli utenti devono effettuare l’accesso prima di poter visualizzare qualsiasi contenuto.

**Autenticazione parziale**: Alcune pagine sono pubbliche, mentre altre richiedono l’autenticazione. Gli utenti possono consultare liberamente i contenuti pubblici e autenticarsi solo quando accedono alle pagine protette.

Quando configuri uno dei metodi di handshake riportati di seguito, seleziona **Autenticazione completa** o **Autenticazione parziale** nelle impostazioni della dashboard.

</div>

Seleziona il metodo di handshake che vuoi configurare.

<Tabs>
  <Tab title="Password">
    <Info>
      L'autenticazione tramite password offre solo controllo degli accessi e **non** supporta la personalizzazione dei contenuti.
    </Info>

    ### Prerequisiti

    - I tuoi requisiti di sicurezza consentono la condivisione delle password tra gli utenti.

    ### Implementazione

    <Steps>
      <Step title="Crea una password.">
        1. Nel dashboard, vai su [Authentication](https://dashboard.mintlify.com/settings/deployment/authentication).
        2. Seleziona **Autenticazione completa** o **Autenticazione parziale**.
        3. Seleziona **Password**.
        4. Inserisci una password sicura.
        5. Seleziona **Salva modifiche**.
      </Step>

      <Step title="Distribuisci l'accesso.">
        Condividi in modo sicuro la password e l'URL della documentazione con gli utenti autorizzati.
      </Step>
    </Steps>

    ## Esempio

    La tua documentazione è ospitata su `docs.foo.com` e ti serve un controllo degli accessi di base senza tracciare i singoli utenti. Vuoi impedire l'accesso pubblico mantenendo la configurazione semplice.

    **Crea una password forte** nel dashboard. **Condividi le credenziali** con gli utenti autorizzati. Tutto qui!
  </Tab>

  <Tab title="Mintlify Dashboard">
    ### Prerequisiti

    - Gli utenti della tua documentazione sono anche i relativi editor.

    ### Implementazione

    <Steps>
      <Step title="Abilita l'autenticazione della dashboard Mintlify.">
        1. Nella dashboard, vai a [Authentication](https://dashboard.mintlify.com/settings/deployment/authentication).
        2. Seleziona **Autenticazione completa** o **Autenticazione parziale**.
        3. Seleziona **Mintlify Auth**.

        1. Nella dashboard, vai a [Members](https://dashboard.mintlify.com/settings/organization/members).

        3. Assegna i ruoli appropriati in base alle loro autorizzazioni di modifica.
      </Step>
    </Steps>

    ### Esempio

    La documentazione è ospitata su `docs.foo.com` e il tuo team utilizza la dashboard per modificarla. Vuoi limitare l’accesso ai soli membri del team.

    **Abilita l’autenticazione Mintlify** nelle impostazioni della dashboard.

    **Verifica l’accesso del team** controllando che tutti i membri siano stati aggiunti all’organizzazione.

  <Tab title="OAuth 2.0">
    ### Prerequisiti

    - Un server OAuth o OIDC che supporti l’Authorization Code Flow.
    - Possibilità di creare un endpoint API accessibile tramite token di accesso OAuth (opzionale, per abilitare le funzionalità di personalizzazione).

    ### Implementazione

    <Steps>
      <Step title="Configura le impostazioni OAuth.">
        1. Nel dashboard, vai su [Authentication](https://dashboard.mintlify.com/settings/deployment/authentication).
        2. Seleziona **Full Authentication** o **Partial Authentication**.
        3. Seleziona **OAuth** e configura questi campi:

        - **Authorization URL**: l’endpoint OAuth.
        - **Client ID**: l’identificatore client di OAuth 2.0.
        - **Client Secret**: il secret del client OAuth 2.0.
        - **Scopes**: autorizzazioni da richiedere. Copia l’**intera** stringa dello scope (ad esempio, per uno scope come `provider.users.docs`, copia per intero `provider.users.docs`). Usa più scope se ti servono diversi livelli di accesso.
        - **Token URL**: l’endpoint di scambio del token OAuth.
        - **Info API URL** (opzionale): endpoint per recuperare le informazioni utente per la personalizzazione. Se omesso, il flusso OAuth verrà usato solo per verificare l’identità e le informazioni utente saranno vuote.
        - **Logout URL**: l’URL di logout nativo del provider OAuth. Se il provider prevede un parametro `returnTo` o simile, reindirizzalo all’URL della tua documentazione.

        4. Seleziona **Save changes**.
      </Step>

      <Step title="Configura il tuo server OAuth.">
        1. Copia la **Redirect URL** dalle [impostazioni di autenticazione](https://dashboard.mintlify.com/settings/deployment/authentication).
        2. Aggiungi la Redirect URL come URL di reindirizzamento autorizzato nel tuo server OAuth.
      </Step>

      <Step title="Crea l’endpoint delle informazioni utente (opzionale).">
        Per abilitare le funzionalità di personalizzazione, crea un endpoint API che:

        - Accetti token di accesso OAuth per l’autenticazione.
        - Restituisca i dati utente nel formato `User`. Consulta [Formato dei dati utente](/it/authentication-personalization/personalization-setup#user-data-format) per maggiori informazioni.

        Aggiungi l’URL di questo endpoint al campo **Info API URL** nelle [impostazioni di autenticazione](https://dashboard.mintlify.com/settings/deployment/authentication).
      </Step>
    </Steps>

    ### Esempio

    La tua documentazione è ospitata su `foo.com/docs` e disponi di un server OAuth esistente su `auth.foo.com` che supporta l’Authorization Code Flow.

    **Configura i dettagli del server OAuth** nel dashboard:

    - **Authorization URL**: `https://auth.foo.com/authorization`
    - **Client ID**: `ydybo4SD8PR73vzWWd6S0ObH`
    - **Scopes**: `['provider.users.docs']`
    - **Token URL**: `https://auth.foo.com/exchange`
    - **Info API URL**: `https://api.foo.com/docs/user-info`
    - **Logout URL**: `https://auth.foo.com/logout?returnTo=https%3A%2F%2Ffoo.com%2Fdocs`

    **Crea un endpoint per le informazioni utente** su `api.foo.com/docs/user-info`, che richieda un token di accesso OAuth con lo scope `provider.users.docs` e restituisca:

    ```json
    {
      "content": {
        "firstName": "Jane",
        "lastName": "Doe"
      },
      "groups": ["engineering", "admin"]
    }
    ```

    **Configura il server OAuth per consentire i reindirizzamenti** all’URL di callback.
  </Tab>

  <Tab title="JWT">
    ### Prerequisiti

    - Un sistema di autenticazione in grado di generare e firmare JWT.
    - Un servizio backend in grado di creare URL di reindirizzamento.

    ### Implementazione

    <Steps>
      <Step title="Genera una chiave privata.">
        1. Nel tuo dashboard, vai su [Authentication](https://dashboard.mintlify.com/settings/deployment/authentication).
        2. Seleziona **Full Authentication** o **Partial Authentication**.
        3. Seleziona **JWT**.
        4. Inserisci l'URL del tuo flusso di accesso esistente e seleziona **Save changes**.
        5. Seleziona **Generate new key**.
        6. Conserva la tua chiave in modo sicuro in un luogo accessibile dal backend.
      </Step>

      <Step title="Integra l'autenticazione Mintlify nel tuo flusso di accesso.">
        Modifica il flusso di accesso esistente per includere questi passaggi dopo l'autenticazione dell'utente:

        - Crea un JWT contenente le informazioni dell'utente autenticato nel formato `User`. Consulta [User data format](/it/authentication-personalization/personalization-setup#user-data-format) per maggiori informazioni.
        - Firma il JWT con la tua chiave segreta, utilizzando l'algoritmo EdDSA.
        - Crea un URL di reindirizzamento al percorso `/login/jwt-callback` della tua documentazione, includendo il JWT come hash.
      </Step>
    </Steps>

    ### Esempio

    La tua documentazione è ospitata su `docs.foo.com` e il sistema di autenticazione esistente su `foo.com`. Vuoi estendere il flusso di accesso per concedere l’accesso alla documentazione mantenendola separata dal dashboard (oppure non disponi di un dashboard).

    Crea un endpoint di accesso su `https://foo.com/docs-login` che estenda l’autenticazione esistente.

    Dopo aver verificato le credenziali dell’utente:

    - Genera un JWT con i dati utente nel formato di Mintlify.
    - Firma il JWT e reindirizza a `https://docs.foo.com/login/jwt-callback#{SIGNED_JWT}`.

    <CodeGroup>
      ```ts TypeScript
      import * as jose from 'jose';
      import { Request, Response } from 'express';

      const TWO_WEEKS_IN_MS = 1000 * 60 * 60 * 24 * 7 * 2;

      const signingKey = await jose.importPKCS8(process.env.MINTLIFY_PRIVATE_KEY, 'EdDSA');

      export async function handleRequest(req: Request, res: Response) {
        const user = {
          expiresAt: Math.floor((Date.now() + TWO_WEEKS_IN_MS) / 1000), // 2 week session expiration
          groups: res.locals.user.groups,
          content: {
            firstName: res.locals.user.firstName,
            lastName: res.locals.user.lastName,
          },
        };

        const jwt = await new jose.SignJWT(user)
          .setProtectedHeader({ alg: 'EdDSA' })
          .setExpirationTime('10 s') // 10 second JWT expiration
          .sign(signingKey);

        return res.redirect(`https://docs.foo.com/login/jwt-callback#${jwt}`);
      }
      ```

      ```python Python
      import jwt # pyjwt
      import os

      from datetime import datetime, timedelta
      from fastapi.responses import RedirectResponse

      private_key = os.getenv(MINTLIFY_JWT_PEM_SECRET_NAME, '')

      @router.get('/auth')
      async def return_mintlify_auth_status(current_user):
        jwt_token = jwt.encode(
          payload={
            'exp': int((datetime.now() + timedelta(seconds=10)).timestamp()),    # 10 second JWT expiration
            'expiresAt': int((datetime.now() + timedelta(weeks=2)).timestamp()), # 1 week session expiration
            'groups': ['admin'] if current_user.is_admin else [],
            'content': {
              'firstName': current_user.first_name,
              'lastName': current_user.last_name,
            },
          },
          key=private_key,
          algorithm='EdDSA'
        )

        return RedirectResponse(url=f'https://docs.foo.com/login/jwt-callback#{jwt_token}', status_code=302)
      ```
    </CodeGroup>

    ### Reindirizzamento degli utenti non autenticati

    Quando un utente non autenticato tenta di accedere a una pagina protetta, la destinazione prevista viene mantenuta nel reindirizzamento al tuo URL di accesso:

    1. L’utente tenta di visitare una pagina protetta: `https://docs.foo.com/quickstart`.
    2. Reindirizzamento al tuo URL di accesso con un parametro di query redirect: `https://foo.com/docs-login?redirect=%2Fquickstart`.
    3. Dopo l’autenticazione, reindirizza a `https://docs.foo.com/login/jwt-callback?redirect=%2Fquickstart#{SIGNED_JWT}`.
    4. L’utente viene reindirizzato alla destinazione originale.

<div id="making-pages-public">
  ## Rendere le pagine pubbliche
</div>

Quando si utilizza l’autenticazione parziale, tutte le pagine sono protette per impostazione predefinita. È possibile rendere specifiche pagine visibili senza autenticazione, a livello di pagina o di gruppo, tramite la proprietà `public`.

<div id="page-level">
  ### A livello di pagina
</div>

Per rendere pubblica una pagina, aggiungi `public: true` al frontmatter della pagina.

```mdx Public page example
---
title: "Public page"
public: true
---
```

<div id="group-level">
  ### Livello di gruppo
</div>

Per rendere pubbliche tutte le pagine di un gruppo, aggiungi `"public": true` sotto il nome del gruppo nell’oggetto `navigation` del file `docs.json`.

```json Public group example
{
  "navigation": {
    "groups": [
      {
        "group": "Public group",
        "public": true,
        "icon": "play",
        "pages": [
          "quickstart",
          "installation",
          "settings"
        ]
      },
      {
        "group": "Private group",
        "icon": "pause",
        "pages": [
          "private-information",
          "secret-settings"
        ]
      }
    ]
  }
}
```
