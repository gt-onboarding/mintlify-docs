<div id="installing-the-cli">
  ## Installazione della CLI
</div>

<Info>
  **Prerequisito**: installa [Node.js](https://nodejs.org/en) prima di procedere.
</Info>

<Steps>
  <Step title="Installa la CLI.">
    Esegui il seguente comando per installare la [CLI](https://www.npmjs.com/package/mint):

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Anteprima locale.">
    Spostati nella directory della documentazione (dove si trova il file `docs.json`) ed esegui il seguente comando:

    ```bash
    mint dev
    ```

    Un'anteprima locale della documentazione sarà disponibile all'indirizzo `http://localhost:3000`.
  </Step>
</Steps>

In alternativa, se non vuoi installare la CLI a livello globale, puoi eseguire uno script una tantum:

```bash
npx mint dev
```

<div id="updates">
  ## Aggiornamenti
</div>

Se l’anteprima locale non è sincronizzata con ciò che vedi online nella versione di produzione, aggiorna la tua CLI locale:

```bash
mint update
```

Se il comando `mint update` non è disponibile nella tua versione locale, reinstalla la CLI all’ultima versione:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Porte personalizzate
</div>

Per impostazione predefinita, la CLI utilizza la porta 3000. Puoi personalizzare la porta utilizzando l’opzione `--port`. Per eseguire la CLI sulla porta 3333, ad esempio, usa questo comando:

```bash
mint dev --port 3333
```

Se provi a eseguirla su una porta già in uso, verrà utilizzata la successiva disponibile:

```mdx
La porta 3000 è già in uso. Provo la 3001.
```

<div id="previewing-as-a-specific-group">
  ## Anteprima come gruppo specifico
</div>

Se utilizzi l'autenticazione parziale per limitare l'accesso alla documentazione, puoi visualizzare l'anteprima come uno specifico gruppo di autenticazione usando il flag `--group [groupname]`.

Ad esempio, se hai un gruppo chiamato `admin`, puoi visualizzare l'anteprima come membro di quel gruppo con il comando:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Comandi aggiuntivi
</div>

Sebbene `mint dev` sia il comando più utilizzato, sono disponibili altri comandi per gestire la documentazione.

<div id="finding-broken-links">
  ### Individuare i link interrotti
</div>

La CLI può aiutare a convalidare i collegamenti di riferimento presenti nella documentazione. Per identificare eventuali link interrotti, usa il seguente comando:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### Verifica della spec OpenAPI
</div>

Puoi usare la CLI per controllare il file OpenAPI alla ricerca di errori con il seguente comando:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Puoi fornire un nome file (ad es. `./openapi.yaml`) oppure un URL (ad es. `https://petstore3.swagger.io/api/v3/openapi.json`).

<div id="renaming-files">
  ### Ridenominare i file
</div>

Puoi rinominare e aggiornare tutti i riferimenti ai file utilizzando il seguente comando:

```bash
mint rename <oldFilename> <newFilename>
```

<div id="migrating-mdx-endpoint-pages">
  ### Migrazione delle pagine degli endpoint MDX
</div>

Puoi migrare le pagine degli endpoint MDX in pagine generate automaticamente dalla tua specifica OpenAPI utilizzando il seguente comando:

```bash
mint migrate-mdx
```

Questo comando converte le singole pagine degli endpoint MDX in pagine generate automaticamente definite nel tuo `docs.json`, sposta il contenuto MDX nell’estensione `x-mint` nella tua specifica OpenAPI e aggiorna la navigazione. Vedi [Migrazione da MDX](/it/api-playground/migrating-from-mdx) per informazioni dettagliate.

<div id="formatting">
  ## Formattazione
</div>

Durante lo sviluppo in locale, consigliamo di utilizzare estensioni nel tuo IDE per riconoscere e formattare i file `MDX`.

Se usi Cursor, Windsurf o VS Code, consigliamo l'[estensione MDX per VS Code](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) per l'evidenziazione della sintassi e [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) per la formattazione del codice.

Se usi JetBrains, consigliamo il [plugin MDX per IntelliJ IDEA](https://plugins.jetbrains.com/plugin/14944-mdx) per l'evidenziazione della sintassi e di configurare [Prettier](https://prettier.io/docs/webstorm) per la formattazione del codice.

<div id="troubleshooting">
  ## Risoluzione dei problemi
</div>

<AccordionGroup>
  <Accordion title="Errore: impossibile caricare il modulo &#x22;sharp&#x22; utilizzando il runtime darwin-arm64">
    Questo potrebbe essere dovuto a una versione obsoleta di Node. Prova quanto segue:

    1. Rimuovi la versione attualmente installata della Mint CLI: `npm uninstall -g mint`
    2. Aggiorna Node.js.
    3. Reinstalla la Mint CLI: `npm install -g mint`
  </Accordion>

  <Accordion title="Problema: si verifica un errore sconosciuto">
    **Soluzione**: Vai alla root del tuo dispositivo ed elimina la cartella `~/.mintlify`. Poi esegui di nuovo `mint dev`.
  </Accordion>

  <Accordion title="Errore: permission denied">
    Questo è dovuto alla mancanza delle autorizzazioni necessarie per installare globalmente i pacchetti Node.

    **Soluzione**: Prova a eseguire `sudo npm i -g mint`. Ti verrà richiesta la password, ovvero quella che usi per sbloccare il computer.
  </Accordion>

  <Accordion title="L'anteprima locale non corrisponde alla documentazione online">
    Con ogni probabilità è dovuto a una versione obsoleta della CLI.

    **Soluzione:** Esegui `mint update` per ottenere le modifiche più recenti.
  </Accordion>

  <Accordion title="mintlify vs. pacchetto mint">
    Se riscontri problemi con il pacchetto CLI, esegui prima `npm ls -g`. Questo comando mostra quali pacchetti sono installati globalmente sul tuo computer.

    Se non usi npm o non lo vedi nell'elenco con -g, prova `which mint` per individuare l’installazione.

    Se hai installati sia un pacchetto chiamato `mint` sia uno chiamato `mintlify`, disinstalla `mintlify`.

    1. Disinstalla il pacchetto precedente:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Svuota la cache di npm:

    ```bash
      npm cache clean --force
    ```

    3. Reinstalla il nuovo pacchetto:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
