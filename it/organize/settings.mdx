Il file `docs.json` consente di trasformare una raccolta di file Markdown in un sito di documentazione navigabile e personalizzato. Questo file di configurazione obbligatorio controlla stile, navigazione, integrazioni e altro ancora. Consideralo come il blueprint della tua documentazione.

Le impostazioni in `docs.json` si applicano globalmente a tutte le pagine.

<div id="setting-up-your-docsjson">
  ## Configurazione del `docs.json`
</div>

Per iniziare, è sufficiente specificare `theme`, `name`, `colors.primary` e `navigation`. Gli altri campi sono opzionali e potrai aggiungerli man mano che aumentano le esigenze della tua documentazione.

Per il miglior experience di modifica, includi il riferimento allo schema all’inizio del file `docs.json`. In questo modo abiliti l’autocompletamento, la convalida e i tooltip informativi nella maggior parte degli editor di codice:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "La tua documentazione",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // La tua struttura di navigazione
  }
  // Il resto della configurazione
}
```

<ResponseField name="theme" required>

  Consulta [Temi](/it/customize/themes) per ulteriori informazioni.
</ResponseField>

<ResponseField name="name" type="string" required>
  Il nome del tuo progetto, della tua organizzazione o del tuo prodotto.
</ResponseField>

<ResponseField name="colors" type="object" required>
  I colori utilizzati nella tua documentazione. L’applicazione dei colori varia a seconda del tema. Se fornisci solo un colore principale, verrà utilizzato per tutti gli elementi cromatici.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      Il colore principale della tua documentazione. In genere usato per l’enfasi in modalità chiara, con alcune variazioni in base al tema.

      Deve essere un codice esadecimale che inizia con `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Il colore utilizzato per l’enfasi in modalità scura.

      Deve essere un codice esadecimale che inizia con `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Il colore utilizzato per pulsanti e stati hover sia in modalità chiara sia in modalità scura, con alcune variazioni in base al tema.

      Deve essere un codice esadecimale che inizia con `#`.

</ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      Configurazione del tema dei blocchi di codice. Valore predefinito: `"system"`.

      **Configurazione semplice:**

      * `"system"`: Allinea alla modalità corrente del sito (chiara o scura)
      * `"dark"`: Usa sempre la modalità scura

      **Configurazione tema personalizzato:**

      * Usa una stringa per specificare un singolo [tema Shiki](https://shiki.style/themes) per tutti i blocchi di codice
      * Usa un oggetto per specificare [temi Shiki](https://shiki.style/themes) distinti per le modalità chiara e scura

      <ResponseField name="theme" type="string">
        Un singolo nome di tema Shiki da usare sia in modalità chiara sia in modalità scura.

        ```json

            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

        Temi distinti per le modalità chiara e scura.

          <ResponseField name="dark" type="string" required>

          ```json
            }
          }
          ```

<div id="upgrading-from-mintjson">
  ## Aggiornamento da `mint.json`
</div>

Se il tuo progetto di documentazione utilizza il file obsoleto `mint.json`, segui questi passaggi per aggiornare a `docs.json`.

<Steps>
  <Step title="Installa o aggiorna la CLI">
    Se non hai installato la [CLI](/it/installation), installala ora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Se hai già installato la CLI, assicurati che sia aggiornata:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Crea il file docs.json">
    Nel repository della tua documentazione, esegui:

    ```bash
    mint upgrade
    ```

    Questo comando creerà un file `docs.json` a partire dal tuo `mint.json` esistente. Controlla il file generato per assicurarti che tutte le impostazioni siano corrette.
  </Step>

  <Step title="Elimina il file mint.json">
    Dopo aver verificato che `docs.json` sia configurato correttamente, puoi eliminare in sicurezza il vecchio file `mint.json`.
  </Step>
</Steps>
