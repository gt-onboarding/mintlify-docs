<Update label="8月16日" tags={["Improvements"]} rss={{ title: "每周更新", description: "性能优化、LLM 集成、网页编辑器优化，以及 API 操作台增强" }}>
  ## 性能与构建优化

  * 在未缓存的 NextJS 无服务器环境中，MDX 转译现已在部署时执行，而非每次页面加载时执行，从而缩短未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希可在 MDX 未变更时避免重复转译，将大型站点用户的更新流程耗时减少约 50%（超过 5 分钟的部署预计可缩短至约一半）
  * 通过在后端添加数据库索引并并行化查询，仪表盘中的预览部署加载更快
  * 通过移除每个页面 `rsc` 负载中的重复 `navigation` 数据来减小页面体积——在页面数量较多或导航结构复杂时，性能提升最为明显
  * 更积极的预取使页面更常实现瞬时加载

  ## API 操作台与 OpenAPI 增强

  * 将 OpenAPI 到 MCP 的转换迁移至后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  * 为 API 操作台新增 Ruby 支持
  * 新增能力：你现在可以[仅通过 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建任何新的 MDX 文件。
  * 在基于 OpenAPI 规范的文档导航中支持[`webhook pages`](/zh/api-playground/openapi-setup#webhooks)
  * 通过在导航至 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中移除锚点参数来优化 AI 模型上下文

  ## 网页编辑器改进

  * 创建/重命名文件时，现在点击其他位置即可保存更改，无需按 Enter 键
  * 修复分支导航问题：当将 URL 更改为某个特定分支时，会重定向到上次活动分支而非预期目标
  * 对包含 `/` 的分支标题进行正确的 URL 编码，防止导航异常
  * 修复了单体仓库仪表盘的编辑器中 `Ctrl+K` 链接快捷键会在前面附加文档仓库路径并造成链接失效的问题

  ## 分析与 LLM 集成

  * 支持自定义 `llms.txt` 与 `llms-full.txt`——将文件添加到文档仓库根目录，即可分别在 `/llms.txt` 与 `/llms-full.txt` 端点提供，便于定制 LLM
