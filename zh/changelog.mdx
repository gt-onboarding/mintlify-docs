<Update label="8 月 31 日 - 9 月 13 日" tags={["New releases", "Improvements"]} rss={{ title: "每周更新", description: "AI 404 建议、助手网页搜索接入外部来源、安全性增强与可靠性修复" }}>
  ## 重大发布

  * **重大增强**：在 404 页面提供 AI 推荐页面，[当用户访问死链 → AI 代理读取路径 → 推荐语义相似的页面](https://x.com/mintlify/status/1966625627773059495)
  * **重大发布**：助手的网页搜索现在可包含外部来源\
    *注意：请联系我们为你的网站启用此功能。*

  ## 助手和 MCP

  * 修复了由于时间窗口未正确滑动导致助手被错误限流的问题
  * 修复了助手工具调用在处理空 `text` 块时的问题
  * 修复了 MCP 服务器名称与工具调用拼接后有时会超过 MCP 客户端强制的 60 字符限制的问题
  * 修复了助手菜单高度远大于视口并无限滚动的问题
  * 修复了助手花费数值在仪表盘中显示超过两位小数的问题

  ## 网页编辑器与部署

  * 增强了编辑器安全性，只有对连接的 git 托管仓库拥有 `write permissions` 的用户才能进行更改
  * 修复了名称中包含 `=` 的分支无法进行预览部署的问题
  * 修复了在创建预览部署时，过长的分支名称会在模态框中溢出的问题
  * 体验优化：在注册邀请中，email 查询参数将预填输入框
  * 修复了在 Safari 上通过上下文菜单复制页面不起作用的问题

  ## API 操作台与导航

  * 多个 API 操作台响应码在聚焦时现在显示于受控样式的选择菜单中，而非系统默认选择菜单
  * 你现在可以在 `docs.json` 的导航分组上使用 [`expanded` 字段，使其默认展开](https://mintlify.com/docs/navigation#default-expanded-state)

  ## SEO 与 UI

  * 通过将 favicon 与各文档站点本身使用相同的 URL 提供，修复了其在搜索引擎中不显示的问题
  * 修复了 YouTube 嵌入在加载时闪烁的问题
  * 修复了将反馈菜单展开以包含书面回复时导致目录发生布局位移的问题
  * 修复了在 Maple 主题中，关闭通知横幅后文字会溢出到顶栏上方的问题
  * 为 Maple 和 Willow 主题在侧边栏新增登录/登出按钮，提升易用性

  ## 分析与导出

  * 修复了助手分析视图与导出的可靠性问题
  * 助手分析导出现改为在后台执行，并通过邮件发送，以提供更可靠的体验
</Update>

<Update label="August 24 - August 30" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "反馈收集增强、导航优化及认证相关错误修复" }}>
  ## 重要发布：增强的反馈收集

  * **重大改进**：读者在选择“点赞/点踩”后，现在可以提供更细化的反馈，包括选项和文字评论。你还可以对代码块收集反馈，并在仪表盘的分析中查看所有反馈。\
    *注意：联系我方以为你的网站启用此功能。*

  ## 导航与易用性改进

  * 易用性改进：忽略尾部斜杠和双斜杠，你无需在 docs.json 中对其严格匹配
  * 你现在可以在 `h1-6` HTML 标签上添加 `noAnchor` 属性，以避免生成锚点链接
  * Palm 主题现在在左下角提供浮动语言选择器，类似于 Stripe 的做法
  * 在 docs.json 中新增字段 [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups)，用于控制当导航分组展开时，是否自动导航到该分组的第一页
  * 易用性改进：使嵌套有序列表在十进制/罗马数字与字母样式之间交替

  ## 错误修复与可靠性

  * 修复了当页面存在 JS 组件时，滚动位置锚点链接无法正常工作的问题
  * 修复了 Google 会索引原始 `*.md` 文件的问题，原因是缺少 `x-robots-tag noindex` 头
  * 修复了受保护文档的 OAuth 问题：成功完成流程后不会重定向回起始页面
  * 修复了受认证保护文档的预览中无法看到完整导航栏的问题
  * 修复了使用新图像 CDN 处理 SVG 的相关问题

  ## 组件与样式增强

  * 为 `SidebarNavGroupDivider` 新增 CSS 选择器，以便自定义样式
  * 为在其上定义了安全性的、由 MDX 定义的 API 页面新增回归测试，以提升稳定性

  ## 性能改进

  * 通过将 KaTeX CSS 从 cdnjs 迁移到我们在 CloudFront 上的自有 CDN，降低延迟并提升性能
</Update>

<Update label="8月10日 - 8月16日" tags={["Improvements"]} rss={{ title: "每周更新", description: "性能优化、LLM 集成、网页编辑器打磨，以及 API 操作台增强" }}>
  ## 性能与构建优化

  * 在未缓存的 NextJS 无服务器环境中，MDX 转译现改为在部署时进行，而非在每次页面加载时进行，从而缩短未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希在 MDX 未变化时可避免重复转译，将大量页面客户的更新流程耗时减少约 50%（超过 5 分钟的部署应能大致减半）
  * 在仪表盘中查看预览部署更快了：我们在后端新增数据库索引并并行化查询
  * 通过移除每个页面 `rsc` 负载中的重复 `navigation` 数据来减小页面体积——在页面数量多或导航结构复杂时性能提升最明显
  * 更激进的预取使即时加载更常见

  ## API 操作台与 OpenAPI 增强

  * 将 OpenAPI 到 MCP 的转换移至后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  * 为 API 操作台新增 Ruby 支持
  * 我们新增了一个功能，现在你可以[仅通过 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建任何新的 MDX 文件。
  * 在文档导航中支持来自 OpenAPI 规范的[`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)
  * 在导航至 Anthropic、OpenAI 等提供方时，从 markdown 链接中移除锚点规范以优化 AI 模型上下文

  ## 网页编辑器改进

  * 创建/重命名文件时，现在点击其他区域即可保存更改，无需按 Enter 键
  * 修复了分支导航：将 URL 改为特定分支时会被重定向到上次活动分支而非目标分支的问题
  * 对包含 `/` 的分支标题进行正确的 URL 编码，防止导航异常
  * 修复了 monorepo 仪表盘编辑器中的 `Ctrl+K` 链接快捷键会在前面添加 docs 仓库路径并导致链接失效的问题

  ## 分析与 LLM 集成

  * 支持自定义 `llms.txt` 和 `llms-full.txt`——将其添加到文档仓库根目录，即可通过 `/llms.txt` 和 `/llms-full.txt` 端点提供，供 LLM 定制使用
  * 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)
  * 增强上下文菜单的分析跟踪（仪表盘视图即将上线）
  * 为 `llms.txt` 和 `llms-full.txt` 添加端到端测试，确保正确提供

  ## 组件与样式增强

  * `h{1-4}` 标签支持自定义 className，便于应用自定义标题样式
  * 修复了在自定义页面模式下，`h{1-4}` 标签会渲染为带徽章的 `Heading` 组件的问题
  * 为[面包屑](/zh/navigation#breadcrumbs)添加 CSS 选择器，便于定向自定义
  * 通过分析尺寸，在 56px 高度下保持比例，修复被拉伸的 open-graph 图片
  * 在启用时，将上下文菜单中的 `VSCode` 更正为 `VS Code`
  * 修复了自定义组件内的标题与语义化标题同时出现在目录中的问题

  ## Bug 修复与可靠性

  * 通过清理导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题
  * 解决了在遇到空的 OpenAPI JSON 文件时出现的 CLI 错误 `Cannot convert undefined or null to object`
  * 修复了自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题
  * 通过对 RSS 链接使用 origin + pathname，修复了落到锚点链接时 RSS 订阅按钮点击的问题
  * 通过移除 sourcemap 改进了 CLI 下载速度

  ## 技术改进

  * 在 CI 流水线中添加可视化测试，以更早发现回归
  * 增强错误处理与调试能力
  * 为新功能与边界情况提供了更全面的测试覆盖
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## 认证改进

  * 组级公开访问：通过 `docs.json` 将整个页面组设为公开，这样无需在每个页面上单独设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，用于删除上游 cookies 并完成登出
  * 当发生 OAuth 错误时，用户会被重定向到你指定的 `logoutURL` 以重新开始认证流程
  * 修复在回调前 OAuth/JWT 流程中短暂闪现 500 错误的问题
  * 在 OAuth/JWT 认证配置中自动去除 URL 中的 `https://` 以防误配置

  ## API 操作台增强

  * 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可在你的文档之上构建代理和 MCP 服务器
  * 现已按配置的路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 现在可以在 OpenAPI 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 来覆盖生成的字段、自定义前言内容，或更改代码示例中的端点 URL
  * 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由以 MCP 工具形式暴露

  ## 助手更新

  * 修复旧消息的操作菜单（包含复制、点赞等选项）在新消息流入时消失的问题
  * 修复上周 [托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server) 后嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  * 文档仓库中的所有图片和视频资源现在都会在你的域名下以正确的路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，则可在 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  * Mintlify 仪表盘登录页的邮箱输入框现已自动聚焦，便于立即输入（体验优化）
  * 在 Redis 中同时处理自定义域名和子域名，以提升导航加载性能（约 50ms 延迟降低）
  * 为 PDF 导出添加重试逻辑以提升稳定性
  * 修复在同意或关闭后 cookie 同意弹窗仍会再次出现的问题——现已尊重首次选择
  * 通过在 `navigator.write` 中指定 MIME `type`，修复在 Safari 上将页面复制到剪贴板的问题

  ## 技术改进

  * 修复 Windows 与 pnpm 的 CLI 问题，并加入 CI 测试以防回归
  * 改进错误日志输出——为工程团队调试带来更佳体验
  * 当缺少 `contentDirectory` 文件时，修复 broken-link CI action 的若干小问题
  * 修复上周认证保护的预览修复导致的回归问题：导航 UI 中未能正确设置活动选项卡
  * 修复主题浅色背景未应用于活动选项卡图标的问题
  * 修复在仪表盘中更改认证类型会先更新但随后回退至先前保存类型的问题——现在保存后新选择会持久保留
  * 面向使用自定义 UI 库的企业客户的内部 DX 改进——我们现在能更快集成你的组件并在更短周期内满足需求
