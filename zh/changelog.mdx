<Update label="8月24日 - 8月30日" tags={["Improvements"]} rss={{ title: "每周更新", description: "增强的反馈收集、导航改进，以及身份验证错误修复" }}>
  ## 重大版本：增强的反馈收集

  - **重大改进**：读者现在在选&#x62E9;_&#x70B9;赞/点&#x8E29;_&#x540E;可以提供更详细的反馈，包括选项和书面评论。你还可以对代码块收集反馈，并在仪表盘分析中查看所有响应。\
    _注意：请联系我们为你的网站启用此功能。_

  ## 导航与易用性改进

  - 易用性改进：忽略结尾斜杠和双斜杠，这样你不必在 docs.json 中严格保证它们完全正确
  - 你现在可以在 `h1-6` HTML 标签中添加 `noAnchor` 属性，以避免它们带有锚点链接
  - Palm 主题现在在左下角提供浮动语言选择器，类似于 Stripe 的做法
  - 在 docs.json 中新增字段 [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups)，用于控制在展开导航分组时是否将用户导航到该分组中的第一页
  - 易用性改进：使嵌套有序列表的样式在十进制/罗马数字与字母之间交替

  ## 错误修复与可靠性

  - 修复了当页面存在 JS 组件时滚动位置锚点链接无法正常工作的错误
  - 修复了 Google 索引原始 `*.md` 文件的问题，原因是缺少 `x-robots-tag noindex` 头
  - 修复了受保护文档的 OAuth 问题：在成功完成流程后不会重定向回起始页面
  - 修复了受保护文档预览中无法看到完整导航栏的问题
  - 修复了通过我们的新图像 CDN 处理 SVG 的相关问题

  ## 组件与样式增强

  - 为 `SidebarNavGroupDivider` 的自定义样式新增了一个 CSS 选择器
  - 为在其上定义了安全性的 MDX 定义 API 页面新增了回归测试，以确保更高的稳定性

  ## 性能改进

  - 将 KaTeX CSS 从 cdnjs 迁移到我们在 CloudFront 上的自有 CDN 以降低延迟
</Update>

<Update label="8月17日 - 8月23日" tags={["Improvements"]} rss={{ title: "每周更新", description: "图像布局修复、身份验证增强、移动端导航改进，以及 Web 编辑器稳定性提升" }}>
  ## 图像处理改进

  - **重大改进**：即使未指定宽度和高度属性，图像默认也不会引发布局偏移——自动尺寸可在页面加载期间防止内容跳动
  - 仓库中的所有静态文件（PDF、TXT、XML 等）现在在部署时会自动上传并提供服务，实现完整的资源覆盖

  ## Web 编辑器与部署增强

  - 修复了 Web 编辑器中的分支创建流程，现可正确导航至新建分支并保持停留
  - 增强了合并冲突对话框，支持正确的退出操作，无需再刷新页面即可关闭冲突
  - 通过仅对部分更新中变更的页面进行缓存失效，优化了更新流程性能，缩短部署时间

  ## 身份验证与导航改进

  - 新增对自定义子目录的身份验证支持；如果你在 `https://yourdomain.com/docs` 提供文档，身份验证现在可无缝工作
  - 修复了在仅配置一个链接时侧边栏显示不正确的错误
  - 全面改造移动端导航：按钮居中并具有适当的外边距/内边距、改进下拉菜单的间距、移除空章节不必要的分隔符和边距，并修复了 Maple 主题的间距/内边距问题

  ## 组件与样式修复

  - 解决了将 `<h1-6>` 标签错误转换为 Heading 组件并破坏自定义样式的问题
  - 在仪表盘中新增一键助手配置开关，便于管理

  ## 技术改进与可靠性

  - 增强了更新工作流的日志系统，以便更快进行调试和问题定位
  - 为拥有 10+ OpenAPI/AsyncAPI 规范的客户修复了 GitHub 速率限制问题，方式是从单文件抓取切换为仓库克隆
  - 提升了助手的可靠性：增加备用 LLM 支持、强化速率限制错误处理，并使搜索工具更稳健
</Update>

<Update label="8 月 10 日 - 8 月 16 日" tags={["Improvements"]} rss={{ title: "每周更新", description: "性能优化、LLM 集成、Web 编辑器打磨，以及 API 游乐场增强" }}>
  ## 性能与构建优化

  - 在未缓存的 NextJS 无服务器环境中，MDX 转译现于部署时执行，而非每次页面加载时执行，从而缩短未缓存页面的首字节时间（TTFB）。
  - 基于内容的哈希在 MDX 未变化时避免重新转译，将大规模站点的更新流程时间减少约 50%（超过 5 分钟的部署应大致减半）
  - 在仪表盘中预览部署的加载速度更快：我们在后端添加了数据库索引并并行化查询
  - 通过移除每个页面 `rsc` 负载中重复的 `navigation` 数据来减小页面体积——在页面数量多或导航结构复杂时，性能收益更显著
  - 更积极的预取策略让瞬时页面加载出现得更频繁

  ## API 游乐场与 OpenAPI 增强

  - 将 OpenAPI 到 MCP 的转换移至后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  - 在 API 游乐场中新增对 Ruby 的支持
  - 我们新增了一个功能，现在你可以[仅使用 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建任何新的 MDX 文件。
  - 在来自 OpenAPI 规范的文档导航中支持 [`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)
  - 通过在导航至 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中移除锚点链接规范，来优化 AI 模型上下文

  ## Web 编辑器改进

  - 文件创建/重命名在点击空白处时即可保存，无需再按 Enter 键
  - 修复了分支导航问题：将 URL 改为特定分支时会重定向到上次活动分支而非预期目标
  - 对包含 `/` 的分支标题正确进行 URL 编码，防止导航异常
  - 修复了 monorepo 仪表盘 Editor 中 `Ctrl+K` 链接快捷键会在前面拼接 docs 仓库路径并导致链接失效的问题

  ## 分析与 LLM 集成

  - 支持自定义 `llms.txt` 和 `llms-full.txt`——将其添加到 docs 仓库根目录，即可在 `/llms.txt` 和 `/llms-full.txt` 端点提供，用于自定义 LLM
  - 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)
  - 增强了上下文菜单的分析追踪（仪表盘可视化即将推出）
  - 为 `llms.txt` 和 `llms-full.txt` 添加了端到端测试，确保正确提供

  ## 组件与样式增强

  - `h{1-4}` 标签支持自定义 className，可应用自定义标题样式
  - 修复了在自定义页面模式下 `h{1-4}` 标签渲染为带徽标的 `Heading` 组件的问题
  - 为[面包屑](/zh/navigation#breadcrumbs)添加了 CSS 选择器，便于进行自定义 CSS 定位
  - 通过分析尺寸，在 56px 高度下保持比例，修复了被拉伸的 open-graph 图像
  - 在启用时，将上下文菜单中的 `VSCode` 更正为 `VS Code`
  - 修复了自定义组件内的标题与语义化标题同时出现在目录中的问题

  ## Bug 修复与可靠性

  - 通过清理导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题
  - 解决了在遇到空的 OpenAPI JSON 文件时出现的 CLI 错误 `Cannot convert undefined or null to object`
  - 修复了自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题
  - 通过对 RSS 链接使用 origin + pathname，修复了在落到锚点链接时点击 RSS 订阅按钮无效的问题
  - 通过移除 sourcemaps 提升了 CLI 下载速度

  ## 技术改进

  - 在 CI 流水线中添加了可视化测试，以更早发现回归
  - 增强了错误处理与调试能力
  - 为新功能和边界情况提供了更全面的测试覆盖
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## 身份验证改进

  - 组级公开访问：通过 `docs.json` 将整组页面设为公开，无需在每个页面单独设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  - 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，用于删除上游 cookies 并完成退出
  - 发生 OAuth 错误时，用户会被重定向到你指定的 `logoutURL` 以重新开始认证流程
  - 修复了在回调前 OAuth/JWT 流程中短暂闪现 500 错误的问题
  - 在 OAuth/JWT 身份验证配置中自动移除 URL 中的 `https://`，以防止配置错误

  ## API 游乐场增强

  - 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，便于在文档之上构建代理和 MCP 服务器
  - 现在会在其指定路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  - 现在你可以在 OpenAPI 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 来覆盖生成字段、自定义前言内容，或在代码示例中更改端点 URL
  - 在 OpenAPI 配置中，[`x-mcp` 现更名为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由作为 MCP 工具对外暴露

  ## 助手更新

  - 修复了旧消息的操作菜单（包含复制和点赞等选项）在新消息流入时消失的问题
  - 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后，嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  - 文档仓库中的所有图片和视频资源现在都会在你的域名上以相应路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，就可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  - Mintlify 仪表盘登录页的邮箱字段现在自动聚焦，便于立即输入（体验优化）
  - 在 Redis 中同时处理了自定义域名和子域名，以提升导航加载性能（约 50ms 延迟降低）
  - 为 PDF 导出添加了重试逻辑以提高可靠性
  - 修复了在同意或关闭后 cookie 同意弹窗仍会再次出现的问题——第一次选择现已生效
  - 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上将页面复制到剪贴板的问题

  ## 技术改进

  - 修复了 Windows 与 pnpm 环境下的 CLI 问题，并添加 CI 测试防止回归
  - 改进了错误日志输出——为工程团队调试带来体验优化
  - 修复了在缺少 `contentDirectory` 文件时，断链检查 CI 动作的若干小问题
  - 修复了上周受身份验证保护预览修复引发的回归：导航 UI 中的活动选项卡未正确设置
  - 修复了主题浅色背景未应用于活动选项卡图标的问题
  - 修复了在仪表盘中更改身份验证类型会先更新后又回退到先前保存类型的问题——现在新选择在保存后会持久化
  - 面向使用自定义 UI 库的企业客户的内部 DX 改进——我们现在更易于在更短周期内纳入你的组件并响应需求

    ![docs.json 截图](/images/changelog/docsjson.png)
  </Frame>

  我们推出了新的 `docs.json` 模式以取代 `mint.json`，以支持更完善的多级版本管理、更易理解的可视化，以及更一致的术语。了解详情，请[查看我们的博客](https://mintlify.com/blog/refactoring-mint-json-into-docs-json)。

  按以下步骤从 `mint.json` 升级到 `docs.json`：

  1. 确保你的 CLI 为最新版本

  ```bash
  npm i mint@latest -g
  ```

  1. 在你的文档仓库中运行

  ```bash
  mint upgrade
  ```

  1. 删除旧的 `mint.json` 文件并推送你的更改
