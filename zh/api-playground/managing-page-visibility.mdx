---
title: "管理页面可见性"
description: "控制 OpenAPI 规范中的哪些端点在文档导航中显示"
---

你可以控制哪些 OpenAPI 操作会发布为文档页面，以及它们在导航中的展示方式。这对仅供内部使用的端点、已弃用的操作、测试版功能，或需要通过直接 URL 访问但不应通过站点导航被发现的端点非常有用。

如果你的页面是由 OpenAPI 文档自动生成的，你可以使用 `x-hidden` 和 `x-excluded` 扩展来管理页面可见性。

<div id="x-hidden">
  ## `x-hidden`
</div>

`x-hidden` 扩展会为某个 endpoint 创建一个页面，但会将其从导航中隐藏。该页面只能通过直接访问其 URL 打开。

`x-hidden` 的常见用例包括：

- 需要编写文档但不希望在导航中曝光的 endpoints。
- 将从其他内容中跳转链接到的页面。
- 面向特定用户的 endpoints。

<div id="x-excluded">
  ## `x-excluded`
</div>

`x-excluded` 扩展会将某个端点从文档中完全排除。

`x-excluded` 的常见用例包括：

- 仅供内部使用的端点。
- 不希望在文档中展示的已弃用端点。
- 尚未准备好对外公开文档的 Beta 功能。

<div id="implementation">
  ## 实现
</div>

在 OpenAPI 规范中，于相应的 HTTP 方法下添加 `x-hidden` 或 `x-excluded` 扩展。

下面展示了在 OpenAPI 架构文档中，如何在某个端点（endpoint）和 webhook 路径上使用每个属性的示例。

```json {11, 19}
"paths": {
  "/plants": {
    "get": {
      "description": "返回商店中的所有植物",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/hidden_plants": {
    "get": {
      "x-hidden": true,
      "description": "返回商店中的所有半保密植物",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/secret_plants": {
    "get": {
      "x-excluded": true,
      "description": "返回商店中的所有绝密植物(请勿发布此端点!)",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  }
},
```

```json {9, 15}
"webhooks": {
  "/plants_hook": {
    "post": {
      "description": "新植物添加到商店时的 Webhook 通知",
    }
  },
  "/hidden_plants_hook": {
    "post": {
      "x-hidden": true,
      "description": "新植物添加到商店时的部分保密信息 Webhook 通知"
    }
  },
  "/secret_plants_hook": {
    "post": {
      "x-excluded": true,
      "description": "新植物添加到商店时的绝密信息 Webhook 通知(请勿发布此端点!)"
    }
  }
}
```
