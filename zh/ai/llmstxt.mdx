[llms.txt 文件](https://llmstxt.org) 是一项行业标准，可帮助 LLM 更高效地索引内容，类似于站点地图对搜索引擎的作用。AI 工具可以使用该文件理解你的文档结构，并定位与用户查询相关的内容。

Mintlify 会在你的项目根目录自动提供一个 `llms.txt` 文件，列出文档中的所有可用页面。该文件始终保持最新且无需维护。你也可以选择在项目根目录添加自定义的 `llms.txt` 文件。

在你的文档站点 URL 后追加 `/llms.txt` 即可查看 `llms.txt`。

<PreviewButton href="https://mintlify.com/docs/llms.txt">打开此站点的 llms.txt</PreviewButton>


<div id="llmstxt-structure">
  ## llms.txt 结构
</div>

`llms.txt` 文件是一个纯 Markdown 文件，包含：

* 以 H1 标题形式呈现的**站点标题**。
* **结构化内容部分**，包含文档中每个页面的链接与描述。

页面按从根目录开始、在代码库中的出现顺序进行字母序排列。

每个页面的描述来自其 frontmatter 中的 `description` 字段。没有 `description` 字段的页面将在 `llms.txt` 文件中显示为无描述。

```mdx Example llms.txt
# 站点标题

## 文档

- [API](https://example.com/docs/api): 端点列表和使用说明
- [Install](https://example.com/docs/install): 安装步骤
- [Getting started](https://example.com/docs/start): 快速入门
```

这种结构化方法使 LLM 能以全局视角高效处理你的文档，并为用户查询定位相关内容，从而提升 AI 辅助文档检索的准确性和速度。


<div id="llms-fulltxt">
  ## llms-full.txt
</div>

`llms-full.txt` 文件会将整个文档站点整合为一个文件，供 AI 工具作为上下文使用，并由 LLM 流量进行索引。

Mintlify 会在你的项目根目录自动托管一个 `llms-full.txt` 文件。你可以在文档站点的 URL 末尾追加 `/llms-full.txt` 来查看它。

<PreviewButton href="https://mintlify.com/docs/llms-full.txt">打开此站点的 llms-full.txt。</PreviewButton>

<div id="custom-files">
  ## 自定义文件
</div>

要添加自定义的 `llms.txt` 或 `llms-full.txt` 文件，请在项目根目录创建一个同名文件。添加自定义文件会覆盖同名的自动生成文件。若删除该自定义文件，将恢复使用默认文件。

你的自定义 `llms.txt` 或 `llms-full.txt` 文件必须包含以 H1 级标题标注的站点标题。其他内容为可选。有关可选部分与最佳实践的更多信息，请参见 `llms.txt` 规范中的[格式](https://llmstxt.org/#format)。
