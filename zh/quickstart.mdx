本快速入门指南将演示如何在几分钟内设置并部署你的文档站点。

完成本指南后，你将拥有一个可在线访问的文档站点，随时进行自定义和扩展。

<Info>
  **前提条件**：开始之前，请先[创建账户](https://mintlify.com/start)并完成初始设置。
</Info>

<div id="getting-started">
  ## 入门
</div>

完成入门流程后，你的文档站点会自动部署到一个唯一的 URL，格式如下：

```
https://<your-project-name>.mintlify.app
```

在你的[仪表板](https://dashboard.mintlify.com/)的 Overview 页面找到该 URL。

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify 域名" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify 域名" className="hidden dark:block" />
</Frame>

你的站点 URL 会立即生效。在你设置文档站点期间，使用该 URL 进行测试并与团队共享。

<div id="install-the-github-app">
  ### 安装 GitHub 应用
</div>

Mintlify 提供了一个 GitHub 应用，可在你将更改推送到仓库时自动部署。

按照入门清单或控制台中的指引安装 GitHub 应用。

1. 在 Mintlify 控制台中进入 **Settings**。
2. 在侧边栏选择 **GitHub App**。
3. 选择 **Install GitHub App**。这将在新标签页中打开 GitHub 应用的安装页面。
4. 选择要安装该应用的组织或个人账户。
5. 选择你要连接的仓库。

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub App Installation" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub App Installation" className="hidden dark:block" />
</Frame>

<Info>
  如果你将文档迁移到其他仓库，请更新 GitHub 应用的权限。
</Info>

<div id="authorize-your-github-account">
  ### 授权你的 GitHub 账户
</div>

1. 在 Mintlify 仪表盘中前往 **Settings**。
2. 在侧边栏选择 **My Profile**。
3. 点击 **Authorize GitHub account**。这会在新标签页中打开 GitHub 授权页面。

<Info>
  根据你所在组织的设置，你的 GitHub 组织管理员可能需要为你的账户授权。
</Info>

<div id="code-based-workflow">
  ## 基于代码的工作流
</div>

基于代码的工作流可与您现有的开发环境和 Git 仓库集成。此工作流最适合希望将文档与代码并行管理的技术团队。

<div id="install-the-cli">
  ### 安装 CLI
</div>

<Info>
  **前提条件**：CLI 需要 [Node.js](https://nodejs.org/en) v19 或更高版本。
</Info>

要在本地编辑与管理文档，请在终端运行以下命令安装名为 [mint](https://www.npmjs.com/package/mint) 的命令行工具（CLI）：

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<div id="edit-the-documentation">
  ### 编辑文档
</div>

完成环境配置后，你就可以开始编辑文档文件了。例如，更新简介页面的标题：

1. 打开你在引导流程中创建的代码仓库。
2. 打开 `index.mdx` 并找到文件顶部：

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. 将 `title` 字段更新为 `"Hello World"`。

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

<div id="preview-the-changes">
  ### 预览更改
</div>

要在本地预览更改，请运行以下命令：

```bash
mint dev
```

预览可在 `localhost:3000` 访问。

  <img src="/images/quickstart/mintlify-dev-dark.png" alt="Mintlify Dev" className="hidden dark:block" />

<div id="push-the-changes">
  ### 推送更改
</div>

当你准备发布更改时，将它们推送到你的代码仓库。

Mintlify 会自动检测更改、构建文档，并将更新部署到你的网站。你可以在 GitHub 仓库的提交历史或[仪表盘](https://dashboard.mintlify.com)中查看部署状态。

部署完成后，你的最新更新将可通过 `<your-project-name>.mintlify.app` 访问。

<Card title="跳转到添加自定义域名" icon="arrow-down" href="#adding-a-custom-domain" horizontal>
  你也可以跳过网页编辑器流程，直接前往添加自定义域名。
</Card>

<div id="web-editor-workflow">
  ## 网页编辑器工作流
</div>

网页编辑器工作流提供一个所见即所得（WYSIWYG）界面，用于创建和编辑文档。该工作流最适合希望直接在浏览器中工作、且不需要额外本地开发工具的用户。

<div id="access-the-web-editor">
  ### 访问网页编辑器
</div>

1. 登录你的[控制台](https://dashboard.mintlify.com)。
2. 在左侧边栏选择 **Editor**。

<Info>
  如果你尚未安装 GitHub 应用，打开网页编辑器时会提示你进行安装。
</Info>

  <img alt="Mintlify 网页编辑器的可视化编辑模式" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

  <img alt="Mintlify 网页编辑器的可视化编辑模式" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

<div id="edit-the-documentation">
  ### 编辑文档
</div>

在网页编辑器中，你可以在侧边栏浏览文档文件。我们来更新一下介绍页：

在文件浏览器中找到并选择 `index.mdx`。

然后在编辑器中，将标题字段更新为“Hello World”。

<Frame>
  <img alt="在网页编辑器中编辑" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="在网页编辑器中编辑" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />

<Tip>
  编辑器提供了丰富的格式化工具和组件。在编辑器中输入 <kbd>/</kbd> 可打开命令菜单以使用这些工具。
</Tip>

<div id="publish-your-changes">
  ### 发布更改
</div>

当您对编辑结果满意时，点击右上角的 **Publish** 按钮。您的更改会立即部署到文档站点。

<Tip>
  使用分支并通过拉取请求在部署到正式站点之前预览和审阅更改。
</Tip>

有关使用网页编辑器的更多信息（包括如何通过分支和拉取请求进行协作与预览更改），请参阅我们的[网页编辑器文档](/zh/editor)。

<div id="adding-a-custom-domain">
  ## 添加自定义域名
</div>

虽然你的 `<your-project-name>.mintlify.app` 子域名适合测试和开发，但大多数团队在生产环境的文档中更倾向于使用自定义域名。

要添加自定义域名，请在控制台前往 [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) 页面。

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Custom Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Custom Domain" className="hidden dark:block" />
</Frame>

输入你的域名（例如 `docs.yourcompany.com`），并按照页面提供的指引在你的域名服务商处配置 DNS 设置。

<Table>
  | 记录类型 | 名称 | 值 | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs（或子域名） | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS 变更最多可能需要 48 小时才能完全生效，但通常会更快完成。
</Info>

<div id="troubleshooting">
  ## 故障排除
</div>

如果在设置过程中遇到问题，请查看以下常见解决方案：

<AccordionGroup>
  <Accordion title="本地预览无法运行">
    请确保已安装 Node.js v19+，并在包含 `docs.json` 文件的目录中运行 `mint dev` 命令。
  </Accordion>

  <Accordion title="更改未在线上站点生效">
    部署可能需要几分钟时间。请检查你的 GitHub Actions（用于代码化工作流）或在 Mintlify 控制台中的部署日志，确认没有构建错误。
  </Accordion>

  <Accordion title="自定义域名无法连接">
    请核对 DNS 记录配置是否正确，并预留足够的 DNS 传播时间（最长可达 48 小时）。你可以使用 [DNSChecker](https://dnschecker.org) 等工具验证 CNAME 记录。
  </Accordion>
</AccordionGroup>
