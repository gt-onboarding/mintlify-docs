---
title: "自定义 404 页面"
description: "自定义 404 错误页面的标题和描述"
icon: "file-warning"
---

当用户访问失效或不存在的链接时出现的 404 错误页面，其标题和描述可由你控制。

在自定义 404 页面时，可在描述中引导用户前往文档中的相关资源或链接，帮助他们找到所需内容。

<div id="configuration">
  ## 配置
</div>

在你的 `docs.json` 文件的 `errors.404` 部分配置 404 页面：

```json
"errors": {
  "404": {
    "redirect": false,
    "title": "I can't be found",
    "description": "What ever **happened** to this _page_?"
  }
}
```

<div id="parameters">
  ## 参数
</div>

<ResponseField name="redirect" type="boolean">
  当页面不存在时，是否自动重定向到首页。
  
  设为 `true` 将重定向到首页。

  设为 `false` 将显示自定义 404 页面。
</ResponseField>

<ResponseField name="title" type="string">
  404 错误页面的自定义标题。将替换默认的 “Page not found” 标题。
</ResponseField>

<ResponseField name="description" type="string">
  404 错误页面的自定义描述。支持 Markdown 格式。
</ResponseField>
