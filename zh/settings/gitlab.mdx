我们结合使用 Access token 和 Webhook 来完成 GitLab 与 Mintlify 之间的身份验证与变更同步。

- 我们使用 Access token 从 GitLab 拉取信息。
- 我们使用 Webhook，使 GitLab 在发生更改时通知 Mintlify。
  - 这样在创建 MR 时，Mintlify 就能创建预览部署。

<div id="set-up-the-connection">
  ## 设置连接
</div>

<Steps>
  <Step title="查找你的项目 ID">
    在 GitLab 项目中，前往 `Settings` > `General`，找到 `Project ID`。
    <Frame>
      <img src="/images/gitlab/gitlab-project-id.png" />
    </Frame>
  </Step>
  <Step title="生成访问令牌">
    a. 前往 `Settings` > `Access Tokens`。

    b. 选择 `Add new token`。
      1. 将令牌命名为“Mintlify”。
      2. 如果是私有仓库，需将角色设置为 `Maintainer`。
      3. 将作用域勾选为 `api` 和 `read_api`。

    c. 最后点击 `Create project access token` 并复制该令牌。

    <Frame>
      <img src="/images/gitlab/gitlab-project-access-token.png" />
    </Frame>

  </Step>
  <Step title="设置连接">
    在 [Mintlify dashboard](https://dashboard.mintlify.com/settings/deployment/git-settings) 中，添加上一步获取的项目 ID 和访问令牌，并完成其他相关配置。完成后点击“Save Changes”。
    <Frame>
      <img src="/images/gitlab/gitlab-config.png" />
    </Frame>
  </Step>
</Steps>

<div id="create-the-webhook">
  ## 创建 Webhook
</div>

Webhook 可让我们在发生更改时接收事件，从而自动触发部署。

<Steps>
  <Step title="前往 Settings > Webhooks 并点击“Add new Webhook”">
    <Frame>
      <img src="/images/gitlab/gitlab-webhook.png" />
    </Frame>
  </Step>
  <Step title="设置 URL 和 Webhook">
    在“URL”字段中输入端点 `https://leaves.mintlify.com/gitlab-webhook`，并将 Webhook 命名为“Mintlify”。

  </Step>
  <Step title="粘贴令牌">
    粘贴在完成连接设置后生成的 Webhook 令牌。
    <Frame>
      <img src="/images/gitlab/gitlab-show-webtoken.png" />
    </Frame>
  </Step>
  <Step title="选择事件">
    选择要触发 Webhook 的事件：
      - Push events（所有分支）
      - Merge requests events
      
    完成后应类似如下：
    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken.png" />
    </Frame>
  </Step>
  <Step title="测试 Webhook">
    创建 Webhook 后，点击“Test”下拉菜单并选择“Push events”，发送示例负载以确认配置正确。若配置无误，将显示“Hook executed successfully: HTTP 200”。

    这有助于验证一切正常运行，并确保您的文档可与 GitLab 仓库正确同步。
    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken-test.png" />
    </Frame>

  </Step>
</Steps>
