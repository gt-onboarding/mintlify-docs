当你在 docs 文件夹中更改某个文件的路径时，该页面的 URL 路径也会随之改变。这通常发生在重构文档或更改侧边栏标题时。

<div id="broken-links">
  ## 失效链接
</div>

使用我们的命令行工具捕获失效链接。[安装命令行工具](/zh/installation)后运行以下命令：

```bash
mint broken-links
```

命令行工具会识别文档中所有指向不存在目标的相对链接。

<div id="redirects">
  ## 重定向
</div>

在 `docs.json` 文件中添加 `redirects` 字段以设置 301 重定向。

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

这会将 `/source/path` 永久重定向到 `/destination/path`，从而保留原页面累积的 SEO 权重。

要匹配通配符路径，请在参数后加上 `*`。在此示例中，`/beta/:slug*` 会匹配 `/beta/introduction` 并将其重定向到 `/v2/introduction`。

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
