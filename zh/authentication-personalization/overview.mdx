<Info>
  [Pro 方案](https://mintlify.com/pricing?ref=authentication)包含密码身份验证。

  [Enterprise 方案](https://mintlify.com/pricing?ref=authentication)包含所有身份验证方式。
</Info>

从以下三种方式中选择，根据用户信息和安全需求管理访问并自定义文档：

- **完全身份验证**：为所有内容提供完整的隐私保护，并支持全面的内容定制。
- **部分身份验证**：按页面进行访问控制，并支持全面的内容定制。
- **个性化**：仅进行内容定制，**不提供任何安全保证**。所有内容依然对公众可见。

<div id="use-case-examples">
  ### 用例示例
</div>

**在以下情况下选择完全身份验证：**

- 构建包含敏感信息的公司内部文档
- 记录需要用户验证的专有 API
- 创建面向特定客户的实施指南

**在以下情况下选择部分身份验证：**

- 提供公开的入门指南，同时包含私有的高级功能
- 运营免费增值产品，让付费用户获得额外文档
- 发布开源文档，并包含私有的企业版内容

**在以下情况下选择个性化：**

- 创建公开的 API 文档，展示与用户相关的示例
- 构建可基于用户画像定制内容的营销网站
- 提供可根据用户偏好或技能水平自适应的公开教程

<div id="handshake-methods">
  ## 握手机制
</div>

选择最符合您现有基础设施与安全要求的机制。

| 方法 | 适用范围 | 配置复杂度 | 最佳适用场景 |
|:-------|:--------------|:-----------------|:----------|
| **JWT** | 所有方案 | 中等 | 自定义登录流程，最大化安全控制 |
| **OAuth 2.0** | 所有方案 | 高 | 第三方认证提供商、企业级安全 |
| **Mintlify Dashboard** | 仅身份验证 | 低 | 已使用 Mintlify 仪表盘的团队 |
| **Password** | 仅身份验证 | 低 | 无需个性化的简单共享访问 |
| **Shared Session** | 仅个性化 | 低 | 已有基于会话的认证的应用 |

<div id="when-to-use-each-method">
  ### 何时使用各方法
</div>

**JWT**：适用于你已有身份验证系统并希望完全掌控登录流程的场景。非常适合自定义用户管理，或需要将文档访问与主应用解耦时。

**OAuth 2.0**：适用于希望借助第三方身份提供商（Google、GitHub 等）或需要企业级安全标准的场景。最适合已部署 OAuth 基础设施的组织。

**Mintlify Dashboard**：适用于文档编辑者同时也是文档读者的情况。非常适合已通过 Mintlify Dashboard 管理内容的内部团队。

**Password**：在无需追踪个体用户或进行内容个性化、仅需简单访问控制时使用。适合承包商、内测用户或临时访问场景。

**Shared Session**：当你希望在应用与文档之间实现无缝登录时使用。理想场景是用户已在主应用中完成身份验证，你希望据此个性化其文档体验。

<div id="content-customization">
  ## 内容自定义
</div>

这三种方式都支持内容个性化功能，包括动态 MDX 内容、API 密钥预填充，以及页面可见性控制。有关详细的实现指南，请参见[个性化设置](/zh/authentication-personalization/personalization-setup)。
