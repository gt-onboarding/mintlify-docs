从你的[仪表盘](https://dashboard.mintlify.com/editor)进入可视化编辑器，直接在浏览器中管理文档。

* **所见即所得编辑**：使用所见即所得（WYSIWYG）编辑器修改文档，并预览发布后的呈现效果。
* **Git 同步**：所有更改会自动同步到你的 Git 仓库，确保版本可追踪。
* **团队协作**：多人可同时协作编辑文档。
* **组件集成**：通过斜杠命令添加提示、代码块等组件。
* **零配置**：直接在仪表盘中立即开始撰写。

<div id="overview">
  ## 概览
</div>

在可视化编辑器中的典型工作流程如下：

<Steps>
  <Step title="选择分支">
    创建一个分支，或直接在部署分支上修改。我们建议先创建分支，以便在更改上线前进行预览。
  </Step>

  <Step title="打开文件">
    在侧边栏定位到现有文件，或通过文件资源管理器创建新文件。
  </Step>

  <Step title="编辑内容">
    在可视化编辑器中进行修改。按 <kbd>/</kbd> 打开组件菜单。
  </Step>

  <Step title="预览更改">
    可视化模式会展示这些更改在线上站点的实际效果。用它来确认一切无误。
  </Step>

  <Step title="发布更改">
    如果你在部署分支上工作，可直接在可视化编辑器中发布更改。在其他分支上，需要先创建 Pull Request 进行审核，再发布。
  </Step>
</Steps>

  ## 编辑器模式
</div>

</div>

<div id="organize-your-navigation">
  ### 组织导航
</div>

编辑 `docs.json` 文件，在站点导航中添加新页面并移除已删除的页面。关于如何组织页面的更多信息，请参阅[导航](/zh/organize/navigation)。

**示例：将 Themes 页面添加到 Profile 组**

在此示例中，你创建了一个名为 Themes 的新页面，并希望将其添加到文档中的 Profile 组。将新页面的路径添加到 `docs.json` 文件中 `Profile` 组的 `pages` 数组中，使其显示在站点导航中。

```json title="Adding a Themes page to the Profile group" highlight="18"
{
    "navigation": {
        "groups": [
            {
                "group": "开始使用",
                "pages": [
                    "index",
                    "quickstart",
                    "installation"
                ]
            },
            {
                "group": "配置",
                "pages": [
                    "settings",
                    "account-types",
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```

  ### 编辑内容
</div>

在编辑器中使用可视化模式或 Markdown 模式来修改页面。

在可视化模式下，按 <kbd>/</kbd> 打开组件菜单。添加内容块、提示框、代码块等组件，以自定义你的文档。

  <img src="/images/editor/component-menu-light.png" alt="Mintlify 可视化编辑器中展开的组件菜单高亮显示" className="block dark:hidden" />

在 Markdown 模式下，你可以直接编辑页面的 `MDX`。在以下情况下尤其有用：

* 设置特定的组件属性

参见[格式化文本](/zh/create/text)和[格式化代码](/zh/create/code)，了解使用 Markdown 语法编写的更多信息。

  ## 发布你的更改
</div>

你正在处理的分支决定了编辑器如何发布你的更改：

* **部署分支**：发布会立即更新你的线上站点。
* **其他分支**：发布会创建一个 pull request，便于你在部署到生产环境前审阅更改。

<div id="pull-requests-and-reviewing-changes">
  ### Pull request 与变更评审
</div>

Pull request 允许你从自己的分支发起修改请求，让他人在合并到线上文档前进行评审。这有助于确保改动正确无误，并为团队提供协作完善内容的机会。

<Tip>
  即使是单人作业，使用 pull request 也有助于预览改动并维护清晰的更新历史。
</Tip>

<div id="create-a-pull-request">
  #### 创建 pull request
</div>

<Steps>
  <Step title="保存你的工作">
    选择 **Save Changes**，保存当前分支上的所有更改。
  </Step>

  <Step title="创建 pull request">
    在编辑器工具栏中选择 **Publish Pull Request**。
  </Step>

  <Step title="添加标题和描述">
    编写清晰的标题和描述，说明：

    * 你做了哪些更改
    * 你为什么这样做
    * 需要重点审核的部分
  </Step>

  <Step title="创建并分享">
    选择 **Publish Pull Request**。编辑器会提供一个链接，用于查看你的 pull request。

    <Frame>
      <img src="/images/editor/pull-request-light.png" alt="在 Mintlify 可视化编辑器中突出显示的 Publish Pull Request 按钮" className="block dark:hidden" />

      <img src="/images/editor/pull-request-dark.png" alt="在 Mintlify 可视化编辑器中突出显示的 Publish Pull Request 按钮" className="hidden dark:block" />
    </Frame>
  </Step>
</Steps>

<div id="review-pull-requests">
  #### 审阅 pull request
</div>

创建 pull request 后：

1. **审阅更改**：你和团队成员可以在 GitHub、GitLab 等 Git 托管平台上审阅你的 pull request。
2. **提供反馈**：添加评论或发起变更请求。
3. **继续修改**：在可视化编辑器中进行进一步修改。保存后，编辑器会将更改推送到你的 pull request。
4. **批准**：当你对更改满意时，批准该 pull request。
5. **合并**：当你准备将更改部署到生产环境时，合并该 pull request。

<div id="troubleshooting">
  ## 故障排查
</div>

查找使用可视化编辑器时常见问题的解决方案。

<AccordionGroup>
  <Accordion title="发布后更改未生效">
    **可能原因：**

    * 部署仍在进行
    * 浏览器缓存问题
    * 构建或部署错误

    **解决方案：**

    1. 在仪表盘中查看部署状态。
    2. 强制刷新浏览器（<kbd>Ctrl</kbd> + <kbd>F5</kbd> 或 <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>R</kbd>）。
    3. 清除浏览器缓存。
  </Accordion>

  <Accordion title="发布时出现权限错误">
    **可能原因：**

    * 对 Git 仓库的权限不足
    * 与 Git 提供商的身份验证问题

    **解决方案：**

    1. 验证你对该仓库是否拥有正确的访问权限。
    2. 检查你的 Git 集成是否已正确配置。
    3. 查看[编辑器权限](/zh/dashboard/permissions)文档。
  </Accordion>

  <Accordion title="编辑器加载问题">
    **可能原因：**

    * 网络连接问题
    * 文档仓库规模较大

    **解决方案：**

    1. 检查你的网络连接。
    2. 刷新页面。
    3. 若问题仍然存在，请联系支持。
  </Accordion>

  <Accordion title="文件未加载或出现错误">
    **可能原因：**

    * 文件中的 MDX 语法无效
    * 文件缺失或损坏
    * 大文件导致超时

    **解决方案：**

    1. 检查文件语法是否存在 MDX 格式错误。
    2. 验证该文件是否存在于你的仓库中。
  </Accordion>
</AccordionGroup>
