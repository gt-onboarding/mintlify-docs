每个页面都是一个 MDX 文件，它将 Markdown 内容与 React 组件相结合，使你能够创建丰富、交互式的文档。

<div id="page-metadata">
  ## 页面元数据
</div>

每个页面都以 frontmatter 开头，即文件开头由 `---` 包裹的 YAML 元数据。该元数据定义页面的呈现方式和行为。

使用元数据可以控制：

- 页面标题和描述
- 侧边栏标题、图标和标签

<ResponseField name="title" type="string" required>
  出现在导航和浏览器标签页中的页面标题。
</ResponseField>

<ResponseField name="description" type="string">
  对本页面内容的简要说明。显示在标题下方，并有助于提升 SEO。
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  显示在侧边栏导航中的简短标题。
</ResponseField>

<IconsOptional />

<ResponseField name="tag" type="string">
  显示在侧边栏中页面标题旁的标签。
</ResponseField>

```yaml Example YAML frontmatter wrap
---
title: "About frontmatter"
description: "Frontmatter is the metadata that controls how your page appears and behaves"
sidebarTitle: "Frontmatter"
icon: "book"
tag: "NEW"
---
```

<div id="page-mode">
  ## 页面模式
</div>

通过 `mode` 设置控制页面的展示方式。

<div id="default">
  ### 默认
</div>

如果未设置模式，将使用包含侧边导航和目录的标准布局。

```yaml
---
title: "Default page title"
---
```

<div id="wide">
  ### 宽屏
</div>

宽屏模式会隐藏目录。对于没有任何标题的页面，或当你想利用更多横向空间时，这很有用。

```yaml
---
title: "宽屏页面标题"
mode: "wide"
---
```

<div id="custom">
  ### 自定义
</div>

自定义模式提供极简布局，除顶部导航栏外移除所有元素。它是一块空白画布，适合创建登录页或任何其他希望导航元素尽量精简的独特布局。

```yaml
---
title: "Custom page title"
mode: "custom"
---
```

<div id="center">
  ### 居中
</div>

居中模式会移除侧边栏和目录，并将内容置于页面中央。对于更新日志或其他需要突出内容的页面，这非常实用。

```yaml
---
title: "Center page title"
mode: "center"
---
```

<div id="api-pages">
  ## API 页面
</div>

在 frontmatter 中添加 API 规范字段 `api` 或 `openapi`，即可创建交互式 API 操作台。

```yaml
---
openapi: "GET /endpoint"
---
```

了解如何构建 [API 文档](/zh/api-playground/overview)。

<div id="external-links">
  ## 外部链接
</div>

在导航中使用 `url` 元数据直接指向外部站点。

```yaml
---
title: "npm Package"
url: "https://www.npmjs.com/package/mint"
---
```

<div id="search-engine-optimization">
  ## 搜索引擎优化
</div>

大多数 SEO 元标签会自动生成。你也可以手动设置 SEO 元标签，以提升站点的 SEO、社交分享效果和浏览器兼容性。

<Note>
  含有冒号的元标签必须用引号括起来。
</Note>

```yaml
---
"twitter:image": "/images/social-preview.jpg"
---
```

有关完整的 SEO 元数据选项，请参见 [SEO](/zh/settings/seo)。

<div id="internal-search-keywords">
  ## 内部搜索关键词
</div>

在元数据中添加 `keywords`，可提升特定页面在内置搜索中的可发现性。这些关键词不会显示为页面内容，也不会直接出现在搜索结果中，但当用户搜索这些词时，该页面会作为结果显示。

```yaml
---
keywords: ['configuration', 'setup', 'getting started']
---
```
