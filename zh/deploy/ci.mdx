<Info>
  [Pro 和 Enterprise 方案](https://mintlify.com/pricing?ref=docs-ci) 包含针对 GitHub 仓库的 CI 检查。
</Info>

使用 CI 检查对文档进行规范校验，在部署前捕获错误并给出警告。CI 检查会在向已配置的部署分支提交时运行，或在针对该分支的拉取请求（pull request）上运行。

<div id="installation">
  ## 安装
</div>

首先，请按照 [GitHub](/zh/deploy/github) 页面上的步骤操作。

对于 GitHub 应用，您可以选择只为单个仓库授予权限。我们强烈建议这样做，因为我们只需要访问托管您文档的那个仓库。

## 配置

前往仪表盘的 [Add-ons](https://dashboard.mintlify.com/products/addons) 页面，配置部署中启用的 CI 检查。启用你希望运行的检查。

在启用检查时，你可以选择将其设置为 `Warning` 或 `Blocking` 级别运行。

* `Warning` 级别的检查即使存在错误或建议，也不会标记为失败。
* `Blocking` 级别的检查如果未通过或提出了更改建议，将标记为失败。

<div id="available-ci-checks">
  ## 可用的 CI 检查项
</div>

<div id="broken-links">
  ### 失效链接
</div>

与在本地运行的 [CLI 链接检查器](/zh/create/broken-links#broken-links)类似，我们会自动检查你的文档是否存在失效链接。

要查看检查结果，请前往该次提交对应的 GitHub 检查结果页面。

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) 是一款开源的规则驱动型文本风格校验工具，支持多种文档类型，包括 Markdown 和 MDX。

Mintlify 支持在 CI 检查中自动运行 Vale，并将结果显示为检查状态。

<div id="configuration">
  #### 配置
</div>

如果在部署的根内容目录中有 `.vale.ini` 文件，我们会自动使用该配置文件。我们也会自动使用你在 `stylesPath` 中指定的任何配置文件。

如果你没有 Vale 配置文件，将自动加载默认配置。

```mdx Default vale.ini configuration expandable
# 顶层样式
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# 词汇表
Vocab = Mintlify

# 由于 Vale 官方不支持 MDX,此项为必需配置
[formats]
mdx = md

# MDX 支持
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # 强制执行严格的大小写规则,建议保持关闭

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# 排除项:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<Warning>
  请注意，出于安全原因，我们不支持任何绝对路径的 `stylesPath`，或包含 `..` 的 `stylesPath`。请使用相对路径，并将该 `stylesPath` 提交到你的代码仓库中。
</Warning>

<div id="packages">
  #### 软件包
</div>

Vale 支持多种[软件包](https://vale.sh/docs/keys/packages)，可用于检查拼写和样式错误。
任何放在代码库中正确的 `stylesPath` 路径下的软件包，都会在你的 Vale 配置中自动安装并启用。

对于未包含在代码库中的软件包，你可以从 [Vale 软件包注册表](https://vale.sh/explorer) 指定任意软件包；它们会被自动下载并在你的 Vale 配置中使用。

<Warning>
  请注意，出于安全考虑，我们无法支持自动下载不来自 [Vale 软件包注册表](https://vale.sh/explorer) 的软件包。
</Warning>

<div id="vale-with-mdx">
  #### 使用 `MDX` 的 Vale
</div>

Vale 并不原生支持 `MDX`，但 Vale 的作者提供了一个[自定义扩展](https://github.com/errata-ai/MDX)以实现支持。

如果你不想使用该扩展，建议在你的 `.vale.ini` 文件中添加以下几行：

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

要在文档中使用 Vale 的文内注释，请使用 MDX 风格的注释 `{/* ... */}`。如果你在配置中启用 `CommentDelimiters = {/*, */}` [设置](https://vale.sh/docs/keys/commentdelimiters)，Vale 会在 lint 时自动识别这些注释。这意味着你可以轻松使用 Vale 的内置功能，比如跳过某些行或部分。

```mdx
{/* vale off */}

此文本将被 Vale 忽略

{/* vale on */}
```

如果你不使用 `CommentDelimiters`，但仍要使用 Vale 的注释，则必须将所有 Vale 注释包裹在 MDX 注释 `{/* ... */}` 中。例如：

```mdx
{/* <!-- vale off --> */}

此文本将被 Vale 忽略

{/* <!-- vale on --> */}
```

这些注释标签在 Mintlify 组件内不受支持，但可在文档的顶层任意位置使用。
