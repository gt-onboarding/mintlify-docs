---
title: "预览部署"
description: "在实时环境中预览文档的更改"
---

<Info>
  预览部署适用于 [Pro 和 Enterprise 方案](https://mintlify.com/pricing?ref=preview-deployments)。
</Info>

通过预览部署，你可以在合并到生产环境之前查看文档变更的实际呈现效果。每个预览都会生成一个可共享的 URL，并会在你推送新更改时自动更新。

<div id="creating-preview-deployments">
  ## 创建预览部署
</div>

预览部署会通过拉取请求自动创建，或可在仪表盘中手动创建。

<div id="automatic-previews">
  ### 自动预览
</div>

当你创建 Pull Request 时，Mintlify 机器人会自动在该 Pull Request 中添加一个链接，用于查看预览部署。每次你向该分支推送新的提交时，预览都会同步更新。

<div id="manual-previews">
  ### 手动预览
</div>

你可以为任何分支手动创建预览。

1. 前往你的[控制台](https://dashboard.mintlify.com/)。
2. 选择 **Previews**。
3. 选择 **Create preview**。
4. 输入分支名称。
5. 选择 **Create deployment**。

<div id="redeploying-a-preview">
  ## 重新部署预览
</div>

当需要刷新内容，或在部署失败后重试时，可重新部署预览。

1. 在你的[控制台](https://dashboard.mintlify.com/)中选择该预览。
2. 点击 **Redeploy**。

<div id="sharing-preview-deployments">
  ## 共享预览部署
</div>

预览部署可以对公众开放，或仅限通过身份验证的用户访问。

**公开访问**：预览 URL 默认对所有人可见。将预览链接分享给需要审阅你更改的任何人。

**受限访问**：启用预览部署身份验证，将预览访问限制为仅面向已通过身份验证的 Mintlify 用户。更多信息请参阅[身份验证设置](/zh/deploy/authentication-setup)。
