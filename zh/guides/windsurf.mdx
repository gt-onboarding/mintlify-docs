<div id="using-windsurf-with-mintlify">
  ## 在 Mintlify 中使用 Windsurf
</div>

Windsurf 的 Cascade AI 助手可以结合 Mintlify 组件，根据你的规范生成文档。Workspace 规则和记忆为项目提供持久化上下文，帮助 Cascade 给出更一致的建议。

* **Workspace 规则**存放在你的文档仓库中，并与团队共享。
* **记忆（Memories）**提供会随时间累积的个人上下文。

我们建议为共享的文档规范设置 Workspace 规则。你可以在工作中逐步积累记忆，但由于记忆不共享，因此团队成员之间可能不一致。

在你的文档仓库的 `.windsurf/rules` 目录中创建 Workspace 规则。更多信息请参见 Windsurf 文档的 [Memories & Rules](https://docs.windsurf.com/windsurf/cascade/memories)。

<div id="example-workspace-rule">
  ## 示例工作区规则
</div>

此规则为 Cascade 提供有关 Mintlify 组件与通用技术写作最佳实践的上下文。

你可以直接使用此示例规则，或根据你的文档进行自定义：

* **写作规范**：更新语言指南以符合你的风格指南。
* **组件模式**：添加项目特定的组件或修改现有示例。
* **代码示例**：将通用示例替换为与你产品相关的真实 API 调用与响应。
* **风格与语气偏好**：调整术语、格式和其他规则。

将你的规则保存为 `.md` 文件，放在文档代码库的 `.windsurf/rules` 目录中。

````mdx
# Mintlify 技术写作规则

## 项目背景

- 这是基于 Mintlify 平台的文档项目
- 我们使用带有 YAML 前言的 MDX 文件  
- 导航在 `docs.json` 中配置
- 我们遵循技术写作最佳实践

## 写作规范

- 在说明中使用第二人称（"you"）
- 使用主动语态与一般现在时
- 以前提条件开始操作流程
- 为主要步骤说明预期结果
- 使用描述性且富含关键词的标题
- 句子保持简洁且信息充分

## 必需的页面结构

每个页面必须以前言区块开始：

```yaml
---
title: "清晰、具体的标题"
description: "用于 SEO 和导航的精炼描述"
---
```

## Mintlify 组件

### docs.json

- 构建 docs.json 文件与站点导航时，请参考 [docs.json schema](https://mintlify.com/docs.json)

### Callouts

- `<Note>`：用于有用的补充信息
- `<Warning>`：用于重要的注意事项和破坏性变更
- `<Tip>`：用于最佳实践和专家建议  
- `<Info>`：用于中性的背景信息
- `<Check>`：用于成功确认

### 代码示例

- 在合适的情况下，包含完整且可运行的示例
- 使用 `<CodeGroup>` 展示多语言示例
- 为所有代码块指定语言标签
- 使用真实数据，而非占位符
- 在 API 文档中使用 `<RequestExample>` 和 `<ResponseExample>`

### 操作流程

- 使用 `<Steps>` 组件进行顺序性说明
- 在相关情况下，使用 `<Check>` 组件添加验证步骤
- 将复杂流程拆解为更小的步骤

### 内容组织

- 针对平台特定内容使用 `<Tabs>`
- 使用 `<Accordion>` 实现渐进式披露
- 使用 `<Card>` 和 `<CardGroup>` 突出内容
- 使用带有描述性 alt 文本的 `<Frame>` 组件包裹图片

## API 文档要求

- 使用 `<ParamField>` 记录所有参数 
- 使用 `<ResponseField>` 展示响应结构
- 同时包含成功与错误示例
- 对嵌套对象属性使用 `<Expandable>`
- 始终包含身份验证示例

## 质量标准

- 在发布前测试所有代码示例
- 内部链接使用相对路径
- 为所有图片添加 alt 文本
- 确保正确的标题层级（从 h2 开始）
- 检查既有模式以保持一致性
````

<div id="working-with-cascade">
  ## 使用 Cascade
</div>

完成规则配置后，您可以使用 Cascade 协助处理各类文档任务。更多信息请参见 Windsurf 文档中的 [Cascade](https://docs.windsurf.com/windsurf/cascade)。

<div id="example-prompts">
  ### 示例提示
</div>

**撰写新内容**：

```text wrap
创建一个新页面，说明如何在我们的 API 中进行身份验证。请包含 JavaScript、Python 和 cURL 的代码示例。
```

**改进现有内容**：

```text wrap
审阅此页面并就表达清晰度和组件使用提出改进建议。重点让步骤更易于理解和跟进。
```

**创建代码示例**：

```text wrap
生成一个完整的代码示例，展示对此 API 端点的错误处理。使用贴近真实的数据，并包含预期的响应。
```

**保持一致性**：

```text wrap
检查此新页面是否符合我们的文档规范，并提出所需的修改建议。
```
