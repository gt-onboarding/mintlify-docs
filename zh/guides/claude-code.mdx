Claude Code 是一款具备代理式能力的命令行工具，可帮助你维护文档。它能够撰写新内容、评审现有页面，并持续保持文档最新。

你可以在项目中添加一个 `CLAUDE.md` 文件，并随时间不断完善，从而训练 Claude Code 理解你的文档规范和工作流程。

<div id="getting-started">
  ## 入门
</div>

**前置条件：**

- 有效的 Claude 订阅（Pro、Max 或 API 访问）

**设置：**

1. 安装 Claude Code：

```bash
  npm install -g @anthropic-ai/claude-code
  ```

2. 进入你的文档目录。
3.（可选）将下方的 `CLAUDE.md` 文件添加到项目中。
4. 运行 `claude` 开始使用。

<div id="claudemd-template">
  ## CLAUDE.md 模板
</div>

在文档目录的根目录保存一个名为 `CLAUDE.md` 的文件，帮助 Claude Code 了解你的项目。该文件用于让 Claude Code 学习你的文档规范、偏好和工作流程。更多信息请参阅 Anthropic 文档中的[管理 Claude 的记忆](https://docs.anthropic.com/en/docs/claude-code/memory)。

复制此示例模板，或根据你的文档规范进行调整：

```mdx
# Mintlify documentation

## Working relationship
- 你可以对想法提出质疑——这有助于产出更好的文档。这样做时请引用来源并说明你的理由
- 一定要先寻求澄清，而不是做出假设
- 绝不要撒谎、猜测或编造信息

## Project context
- 格式：带有 YAML frontmatter 的 MDX 文件
- 配置：使用 docs.json 定义导航、主题和设置
- 组件：Mintlify 组件

## Content strategy
- 只记录确保用户成功所需的内容——不多也不少
- 优先保证信息的准确性与可用性
- 尽可能让内容具有长期有效性
- 在添加新内容前先搜索是否已存在相关信息。除非基于明确策略，否则避免重复
- 检查既有模式以确保一致性
- 从最小、最合理的改动开始

## docs.json

- 构建 docs.json 文件和站点导航时，请参考 [docs.json 模式](https://mintlify.com/docs.json)

## Frontmatter requirements for pages
- title：清晰、可描述的页面标题
- description：用于 SEO/导航的精炼摘要

## Writing standards
- 使用第二人称语气（“你”）
- 在步骤式内容开头列出前置条件
- 在发布前测试所有代码示例
- 匹配现有页面的风格与格式
- 同时涵盖基础与高级用例
- 为所有代码块添加语言标记
- 为所有图片添加替代文本（alt 文本）
- 站内链接使用相对路径

## Git workflow
- 提交时绝不要使用 --no-verify
- 在开始前先确认如何处理未提交的更改
- 若没有明确的分支可用，创建新的分支
- 在开发过程中频繁提交
- 绝不要跳过或禁用 pre-commit hooks

## Do not
- 任何 MDX 文件都不要省略 frontmatter
- 站内链接不要使用绝对 URL
- 不要包含未经测试的代码示例
- 不要做出假设——始终先寻求澄清
```

  ## 示例提示

完成 Claude Code 的设置后，可尝试以下提示，了解它如何协助处理常见文档任务。你可以直接复制这些示例并粘贴使用，或根据你的具体需求进行调整。

<div id="convert-notes-to-polished-docs">
  ### 将笔记转换为精致文档
</div>

把粗略草稿转换为包含组件和 frontmatter 的规范 Markdown 页面。

**示例提示：**

```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

<div id="review-docs-for-consistency">
  ### 审阅文档以确保一致性
</div>

获取关于改进写作风格、格式与组件用法的建议。

**示例提示：**

```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### 当功能发生变化时更新文档
</div>

在产品迭代时保持文档的实时性。

**示例提示：**

```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

<div id="generate-comprehensive-code-examples">

</div>

创建包含错误处理的多语言示例。

**示例提示：**

```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## 扩展 Claude Code
</div>

除了手动与 Claude Code 交互外，你还可以将其集成到现有工作流程中。

<div id="automation-with-github-actions">
  ### 使用 GitHub Actions 实现自动化
</div>

当代码发生变更时自动运行 Claude Code，确保文档保持最新。你可以在 pull request 上触发文档评审，或在检测到 API 变更时自动更新示例。

<div id="multi-instance-workflows">
  ### 多实例工作流
</div>

针对不同任务分别开启独立的 Claude Code 会话：一个用于撰写新内容，另一个用于审阅与质量把控。这样有助于保持一致性，并捕捉单一会话可能遗漏的问题。

<div id="team-collaboration">
  ### 团队协作
</div>

将完善后的 `CLAUDE.md` 文件与团队共享，确保所有贡献者遵循一致的文档标准。团队通常会制定项目特定的提示和工作流程，并将其纳入文档实践。

<div id="custom-commands">
  ### 自定义命令
</div>

在 `.claude/commands/` 中创建可复用的斜杠命令，用于处理与你的项目或团队特定相关的常用文档任务。
