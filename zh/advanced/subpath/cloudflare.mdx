要在 Cloudflare 上将文档托管到 `/docs` 子路径下，您需要创建并配置一个 Cloudflare Worker。

<Info>
  开始之前，您需要一个 Cloudflare 账号和一个域名（可在 Cloudflare 内或外进行管理）。
</Info>


<div id="set-up-a-cloudflare-worker">
  ## 设置 Cloudflare Worker
</div>

如果尚未创建，请按照 [Cloudflare Workers 入门指南](https://developers.cloudflare.com/workers/get-started/dashboard/) 创建一个 Cloudflare Worker。

<Warning>
  如果 DNS 提供商为 Cloudflare，请不要为该 CNAME 记录启用代理。
</Warning>

<div id="proxies-with-vercel-deployments">
  ### 使用 Vercel 部署时的代理
</div>

如果您在 Vercel 部署中使用 Cloudflare 作为代理，必须确保正确配置，以避免与 Vercel 的域名验证和 SSL 证书签发发生冲突。

错误的代理配置可能会导致 Vercel 无法为您签发 Let's Encrypt SSL 证书，并引发域名验证失败。

<div id="required-path-allowlist">
  #### 必需的路径白名单
</div>

你的 Cloudflare Worker 必须允许以下特定路径的流量通过，且不得拦截或重定向：

- `/.well-known/acme-challenge/*` - Let's Encrypt 证书验证所必需
- `/.well-known/vercel/*` - Vercel 域名验证所必需

尽管 Cloudflare 会自动处理许多验证规则，但额外创建自定义规则可能会无意间阻断这些关键流量。

<div id="header-forwarding-requirements">
  #### 标头转发要求
</div>

请确保在 Worker 配置中正确转发 `HOST` 标头。若未正确转发该标头，验证请求将会失败。

<div id="configure-routing">
  ### 配置路由
</div>

在 Cloudflare 控制台中，选择 **Edit Code**，并将以下脚本添加到你的 Worker 代码里。有关编辑 Worker 的更多信息，请参阅 [Cloudflare 文档](https://developers.cloudflare.com/workers-ai/get-started/dashboard/#development)。

<Tip>
  将 `[SUBDOMAIN]` 替换为你的专属子域名，将 `[YOUR_DOMAIN]` 替换为你网站的基础 URL。
</Tip>

```javascript
addEventListener("fetch", (event) => {
  event.respondWith(handleRequest(event.request));
});

async function handleRequest(request) {
  try {
    const urlObject = new URL(request.url);
    
    // 若请求指向 Vercel 的校验路径，直接放行
    if (urlObject.pathname.startsWith('/.well-known/')) {
      return await fetch(request);
    }
    
    // 若请求指向 docs 子路径
    if (/^\/docs/.test(urlObject.pathname)) {
      // 代理到 Mintlify
      const DOCS_URL = "[SUBDOMAIN].mintlify.dev";
      const CUSTOM_URL = "[YOUR_DOMAIN]";

      let url = new URL(request.url);
      url.hostname = DOCS_URL;

      let proxyRequest = new Request(url, request);

      proxyRequest.headers.set("Host", DOCS_URL);
      proxyRequest.headers.set("X-Forwarded-Host", CUSTOM_URL);
      proxyRequest.headers.set("X-Forwarded-Proto", "https");
      // 若部署到 Vercel，保留客户端 IP
      proxyRequest.headers.set("CF-Connecting-IP", request.headers.get("CF-Connecting-IP"));

      return await fetch(proxyRequest);
    }
  } catch (error) {
    // 未匹配到规则时，按原请求转发
    return await fetch(request);
  }
}
```

选择 **Deploy** 并等待更改生效。

<div id="test-your-worker">
  ### 测试你的 Worker
</div>

代码部署完成后，测试你的 Worker，确保它能正确路由到你的 Mintlify 文档。

1. 使用 Worker 的预览 URL 进行测试：`your-worker.your-subdomain.workers.dev/docs`
2. 确认 Worker 能正确路由到你的 Mintlify 文档和你的网站。

<div id="add-custom-domain">
  ### 添加自定义域名
</div>

1. 在您的 [Cloudflare 控制台](https://dash.cloudflare.com/)中，进入您的 Worker。
2. 依次打开 **Settings > Domains & Routes > Add > Custom Domain**。
3. 添加您的域名。

<Tip>
  我们建议同时添加带有和不带有 `www.` 前缀的域名。
</Tip>

更多信息请参见 Cloudflare 文档：[Add a custom domain](https://developers.cloudflare.com/workers/configuration/routing/custom-domains/#add-a-custom-domain)。 

<div id="resolve-dns-conflicts">
  ### 解决 DNS 冲突
</div>

如果你的域名已指向其他服务，你必须移除现有的 DNS 记录。必须将 Cloudflare Worker 配置为掌控该域名的全部流量。

1. 删除该域名现有的 DNS 记录。更多信息请参阅 Cloudflare 文档中的[删除 DNS 记录](https://developers.cloudflare.com/dns/manage-dns-records/how-to/create-dns-records/#delete-dns-records)。
2. 返回你的 Worker 并添加自定义域名。

<div id="webflow-custom-routing">
  ## Webflow 自定义路由
</div>

如果你使用 Webflow 托管主站点，并希望在同一域名下的 `/docs` 提供 Mintlify 文档，你需要通过 Cloudflare Workers 配置自定义路由，将所有非文档流量代理到主站点。

<Warning>
  在部署此 Worker 之前，请确保你的主站点已设置为落地页，否则访问主站点的访客会看到错误。
</Warning>

1. 在 Webflow 中，为主站点设置一个落地页，例如 `landing.yoursite.com`。当访客访问你的网站时，他们将看到此页面。
2. 将主站点部署到该落地页。这样可确保在配置 Worker 的同时，主站点仍然可访问。
3. 为避免冲突，将主站点中的任何绝对 URL 更新为相对路径。
4. 在 Cloudflare 中选择 **Edit Code**，并将以下脚本添加到 Worker 的代码中。

<Tip> 将 `[SUBDOMAIN]` 替换为你的唯一子域名，将 `[YOUR_DOMAIN]` 替换为你网站的基础 URL，将 `[LANDING_DOMAIN]` 替换为你的落地页 URL。 </Tip>

```javascript
  addEventListener("fetch", (event) => {
  event.respondWith(handleRequest(event.request));
  });
  async function handleRequest(request) {
  try {
    const urlObject = new URL(request.url);
    
    // 如果请求是到 Vercel 验证路径，允许其通过
    if (urlObject.pathname.startsWith('/.well-known/')) {
      return await fetch(request);
    }
    
    // 如果请求是到 docs 子目录
    if (/^\/docs/.test(urlObject.pathname)) {
      // 代理到 Mintlify
      const DOCS_URL = "[SUBDOMAIN].mintlify.dev";
      const CUSTOM_URL = "[YOUR_DOMAIN]";
      let url = new URL(request.url);
      url.hostname = DOCS_URL;
      let proxyRequest = new Request(url, request);
      proxyRequest.headers.set("Host", DOCS_URL);
      proxyRequest.headers.set("X-Forwarded-Host", CUSTOM_URL);
      proxyRequest.headers.set("X-Forwarded-Proto", "https");
      // 如果部署到 Vercel，保留客户端 IP
      proxyRequest.headers.set("CF-Connecting-IP", request.headers.get("CF-Connecting-IP"));
      return await fetch(proxyRequest);
    }
    // 其余所有请求路由到主站点
    const MAIN_SITE_URL = "[LANDING_DOMAIN]";
    if (MAIN_SITE_URL && MAIN_SITE_URL !== "[LANDING_DOMAIN]") {
      let mainSiteUrl = new URL(request.url);
      mainSiteUrl.hostname = MAIN_SITE_URL;
      return await fetch(mainSiteUrl, {
        method: request.method,
        headers: request.headers,
        body: request.body
      });
    }
  } catch (error) {
    // 如果未匹配到任何规则，则按常规请求处理
    return await fetch(request);
  }
  }
  ```

5. 选择 **Deploy** 并等待更改传播生效。
