Die Komponenten `<RequestExample>` und `<ResponseExample>` zeigen Codeblöcke in der rechten Seitenleiste an und erzeugen so ein zweispaltiges Layout, das Beispiele sichtbar hält, während Nutzer durch Ihre Inhalte scrollen. Diese Komponenten sind für API-Dokumentation konzipiert, funktionieren jedoch auf allen Seiten.

Häufige Anwendungsfälle:

- Dokumentation von API-Endpunkten mit Anfrage- und Antwortbeispielen
- Konfigurationsbeispiele neben erläuternden Texten
- Codebeispiele, auf die sich Nutzer beim Befolgen von Anleitungen beziehen
- Vorher-/Nachher-Beispiele in Tutorials

Auf Mobilgeräten werden die Komponenten `<RequestExample>` und `<ResponseExample>` als normale Codeblöcke angezeigt und können einfach weitergescrollt werden.

<div id="requestexample">
  ## RequestExample
</div>

Verwenden Sie `<RequestExample>`, um Codebeispiele in der rechten Seitenleiste zu fixieren. Diese Komponente funktioniert ähnlich wie die Komponente [CodeGroup](/de/components/code-groups), zeigt den Code jedoch in der Seitenleiste statt inline an.

Sie können mehrere Codeblöcke in einem einzelnen `<RequestExample>` einfügen. Jeder Codeblock muss ein title-Attribut besitzen.

````mdx RequestExample
<RequestExample>

```bash Request
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

<div id="responseexample">
  ## ResponseExample
</div>

Die `<ResponseExample>`-Komponente fixiert Codebeispiele in der rechten Seitenleiste unterhalb der `<RequestExample>`-Inhalte auf derselben Seite.

````mdx ResponseExample
<ResponseExample>

```json Response
{ "status": "success" }
```

</ResponseExample>
````
