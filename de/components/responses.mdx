---
title: "Antwortfelder"
description: "API-Antwortwerte anzeigen"
---

Die Komponente `<ResponseField>` dient dazu, die Rückgabewerte einer API zu definieren. In vielen Dokumentationen wird `<ResponseField>` auch auf Seiten verwendet, auf denen Typen aufgelistet werden müssen.

<ResponseField name="response" type="string" required>
  Beispiel für ein Antwortfeld
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Beispiel für ein Antwortfeld
</ResponseField>
```

<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Der Name des Antwortwerts.
</ResponseField>

<ResponseField name="type" type="string" required>
  Erwarteter Typ des Antwortwerts – dies kann eine beliebige Zeichenfolge sein.
</ResponseField>

<ResponseField name="default" type="string">
  Der Standardwert.
</ResponseField>

<ResponseField name="required" type="boolean">
  „required“ neben dem Feldnamen anzeigen.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Ob ein Feld als veraltet gilt oder nicht.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Labels, die vor dem Feldnamen angezeigt werden
</ResponseField>

<ResponseField name="post" type="string[]">
  Labels, die nach dem Feldnamen angezeigt werden
</ResponseField>
