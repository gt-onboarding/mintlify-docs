Mintlify übernimmt automatisch viele SEO-Best Practices, darunter:

- Generierung von Meta-Tags
- Generierung von Sitemap und `robots.txt`
- Semantische HTML-Struktur
- Mobile Optimierung

Sie können die Meta-Tags Ihrer Website vollständig anpassen, indem Sie das Feld `metatags` zu Ihrer `docs.json` oder zum Frontmatter einer Seite hinzufügen.

<div id="global-meta-tags">
  ## Globale Metatags
</div>

Sie können standardmäßige Metatags festlegen, indem Sie das Feld `metadata` zu Ihrer `docs.json` hinzufügen.

```json
"seo": {
    "metatags": {
        "og:image": "link to your default meta tag image"
    }
}
```

<div id="page-specific-meta-tags">
  ## Seitenspezifische Meta-Tags
</div>

Fügen Sie Ihre Meta-Tags wie folgt in die Metadaten der Seite ein:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  Meta-Tags mit Doppelpunkten müssen in Anführungszeichen stehen, wenn Sie sie in die Seitenmetadaten aufnehmen.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps und robots.txt-Dateien
</div>

Mintlify generiert automatisch eine `sitemap.xml`-Datei und eine `robots.txt`-Datei. Sie können Ihre Sitemap anzeigen, indem Sie `/sitemap.xml` an die URL Ihrer Dokumentationsseite anhängen.

Standardmäßig werden nur Seiten berücksichtigt, die in Ihrer `docs.json` aufgeführt sind. Um versteckte Links einzuschließen, fügen Sie `seo.indexing` zu Ihrer `docs.json` hinzu:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Benutzerdefinierte Sitemaps und robots.txt-Dateien
</div>

Um eine benutzerdefinierte `sitemap.xml`- oder `robots.txt`-Datei hinzuzufügen, erstellen Sie eine entsprechende Datei im Stammverzeichnis Ihres Projekts. Eine benutzerdefinierte Datei überschreibt die automatisch generierte Datei gleichen Namens. Wenn Sie die benutzerdefinierte Datei löschen, wird wieder die Standarddatei verwendet.

<div id="disabling-indexing">
  ## Indexierung deaktivieren
</div>

Wenn Sie verhindern möchten, dass eine Seite von Suchmaschinen indexiert wird, können Sie Folgendes im [Frontmatter](de/pages) Ihrer Seite einfügen:

```
---
noindex: true
---
```

Sie können `noindex` auch für alle Seiten Ihrer Dokumentation festlegen, indem Sie das Feld `metadata.robots` in Ihrer `docs.json` auf `"noindex"` setzen:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO-Best Practices
</div>

<AccordionGroup>
  <Accordion title="Aussagekräftige Titel und Beschreibungen verfassen">
    - Verwenden Sie klare, aussagekräftige Seitentitel (50–60 Zeichen)
    - Schreiben Sie prägnante, überzeugende Beschreibungen (150–160 Zeichen)
    - Integrieren Sie relevante Keywords
    - Vergeben Sie für jede Seite einen einzigartigen Titel und eine einzigartige Beschreibung
  </Accordion>

  <Accordion title="Inhaltsstruktur optimieren">
    - Verwenden Sie eine saubere Überschriftenhierarchie (H1 → H2 → H3)
    - Schreiben Sie zuerst für Menschen, danach für Suchmaschinen
    - Platzieren Sie relevante Keywords in Überschriften und Inhalt
    - Halten Sie URLs kurz, aussagekräftig und hierarchisch organisiert
    - Gliedern Sie lange Inhalte mit Zwischenüberschriften und Listen
  </Accordion>

  <Accordion title="Interne Verlinkungsstrategie">
    - Verlinken Sie auf thematisch verwandte Seiten innerhalb Ihrer Dokumentation
    - Nutzen Sie beschreibenden Linktext statt „Hier klicken“
    - Bilden Sie Themencluster, indem Sie verwandte Konzepte verknüpfen
    - Nutzen Sie die automatischen Querverweis-Funktionen
  </Accordion>

  <Accordion title="Bild-SEO">
    - Verwenden Sie aussagekräftige Dateinamen für Bilder
    - Fügen Sie immer Alt-Text für Barrierefreiheit und SEO hinzu
    - Optimieren Sie Bildgrößen für schnellere Ladezeiten
    - Verwenden Sie relevante Bilder, die Ihren Inhalt unterstützen
  </Accordion>
</AccordionGroup>
