<Info>
  [Pro- und Enterprise-Pläne](https://mintlify.com/pricing?ref=docs-ci) enthalten CI-Prüfungen für GitHub-Repositories.
</Info>

Verwenden Sie CI-Prüfungen, um Ihre Dokumentation zu linten, Fehler zu erkennen und Warnungen auszugeben, bevor Sie deployen. CI-Prüfungen sind so konfiguriert, dass sie bei Commits in den konfigurierten Deployment-Branch oder bei Pull Requests gegen diesen Branch ausgeführt werden.

<div id="installation">
  ## Installation
</div>

Beginnen Sie, indem Sie den Schritten auf der Seite [GitHub](/de/settings/github) folgen.

Für GitHub-Apps können Sie die Berechtigungen auf ein einzelnes Repository beschränken. Wir empfehlen dies ausdrücklich, da wir nur Zugriff auf das Repository benötigen, in dem Ihre Dokumentation gehostet wird.

<div id="configuration">
  ## Konfiguration
</div>

Konfigurieren Sie die in der CI für eine Bereitstellung aktivierten Prüfungen, indem Sie in Ihrem Dashboard zur Seite [Add-ons](https://dashboard.mintlify.com/products/addons) navigieren. Aktivieren Sie die Prüfungen, die Sie ausführen möchten.

Beim Aktivieren können Sie wählen, ob die Prüfungen auf der Stufe `Warning` oder `Blocking` laufen sollen.

- Eine Prüfung auf der Stufe `Warning` führt niemals zu einem Fehlerstatus, selbst wenn Fehler oder Empfehlungen vorliegen.
- Eine Prüfung auf der Stufe `Blocking` führt zu einem Fehlerstatus, wenn sie nicht bestanden wird oder Änderungen empfohlen werden.

<div id="available-ci-checks">
  ## Verfügbare CI-Checks
</div>

<div id="broken-links">
  ### Defekte Links
</div>

Ähnlich wie der [CLI-Link-Checker](/de/settings/broken-links#broken-links) auf Ihrem lokalen Rechner prüfen wir Ihre Dokumentation automatisch auf defekte Links.

Um die Ergebnisse dieser Prüfung einzusehen, besuchen Sie die Seite mit den Prüfergebnissen auf GitHub für einen bestimmten Commit.

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) ist ein regelbasierter Open-Source-Linter für Prosa, der verschiedene Dokumenttypen unterstützt, darunter Markdown und MDX.

Mintlify kann Vale automatisch in einem CI-Check ausführen und die Ergebnisse als Prüfstatus anzeigen.

<div id="configuration">
  #### Konfiguration
</div>

Wenn sich eine `.vale.ini`-Datei im Stammverzeichnis der Inhalte Ihrer Bereitstellung befindet, verwenden wir diese Konfigurationsdatei automatisch. Wir verwenden außerdem automatisch alle Konfigurationsdateien im angegebenen `stylesPath`.

Wenn Sie keine Vale-Konfigurationsdatei haben, wird die Standardkonfiguration automatisch geladen.

```mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<div id="packages">
  #### Pakete
</div>

Vale unterstützt eine Reihe von [Paketen](https://vale.sh/docs/keys/packages), mit denen sich Rechtschreib- und Stilfehler prüfen lassen.
Alle Pakete, die Sie in Ihrem Repository unter dem korrekten `stylesPath` ablegen, werden automatisch installiert und in Ihrer Vale-Konfiguration verwendet.

Für Pakete, die nicht in Ihrem Repository enthalten sind, können Sie beliebige Pakete aus dem [Vale-Paketregister](https://vale.sh/explorer) angeben; diese werden automatisch heruntergeladen und in Ihrer Vale-Konfiguration genutzt.

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen das automatische Herunterladen von Paketen, die nicht aus dem [Vale-Paketregister](https://vale.sh/explorer) stammen, nicht unterstützen.
</Warning>

<div id="vale-with-mdx">
  #### Vale mit `MDX`
</div>

Vale unterstützt `MDX` nicht nativ, aber der Autor von Vale stellt eine [benutzerdefinierte Erweiterung](https://github.com/errata-ai/MDX) zur Verfügung.

Wenn Sie diese Erweiterung nicht verwenden möchten, empfehlen wir die folgenden Zeilen in Ihrer `.vale.ini`-Datei:

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Um die Dokumentkommentare von Vale zu nutzen, verwenden Sie MDX‑Kommentare `{/* ... */}`. Wenn Sie die [Einstellung](https://vale.sh/docs/keys/commentdelimiters) `CommentDelimiters = {/*, */}` in Ihrer Konfiguration setzen, interpretiert Vale diese Kommentare beim Linting automatisch. So können Sie Vales integrierte Funktionen wie das Überspringen von Zeilen oder Abschnitten bequem verwenden.

```mdx
{/* vale off */}

Dieser Text wird von Vale ignoriert

{/* vale on */}
```

Wenn Sie `CommentDelimiters` nicht verwenden, aber dennoch die Kommentare von Vale einsetzen möchten, müssen Sie alle Vale‑Kommentare in MDX‑Kommentare `{/* ... */}` einschließen. Zum Beispiel:

```mdx
{/* <!-- vale off --> */}

Dieser Text wird von Vale ignoriert

{/* <!-- vale on --> */}
```

Diese Kommentar-Tags werden in Mintlify‑Komponenten nicht unterstützt, können jedoch überall auf der Basisebene eines Dokuments verwendet werden.
