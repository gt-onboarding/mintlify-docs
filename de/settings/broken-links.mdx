Wenn Sie den Pfad einer Datei im Ordner docs ändern, ändert sich auch der URL-Pfad zu dieser Seite. Das kann beim Umstrukturieren Ihrer Dokumentation oder beim Anpassen des Titels in der Seitenleiste passieren.

<div id="broken-links">
  ## Kaputte Links
</div>

Finde kaputte Links mit unserer CLI. [Installiere die CLI](/de/installation) und führe den folgenden Befehl aus:

```bash
mint broken-links
```

Die CLI erkennt alle relativen Links in deinen Docs, die nicht existieren.

<div id="redirects">
  ## Weiterleitungen
</div>

Richte 301-Weiterleitungen ein, indem du das Feld `redirects` zu deiner `docs.json` hinzufügst.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Das leitet `/source/path` dauerhaft auf `/destination/path` um, sodass du das bisherige SEO-Ranking der ursprünglichen Seite nicht verlierst.

Um einen Wildcard-Pfad zu matchen, verwende `*` nach einem Parameter. In diesem Beispiel matcht `/beta/:slug*` `/beta/introduction` und leitet nach `/v2/introduction` um.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
