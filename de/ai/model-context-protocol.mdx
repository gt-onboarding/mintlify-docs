<div id="about-mcp-servers">
  ## Über MCP-Server
</div>

Das Model Context Protocol (MCP) ist ein offenes Protokoll, das standardisierte Verbindungen zwischen KI-Anwendungen und externen Diensten, etwa Ihrer Dokumentation, ermöglicht. Mintlify generiert aus Ihrer Dokumentation und Ihren OpenAPI-Spezifikationen einen MCP-Server und bereitet Ihre Inhalte für das breitere KI-Ökosystem auf, in dem sich jeder MCP-Client (wie Claude, Cursor, Goose und andere) mit Ihrer Dokumentation und Ihren APIs verbinden kann.

Ihr MCP-Server stellt Tools bereit, mit denen KI-Anwendungen Ihre Dokumentation durchsuchen und mit Ihren APIs interagieren können.

<div id="accessing-your-mcp-server">
  ## Zugriff auf Ihren MCP-Server
</div>

<Note>
  MCP-Server können nur für öffentliche Dokumentation erstellt werden. Dokumentation, die hinter einer Endnutzer-Authentifizierung liegt, kann nicht für die Servererstellung genutzt werden.
</Note>

Mintlify erstellt automatisch einen MCP-Server für Ihre Dokumentation und hostet ihn unter Ihrer Dokumentations-URL mit dem Pfad `/mcp`. Der MCP-Server von Mintlify ist beispielsweise unter `https://mintlify.com/docs/mcp` verfügbar.

Sie können die URL Ihres MCP-Servers in Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) anzeigen und kopieren.

Der Pfad `/mcp` ist für gehostete MCP-Server reserviert und kann nicht für andere Navigationselemente verwendet werden.

<div id="configuring-your-mcp-server">
  ## Konfigurieren Ihres MCP-Servers
</div>

Alle MCP-Server enthalten standardmäßig das Tool `search`, mit dem Nutzer Informationen aus Ihrer Dokumentation in anderen Tools abfragen können.

Wenn Sie einen [Pro- oder Enterprise-Plan](https://mintlify.com/pricing?ref=mcp) haben, können Sie Endpunkte aus Ihrer OpenAPI-Spezifikation als MCP-Tools verfügbar machen.

Um Endpunkte als MCP-Tools bereitzustellen, verwenden Sie das Objekt `mcp` innerhalb der Erweiterung `x-mint` auf Datei- oder Endpunktebene. Der Mintlify-MCP-Server umfasst beispielsweise Tools zum Erstellen von Assistenten-Chats, zum Abrufen von Statusupdates und zum Auslösen von Aktualisierungen.

MCP-Server folgen einem Security-first-Ansatz, bei dem API-Endpunkte nicht standardmäßig offengelegt werden. Sie müssen Endpunkte explizit aktivieren, um sie als MCP-Tools bereitzustellen. Legen Sie nur Endpunkte offen, die für den öffentlichen Zugriff über KI-Tools sicher sind.

<ResponseField name="mcp" type="object">
  Die MCP-Konfiguration für den Endpunkt.

  <Expandable title="MCP">
    <ResponseField name="enabled" type="boolean">
      Ob der Endpunkt als MCP-Tool bereitgestellt werden soll. Hat Vorrang vor der Konfiguration auf Dateiebene.
    </ResponseField>

    <ResponseField name="name" type="string">
      Der Name des MCP-Tools.
    </ResponseField>

    <ResponseField name="description" type="string">
      Die Beschreibung des MCP-Tools.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="file-level-configuration">
  ### Konfiguration auf Dateiebene
</div>

Aktiviere MCP standardmäßig für alle Endpunkte in einer OpenAPI-Spezifikationsdatei und schließe einzelne Endpunkte gezielt aus:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": false // Deaktiviert MCP für diesen Endpunkt
          }
        },
        // ...
      }
    }
  }
}
```

<div id="endpoint-level-configuration">
  ### Konfiguration auf Endpunktebene
</div>

Aktiviere MCP für bestimmte Endpunkte:

```json
{
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": true,
            "name": "get-users",
            "description": "Get a list of users"
          },
          // ...
        }
      }
    },
    "/api/v1/delete": {
      "delete": {
        // Kein `x-mint: mcp`, daher wird dieser Endpunkt nicht als MCP-Tool exponiert
        // ...
      }
    }
  }
}
```

<div id="using-your-mcp-server">
  ## Verwendung Ihres MCP-Servers
</div>

Ihre Nutzer müssen Ihren MCP-Server mit ihren bevorzugten KI-Tools verbinden.

1. Stellen Sie die URL Ihres MCP-Servers öffentlich bereit.
2. Nutzer kopieren die URL Ihres MCP-Servers und fügen sie in ihre Tools ein.
3. Nutzer greifen über ihre Tools auf Ihre Dokumentation und API-Endpunkte zu.

So können Sie Ihren Nutzern helfen, sich mit Ihrem MCP-Server zu verbinden:

<Tabs>
  <Tab title="Kontextmenü">
    Fügen Sie Optionen im [Kontextmenü](/de/ai/contextual-menu) hinzu, damit Ihre Nutzerinnen und Nutzer von jeder Seite Ihrer Dokumentation aus eine Verbindung zu Ihrem MCP-Server herstellen können.

    | Option | Bezeichner | Beschreibung |
    | :----- | :--------- | :---------- |
    | **MCP-Server-URL kopieren** | `mcp` | Kopiert Ihre MCP-Server-URL in die Zwischenablage des Nutzers. |
    | **Mit Cursor verbinden** | `cursor` | Installiert Ihren MCP-Server in Cursor. |
    | **Mit VS Code verbinden** | `vscode` | Installiert Ihren MCP-Server in VS Code. |
  </Tab>

  <Tab title="Claude">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung, die Ihre MCP-Server-URL sowie die Schritte zur Verbindung mit Claude enthält.

        1. Öffnen Sie in den Claude-Einstellungen die Seite [Connectors](https://claude.ai/settings/connectors).
        2. Wählen Sie **Add custom connector**.
        3. Fügen Sie den Namen und die URL Ihres MCP-Servers hinzu.
        4. Wählen Sie **Add**.
        5. Wählen Sie in Claude die Schaltfläche für Anhänge (das Plus-Symbol).
        6. Wählen Sie Ihren MCP-Server aus.
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Model Context Protocol-Dokumentation](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung, die Ihre MCP-Server-URL und den Befehl zur Verbindung mit Claude Code enthält.

        ```bash
        claude mcp add --transport http <name> <url>
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Claude Code-Dokumentation](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung, die Ihre MCP-Server-URL sowie die Schritte zur Verbindung mit Cursor enthält.

        1. Verwenden Sie <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> (<kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> unter Windows), um die Befehlspalette zu öffnen.
        2. Suchen Sie nach „Open MCP settings“.
        3. Wählen Sie **Add custom MCP**. Dadurch wird die Datei `mcp.json` geöffnet.
        4. Konfigurieren Sie in `mcp.json` Ihren Server:

        ```json
        {
          "mcpServers": {
            "<your-mcp-server-name>": {
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Cursor-Dokumentation](https://docs.cursor.com/en/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung, die Ihre MCP-Server-URL sowie die Schritte zur Verbindung mit VS Code enthält.

        1. Erstellen Sie die Datei `.vscode/mcp.json`.
        2. Konfigurieren Sie in `mcp.json` Ihren Server:

        ```json
        {
          "servers": {
            "<your-mcp-server-name>": {
              "type": "http",
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [VS Code-Dokumentation](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>
</Tabs>

<div id="authentication">
  ## Authentifizierung
</div>

Wenn Sie einen API-Endpunkt für MCP aktivieren, übernimmt der Server die in Ihren OpenAPI-Definitionen `securitySchemes` und `securityRequirement` festgelegten Authentifizierungsanforderungen. Etwaige Schlüssel werden ausschließlich vom Tool gehandhabt und weder von Mintlify gespeichert noch verarbeitet.

Fordert ein Benutzer sein KI-Tool auf, einen geschützten Endpunkt aufzurufen, verlangt das Tool in diesem Moment die notwendigen Anmeldedaten vom Benutzer.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="MCP-Server zeigt nur das Suchtool">
    Wenn Ihr MCP-Server nur das Suchtool bereitstellt, obwohl eine OpenAPI-Spezifikation vorhanden ist:

    1. Überprüfen Sie, ob Ihre OpenAPI-Spezifikation gültig und erreichbar ist.
    2. Stellen Sie sicher, dass Sie MCP für bestimmte Endpunkte ausdrücklich mit `x-mint.mcp.enabled: true` aktiviert haben.
    3. Prüfen Sie Ihre Bereitstellungsprotokolle auf Fehler bei der OpenAPI-Verarbeitung.

    Wenn die OpenAPI-Verarbeitung fehlschlägt, läuft der Server zur Wahrung der Funktionsfähigkeit nur mit dem Suchtool weiter.
  </Accordion>

  <Accordion title="Probleme mit der Authentifizierung">
    Wenn Nutzer von Authentifizierungsproblemen berichten:

    1. Prüfen Sie, ob Ihre OpenAPI-Spezifikation korrekte Definitionen für `securitySchemes` enthält.
    2. Bestätigen Sie, dass aktivierte Endpunkte mit den angegebenen Authentifizierungsmethoden funktionieren.
  </Accordion>

  <Accordion title="Toolbeschreibungen fehlen oder sind unklar">
    Wenn KI-Tools Ihre API-Endpunkte nicht effektiv nutzen:

    1. Fügen Sie Ihren Endpunkten ausführliche `summary`- und `description`-Felder hinzu.
    2. Stellen Sie sicher, dass Parametername und -beschreibung selbsterklärend sind.
    3. Verwenden Sie das MCP-Dashboard, um zu prüfen, wie Ihre Endpunkte als Tools erscheinen.
  </Accordion>
</AccordionGroup>
