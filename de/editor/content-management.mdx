<div id="navigating-your-files">
  ## In deinen Dateien navigieren
</div>

Verwende den Dateiexplorer in der Seitenleiste, um deine Dokumentationsdateien zu durchsuchen. Klicke auf eine Datei, um sie im Editor zu öffnen.

Drücke <kbd>Command</kbd> + <kbd>P</kbd> (<kbd>Strg</kbd> + <kbd>P</kbd> unter Windows), um Dateien nach Namen zu suchen.

<div id="creating-new-pages">
  ## Neue Seiten erstellen
</div>

<Steps>
  <Step title="Neue Datei erstellen">
    Wähle in der Dateiexplorer-Seitenleiste das Symbol **Neue Datei erstellen** aus.
    
    <img
      src="/images/editor/files-menu-light.png"
      className="block dark:hidden rounded-2xl border border-gray-100 shadow-lg"
      style={{
        width: '268px',
        height: 'auto',
      }}
      alt="Dateimenü im Web-Editor"/>
    <img
      src="/images/editor/files-menu-dark.png"
      className="hidden dark:block rounded-2xl border border-white/10 shadow-lg"
      style={{
        width: '268px',
        height: 'auto',
      }}
      alt="Dateimenü im Web-Editor im Dark Mode"/>
  </Step>
  <Step title="Dateinamen eingeben">
    Dateinamen müssen auf `.mdx` enden. Zum Beispiel `api-reference.mdx` oder `getting-started.mdx`.
  </Step>
  <Step title="Erstellen und öffnen">
    Drücke <kbd>Enter</kbd>, um die Datei zu erstellen. Sie wird automatisch im Editor geöffnet und ist bereit für das Hinzufügen von Inhalten.
  </Step>
</Steps>

<div id="organizing-your-navigation">
  ## Navigation organisieren
</div>

Fügen Sie neue Seiten zur Navigation Ihrer Site hinzu, indem Sie die Datei `docs.json` bearbeiten.

<Steps>
  <Step title="docs.json öffnen">
    Suchen und öffnen Sie die Datei `docs.json` in Ihrem Root-Verzeichnis mit dem Dateiexplorer.
  </Step>
  <Step title="Navigationsabschnitt finden">
    Suchen Sie die Stelle in der Datei `docs.json`, an der Sie Ihre neue Seite hinzufügen möchten.
  </Step>
  <Step title="Seite hinzufügen">
    Fügen Sie den Dateinamen Ihrer Seite (ohne die `.mdx`-Erweiterung) im `pages`-Array der entsprechenden Gruppe ein.
  </Step>
</Steps>

<div id="example-adding-a-themes-page">
  ### Beispiel: Hinzufügen einer Themes-Seite
</div>

```json title="Hinzufügen einer Themes-Seite zur Gruppe Profile" highlight="18"
{
    "navigation": {
        "groups": [
            {
                "group": "Getting started",
                "pages": [
                    "index",
                    "quickstart", 
                    "installation"
                ]
            },
            {
                "group": "Profile",
                "pages": [
                    "settings",
                    "account-types", 
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```

<div id="editing-content">
  ## Inhalte bearbeiten
</div>

Nehmen Sie Änderungen an Ihren Seiten im visuellen Modus oder im Markdown-Modus im Editor vor.

Drücken Sie im visuellen Modus <kbd>/</kbd>, um das Komponentenmenü zu öffnen. Fügen Sie Inhaltsblöcke, Hinweise, Codeblöcke und andere Komponenten hinzu, um Ihre Dokumentation anzupassen.

Im Markdown-Modus können Sie das `MDX` Ihrer Seiten direkt bearbeiten. Das ist hilfreich, wenn Sie Folgendes benötigen:

- Bestimmte Eigenschaften von Komponenten festlegen
- Mit komplex verschachtelten Komponenten arbeiten
- `MDX`-Inhalte aus anderen Quellen kopieren und einfügen
