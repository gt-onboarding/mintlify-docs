Eines der Grundprinzipien der Softwareentwicklung ist DRY (Don't Repeat
Yourself), das auch für Dokumentation gilt. Wenn Sie feststellen, dass Sie denselben Inhalt an mehreren Stellen wiederholen,
sollten Sie ein benutzerdefiniertes Snippet erstellen, um Ihre Inhalte konsistent zu halten.

<div id="creating-a-custom-snippet">
  ## Erstellen eines benutzerdefinierten Snippets
</div>

**Voraussetzung**: Sie müssen Ihre Snippet-Datei im Verzeichnis `snippets` erstellen, damit der Import funktioniert.

Alle Seiten im Verzeichnis `snippets` werden als Snippets behandelt und nicht
als eigenständige Seiten gerendert. Wenn Sie aus einem Snippet eine eigenständige Seite
erstellen möchten, importieren Sie das Snippet in eine andere Datei und binden Sie es als
Komponente ein.

<div id="default-export">
  ### Standardexport
</div>

1. Fügen Sie Ihrem Snippet-Datei Inhalte hinzu, die Sie wiederverwenden möchten. Optional können Sie Variablen hinzufügen, die beim Import des Snippets über Props befüllt werden können.
   In diesem Beispiel heißt unsere Variable word.

```typescript snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages. 
```

2. Importieren Sie das Snippet in Ihre Zieldatei.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Überschrift

Lorem impsum dolor sit amet.

<MySnippet/>

```

<div id="exporting-with-variables">
  ### Exportieren mit Variablen
</div>

1. Optional können Sie Variablen hinzufügen, die beim Import des Snippets über Props befüllt werden können. In diesem Beispiel ist unsere Variable word.

```typescript snippets/my-snippet.mdx
My keyword of the day is {word}.
```

2. Importieren Sie das Snippet mit der Variable in Ihre Zieldatei. Die Prop wird entsprechend Ihrer Angabe befüllt.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Header

Lorem impsum dolor sit amet.

<MySnippet word="bananas" />

```

<div id="reusable-variables">
  ### Wiederverwendbare Variablen
</div>

1. Exportiere eine Variable aus deiner Snippet-Datei:

```typescript snippets/path/to/custom-variables.mdx
export const myName = 'my name';

export const myObject = { fruit: 'strawberries' };
```

2. Importiere das Snippet in deiner Zieldatei und verwende die Variable:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

Hallo, mein Name ist {myName} und ich mag {myObject.fruit}.
```

<div id="jsx-snippets">
  ### JSX-Snippets
</div>

1. Exportieren Sie eine JSX-Komponente aus Ihrer Snippet-Datei. (Siehe [React-Komponenten](/de/react-components) für weitere Informationen):

```js icon=square-js snippets/my-jsx-snippet.jsx
export const MyJSXSnippet = () => {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  )
}
```

<Note>
  Wichtig: Verwenden Sie beim Erstellen von JSX-Snippets die Arrow-Function-Syntax (`=>`) statt von Funktionsdeklarationen. Das Schlüsselwort `function` wird in diesem Kontext nicht unterstützt.
</Note>

2. Importieren Sie das Snippet in Ihrer Zieldatei und verwenden Sie die Komponente:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { MyJSXSnippet } from '/snippets/my-jsx-snippet.jsx';

<MyJSXSnippet />
```
