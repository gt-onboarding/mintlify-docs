<div id="installing-the-cli">
  ## Installation der CLI
</div>

<Info>
  **Voraussetzung**: Bitte installieren Sie [Node.js](https://nodejs.org/en), bevor Sie fortfahren.
</Info>

<Steps>
  <Step title="CLI installieren.">
    Führen Sie den folgenden Befehl aus, um die [CLI](https://www.npmjs.com/package/mint) zu installieren:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Lokal in der Vorschau ansehen.">
    Navigieren Sie zu Ihrem Docs-Verzeichnis (in dem sich Ihre Datei `docs.json` befindet) und führen Sie den folgenden Befehl aus:

    ```bash
    mint dev
    ```

    Eine lokale Vorschau Ihrer Dokumentation ist unter `http://localhost:3000` verfügbar.
  </Step>
</Steps>

Alternativ können Sie, wenn Sie die CLI nicht global installieren möchten, ein einmaliges Skript ausführen:

```bash
npx mint dev
```

<div id="updates">
  ## Updates
</div>

Wenn Ihre lokale Vorschau nicht mit der Produktionsversion im Web übereinstimmt, aktualisieren Sie Ihre lokale CLI:

```bash
mint update
```

Wenn der Befehl `mint update` in Ihrer lokalen Version nicht verfügbar ist, installieren Sie die CLI in der neuesten Version neu:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Benutzerdefinierte Ports
</div>

Standardmäßig verwendet die CLI Port 3000. Sie können den Port mit dem Flag `--port` anpassen. Um die CLI beispielsweise auf Port 3333 auszuführen, verwenden Sie folgenden Befehl:

```bash
mint dev --port 3333
```

Wenn Sie versuchen, einen bereits belegten Port zu verwenden, nimmt die CLI automatisch den nächsten verfügbaren Port:

```mdx
Port 3000 ist bereits belegt. Versuche stattdessen 3001.
```

<div id="previewing-as-a-specific-group">
  ## Vorschau als bestimmte Gruppe
</div>

Wenn Sie teilweise Authentifizierung verwenden, um den Zugriff auf Ihre Dokumentation einzuschränken, können Sie mit dem Flag `--group [groupname]` eine Vorschau als bestimmte Authentifizierungsgruppe anzeigen.

Wenn Sie beispielsweise eine Gruppe namens `admin` haben, können Sie mit dem folgenden Befehl eine Vorschau als Mitglied dieser Gruppe anzeigen:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Zusätzliche Befehle
</div>

Obwohl `mint dev` der am häufigsten verwendete Befehl ist, gibt es weitere Befehle, mit denen Sie Ihre Dokumentation verwalten können.

<div id="finding-broken-links">
  ### Defekte Links finden
</div>

Die CLI unterstützt Sie beim Validieren von Referenzlinks in Ihrer Dokumentation. Verwenden Sie den folgenden Befehl, um defekte Links zu ermitteln:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### OpenAPI-Spezifikation prüfen
</div>

Sie können die CLI verwenden, um Ihre OpenAPI-Datei mit dem folgenden Befehl auf Fehler zu prüfen:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Sie können entweder einen Dateinamen (z. B. `./openapi.yaml`) oder eine URL (z. B. `https://petstore3.swagger.io/api/v3/openapi.json`) angeben.

<div id="renaming-files">
  ### Dateien umbenennen
</div>

Sie können Dateien umbenennen und alle Verweise darauf mit dem folgenden Befehl aktualisieren:

```bash
mint rename <oldFilename> <newFilename>
```

<div id="migrating-mdx-endpoint-pages">
  ### Migrieren von MDX-Endpunktseiten
</div>

Sie können MDX-Endpunktseiten mithilfe des folgenden Befehls auf automatisch generierte Seiten aus Ihrer OpenAPI-Spezifikation migrieren:

```bash
mint migrate-mdx
```

Dieser Befehl konvertiert einzelne MDX-Endpunktseiten in automatisch generierte Seiten, die in Ihrer `docs.json` definiert sind, verschiebt MDX-Inhalte in die Erweiterung `x-mint` Ihrer OpenAPI-Spezifikation und aktualisiert Ihre Navigation. Weitere Informationen finden Sie unter [Migration von MDX](/de/api-playground/migrating-from-mdx).

<div id="formatting">
  ## Formatierung
</div>

Für die lokale Entwicklung empfehlen wir, Erweiterungen in Ihrer IDE zu verwenden, um `MDX`-Dateien zu erkennen und zu formatieren.

Wenn Sie Cursor, Windsurf oder VS Code verwenden, empfehlen wir die [MDX VS Code-Erweiterung](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) für Syntaxhervorhebung und [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) für die Codeformatierung.

Wenn Sie JetBrains verwenden, empfehlen wir das [MDX IntelliJ IDEA-Plugin](https://plugins.jetbrains.com/plugin/14944-mdx) für Syntaxhervorhebung sowie die Einrichtung von [Prettier](https://prettier.io/docs/webstorm) für die Codeformatierung.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="Fehler: Das Modul &#x22;sharp&#x22; konnte mit der darwin-arm64-Runtime nicht geladen werden">
    Dies kann auf eine veraltete Node-Version zurückzuführen sein. Versuchen Sie Folgendes:

    1. Entfernen Sie die aktuell installierte Version der Mint-CLI: `npm uninstall -g mint`
    2. Aktualisieren Sie Node.js.
    3. Installieren Sie die Mint-CLI erneut: `npm install -g mint`
  </Accordion>

  <Accordion title="Problem: Unbekannter Fehler">
    **Lösung**: Gehen Sie in Ihr Benutzerverzeichnis und löschen Sie den Ordner `~/.mintlify`. Führen Sie anschließend erneut `mint dev` aus.
  </Accordion>

  <Accordion title="Fehler: permission denied">
    Dies liegt daran, dass die erforderlichen Berechtigungen zum globalen Installieren von Node-Paketen fehlen.

    **Lösung**: Versuchen Sie, `sudo npm i -g mint` auszuführen. Sie werden nach Ihrem Passwort gefragt – dem, mit dem Sie Ihren Computer entsperren.
  </Accordion>

  <Accordion title="Die lokale Vorschau sieht nicht so aus wie meine Docs im Web">
    Dies liegt wahrscheinlich an einer veralteten CLI-Version.

    **Lösung:** Führen Sie `mint update` aus, um die neuesten Änderungen zu erhalten.
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    Wenn Sie Probleme mit dem CLI-Paket haben, führen Sie zuerst `npm ls -g` aus. Dieser Befehl zeigt, welche Pakete global auf Ihrem Rechner installiert sind.

    Wenn Sie npm nicht verwenden oder es nicht in der -g-Liste sehen, versuchen Sie `which mint`, um die Installation zu finden.

    Wenn sowohl ein Paket namens `mint` als auch eines namens `mintlify` installiert ist, sollten Sie `mintlify` deinstallieren.

    1. Deinstallieren Sie das alte Paket:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Leeren Sie den npm-Cache:

    ```bash
      npm cache clean --force
    ```

    3. Installieren Sie das neue Paket:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
