<div id="installing-the-cli">
  ## Installation der CLI
</div>

<Info>
  **Voraussetzung**: Bitte installiere [Node.js](https://nodejs.org/en), bevor du fortfährst.
</Info>

<Steps>
  <Step title="CLI installieren.">
    Führe den folgenden Befehl aus, um die [CLI](https://www.npmjs.com/package/mint) zu installieren:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Lokal prüfen.">
    Wechsle in dein Docs-Verzeichnis (wo sich deine `docs.json`-Datei befindet) und führe den folgenden Befehl aus:

    ```bash
    mint dev
    ```

    Eine lokale Vorschau deiner Dokumentation ist unter `http://localhost:3000` verfügbar.
  </Step>
</Steps>

Alternativ kannst du, wenn du die CLI nicht global installieren möchtest, ein einmaliges Skript ausführen:

```bash
npx mint dev
```

<div id="updates">
  ## Aktualisierungen
</div>

Wenn Ihre lokale Vorschau nicht mit der Produktionsversion im Web übereinstimmt, aktualisieren Sie Ihre lokale CLI:

```bash
mint update
```

Wenn der Befehl `mint update` in Ihrer lokalen Version nicht verfügbar ist, installieren Sie die CLI in der neuesten Version neu:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Benutzerdefinierte Ports
</div>

Standardmäßig verwendet die CLI Port 3000. Sie können den Port mit dem Flag `--port` anpassen. Um die CLI beispielsweise auf Port 3333 auszuführen, verwenden Sie diesen Befehl:

```bash
mint dev --port 3333
```

Wenn Sie versuchen, einen bereits belegten Port zu verwenden, wird der nächste verfügbare Port genutzt:

```mdx
Port 3000 ist bereits belegt. Versuche stattdessen 3001.
```

<div id="previewing-as-a-specific-group">
  ## Als bestimmte Gruppe in der Vorschau
</div>

Wenn Sie die Partielle Authentifizierung verwenden, um den Zugriff auf Ihre Dokumentation einzuschränken, können Sie mit dem Flag `--group [groupname]` als eine bestimmte Authentifizierungsgruppe eine Vorschau anzeigen.

Wenn Sie beispielsweise eine Gruppe namens `admin` haben, können Sie als Mitglied dieser Gruppe mit dem folgenden Befehl eine Vorschau anzeigen:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Zusätzliche Befehle
</div>

Auch wenn `mint dev` der am häufigsten verwendete Befehl ist, gibt es weitere Befehle, mit denen Sie Ihre Dokumentation verwalten können.

<div id="finding-broken-links">
  ### Kaputte Links finden
</div>

Die CLI unterstützt Sie beim Validieren von Referenzlinks in Ihrer Dokumentation. Um kaputte Links zu erkennen, verwenden Sie den folgenden Befehl:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### OpenAPI-Spezifikation prüfen
</div>

Sie können die CLI verwenden, um Ihre OpenAPI-Datei mit dem folgenden Befehl auf Fehler zu prüfen:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Sie können einen Dateinamen (z. B. `./openapi.yaml`) oder eine URL (z. B. `https://petstore3.swagger.io/api/v3/openapi.json`) übergeben.

<div id="renaming-files">
  ### Dateien umbenennen
</div>

Sie können Dateien umbenennen und alle Verweise darauf mit dem folgenden Befehl aktualisieren:

```bash
mint rename <oldFilename> <newFilename>
```

<div id="migrating-mdx-endpoint-pages">
  ### Migrieren von MDX-Endpunktseiten
</div>

Sie können MDX-Endpunktseiten mit dem folgenden Befehl aus Ihrer OpenAPI-Spezifikation in automatisch generierte Seiten überführen:

```bash
mint migrate-mdx
```

Dieser Befehl wandelt einzelne MDX-Endpunktseiten in automatisch generierte, in Ihrer `docs.json` definierte Seiten um, verschiebt MDX-Inhalte in die `x-mint`-Erweiterung Ihrer OpenAPI-Spezifikation und aktualisiert Ihre Navigation. Weitere Informationen finden Sie unter [Migration von MDX](/de/api-playground/migrating-from-mdx).

<div id="formatting">
  ## Formatierung
</div>

Für die lokale Entwicklung empfehlen wir Erweiterungen in Ihrer IDE, um `MDX`-Dateien zu erkennen und zu formatieren.

Wenn Sie Cursor, Windsurf oder VS Code verwenden, empfehlen wir die [MDX VS Code-Erweiterung](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) für Syntaxhervorhebung sowie [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) für die Codeformatierung.

Wenn Sie JetBrains verwenden, empfehlen wir das [MDX IntelliJ-IDEA-Plugin](https://plugins.jetbrains.com/plugin/14944-mdx) für Syntaxhervorhebung und die Einrichtung von [Prettier](https://prettier.io/docs/webstorm) für die Codeformatierung.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="Fehler: Das Modul &#x22;sharp&#x22; konnte mit der darwin-arm64-Runtime nicht geladen werden">
    Dies kann an einer veralteten Version von Node liegen. Probiere Folgendes:

    1. Entferne die aktuell installierte Version der mint-CLI: `npm uninstall -g mint`
    2. Aktualisiere auf die neueste Version von Node.js.
    3. Installiere die mint-CLI erneut: `npm install -g mint`
  </Accordion>

  <Accordion title="Problem: Unbekannter Fehler tritt auf">
    **Lösung**: Wechsle ins Stammverzeichnis deines Geräts und lösche den Ordner `~/.mintlify`. Führe anschließend `mint dev` erneut aus.
  </Accordion>

  <Accordion title="Fehler: permission denied">
    Das liegt daran, dass die erforderlichen Berechtigungen für die globale Installation von Node-Paketen fehlen.

    **Lösung**: Führe `sudo npm i -g mint` aus. Du wirst nach deinem Passwort gefragt – das ist dasselbe, mit dem du deinen Computer entsperrst.
  </Accordion>

  <Accordion title="Die lokale Vorschau sieht nicht genauso aus wie meine Docs im Web">
    Das liegt wahrscheinlich an einer veralteten Version der CLI.

    **Lösung:** Führe `mint update` aus, um die neuesten Änderungen zu erhalten.
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    Wenn du Probleme mit dem CLI-Paket hast, führe zuerst `npm ls -g` aus. Dieser Befehl zeigt, welche Pakete auf deinem Rechner global installiert sind.

    Wenn du npm nicht verwendest oder es nicht in der -g-Liste siehst, probiere `which mint`, um die Installation zu finden.

    Wenn sowohl ein Paket namens `mint` als auch eines namens `mintlify` installiert ist, solltest du `mintlify` deinstallieren.

    1. Deinstalliere das alte Paket:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Leere deinen npm-Cache:

    ```bash
      npm cache clean --force
    ```

    3. Installiere das neue Paket:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
