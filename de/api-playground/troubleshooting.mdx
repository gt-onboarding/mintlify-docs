Wenn Ihre API-Seiten nicht richtig angezeigt werden, überprüfen Sie diese häufigen Konfigurationsfehler:

<AccordionGroup>
  <Accordion title="All of my OpenAPI pages are completely blank">
    In diesem Szenario kann Mintlify entweder Ihr OpenAPI-Dokument nicht finden, oder Ihr OpenAPI-Dokument ist ungültig.

    Das lokale Ausführen von `mint dev` sollte einige dieser Probleme aufdecken.

    So überprüfen Sie, ob Ihr OpenAPI-Dokument die Validierung besteht:

    1. Besuchen Sie [diesen Validator](https://editor.swagger.io/).
    2. Wechseln Sie zur Registerkarte „Validate text“.
    3. Fügen Sie Ihr OpenAPI-Dokument ein.
    4. Klicken Sie auf „Validate it!“.

    Wenn das unten erscheinende Textfeld einen grünen Rand hat, wurde Ihr Dokument erfolgreich validiert. Dies ist genau das Validierungspaket, das Mintlify zur Validierung von OpenAPI-Dokumenten verwendet. Besteht Ihr Dokument hier die Validierung, ist es sehr wahrscheinlich, dass das Problem an anderer Stelle liegt.

    Außerdem unterstützt Mintlify OpenAPI 2.0 nicht. Wenn Ihr Dokument diese Version der Spezifikation verwendet, kann dieses Problem auftreten. Sie können Ihr Dokument auf [editor.swagger.io](https://editor.swagger.io/) konvertieren (unter Edit > Convert to OpenAPI 3):

    </Frame>
  </Accordion>

  <Accordion title="One of my OpenAPI pages is completely blank">
    Dies wird normalerweise durch ein falsch geschriebenes `openapi`-Feld in den Seitenmetadaten verursacht. Stellen Sie sicher, dass HTTP-Methode und Pfad exakt mit der HTTP-Methode und dem Pfad im OpenAPI-Dokument übereinstimmen.

    Hier ist ein Beispiel, wie etwas schiefgehen kann:

    ```mdx get-user.mdx
    ---
    openapi: "GET /users/{id}/"
    ---
    ```

    ```yaml openapi.yaml
    paths:
      "/users/{id}":
        get: ...
    ```

    Beachten Sie, dass der Pfad im `openapi`-Feld einen nachgestellten Schrägstrich hat, während der Pfad im OpenAPI-Dokument keinen hat.

    Ein weiteres häufiges Problem ist ein falsch geschriebener Dateiname. Wenn Sie ein bestimmtes OpenAPI-Dokument im `openapi`-Feld angeben, stellen Sie sicher, dass der Dateiname korrekt ist. Wenn Sie beispielsweise zwei OpenAPI-Dokumente `openapi/v1.json` und `openapi/v2.json` haben, könnten Ihre Metadaten folgendermaßen aussehen:

    ```mdx api-reference/v1/users/get-user.mdx
    ---
    openapi: "v1 GET /users/{id}"
    ---
    ```
  </Accordion>

  <Accordion title="Requests from the API Playground don't work">
    Wenn Sie eine benutzerdefinierte Domain konfiguriert haben, könnte das ein Problem mit Ihrem Reverse-Proxy sein. Standardmäßig beginnen Anfragen, die über die API-Spielwiese gestellt werden, mit einer `POST`-Anfrage an den Pfad `/api/request` auf der Docs-Website. Wenn Ihr Reverse-Proxy so konfiguriert ist, dass er nur `GET`-Anfragen zulässt, schlagen all diese Anfragen fehl. Um das zu beheben, konfigurieren Sie Ihren Reverse-Proxy so, dass er `POST`-Anfragen an den Pfad `/api/request` zulässt.

    Alternativ können Sie, wenn Ihr Reverse-Proxy das Akzeptieren von `POST`-Anfragen verhindert, Mintlify so konfigurieren, dass Anfragen mit der Einstellung `api.playground.proxy` in der `docs.json` direkt an Ihr Backend gesendet werden, wie in der [Einstellungsdokumentation](/de/settings#param-proxy) beschrieben. Bei Verwendung dieser Konfiguration müssen Sie CORS auf Ihrem Server einrichten, da Anfragen direkt aus den Browsern der Nutzer und nicht über Ihren Proxy kommen.
  </Accordion>

  <Accordion title="OpenAPI navigation entries are not generating pages">
    Wenn Sie eine OpenAPI-Navigationskonfiguration verwenden, die Seiten aber nicht generiert werden, prüfen Sie diese häufigen Ursachen:

    1. **Fehlende Standard-OpenAPI-Spezifikation**: Stellen Sie sicher, dass für das Navigationselement ein `openapi`-Feld gesetzt ist:

    ```json {5}
    "navigation": {
      "groups": [
        {
          "group": "API reference",
          "openapi": "/path/to/openapi.json",
          "pages": [
            "GET /users",
            "POST /users"
          ]
        }
      ]
    }
    ```

    2. **Vererbung der OpenAPI-Spezifikation**: Wenn Sie eine verschachtelte Navigation verwenden, stellen Sie sicher, dass untergeordnete Gruppen die richtige OpenAPI-Spezifikation erben oder eine eigene angeben.

    3. **Validierungsprobleme**: Verwenden Sie `mint openapi-check <path-to-openapi-file>`, um zu prüfen, ob Ihr OpenAPI-Dokument gültig ist.
  </Accordion>

  <Accordion title="Some OpenAPI operations appear in navigation but others don't">
    1. **Ausgeblendete Operationen**: Operationen, die in Ihrer OpenAPI-Spezifikation mit `x-hidden: true` markiert sind, erscheinen nicht in der automatisch generierten Navigation.
    2. **Ungültige Operationen**: Operationen mit Validierungsfehlern in der OpenAPI-Spezifikation können übersprungen werden. Prüfen Sie Ihr OpenAPI-Dokument auf Syntaxfehler.
    3. **Manuelle vs. automatische Aufnahme**: Wenn Sie Endpunkte aus einer OpenAPI-Spezifikation referenzieren, erscheinen in der Navigation nur die explizit referenzierten Operationen. Es werden keine anderen Seiten automatisch hinzugefügt. Dies schließt auch Operationen ein, die in untergeordneten Navigationselementen referenziert werden.
  </Accordion>

  <Accordion title="Mixed navigation (OpenAPI and MDX pages) not working correctly">
    Beim Kombinieren von OpenAPI-Operationen mit regulären Dokumentationsseiten in der Navigation:

    1. **Dateikonflikte**: Sie können nicht sowohl eine `MDX`-Datei als auch einen Navigationseintrag für dieselbe Operation haben. Wenn Sie beispielsweise `get-users.mdx` haben, fügen Sie nicht zusätzlich „GET /users“ in Ihrer Navigation hinzu. Wenn Sie eine Datei benötigen, die denselben Namen wie eine Operation trägt, verwenden Sie die Erweiterung `x-mint` für den Endpunkt, damit das href auf einen anderen Speicherort verweist.
    2. **Pfadauflösung**: Navigationseinträge, die nicht mit OpenAPI-Operationen übereinstimmen, werden als Dateipfade behandelt. Stellen Sie sicher, dass Ihre `MDX`-Dateien an den erwarteten Speicherorten vorhanden sind.
    3. **Groß-/Kleinschreibung**: Das Abgleichen von OpenAPI-Operationen ist groß-/kleinschreibungssensitiv. Stellen Sie sicher, dass HTTP-Methoden in Navigationseinträgen in Großbuchstaben angegeben sind.
  </Accordion>
</AccordionGroup>
