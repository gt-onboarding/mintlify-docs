title: "Komplexe Datentypen"
description: "Beschreiben Sie APIs mit flexiblen Schemata, optionalen Eigenschaften und mehreren Datenformaten mithilfe der Schlüsselwörter `oneOf`, `anyOf` und `allOf`."
---

Wenn Ihre API mehrere Datenformate akzeptiert, bedingte Felder enthält oder Vererbungsmuster verwendet, helfen die Schema-Kompositionsschlüsselwörter von OpenAPI dabei, diese flexiblen Strukturen zu dokumentieren. Mit `oneOf`, `anyOf` und `allOf` können Sie APIs beschreiben, die unterschiedliche Eingabetypen verarbeiten oder mehrere Schemata zu umfassenden Datenmodellen kombinieren.

<div id="oneof-anyof-allof-keywords">
  ## Schlüsselwörter `oneOf`, `anyOf`, `allOf`
</div>

Für komplexe Datentypen stellt OpenAPI Schlüsselwörter zum Kombinieren von Schemas bereit:

- `allOf`: Kombiniert mehrere Schemas (ähnlich dem Zusammenführen von Objekten oder dem Erweitern eines Basisschemas). Entspricht einem `and`-Operator. 
- `anyOf`: Akzeptiert Daten, die einem der angegebenen Schemas entsprechen. Entspricht einem `or`-Operator.
- `oneOf`: Akzeptiert Daten, die genau einem der angegebenen Schemas entsprechen. Entspricht einem `exclusive-or`-Operator.

<Warning>Mintlify behandelt `oneOf` und `anyOf` identisch, da der praktische Unterschied die Nutzung der API nur selten beeinflusst.</Warning>

Ausführliche Spezifikationen dieser Schlüsselwörter finden Sie in der [OpenAPI-Dokumentation](https://swagger.io/docs/specification/data-models/oneof-anyof-allof-not/).

<Info>Das Schlüsselwort `not` wird derzeit nicht unterstützt.</Info>

<div id="combining-schemas-with-allof">
  ### Schemas mit `allOf` kombinieren
</div>

Wenn Sie `allOf` verwenden, führt Mintlify eine Vorverarbeitung Ihres OpenAPI-Dokuments durch, um komplexe Kombinationen lesbar darzustellen. Kombinieren Sie beispielsweise zwei Objekt-Schemas mit `allOf`, fasst Mintlify die Eigenschaften beider zu einem einzigen Objekt zusammen. Das ist besonders nützlich, wenn Sie die wiederverwendbaren [Components](https://swagger.io/docs/specification/components/) von OpenAPI nutzen.

```yaml
org_with_users:
  allOf:
    - $ref: '#/components/schemas/Org'
    - type: object
      properties:
        users:
          type: array
          description: Ein Array mit allen Benutzern der Organisation
# ...
components:
  schemas:
    Org:
      type: object
      properties:
        id:
          type: string
          description: Die ID der Organisation
```

    </ParamField>

    <ParamField body="users" type="object[]">
      Ein Array, das alle Nutzer in der Organisation enthält

<div id="providing-options-with-oneof-and-anyof">
  ### Optionen mit `oneOf` und `anyOf` bereitstellen
</div>

Wenn Sie `oneOf` oder `anyOf` verwenden, werden die Optionen in einem Tab-Container angezeigt. Geben Sie in jedem Teilschema ein `title`-Feld an, um den Optionen Namen zuzuweisen. So könnten Sie beispielsweise zwei verschiedene Arten von Lieferadressen darstellen:

```yaml
delivery_address:
  oneOf:
    - title: StreetAddress
      type: object
      properties:
        address_line_1:
          type: string
          description: Die Straßenadresse des Empfängers
        # ...
    - title: POBox
      type: object
      properties:
        box_number:
          type: string
          description: Die Postfachnummer
        # ...
```

        </ParamField>
      </Tab>
    </Tabs>
  </div>
</ParamField>
