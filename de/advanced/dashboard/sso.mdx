---
title: "Single Sign-on (SSO)"
description: "Passen Sie an, wie sich Ihr Team am Admin-Dashboard anmelden kann"
---

<Info>
  Die SSO-Funktion ist im [Enterprise-Tarif](https://mintlify.com/pricing?ref=sso) verfügbar.
</Info>

Nutzen Sie Single Sign-on für Ihr Dashboard via SAML und OIDC. Wenn Sie Okta, Google Workspace oder Microsoft Entra verwenden, finden Sie bei uns anbieterspezifische Anleitungen zur Einrichtung von SSO. Wenn Sie einen anderen Anbieter nutzen, [kontaktieren Sie uns](mailto:support@mintlify.com) bitte.

<div id="okta">
  ## Okta
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                Klicken Sie unter `Applications`, um eine neue App-Integration mit SAML 2.0 zu erstellen.
            </Step>
            <Step title="Integration konfigurieren">
                Geben Sie Folgendes ein:
                * Single-Sign-On-URL (bereitgestellt von Mintlify)
                * Audience-URI (bereitgestellt von Mintlify)
                * NameID-Format: `EmailAddress`
                * Attributzuweisungen:
                    | Name | Namensformat | Wert
                    | ---- | ------------ | -----
                    | `firstName` | Basic | `user.firstName` |
                    | `lastName` | Basic | `user.lastName` |
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „Sign-On“ und senden Sie uns die Metadata-URL.
                Wir aktivieren die Verbindung auf unserer Seite anhand dieser Informationen.
            </Step>
        </Steps>
    </Tab>
    <Tab title="OIDC">
        <Steps>
            <Step title="Anwendung erstellen">
                Klicken Sie unter `Applications`, um eine neue App-Integration mit OIDC zu erstellen.
                Wählen Sie den Anwendungstyp `Web Application`.
            </Step>
            <Step title="Integration konfigurieren">
                Wählen Sie den Authorization Code Grant Type und geben Sie die von Mintlify bereitgestellte Redirect-URI ein.
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „General“ und ermitteln Sie die Client ID und das Client Secret.
                Bitte übermitteln Sie uns diese zusammen mit der URL Ihrer Okta-Instanz (z. B. `<your-tenant-name>.okta.com`) auf sicherem Weg. Sie können diese über einen Dienst wie 1Password oder SendSafely senden.
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="google-workspace">
  ## Google Workspace
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                Unter `Web and mobile apps` die Option `Add custom SAML app` aus dem Dropdown `Add app` auswählen.  
                <Frame>
                    ![](/images/gsuite-add-custom-saml-app.png)
                </Frame>              
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Kopieren Sie die angegebene SSO-URL, die Entity ID und das x509-Zertifikat und senden Sie diese an das Mintlify-Team.
                <Frame>
                    ![](/images/gsuite-saml-metadata.png)                    
                </Frame>
            </Step>
            <Step title="Integration konfigurieren">
                Geben Sie auf der Seite „Service provider details“ Folgendes ein:
                * ACS URL (von Mintlify bereitgestellt)
                * Entity ID (von Mintlify bereitgestellt)
                * Name ID format: `EMAIL`
                * Name ID: `Basic Information > Primary email`

                <Frame>
                    ![](/images/gsuite-sp-details.png)
                </Frame>

                Geben Sie auf der nächsten Seite die folgenden Attributzuweisungen ein:
                | Google Directory Attribute | App-Attribut |
                | -------------------------- | ------------- |
                | `First name` | `firstName`  |
                | `Last name` | `lastName` |

                Sobald dieser Schritt abgeschlossen ist und Nutzer der Anwendung zugewiesen wurden, informieren Sie unser Team, und wir aktivieren SSO für Ihr Konto!
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="microsoft-entra">
  ## Microsoft Entra
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                1. Unter „Enterprise applications“ **New application** auswählen.
                2. **Create your own application** auswählen und „Integrate any other application you don't find in the gallery (Non-gallery).“ wählen.
            </Step>
            <Step title="SAML konfigurieren">
                Navigieren Sie zur Single Sign-On-Einrichtungsseite und wählen Sie **SAML**. Unter „Basic SAML Configuration“ geben Sie Folgendes ein:
                * Identifier (Entity ID): Die von Mintlify bereitgestellte Audience-URI.
                * Reply URL (Assertion Consumer Service URL): Die von Mintlify bereitgestellte ACS-URL.
                
                Lassen Sie die übrigen Felder leer und wählen Sie **Save**.
            </Step>
            <Step title="Attribute & Claims konfigurieren">
                Bearbeiten Sie den Abschnitt „Attributes & Claims“:
                1. Wählen Sie **Unique User Identifier (Name ID)** unter „Required claim“.
                2. Ändern Sie das Source-Attribut auf `user.primaryauthoritativeemail`.
                3. Legen Sie unter „Additional claims“ die folgenden Claims an:
                    | Name | Value |
                    | ---- | ----- |
                    | `firstName` | `user.givenname` |
                    | `lastName` | `user.surname` |
            </Step>
            <Step title="IdP-Informationen an Mintlify senden">
                Sobald die Anwendung eingerichtet ist, navigieren Sie zum Abschnitt „SAML Certificates“ und senden Sie uns die App Federation Metadata URL.
                Wir aktivieren die Verbindung unsererseits anhand dieser Informationen.
            </Step>
            <Step title="Benutzer zuweisen">
                Navigieren Sie zu „Users and groups“ in Ihrer Entra-Anwendung und fügen Sie die Benutzer hinzu, die Zugriff auf Ihr Dashboard erhalten sollen.
            </Step>
        </Steps>
    </Tab>
</Tabs>
