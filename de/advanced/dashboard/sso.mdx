---
title: "Single Sign-on (SSO)"
description: "Passen Sie an, wie sich Ihr Team beim Admin-Dashboard anmelden kann"
---

<Info>
  Die SSO-Funktion ist im [Enterprise-Tarif](https://mintlify.com/pricing?ref=sso) verfügbar.
</Info>

Verwenden Sie Single Sign-on für Ihr Dashboard via SAML und OIDC. Wenn Sie Okta, Google Workspace oder Microsoft Entra nutzen, finden Sie bei uns anbieterspezifische Anleitungen zur Einrichtung von SSO. Wenn Sie einen anderen Anbieter nutzen, [kontaktieren Sie uns](mailto:support@mintlify.com) bitte.

<div id="okta">
  ## Okta
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                Unter `Applications` klicken Sie, um eine neue App-Integration mit SAML 2.0 zu erstellen.
            </Step>
            <Step title="Integration konfigurieren">
                Geben Sie Folgendes ein:
                * Single-Sign-On-URL (bereitgestellt von Mintlify)
                * Audience-URI (bereitgestellt von Mintlify)
                * Name-ID-Format: `EmailAddress`
                * Attributaussagen:
                    | Name | Namensformat | Wert
                    | ---- | ----------- | -----
                    | `firstName` | Basic | `user.firstName` |
                    | `lastName` | Basic | `user.lastName` |
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „Sign-On“ und senden Sie uns die Metadaten-URL.
                Wir aktivieren die Verbindung auf unserer Seite anhand dieser Informationen.
            </Step>
        </Steps>
    </Tab>
    <Tab title="OIDC">
        <Steps>
            <Step title="Anwendung erstellen">
                Unter `Applications` klicken Sie, um eine neue App-Integration mit OIDC zu erstellen.
                Wählen Sie den Anwendungstyp `Web Application`.
            </Step>
            <Step title="Integration konfigurieren">
                Wählen Sie den Authorization-Code-Grant-Typ und geben Sie die von Mintlify bereitgestellte Redirect-URI ein.
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „General“ und ermitteln Sie die Client-ID und das Client-Secret.
                Bitte stellen Sie uns diese sowie die URL Ihrer Okta-Instanz (z. B. `<your-tenant-name>.okta.com`) auf sicherem Weg zur Verfügung. Sie können diese über einen Dienst wie 1Password oder SendSafely senden.
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="google-workspace">
  ## Google Workspace
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                Unter `Web and mobile apps` wählen Sie im Dropdown `Add app` die Option `Add custom SAML app`.  
                <Frame>
                    ![](/images/gsuite-add-custom-saml-app.png)
                </Frame>              
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Kopieren Sie die bereitgestellte SSO-URL, die Entity ID und das x509-Zertifikat und senden Sie diese an das Mintlify-Team.
                <Frame>
                    ![](/images/gsuite-saml-metadata.png)                    
                </Frame>
            </Step>
            <Step title="Integration konfigurieren">
                Geben Sie auf der Seite Service provider details Folgendes ein:
                * ACS-URL (bereitgestellt von Mintlify)
                * Entity ID (bereitgestellt von Mintlify)
                * Name ID format: `EMAIL`
                * Name ID: `Basic Information > Primary email`

                <Frame>
                    ![](/images/gsuite-sp-details.png)
                </Frame>

                Geben Sie auf der nächsten Seite die folgenden Attributzuweisungen ein:
                | Google Directory Attribute | App Attribute |
                | -------------------------- | ------------- |
                | `First name` | `firstName`  |
                | `Last name` | `lastName` |

                Sobald dieser Schritt abgeschlossen ist und Nutzende der Anwendung zugewiesen sind, geben Sie unserem Team Bescheid, und wir aktivieren SSO für Ihr Konto!
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="microsoft-entra">
  ## Microsoft Entra
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                1. Unter „Enterprise applications“ **New application** auswählen.
                2. **Create your own application** auswählen und „Integrate any other application you don't find in the gallery (Non-gallery)“ wählen.
            </Step>
            <Step title="SAML konfigurieren">
                Navigieren Sie zur Single Sign-On-Einrichtungsseite und wählen Sie **SAML**. Unter „Basic SAML Configuration“ geben Sie Folgendes ein:
                * Identifier (Entity ID): Die von Mintlify bereitgestellte Audience-URI.
                * Reply URL (Assertion Consumer Service URL): Die von Mintlify bereitgestellte ACS-URL.
                
                Lassen Sie die übrigen Felder leer und wählen Sie **Save**.
            </Step>
            <Step title="Attribute und Claims konfigurieren">
                Bearbeiten Sie den Abschnitt „Attributes & Claims“:
                1. Wählen Sie unter „Required claim“ **Unique User Identifier (Name ID)**.
                2. Ändern Sie das Quellattribut auf `user.primaryauthoritativeemail`.
                3. Erstellen Sie unter „Additional claims“ die folgenden Claims:
                    | Name | Wert |
                    | ---- | ----- |
                    | `firstName` | `user.givenname` |
                    | `lastName` | `user.surname` |
            </Step>
            <Step title="IdP-Informationen an Mintlify senden">
                Sobald die Anwendung eingerichtet ist, navigieren Sie zum Abschnitt „SAML Certificates“ und senden Sie uns die App Federation Metadata URL.
                Wir aktivieren die Verbindung auf unserer Seite anhand dieser Informationen.
            </Step>
            <Step title="Benutzer zuweisen">
                Navigieren Sie zu „Users and groups“ in Ihrer Entra-Anwendung und fügen Sie die Benutzer hinzu, die Zugriff auf Ihr Dashboard erhalten sollen.
            </Step>
        </Steps>
    </Tab>
</Tabs>
