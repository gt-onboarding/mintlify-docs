<div id="verceljson-file">
  ## vercel.json-Datei
</div>

Die `vercel.json`-Datei ist Vercels Konfigurationsdatei, mit der Sie steuern können, wie Ihr Projekt erstellt und bereitgestellt wird. Sie liegt im Stammverzeichnis Ihres Projekts und kontrolliert verschiedene Aspekte Ihrer Bereitstellung, einschließlich Routing, Weiterleitungen, Header und Build-Einstellungen.

Wir verwenden die `rewrites`-Konfiguration, um Anfragen von Ihrer Hauptdomain zu Ihrer Dokumentation zu proxien.

Rewrites ermöglichen es, eingehende Anfragen auf andere Ziele abzubilden, ohne die URL im Browser zu ändern. Wenn jemand `yoursite.com/docs` besucht, ruft Vercel intern Inhalte von `your-subdomain.mintlify.dev/docs` ab, während der Nutzer weiterhin `yoursite.com/docs` im Browser sieht. Das unterscheidet sich von Weiterleitungen, die Nutzer vollständig auf eine andere URL schicken würden.

<div id="configuration">
  ## Konfiguration
</div>

Um Ihre Dokumentation unter einem benutzerdefinierten Unterpfad `/docs` mit Vercel zu hosten, fügen Sie die folgende Konfiguration zu Ihrer Datei `vercel.json` hinzu:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

- **`source`**: Das Pfadmuster auf Ihrer Domain, das die Umschreibung auslöst.
- **`destination`**: Das Ziel, an das die Anfrage per Proxy weitergeleitet wird.
- **`:match*`**: Ein Platzhalter, der alle Pfadsegmente nach `/docs/` erfasst.

Weitere Informationen finden Sie unter [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) in der Vercel-Dokumentation.

<div id="using-external-proxies-with-vercel">
  ## Verwendung externer Proxys mit Vercel
</div>

Wenn Sie einen externen Proxy (wie Cloudflare oder AWS CloudFront) vor Ihrer Vercel-Deployment verwenden, müssen Sie ihn korrekt konfigurieren, um Konflikte mit der Domainverifizierung von Vercel und der Ausstellung von SSL-Zertifikaten zu vermeiden.

Eine fehlerhafte Proxy-Konfiguration kann verhindern, dass Vercel SSL-Zertifikate von Let’s Encrypt ausstellt, und zu Fehlern bei der Domainverifizierung führen.

Siehe die [unterstützten Anbieter](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) in der Vercel-Dokumentation.

<div id="required-path-allowlist">
  ### Erforderliche Pfad-Positivliste
</div>

Ihr externer Proxy muss den Datenverkehr zu diesen spezifischen Pfaden zulassen, ohne zu blockieren, umzuleiten oder aggressiv zu cachen:

- `/.well-known/acme-challenge/*` – erforderlich für die Let's-Encrypt-Zertifikatsprüfung
- `/.well-known/vercel/*` – erforderlich für die Vercel-Domainverifizierung

Diese Pfade sollten unverändert direkt an Ihre Vercel-Bereitstellung durchgereicht werden.

<div id="header-forwarding-requirements">
  ### Anforderungen an das Weiterleiten von Headern
</div>

Stellen Sie sicher, dass Ihr Proxy den `HOST`-Header korrekt weiterleitet. Ohne korrektes Weiterleiten des Headers schlagen Verifizierungsanfragen fehl.

<div id="testing-your-proxy-setup">
  ### Testen Ihrer Proxy-Konfiguration
</div>

So überprüfen Sie, ob Ihr Proxy korrekt konfiguriert ist:

1. Prüfen Sie, ob `https://[yourdomain].com/.well-known/vercel/` eine Antwort zurückgibt.
2. Stellen Sie sicher, dass SSL-Zertifikate in Ihrem Vercel-Dashboard korrekt ausgestellt werden.
3. Vergewissern Sie sich, dass die Domainverifizierung erfolgreich abgeschlossen wird.
