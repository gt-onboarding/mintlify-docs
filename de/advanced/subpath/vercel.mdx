<div id="verceljson-file">
  ## vercel.json-Datei
</div>

Die Datei `vercel.json` ist die Konfigurationsdatei von Vercel, mit der Sie anpassen können, wie Ihr Projekt gebaut und bereitgestellt wird. Sie liegt im Stammverzeichnis Ihres Projekts und steuert verschiedene Aspekte der Bereitstellung, darunter Routing, Redirects, Header und Build-Einstellungen.

Wir verwenden die `rewrites`-Konfiguration, um Anfragen von Ihrer Hauptdomain an Ihre Dokumentation durch einen Proxy weiterzuleiten.

Rewrites ermöglichen es, eingehende Anfragen auf andere Ziele abzubilden, ohne die URL im Browser zu ändern. Wenn jemand `yoursite.com/docs` besucht, ruft Vercel intern Inhalte von `your-subdomain.mintlify.dev/docs` ab, im Browser des Nutzers bleibt jedoch `yoursite.com/docs` sichtbar. Das unterscheidet sich von Redirects, die Nutzer vollständig auf eine andere URL umleiten würden.

<div id="configuration">
  ## Konfiguration
</div>

Um Ihre Dokumentation unter einem benutzerdefinierten Unterpfad `/docs` mit Vercel zu hosten, fügen Sie die folgende Konfiguration zu Ihrer Datei `vercel.json` hinzu:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

- **`source`**: Das Pfadmuster auf Ihrer Domain, das das Rewrite auslöst.
- **`destination`**: Das Ziel, an das die Anfrage weitergeleitet wird.
- **`:match*`**: Ein Platzhalter, der alle Pfadsegmente nach `/docs/` erfasst.

Weitere Informationen finden Sie unter [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) in der Vercel-Dokumentation.

<div id="using-external-proxies-with-vercel">
  ## Verwendung externer Proxys mit Vercel
</div>

Wenn Sie einen externen Proxy (z. B. Cloudflare oder AWS CloudFront) vor Ihrer Vercel-Deployment verwenden, müssen Sie ihn korrekt konfigurieren, um Konflikte mit der Domain-Verifizierung von Vercel und der Ausstellung von SSL-Zertifikaten zu vermeiden.

Eine fehlerhafte Proxy-Konfiguration kann verhindern, dass Vercel SSL-Zertifikate von Let's Encrypt ausstellt, und zu Fehlern bei der Domain-Verifizierung führen.

Weitere Informationen finden Sie unter [Unterstützte Anbieter](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) in der Vercel-Dokumentation.

<div id="required-path-allowlist">
  ### Erforderliche Pfad-Allowlist
</div>

Ihr externer Proxy muss den Datenverkehr zu diesen spezifischen Pfaden zulassen, ohne zu blockieren, umzuleiten oder stark zu cachen:

- `/.well-known/acme-challenge/*` – Erforderlich für die Zertifikatsüberprüfung durch Let's Encrypt
- `/.well-known/vercel/*` – Erforderlich für die Domain-Verifizierung bei Vercel

Diese Pfade sollten unverändert direkt an Ihre Vercel-Deployment-Umgebung durchgereicht werden.

<div id="header-forwarding-requirements">
  ### Anforderungen an die Weiterleitung von Headern
</div>

Stellen Sie sicher, dass Ihr Proxy den `HOST`-Header korrekt weiterleitet. Ohne eine korrekte Weiterleitung des Headers schlagen Prüfanfragen fehl.

<div id="testing-your-proxy-setup">
  ### Testen Ihrer Proxy-Konfiguration
</div>

So verifizieren Sie, dass Ihr Proxy korrekt konfiguriert ist:

1. Prüfen Sie, ob `https://[yourdomain].com/.well-known/vercel/` eine Antwort zurückgibt.
2. Stellen Sie sicher, dass SSL-Zertifikate in Ihrem Vercel-Dashboard korrekt bereitgestellt werden.
3. Überprüfen Sie, dass die Domainverifizierung erfolgreich abgeschlossen wird.
