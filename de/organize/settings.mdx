Die Datei `docs.json` ermöglicht es, eine Sammlung von Markdown-Dateien in eine navigierbare, individuell anpassbare Dokumentationswebsite zu verwandeln. Diese erforderliche Konfigurationsdatei steuert Styling, Navigation, Integrationen und mehr. Denken Sie an sie als den Bauplan Ihrer Dokumentation.

Die Einstellungen in `docs.json` gelten global für alle Seiten.

<div id="setting-up-your-docsjson">
  ## Einrichten Ihrer `docs.json`
</div>

Für den Einstieg müssen Sie nur `theme`, `name`, `colors.primary` und `navigation` angeben. Weitere Felder sind optional und können hinzugefügt werden, wenn Ihr Dokumentationsbedarf wächst.

Für das beste Bearbeitungserlebnis fügen Sie die Schema-Referenz oben in Ihrer `docs.json`-Datei ein. Dadurch werden in den meisten Code-Editoren Autovervollständigung, Validierung und hilfreiche Tooltips aktiviert:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Ihre Dokumentation",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Ihre Navigationsstruktur
  }
  // Der Rest Ihrer Konfiguration
}
```

</div>

<ResponseField name="theme" required>
  Das Layout-Theme Ihrer Seite.

  Eines der folgenden: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Weitere Informationen finden Sie unter [Themes](/de/customize/themes).
</ResponseField>

<ResponseField name="name" type="string" required>
  Der Name Ihres Projekts, Ihrer Organisation oder Ihres Produkts.
</ResponseField>

<ResponseField name="colors" type="object" required>
  Die in Ihrer Dokumentation verwendeten Farben. Farben werden je nach Theme unterschiedlich angewendet. Wenn Sie nur eine Primärfarbe angeben, wird diese für alle Farbelemente verwendet.

  <Expandable title="Farben">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      Die Primärfarbe für Ihre Dokumentation. Wird im Allgemeinen im hellen Modus für Hervorhebungen verwendet, mit einigen Variationen je nach Theme.

      Muss ein Hex-Code sein, der mit `#` beginnt.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Die im dunklen Modus für Hervorhebungen verwendete Farbe.

      Muss ein Hex-Code sein, der mit `#` beginnt.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      Die Farbe, die für Buttons und Hover-Zustände in hellem wie auch dunklem Modus verwendet wird, mit einigen Variationen je nach Theme.

      Muss ein Hex-Code sein, der mit `#` beginnt.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  Beschreibung Ihrer Seite für SEO und KI-Indexierung.
</ResponseField>

<ResponseField name="logo" type="string or object">

    <ResponseField name="href" type="string (uri)">
      Die URL, zu der beim Klicken auf das Logo weitergeleitet wird. Wenn nicht angegeben, verlinkt das Logo auf Ihre Startseite. Beispiel: `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

  <Expandable title="Favicon">

<ResponseField name="styling" type="object">
  Konfigurationen für das visuelle Erscheinungsbild.

  <Expandable title="Styling">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      Stil der Page-Eyebrow. Wähle `section`, um den Abschnittsnamen anzuzeigen, oder `breadcrumbs`, um den vollständigen Navigationspfad anzuzeigen. Standardmäßig `section`.
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      Konfiguration des Codeblock-Themes. Standardmäßig `"system"`.

      **Einfache Konfiguration:**

      * `"system"`: Aktuellen Site-Modus übernehmen (hell oder dunkel)
      * `"dark"`: Immer den Dunkelmodus verwenden

      **Benutzerdefinierte Theme-Konfiguration:**

      * Verwende einen String, um ein einzelnes [Shiki-Theme](https://shiki.style/themes) für alle Codeblöcke festzulegen
      * Verwende ein Objekt, um getrennte [Shiki-Themes](https://shiki.style/themes) für hellen und dunklen Modus festzulegen

      <ResponseField name="theme" type="string">
        Ein einzelner Shiki-Theme-Name, der sowohl für hellen als auch für dunklen Modus verwendet wird.

        ```json
        "styling": {
          "codeblocks": {
            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

      <ResponseField name="theme" type="object">
        Unterschiedliche Themes für hellen und dunklen Modus.

        <Expandable title="theme">
          <ResponseField name="light" type="string" required>
            Ein Shiki-Theme-Name für den hellen Modus.
          </ResponseField>

          <ResponseField name="dark" type="string" required>
            Ein Shiki-Theme-Name für den dunklen Modus.
          </ResponseField>

          ```json
          "styling": {
            "codeblocks": {
              "theme": {
                "light": "github-light",
                "dark": "github-dark"
              }
            }
          }
          ```
        </Expandable>
      </ResponseField>
    </ResponseField>
  </Expandable>
</ResponseField>

      Aktionen, die im Kontextmenü verfügbar sind. Die erste Option wird standardmäßig angezeigt.

      * `copy`: Kopiert die aktuelle Seite als Markdown in die Zwischenablage.
      * `view`: Zeigt die aktuelle Seite als Markdown in einem neuen Tab an.
      * `chatgpt`: Sendet den aktuellen Seiteninhalt an ChatGPT.
      * `claude`: Sendet den aktuellen Seiteninhalt an Claude.
      * `perplexity`: Sendet den aktuellen Seiteninhalt an Perplexity.
      * `mcp`: Kopiert deine MCP-Server-URL in die Zwischenablage.
      * `cursor`: Installiert deinen gehosteten MCP-Server in Cursor.
      * `vscode`: Installiert deinen gehosteten MCP-Server in VSCode.

<div id="upgrading-from-mintjson">
  ## Upgrade von `mint.json`
</div>

Wenn Ihr Docs-Projekt die veraltete Datei `mint.json` verwendet, führen Sie diese Schritte aus, um auf `docs.json` zu migrieren.

<Steps>
  <Step title="CLI installieren oder aktualisieren">
    Falls Sie die [CLI](/de/installation) noch nicht installiert haben, installieren Sie sie jetzt:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Wenn die CLI bereits installiert ist, stellen Sie sicher, dass sie auf dem neuesten Stand ist:

    ```bash
    mint update
    ```
  </Step>

  <Step title="docs.json-Datei erstellen">
    Führen Sie in Ihrem Docs-Repository Folgendes aus:

    ```bash
    mint upgrade
    ```

    Dieser Befehl erstellt aus Ihrer vorhandenen `mint.json` eine `docs.json`. Prüfen Sie die generierte Datei, um sicherzustellen, dass alle Einstellungen korrekt sind.
  </Step>

  <Step title="mint.json-Datei löschen">
    Nachdem Sie bestätigt haben, dass Ihre `docs.json` korrekt konfiguriert ist, können Sie die alte `mint.json` sicher löschen.
  </Step>
</Steps>
