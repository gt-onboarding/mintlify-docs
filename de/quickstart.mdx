Diese Schnellstartanleitung zeigt Ihnen, wie Sie Ihre Dokumentationsseite in wenigen Minuten einrichten und veröffentlichen.

Nach Abschluss dieser Anleitung haben Sie eine live geschaltete Dokumentationsseite, die Sie anpassen und erweitern können.

<Info>
  **Voraussetzungen**: Bevor Sie beginnen, [erstellen Sie ein Konto](https://mintlify.com/start) und schließen Sie das Onboarding ab.
</Info>

<div id="getting-started">
  ## Erste Schritte
</div>

Nachdem Sie den Onboarding-Prozess abgeschlossen haben, wird Ihre Dokumentationssite automatisch unter einer eindeutigen URL im folgenden Format bereitgestellt:

```
https://<your-project-name>.mintlify.app
```

Sie finden Ihre URL auf der Übersichtsseite Ihres [Dashboards](https://dashboard.mintlify.com/).

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify Domain" className="hidden dark:block" />
</Frame>

Die URL Ihrer Site ist sofort verfügbar. Verwenden Sie sie zum Testen und zum Teilen mit Ihrem Team, während Sie Ihre Dokumentationssite einrichten.

<div id="install-the-github-app">
  ### GitHub-App installieren
</div>

Mintlify bietet eine GitHub-App, die die Bereitstellung automatisch auslöst, sobald Sie Änderungen in Ihr Repository pushen.

Installieren Sie die GitHub-App, indem Sie der Onboarding-Checkliste oder den Anweisungen in Ihrem Dashboard folgen.

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **GitHub App** in der Seitenleiste.
3. Klicken Sie auf **Install GitHub App**. Es öffnet sich ein neuer Tab mit der Installationsseite der GitHub-App.
4. Wählen Sie die Organisation oder das Benutzerkonto aus, in dem Sie die App installieren möchten.
5. Wählen Sie die Repositories aus, die Sie verbinden möchten.

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub-App-Installation" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub-App-Installation" className="hidden dark:block" />
</Frame>

<Info>
  Aktualisieren Sie die Berechtigungen der GitHub-App, wenn Sie Ihre Dokumentation in ein anderes Repository verschieben.
</Info>

<div id="authorize-your-github-account">
  ### Autorisieren Sie Ihr GitHub-Konto
</div>

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **My Profile** in der Seitenleiste.
3. Wählen Sie **Authorize GitHub account**. Dadurch wird ein neuer Tab mit der GitHub-Autorisierungsseite geöffnet.

<Info>
  Abhängig von den Einstellungen Ihrer Organisation muss Ihr Konto möglicherweise von einem Administrator Ihrer GitHub-Organisation autorisiert werden.
</Info>

<div id="code-based-workflow">
  ## Codebasierter Workflow
</div>

Der codebasierte Workflow integriert sich in Ihre bestehende Entwicklungsumgebung und Git-Repositories. Dieser Workflow ist ideal für technische Teams, die Dokumentation gemeinsam mit dem Code verwalten möchten.

<div id="install-the-cli">
  ### CLI installieren
</div>

Um lokal mit Ihrer Dokumentation zu arbeiten, installieren Sie die Command-Line-Interface (CLI) [mint](https://www.npmjs.com/package/mint), indem Sie folgenden Befehl in Ihrem Terminal ausführen:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  Auf Ihrem Rechner muss Node.js installiert sein. Wenn bei der Installation Probleme auftreten, lesen Sie den Leitfaden zur Fehlerbehebung.
</Info>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Nachdem Sie Ihre Umgebung eingerichtet haben, können Sie mit der Bearbeitung Ihrer Dokumentationsdateien beginnen. Aktualisieren Sie beispielsweise den Titel der Einführungsseite:

1. Öffnen Sie das Repository, das während des Onboardings erstellt wurde.
2. Öffnen Sie `index.mdx` und gehen Sie zum Anfang der Datei:

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. Aktualisieren Sie das Feld `title` auf „Hello World“.

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

<div id="preview-the-changes">
  ### Änderungen in der Vorschau anzeigen
</div>

Um die Änderungen lokal anzusehen, führen Sie den folgenden Befehl aus:

```bash
mint dev
```

Die Vorschau ist unter `localhost:3000` verfügbar.

<div id="push-the-changes">
  ### Änderungen pushen
</div>

Wenn Sie bereit sind, Ihre Änderungen zu veröffentlichen, pushen Sie sie in Ihr Repository.

Mintlify erkennt die Änderungen automatisch, erstellt Ihre Dokumentation und deployt die Updates auf Ihrer Site. Überwachen Sie den Deployment-Status in der Commit-Historie Ihres GitHub-Repositories oder im [Dashboard](https://dashboard.mintlify.com).

Nach Abschluss des Deployments ist Ihr jüngstes Update unter `<your-project-name>.mintlify.app` verfügbar.

<div id="web-editor-workflow">
  ## Web-Editor-Workflow
</div>

Der Web-Editor-Workflow bietet eine WYSIWYG-Oberfläche (What You See Is What You Get) zum Erstellen und Bearbeiten von Dokumentationen. Dieser Workflow eignet sich besonders für Nutzer, die im Webbrowser ohne zusätzliche lokale Entwicklungstools arbeiten möchten.

<div id="access-the-web-editor">
  ### Auf den Web-Editor zugreifen
</div>

1. Melden Sie sich in Ihrem [Dashboard](https://dashboard.mintlify.com) an.
2. Wählen Sie **Editor** in der linken Seitenleiste.

<Info>
  Wenn Sie die GitHub-App noch nicht installiert haben, werden Sie beim Öffnen des Web-Editors dazu aufgefordert.
</Info>

  <img alt="Der Mintlify Web-Editor im visuellen Editor-Modus" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

  <img alt="Der Mintlify Web-Editor im visuellen Editor-Modus" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Im Web-Editor können Sie in der Seitenleiste durch Ihre Dokumentationsdateien navigieren. Aktualisieren wir die Einführungsseite:

Suchen und wählen Sie `index.mdx` im Dateiexplorer aus.

Aktualisieren Sie dann im Editor das Titelfeld auf „Hello World“.

<Frame>
  <img alt="Bearbeiten im Web-Editor" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="Bearbeiten im Web-Editor" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />

<Tip>
  Der Editor bietet eine umfangreiche Auswahl an Formatierungswerkzeugen und Komponenten. Drücken Sie <kbd>/</kbd> im Editor, um das Befehlsmenü zu öffnen und auf diese Werkzeuge zuzugreifen.
</Tip>

<div id="publish-your-changes">
  ### Änderungen veröffentlichen
</div>

Wenn Sie mit Ihren Bearbeitungen zufrieden sind, klicken Sie oben rechts auf **Publish**. Ihre Änderungen werden sofort auf Ihrer Dokumentationsseite veröffentlicht.

<Tip>
  Verwenden Sie Branches, um Änderungen über Pull Requests vorab zu prüfen und zu überprüfen, bevor Sie sie auf Ihrer Live‑Seite veröffentlichen.
</Tip>

Weitere Informationen zur Verwendung des Web-Editors, einschließlich des Einsatzes von Branches und Pull Requests zur Zusammenarbeit und zur Vorschau von Änderungen, finden Sie in unserer [Web-Editor-Dokumentation](/de/editor).

<div id="adding-a-custom-domain">
  ## Hinzufügen einer benutzerdefinierten Domain
</div>

Während Ihre `<your-project-name>.mintlify.app`-Subdomain sich gut für Tests und Entwicklung eignet, verwenden die meisten Teams für die produktive Dokumentation eine eigene Domain.

Um eine benutzerdefinierte Domain hinzuzufügen, navigieren Sie in Ihrem Dashboard zur Seite [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain).

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Custom Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Custom Domain" className="hidden dark:block" />
</Frame>

Geben Sie Ihre Domain ein (zum Beispiel `docs.yourcompany.com`) und folgen Sie den Anweisungen, um die DNS-Einstellungen bei Ihrem Domainanbieter zu konfigurieren.

<Table>
  | Record Type | Name | Value | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs (oder Subdomain) | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS-Änderungen können bis zu 48 Stunden dauern, bis sie weltweit wirksam werden, auch wenn sie häufig deutlich schneller abgeschlossen sind.
</Info>

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

Wenn während der Einrichtung Probleme auftreten, prüfen Sie diese häufigen Lösungen:

<AccordionGroup>
  <Accordion title="Lokale Vorschau funktioniert nicht">
    Stellen Sie sicher, dass Node.js v19+ installiert ist und dass Sie den Befehl `mint dev` aus dem Verzeichnis ausführen, das Ihre `docs.json` enthält.
  </Accordion>

  <Accordion title="Änderungen werden auf der Live-Website nicht angezeigt">
    Die Bereitstellung kann mehrere Minuten dauern. Überprüfen Sie Ihre GitHub Actions (für codebasierte Workflows) oder die Bereitstellungsprotokolle im Mintlify-Dashboard, um sicherzustellen, dass keine Build-Fehler vorliegen.
  </Accordion>

  <Accordion title="Eigene Domain wird nicht verbunden">
    Überprüfen Sie, ob Ihre DNS-Einträge korrekt konfiguriert sind, und lassen Sie genügend Zeit für die DNS-Propagation (bis zu 48 Stunden). Sie können Tools wie [DNSChecker](https://dnschecker.org) verwenden, um Ihren CNAME-Eintrag zu prüfen.
  </Accordion>
</AccordionGroup>
