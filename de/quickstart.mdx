Dieser Schnellstart zeigt Ihnen, wie Sie Ihre Dokumentationsseite in wenigen Minuten einrichten und bereitstellen.

Nach Abschluss dieser Anleitung haben Sie eine live geschaltete Dokumentationsseite, die Sie anpassen und erweitern können.

<Info>
  **Voraussetzungen**: Bevor Sie beginnen, [erstellen Sie ein Konto](https://mintlify.com/start) und schließen Sie die Ersteinrichtung ab.
</Info>

<div id="getting-started">
  ## Erste Schritte
</div>

Nachdem du den Onboarding-Prozess abgeschlossen hast, wird deine Dokumentationsseite automatisch unter einer eindeutigen URL mit folgendem Format bereitgestellt:

```
https://<your-project-name>.mintlify.app
```

Du findest deine URL auf der Übersichtsseite deines [Dashboards](https://dashboard.mintlify.com/).

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify-Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify-Domain" className="hidden dark:block" />
</Frame>

Die URL deiner Seite ist sofort verfügbar. Verwende diese URL zum Testen und zum Teilen mit deinem Team, während du deine Docs-Site einrichtest.

<div id="install-the-github-app">
  ### GitHub-App installieren
</div>

Mintlify stellt eine GitHub-App bereit, die Deployments automatisch ausführt, sobald du Änderungen in dein Repository pushst.

Installiere die GitHub-App, indem du den Anweisungen in der Onboarding-Checkliste oder in deinem Dashboard folgst.

1. Navigiere zu **Settings** in deinem Mintlify-Dashboard.
2. Wähle **GitHub App** in der Seitenleiste.
3. Wähle **Install GitHub App**. Es öffnet sich ein neuer Tab mit der Installationsseite der GitHub-App.
4. Wähle die Organisation oder das Benutzerkonto aus, in dem du die App installieren möchtest.
5. Wähle die Repositories aus, die du verbinden möchtest.

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub-App-Installation" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub-App-Installation" className="hidden dark:block" />
</Frame>

<Info>
  Aktualisiere die Berechtigungen der GitHub-App, wenn du deine Dokumentation in ein anderes Repository verschiebst.
</Info>

<div id="authorize-your-github-account">
  ### Autorisieren Sie Ihr GitHub-Konto
</div>

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **My Profile** in der Seitenleiste.
3. Wählen Sie **Authorize GitHub account**. Es öffnet sich ein neuer Tab mit der GitHub-Autorisierungsseite.

<Info>
  Je nach Organisationseinstellungen muss ein Administrator Ihrer GitHub-Organisation Ihr Konto möglicherweise freigeben.
</Info>

<div id="code-based-workflow">
  ## Codebasierter Workflow
</div>

Der codebasierte Workflow integriert sich in Ihre bestehende Entwicklungsumgebung und Ihre Git-Repositories. Dieser Workflow eignet sich am besten für technische Teams, die Dokumentation zusammen mit dem Code verwalten möchten.

<div id="install-the-cli">
  ### CLI installieren
</div>

Um lokal mit deiner Dokumentation zu arbeiten, installiere die Befehlszeilenschnittstelle (CLI) namens [mint](https://www.npmjs.com/package/mint), indem du diesen Befehl in deinem Terminal ausführst:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  Du benötigst Node.js auf deinem Rechner. Wenn bei der Installation Probleme auftreten, sieh dir den Troubleshooting-Leitfaden an.
</Info>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Nachdem Sie Ihre Umgebung eingerichtet haben, können Sie mit der Bearbeitung Ihrer Dokumentationsdateien beginnen. Aktualisieren Sie zum Beispiel den Titel der Einführungsseite:

1. Öffnen Sie das Repository, das während des Onboardings erstellt wurde.
2. Öffnen Sie `index.mdx` und gehen Sie zum Anfang der Datei:

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. Aktualisieren Sie das Feld `title` auf `"Hello World"`.

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

<div id="preview-the-changes">
  ### Änderungen in der Vorschau anzeigen
</div>

Um die Änderungen lokal in der Vorschau zu sehen, führe den folgenden Befehl aus:

```bash
mint dev
```

Die Vorschau ist unter `localhost:3000` verfügbar.

<div id="push-the-changes">
  ### Änderungen pushen
</div>

Wenn Sie bereit sind, Ihre Änderungen zu veröffentlichen, pushen Sie sie in Ihr Repository.

Mintlify erkennt die Änderungen automatisch, baut Ihre Dokumentation und deployt die Updates auf Ihrer Site. Verfolgen Sie den Deploy-Status in der Commit-Historie Ihres GitHub-Repositories oder im [Dashboard](https://dashboard.mintlify.com).

Nach Abschluss des Deployments ist Ihr neuestes Update unter `<your-project-name>.mintlify.app` verfügbar.

<div id="web-editor-workflow">
  ## Web-Editor-Workflow
</div>

Der Web-Editor-Workflow bietet eine WYSIWYG-Oberfläche („What you see is what you get“) zum Erstellen und Bearbeiten von Dokumentation. Dieser Workflow eignet sich besonders für Nutzer, die direkt im Webbrowser arbeiten möchten – ohne zusätzliche lokale Entwicklungstools.

<div id="access-the-web-editor">
  ### Zugriff auf den Web-Editor
</div>

1. Melde dich in deinem [Dashboard](https://dashboard.mintlify.com) an.
2. Wähle **Editor** in der linken Seitenleiste.

<Info>
  Wenn du die GitHub-App noch nicht installiert hast, wirst du beim Öffnen des Web-Editors aufgefordert, sie zu installieren.
</Info>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Im Web-Editor kannst du in der Seitenleiste durch deine Dokumentationsdateien navigieren. Aktualisieren wir die Einführungsseite:

Suche im Dateibrowser nach `index.mdx` und wähle die Datei aus.

Aktualisiere anschließend im Editor das Titelfeld auf „Hello World“.

<Frame>
  <img alt="Bearbeiten im Web-Editor" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

<Tip>
  Der Editor bietet eine umfangreiche Auswahl an Formatierungswerkzeugen und Komponenten. Drücke <kbd>/</kbd> im Editor, um das Befehlsmenü zu öffnen und auf diese Werkzeuge zuzugreifen.
</Tip>

<div id="publish-your-changes">
  ### Veröffentlichen Sie Ihre Änderungen
</div>

Wenn Sie mit Ihren Änderungen zufrieden sind, klicken Sie oben rechts auf **Publish**. Ihre Änderungen werden sofort auf Ihrer Dokumentationsseite veröffentlicht.

<Tip>
  Verwenden Sie Branches, um Änderungen über Pull Requests vorab anzusehen und zu überprüfen, bevor Sie sie auf Ihrer Live‑Website veröffentlichen.
</Tip>

Weitere Informationen zur Verwendung des Web-Editors, einschließlich der Zusammenarbeit und Vorschau mit Branches und Pull Requests, finden Sie in unserer [Web-Editor-Dokumentation](/de/editor).

<div id="adding-a-custom-domain">
  ## Hinzufügen einer benutzerdefinierten Domain
</div>

Während Ihre `<your-project-name>.mintlify.app`-Subdomain sich gut für Tests und Entwicklung eignet, verwenden die meisten Teams für die produktive Dokumentation eine eigene Domain.

Um eine benutzerdefinierte Domain hinzuzufügen, navigieren Sie in Ihrem Dashboard zur Seite [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain).

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Benutzerdefinierte Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Benutzerdefinierte Domain" className="hidden dark:block" />
</Frame>

Geben Sie Ihre Domain ein (zum Beispiel `docs.yourcompany.com`) und folgen Sie den Anweisungen, um die DNS-Einstellungen bei Ihrem Domainanbieter zu konfigurieren.

<Table>
  | Record-Typ | Name | Wert | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs (oder Subdomain) | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS-Änderungen können bis zu 48 Stunden benötigen, um sich zu propagieren, häufig sind sie jedoch deutlich schneller wirksam.
</Info>

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

Wenn während der Einrichtung Probleme auftreten, prüfen Sie diese häufigen Lösungen:

<AccordionGroup>
  <Accordion title="Lokale Vorschau funktioniert nicht">
    Stellen Sie sicher, dass Node.js v19+ installiert ist und dass Sie den Befehl `mint dev` aus dem Verzeichnis ausführen, das Ihre Datei `docs.json` enthält.
  </Accordion>

  <Accordion title="Änderungen erscheinen nicht auf der Live‑Site">
    Die Bereitstellung kann einige Minuten dauern. Überprüfen Sie Ihre GitHub Actions (für den codebasierten Workflow) oder die Bereitstellungsprotokolle im Mintlify‑Dashboard, um sicherzustellen, dass keine Build‑Fehler auftreten.
  </Accordion>

  <Accordion title="Eigene Domain wird nicht verbunden">
    Vergewissern Sie sich, dass Ihre DNS‑Einträge korrekt eingerichtet sind, und lassen Sie ausreichend Zeit für die DNS‑Propagation (bis zu 48 Stunden). Sie können Tools wie [DNSChecker](https://dnschecker.org) verwenden, um Ihren CNAME‑Eintrag zu überprüfen.
  </Accordion>
</AccordionGroup>
