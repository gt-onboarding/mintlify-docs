Mintlify kümmert sich automatisch um viele SEO-Best Practices, darunter:

* Generierung von Meta-Tags
* Erstellung von Sitemap und `robots.txt`
* Semantische HTML-Struktur
* Mobile Optimierung

Sie können die Meta-Tags Ihrer Website vollständig anpassen, indem Sie das Feld `metatags` zu Ihrer `docs.json` oder zum Frontmatter einer Seite hinzufügen.

<div id="global-meta-tags">
  ## Globale Meta-Tags
</div>

Um standardmäßige Meta-Tags für alle Seiten festzulegen, füge das Feld `metatags` zu deiner `docs.json` hinzu.

```json
"seo": {
    "metatags": {
        "og:image": "Link zu Ihrem Standard-Meta-Tag-Bild"
    }
}
```

<div id="set-a-canonical-url">
  ### Kanonische URL festlegen
</div>

Wenn du eine benutzerdefinierte Domain verwendest, setze das Meta-Tag `canonical`, damit Suchmaschinen deine bevorzugte Domain indexieren. Eine kanonische URL gibt Suchmaschinen an, welche Version deiner Dokumentation die maßgebliche ist. Das verbessert die SEO, wenn deine Dokumentation über mehrere URLs zugänglich ist, und verhindert Probleme mit doppelten Inhalten.

```json
"seo": {
    "metatags": {
        "canonical": "https://www.ihre-eigene-domain-hier.com"
    }
}
```

<div id="page-specific-meta-tags">
  ## Seitenspezifische Meta-Tags
</div>

Um seitenspezifische Meta-Tags festzulegen, füge sie zum Frontmatter der Seite hinzu.

```mdx
---
title: "Ihr Beispiel-Seitentitel"
"og:image": "Link zu Ihrem Meta-Tag-Bild"
---
```

<Note>
  Meta-Tags mit Doppelpunkten müssen in Anführungszeichen stehen.
</Note>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps und robots.txt-Dateien
</div>

Mintlify generiert automatisch eine `sitemap.xml`-Datei und eine `robots.txt`-Datei. Du kannst deine Sitemap anzeigen, indem du an die URL deiner Dokumentationsseite `/sitemap.xml` anhängst.

Standardmäßig werden nur Seiten berücksichtigt, die in deiner `docs.json` enthalten sind. Um versteckte Links einzubeziehen, füge `seo.indexing` zu deiner `docs.json` hinzu:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Benutzerdefinierte Sitemaps und robots.txt-Dateien
</div>

Um eine benutzerdefinierte `sitemap.xml` oder `robots.txt` hinzuzufügen, erstellen Sie eine `sitemap.xml`- oder `robots.txt`-Datei im Root-Verzeichnis Ihres Projekts. Eine benutzerdefinierte Datei überschreibt die automatisch generierte Datei gleichen Namens. Wenn Sie eine benutzerdefinierte Datei löschen, wird wieder die Standarddatei verwendet.

<div id="disabling-indexing">
  ## Deaktivieren der Indexierung
</div>

Wenn Sie verhindern möchten, dass eine Seite von Suchmaschinen indexiert wird, können Sie Folgendes in das [Frontmatter](/de/organize/pages) Ihrer Seite einfügen:

```
---
noindex: true
---
```

Sie können auch für alle Seiten in Ihrer Dokumentation `noindex` festlegen, indem Sie das Feld `metatags.robots` in Ihrer `docs.json` auf `"noindex"` setzen:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO-Best Practices
</div>

<AccordionGroup>
  <Accordion title="Aussagekräftige Titel und Beschreibungen schreiben">
    * Klare, präzise Seitentitel verwenden (50–60 Zeichen)
    * Prägnante, überzeugende Beschreibungen verfassen (150–160 Zeichen)
    * Relevante Keywords einbinden
    * Jede Seite mit einzigartigem Titel und einzigartiger Beschreibung versehen
  </Accordion>

  <Accordion title="Inhaltsstruktur optimieren">
    * Korrekte Überschriftenhierarchie nutzen (H1 → H2 → H3)
    * Zuerst für Menschen schreiben, erst danach für Suchmaschinen
    * Relevante Keywords in Überschriften und Inhalt platzieren
    * URLs kurz, aussagekräftig und hierarchisch strukturieren
    * Lange Inhalte mit Zwischenüberschriften und Listen gliedern
  </Accordion>

  <Accordion title="Interne Verlinkungsstrategie">
    * Auf thematisch verwandte Seiten innerhalb der Dokumentation verlinken
    * Aussagekräftigen Linktext statt „Hier klicken“ verwenden
    * Themencluster bilden, indem verwandte Konzepte verknüpft werden
    * Die automatischen Querverweisfunktionen nutzen
  </Accordion>

  <Accordion title="Bild-SEO">
    * Aussagekräftige Dateinamen für Bilder verwenden
    * Immer Alt-Text für Barrierefreiheit und SEO hinzufügen
    * Bilddateigrößen für schnelleres Laden optimieren
    * Relevante Bilder verwenden, die den Inhalt unterstützen
  </Accordion>
</AccordionGroup>
