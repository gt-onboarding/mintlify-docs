---
title: "Changelogs"
description: "Veröffentlichen Sie Produktupdates in Ihrer Dokumentation mit einem abonnierbaren RSS-Feed"
---

Erstellen Sie ein Changelog für Ihre Dokumentation, indem Sie [Update-Komponenten](/de/components/update) zu einer Seite hinzufügen.

Sehen Sie sich das [Mintlify Changelog](/de/changelog) als Beispiel an: In jedem Update können Sie Links, Bilder, Text und Demos Ihrer neuen Funktionen einbinden.

<div id="setting-up-your-changelog">
  ## Changelog einrichten
</div>

<Steps>
  <Step title="Erstellen Sie eine Seite für Ihr Changelog">
    1. Erstellen Sie eine neue Seite in Ihrer Dokumentation, z. B. `changelog.mdx` oder `updates.mdx`.
    2. Fügen Sie die Changelog-Seite Ihrem Navigationsschema in der `docs.json` hinzu.
  </Step>

  <Step title="Fügen Sie Update-Komponenten zu Ihrem Changelog hinzu">
    Fügen Sie für jeden Changelog-Eintrag eine `Update`‑Komponente hinzu.

    Geben Sie relevante Informationen wie Feature-Releases, Bugfixes oder andere Ankündigungen an.
  </Step>
</Steps>

```mdx Example changelog.mdx
---
title: "Changelog"
description: "Produktaktualisierungen und Ankündigungen"
---
<Update label="März 2025" description="v0.0.10">
  Neue Wintergreen-Variante hinzugefügt.

  Neue Version der Spearmint-Variante veröffentlicht, jetzt mit 10% mehr Minze.
</Update>

<Update label="Februar 2025" description="v0.0.09">
  Neue Version der Spearmint-Variante veröffentlicht.
</Update>
```

<div id="customizing-your-changelog">
  ## Changelog anpassen
</div>

Steuern Sie, wie Nutzer durch Ihr Changelog navigieren und über Ihre Produktinformationen auf dem Laufenden bleiben.

<div id="table-of-contents">
  ### Inhaltsverzeichnis
</div>

Jede `label`-Eigenschaft eines `Update` erzeugt automatisch einen Eintrag im Inhaltsverzeichnis der rechten Seitenleiste. Dies ist die Standardnavigation für dein Changelog.

<div id="tag-filters">
  ### Tag-Filter
</div>

Fügen Sie Ihren `Update`-Komponenten `tags` hinzu, um das Inhaltsverzeichnis durch Tag-Filter zu ersetzen. Nutzer können das Changelog filtern, indem sie ein oder mehrere Tags auswählen:

```mdx Tag filters example wrap
<Update label="März 2025" description="v0.0.10" tags={["Wintergreen", "Spearmint"]}>
  Neue Wintergreen-Variante hinzugefügt.

  Neue Version der Spearmint-Variante veröffentlicht, jetzt mit 10 % mehr Minze.
</Update>

<Update label="Februar 2025" description="v0.0.09" tags={["Spearmint"]}>
  Neue Version der Spearmint-Variante veröffentlicht.
</Update>

<Update label="Januar 2025" description="v0.0.08" tags={["Peppermint", "Spearmint"]}>
  Peppermint-Variante als veraltet markiert.

  Neue Version der Spearmint-Variante veröffentlicht.
</Update>
```

  Inhaltsverzeichnis und Changelog-Filter werden ausgeblendet, wenn die Seitenmodi `custom`, `center` oder `wide` verwendet werden. Weitere Informationen zu [Seitenmodi](/de/organize/pages#page-mode).
</Tip>

<div id="subscribable-changelogs">
  ### Abonnierbare Changelogs
</div>

Durch die Verwendung von `Update`-Komponenten wird ein abonnierbarer RSS-Feed unter der URL deiner Seite mit angehängtem `/rss.xml` bereitgestellt. Zum Beispiel: `mintlify.com/docs/changelog/rss.xml`.

Dem RSS-Feed werden Einträge hinzugefügt, wenn neue `Update`-Komponenten veröffentlicht werden und wenn neue Überschriften innerhalb bestehender `<Update>`-Tags hinzugefügt werden.

Die obersten Überschriften der `Update`-Komponenten werden als Titel der RSS-Feed-Einträge verwendet, und die Einträge verlinken auf die entsprechenden Anker dieser Überschriften in deiner Dokumentation.

```xml Example RSS feed
<?xml version="1.0" encoding="UTF-8"?>
<rss xmlns:dc="http://purl.org/dc/elements/1.1/" xmlns:content="http://purl.org/rss/1.0/modules/content/" xmlns:atom="http://www.w3.org/2005/Atom" version="2.0">
  <channel>
    <title><![CDATA[Produkt-Updates]]></title>
    <description><![CDATA[Neue Updates und Verbesserungen]]></description>
    <link>https://mintlify.com/docs</link>
    <generator>RSS for Node</generator>
    <lastBuildDate>Mon, 21 Jul 2025 21:21:47 GMT</lastBuildDate>
    <atom:link href="https://mintlify.com/docs/changelog/rss.xml" rel="self" type="application/rss+xml"/>
    <copyright><![CDATA[Mintlify]]></copyright>
    <docs>https://mintlify.com/docs</docs>
    <item>
      <title><![CDATA[Juni 2025]]></title>
      <link>https://mintlify.com/docs/changelog#june-2025</link>
      <guid isPermaLink="true">https://mintlify.com/docs/changelog#june-2025</guid>
      <pubDate>Mon, 23 Jun 2025 16:54:22 GMT</pubDate>
    </item>
  </channel>
</rss>
```

RSS-Feeds können mit Slack, E-Mail oder anderen Abo-Tools integriert werden, um Nutzer über Produktänderungen zu informieren. Einige Optionen sind:

* [Slack](https://slack.com/help/articles/218688467-Add-RSS-feeds-to-Slack)
* [E-Mail](https://zapier.com/apps/email/integrations/rss/1441/send-new-rss-feed-entries-via-email) via Zapier
* Discord-Bots wie [Readybot](https://readybot.io) oder [RSS Feeds to Discord Bot](https://rss.app/en/bots/rssfeeds-discord-bot)

Damit der RSS-Feed leicht gefunden wird, können Sie oben auf der Seite ein RSS-Symbol als Button anzeigen, das auf den Feed verlinkt. Fügen Sie `rss: true` zum Frontmatter der Seite hinzu:

```mdx
---
rss: true
---
```
