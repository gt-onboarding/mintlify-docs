Erstellen Sie ein Changelog für Ihre Dokumentation, indem Sie [Update-Komponenten](/de/components/update) zu einer Seite hinzufügen.

Sehen Sie sich das [Mintlify-Changelog](/de/changelog) als Beispiel an: In jedem Update können Sie Links, Bilder, Text und Demos Ihrer neuen Funktionen einbinden.

<div id="setting-up-your-changelog">
  ## Changelog einrichten
</div>

<Steps>
  <Step title="Seite für den Changelog erstellen">
    1. Erstellen Sie eine neue Seite in Ihrer Dokumentation, z. B. `changelog.mdx` oder `updates.mdx`.
    2. Fügen Sie die Changelog-Seite Ihrem Navigationsschema in Ihrer `docs.json` hinzu.
  </Step>

  <Step title="Update-Komponenten zum Changelog hinzufügen">
    Fügen Sie für jeden Eintrag im Changelog ein `Update` hinzu.

    Ergänzen Sie relevante Informationen wie Feature-Releases, Bugfixes oder andere Ankündigungen.
  </Step>
</Steps>

```mdx Example changelog.mdx
---
title: "Changelog"
description: "Produktupdates und Ankündigungen"
---
<Update label="März 2025" description="v0.0.10">
  Eine neue Geschmacksrichtung „Wintergreen“ hinzugefügt.

  Eine neue Version der Geschmacksrichtung „Spearmint“ veröffentlicht, jetzt mit 10 % mehr Minze.
</Update>

<Update label="Februar 2025" description="v0.0.09">
  Eine neue Version der Geschmacksrichtung „Spearmint“ veröffentlicht.
</Update>
```

<div id="customizing-your-changelog">
  ## Anpassen Ihres Changelogs
</div>

Steuern Sie, wie Nutzer durch Ihren Changelog navigieren und bei Ihren Produktinformationen auf dem neuesten Stand bleiben.

<div id="table-of-contents">
  ### Inhaltsverzeichnis
</div>

Jede `label`-Eigenschaft eines `Update` erzeugt automatisch einen Eintrag im Inhaltsverzeichnis der rechten Seitenleiste. Dies ist die Standardnavigation für Ihren Changelog.

<div id="tag-filters">
  ### Tag-Filter
</div>

Fügen Sie `tags` zu Ihren `Update`-Komponenten hinzu, um das Inhaltsverzeichnis durch Tag-Filter zu ersetzen. Nutzende können das Changelog filtern, indem sie ein oder mehrere Tags auswählen:

```mdx Tag filters example wrap
<Update label="March 2025" description="v0.0.10" tags={["Wintergreen", "Spearmint"]}>
  Added a new Wintergreen flavor.

  Released a new version of the Spearmint flavor, now with 10% more mint.
</Update>

<Update label="February 2025" description="v0.0.09" tags={["Spearmint"]}>
  Released a new version of the Spearmint flavor.
</Update>

<Update label="January 2025" description="v0.0.08" tags={["Peppermint", "Spearmint"]}>
  Deprecated the Peppermint flavor.

  Released a new version of the Spearmint flavor.
</Update>
```

<Tip>
  Inhaltsverzeichnis und Changelog-Filter werden ausgeblendet, wenn die Seitenmodi „Benutzerdefiniert“, „Zentriert“ oder „Breit“ verwendet werden. Mehr zu [Seitenmodi](/de/pages#page-mode).
</Tip>

<div id="subscribable-changelogs">
  ### Abonnierbare Changelogs
</div>

Die Verwendung von `Update`-Komponenten erstellt einen abonnierbaren RSS-Feed unter der Seiten-URL mit angehängtem `/rss.xml`. Zum Beispiel: `mintlify.com/docs/changelog/rss.xml`.

Einträge werden dem RSS-Feed hinzugefügt, wenn neue `Update`-Komponenten veröffentlicht werden und wenn neue Überschriften innerhalb bestehender `<Update>`-Tags ergänzt werden.

Die obersten Überschriften von `Update`-Komponenten dienen als Titel der RSS-Feed-Einträge, und die Einträge verlinken auf die Anker dieser Überschriften in Ihren Docs.

```xml Example RSS feed
<?xml version="1.0" encoding="UTF-8"?>
<rss xmlns:dc="http://purl.org/dc/elements/1.1/" xmlns:content="http://purl.org/rss/1.0/modules/content/" xmlns:atom="http://www.w3.org/2005/Atom" version="2.0">
  <channel>
    <title><![CDATA[Product updates]]></title>
    <description><![CDATA[New updates and improvements]]></description>
    <link>https://mintlify.com/docs</link>
    <generator>RSS for Node</generator>
    <lastBuildDate>Mon, 21 Jul 2025 21:21:47 GMT</lastBuildDate>
    <atom:link href="https://mintlify.com/docs/changelog/rss.xml" rel="self" type="application/rss+xml"/>
    <copyright><![CDATA[Mintlify]]></copyright>
    <docs>https://mintlify.com/docs</docs>
    <item>
      <title><![CDATA[June 2025]]></title>
      <link>https://mintlify.com/docs/changelog#june-2025</link>
      <guid isPermaLink="true">https://mintlify.com/docs/changelog#june-2025</guid>
      <pubDate>Mon, 23 Jun 2025 16:54:22 GMT</pubDate>
    </item>
  </channel>
</rss>
```

RSS-Feeds können mit Slack, E-Mail oder anderen Abonnement-Tools integriert werden, um Nutzer über Produktänderungen zu informieren. Einige Optionen sind:

- [Slack](https://slack.com/help/articles/218688467-Add-RSS-feeds-to-Slack)
- [E-Mail](https://zapier.com/apps/email/integrations/rss/1441/send-new-rss-feed-entries-via-email) via Zapier
- Discord-Bots wie [Readybot](https://readybot.io) oder [RSS Feeds to Discord Bot](https://rss.app/en/bots/rssfeeds-discord-bot)

Damit der RSS-Feed leichter gefunden wird, können Sie oben auf der Seite ein RSS-Symbol als Schaltfläche anzeigen, das auf den Feed verlinkt. Fügen Sie `rss: true` zum Frontmatter der Seite hinzu:

```mdx
---
rss: true
---
```
