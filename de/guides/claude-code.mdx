Claude Code ist ein agentisches Kommandozeilentool, das dir bei der Pflege deiner Dokumentation helfen kann. Es kann neue Inhalte erstellen, bestehende Seiten prüfen und die Dokus aktuell halten.

Du kannst Claude Code darauf trainieren, deine Dokumentationsstandards und Workflows zu verstehen, indem du deinem Projekt eine `CLAUDE.md`-Datei hinzufügst und sie im Laufe der Zeit verfeinerst.

<div id="getting-started">
  ## Erste Schritte
</div>

**Voraussetzungen:**

- Aktives Claude-Abonnement (Pro, Max oder API-Zugriff)

**Einrichtung:**

1. Claude Code installieren:

```bash
  npm install -g @anthropic-ai/claude-code
  ```

2. Zum docs-Verzeichnis navigieren.
3. (Optional) Die untenstehende Datei `CLAUDE.md` zum Projekt hinzufügen.
4. `claude` ausführen, um zu starten.

<div id="claudemd-template">
  ## CLAUDE.md-Vorlage
</div>

Speichern Sie die Datei `CLAUDE.md` im Root-Verzeichnis Ihrer Docs, damit Claude Code Ihr Projekt besser versteht. Diese Datei schult Claude Code auf Ihre Dokumentationsstandards, Präferenzen und Workflows. Weitere Informationen finden Sie unter [Manage Claude's memory](https://docs.anthropic.com/en/docs/claude-code/memory) in der Anthropic-Dokumentation.

Kopieren Sie diese Beispielvorlage oder passen Sie sie an Ihre Docs-Spezifikationen an:

```mdx
# Mintlify documentation

## Working relationship
- You can push back on ideas—this can lead to better documentation. Cite sources and explain your reasoning when you do so
- ALWAYS ask for clarification rather than making assumptions
- NEVER lie, guess, or make up information

## Project context
- Format: MDX files with YAML frontmatter
- Config: docs.json for navigation, theme, settings
- Components: Mintlify components

## Content strategy
- Document just enough for user success - not too much, not too little
- Prioritize accuracy and usability of information
- Make content evergreen when possible
- Search for existing information before adding new content. Avoid duplication unless it is done for a strategic reason
- Check existing patterns for consistency
- Start by making the smallest reasonable changes

## docs.json

- Refer to the [docs.json schema](https://mintlify.com/docs.json) when building the docs.json file and site navigation

## Frontmatter requirements for pages
- title: Clear, descriptive page title
- description: Concise summary for SEO/navigation

## Writing standards
- Second-person voice ("you")
- Prerequisites at start of procedural content
- Test all code examples before publishing
- Match style and formatting of existing pages
- Include both basic and advanced use cases
- Language tags on all code blocks
- Alt text on all images
- Relative paths for internal links

## Git workflow
- NEVER use --no-verify when committing
- Ask how to handle uncommitted changes before starting
- Create a new branch when no clear branch exists for changes
- Commit frequently throughout development
- NEVER skip or disable pre-commit hooks

## Do not
- Skip frontmatter on any MDX file
- Use absolute URLs for internal links
- Include untested code examples
- Make assumptions - always ask for clarification
```

<div id="sample-prompts">
  ## Beispielaufforderungen
</div>

Sobald Sie Claude Code eingerichtet haben, probieren Sie diese Eingabeaufforderungen aus, um zu sehen, wie es bei gängigen Dokumentationsaufgaben helfen kann. Sie können diese Beispiele direkt kopieren und einfügen oder für Ihre spezifischen Anforderungen anpassen.

<div id="convert-notes-to-polished-docs">
  ### Notizen in ausgearbeitete Dokumente umwandeln
</div>

Verwandeln Sie Rohentwürfe in sauber formatierte Markdown-Seiten mit Komponenten und Frontmatter.

**Beispielprompt:**

```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

<div id="review-docs-for-consistency">
  ### Dokumentation auf Konsistenz prüfen
</div>

Erhalten Sie Vorschläge zur Verbesserung von Stil, Formatierung und der Verwendung von Komponenten.

**Beispielprompt:**

```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### Dokumentation aktualisieren, wenn sich Features ändern
</div>

Halten Sie die Dokumentation aktuell, wenn sich Ihr Produkt weiterentwickelt.

**Beispielprompt:**

```text wrap
Unsere API erfordert jetzt einen Versionsparameter. Aktualisieren Sie unsere Dokumentation, sodass in allen Beispielen version=2024-01 enthalten ist.
```

<div id="generate-comprehensive-code-examples">
  ### Umfassende Codebeispiele erstellen
</div>

Erstellen Sie Beispiele in mehreren Sprachen mit Fehlerbehandlung.

**Beispiel-Prompt:**

```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Erweiterung von Claude Code
</div>

Über das manuelle Prompten von Claude Code hinaus können Sie es in Ihre bestehenden Workflows integrieren.

<div id="automation-with-github-actions">
  ### Automatisierung mit GitHub Actions
</div>

Führen Sie Claude Code automatisch aus, sobald sich Code ändert, um die Dokumentation aktuell zu halten. Sie können Dokumentationsreviews bei Pull Requests auslösen oder Beispiele aktualisieren, wenn API-Änderungen erkannt werden.

<div id="multi-instance-workflows">
  ### Workflows mit mehreren Instanzen
</div>

Verwenden Sie separate Claude-Code-Sitzungen für unterschiedliche Aufgaben – eine zum Erstellen neuer Inhalte und eine weitere für die Überprüfung und Qualitätssicherung. So bleiben Sie konsistent und erkennen Probleme, die in einer einzelnen Sitzung übersehen werden könnten.

<div id="team-collaboration">
  ### Teamzusammenarbeit
</div>

Teilen Sie Ihre überarbeitete `CLAUDE.md`-Datei mit Ihrem Team, um einheitliche Dokumentationsstandards für alle Beitragenden sicherzustellen. Teams entwickeln häufig projektspezifische Prompts und Workflows, die fester Bestandteil ihres Dokumentationsprozesses werden.

<div id="custom-commands">
  ### Benutzerdefinierte Befehle
</div>

Erstellen Sie wiederverwendbare Slash-Befehle in `.claude/commands/` für häufige, projektspezifische Dokumentationsaufgaben Ihres Projekts oder Teams.
