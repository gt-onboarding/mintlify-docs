Ihre Dokumentationsseite wird automatisch deployed, wenn Sie Änderungen in das verbundene Repository pushen. Dafür muss die Mintlify-GitHub-App korrekt installiert und verbunden sein.

Wenn Ihre neuesten Änderungen nicht auf der Live-Seite erscheinen, prüfen Sie zuerst, ob die GitHub-App auf dem Konto oder in der Organisation installiert ist, die das Docs-Repository besitzt. Weitere Informationen finden Sie unter [GitHub-Problembehandlung](/de/settings/github#troubleshooting).

Wenn die GitHub-App verbunden ist, Änderungen jedoch weiterhin nicht deployed werden, können Sie von Ihrem Dashboard aus manuell einen Rebuild auslösen.

<div id="manually-triggering-a-deployment">
  ## Eine Bereitstellung manuell auslösen
</div>

<Steps>
  <Step title="Prüfen Sie, ob Ihr letzter Commit erfolgreich war.">
    Vergewissern Sie sich, dass Ihr letzter Commit in Ihrem Docs-Repository erscheint und keine Fehler aufgetreten sind.
  </Step>

  <Step title="Bereitstellung manuell auslösen.">
    Gehen Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com) und wählen Sie die Schaltfläche „Deploy“.

</Steps>
